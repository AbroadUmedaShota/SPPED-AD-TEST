# Handoff: 料金プラン「速度別単価」比較テーブル型カード

## 概要
名刺データ化サービスの料金ページに置く「速度別単価」セクションのデザイン参照です。
5つの速度プラン（お試し / 通常 / 特急 / 超特急 / オンデマンド）を**横並びの比較テーブル風カラム**で提示し、
各プランの「単価」「データ化項目数」「データ化タイミング（納期）」「適したケース」を一目で比較できるようにします。

オンデマンドのみ上位（プレミアム）契約者専用で、**カード右上に「プレミアム限定」テキスト**と専用カラー（ゴールド/クリーム）で差別化します。

## このデザインファイルについて
同梱の HTML/CSS/JSX は **HTML で作成したデザインリファレンス**です。意図する見た目・構造・寸法を示すプロトタイプであり、
そのまま本番に貼り付けるためのコードではありません。実装タスクは、これを**対象コードベースの既存環境
（React / Vue / Svelte / 既存テンプレート等）のパターンとデザイントークンで再現すること**です。
プロトタイプは React(UMD)+Babel+素のCSSで書かれていますが、その構成に縛られる必要はありません。

## 忠実度
**High-fidelity（ハイファイ）**。色・タイポ・余白・角丸・影まで最終想定値です。下記トークン/数値どおりに再現してください
（既存デザインシステムがあれば近いトークンへマッピング可）。

## レイアウト（単一セクション）

### 全体
- 外側フレーム：白カード。`max-width:1240px` / `border-radius:20px` / `padding:44px 40px` / `box-shadow:0 18px 50px rgba(16,41,77,.10)`。薄グレー背景 `#EEF1F5` の中央に配置。
- テーブル本体：**5等幅カラムの CSS Grid**（`grid-template-columns:repeat(5,1fr)`）。

### デスクトップ（≥1081px）— 連結テーブル表示
- `gap:0`。各カラム（カード `.c`）は**それぞれ独立した枠線**を持つ（`border:1px solid #E6ECF3`）。
- 隣接カードは `margin-left:-1px` で枠線を重ね、**距離0・二重線なし**で連結して見える。
- 角丸は**両端のみ**：先頭カードは左角、末尾カードは右角を `14px`。中間は角丸なし。
- オンデマンド（`.c--prem`）は `position:relative; z-index:1`（クリーム背景を前面に）。

### レスポンシブ（段階的に折り返し）
折り返したら連結をやめ、各カードを**独立した角丸カード（余白＋淡い影）**に切り替える。
| ブレークポイント | 列数 | 補足 |
|---|---|---|
| ≤1080px | 3列 | `gap:14px` / 各カード `border-radius:14px` / `box-shadow:0 4px 14px rgba(16,41,77,.05)` / `margin-left:0` |
| ≤760px | 2列 | フレーム `padding:24px 20px` |
| ≤520px | 1列 | 縦積み |

> 実装側の既存ブレークポイントに合わせて調整可。いずれの幅でも「価格・項目数行の整列」「値のみ太字」「オンデマンドのプレミアム限定表示」を維持すること。

## カード内の構成（上から）
`.c`（`padding:14px`、`display:flex; flex-direction:column; align-items:center; text-align:center`）

1. **アイコン** `.c-ico`
   - 直径 `36px` の円。中の線アイコン `22px`、`stroke-width:1.7`、塗りなし（稲妻のみ塗り）。`margin-bottom:7px`。
   - 通常：背景 `#EEF4FD` / 線 `#3B6FD4`。オンデマンド：背景 `#F6E6BD` / 線 `#A1761F`。
   - 種別：お試し=フラスコ / 通常=時計 / 特急=ゲージ / 超特急=ナビ矢印 / オンデマンド=稲妻。

2. **プラン名** `.c-name`：`font-weight:700; font-size:17px; color:#27445E`、`margin-bottom:5px`。

3. **価格** `.c-price`：`display:flex; align-items:baseline; gap:3px`、`margin-bottom:7px`。
   - 数字 `.c-num`：`font-size:34px; font-weight:900; color:#10294D`（オンデマンドは `#A1761F`）。
   - 単位「円/枚」`.c-unit`：`font-size:13px; font-weight:500; color:#5C6B7A`。

4. **プレミアム限定（制約表示）** `.c-gate`：**オンデマンドのみ**。
   - **カード右上に絶対配置**：`position:absolute; top:11px; right:11px`。
   - テキスト表現（ピル/チップではない）：`font-size:10px; font-weight:800; color:#A1761F（var(--gold-deep)）; letter-spacing:.02em`。
   - フロー上の場所を取らないため、**他カードの縦位置に影響しない**（＝価格・項目数行の整列が崩れない）。
   - 上辺overlap型リボンは不使用（枠で切れるのを避ける）。アイコンは付けない。

5. **スペックブロック** `.c-spec > .t5-spec`（`text-align:left`）
   - 行1：`データ化項目数：<b>{項目数}</b>`
     - `font-size:13px; line-height:1.6; font-weight:500; color:#27445E`。**太字は値のみ**（`<b>`=`font-weight:800; font-size:15px`）。ラベルは通常。
   - 区切り罫線 `.t5-hr`：`height:1px; background:#E6ECF3; margin:7px 0`。
   - 行2 `.t5-line2`（`font-size:12px; letter-spacing:-0.02em`：5枚すべて1行に収め、説明文の開始位置を揃えるため）
     - 通常：`会期終了から<b>{N}営業日</b>でデータ化`
     - オンデマンド：`<b>会期当日</b>にデータ化`（`.c--prem` は中央揃え）
     - **太字は値のみ**（`{N}営業日`/`会期当日`、`font-weight:800; font-size:15px`）。オンデマンドの `<b>` 色は `#A1761F`。

6. **説明文** `.c-desc`：`font-size:12px; line-height:1.7; color:#5C6B7A; text-align:left; margin-top:10px`（上段揃え＝スペック直下）。`text-wrap:pretty` 推奨。

- オンデマンドカード `.c--prem`：背景 `#FDF6E8`（クリーム）。

## 整列の要件（厳守）
- 5枚すべてで **価格 / データ化項目数 / 説明文の各行が横一直線**に揃うこと（プロトタイプでは全カードで一致を確認済み）。
- 「プレミアム限定」は**1枚だけに増える要素**だが、**絶対配置（右上）**でフローを占めないため整列に影響しない。
- 納期行は5枚とも1行に収める（`font-size:12px` ＋ `letter-spacing:-0.02em`）。太字値は `15px` を維持。

## Assets / アイコン
- 画像アセットなし。アイコンは**インライン SVG の線アイコン**（フラスコ/時計/ゲージ/ナビ矢印/稲妻）。実装側にアイコンライブラリ（lucide/Material/SF Symbols 等）があれば代替可。SVGパスは `plans-data.jsx` の `Icon` を参照。
- フォント：Google Fonts「Noto Sans JP」。

## Files（このバンドル）
- `料金カード 比較テーブル型.html` — デザイン本体（プロトタイプ）。`Spec`/`TCard`（カードのマークアップ）とテーブル化の `<style>` を内包。
- `styles.css` — 基本カードスタイルの正本。**このデザインで使うのは `:root` トークン＋基本カード `.c*` 系＋ `.row .c`／`.d5`**（`.d1`〜`.d10` 等の他テーマは別案用で未使用）。
- `plans-data.jsx` — プランデータ配列 `PLANS`（`key,name,icon,price,items,days,desc,premium,tone` …）と `Icon` コンポーネント。**コピーの数値とアイコンSVGはここを参照**。
  - 納期行の表示は「会期終了から `${days}営業日` でデータ化」、premium は「会期当日にデータ化」。

> 3ファイルを同じ階層に置き、HTMLをブラウザで開くと確認できます（React/Babel は CDN 読込）。
