# Handoff: 料金プラン 比較テーブル型カード（Pricing Comparison Table）

## Overview
展示会で集めた名刺などを「データ化」するサービスの**料金プラン表示**セクションの改修デザインです。
5つのプラン（お試し / 通常 / 特急 / 超特急 / オンデマンド）を横並びの比較テーブル風カラムで提示し、
各プランの「料金」「データ化項目数」「データ化のタイミング（納期）」「適したケース」を一目で比較できるようにします。

オンデマンドのみ上位（プレミアム）プラン専用で、`プレミアムOnly` ラベルと専用カラー（ゴールド）で差別化します。

## About the Design Files
このバンドルに含まれる HTML / CSS / JSX は **HTML で作成したデザインリファレンス**です。
意図する見た目と構造を示すプロトタイプであり、そのまま本番に貼り付けるためのコードではありません。

実装タスクは、**これらのデザインを対象コードベースの既存環境（React / Vue / Svelte / SwiftUI / ネイティブ等）の
確立されたパターン・コンポーネント・デザイントークンを使って再現すること**です。
まだ環境が無い場合は、プロジェクトに最適なフレームワークを選んで実装してください。
（このプロトタイプは React + 素の CSS で書かれていますが、その構成に縛られる必要はありません。）

## Fidelity
**High-fidelity（ハイファイ）**。色・タイポグラフィ・余白・角丸・影まで最終想定の値です。
下記 Design Tokens / Components の数値どおりにピクセル単位で再現してください（既存のデザインシステムがある場合は、近いトークンへマッピングして構いません）。

## Screens / Views

### View: 料金プラン比較テーブル（単一セクション）
- **Name**: Pricing Comparison Table
- **Purpose**: ユーザーが5プランを横断比較し、納期（データ化タイミング）と料金から最適なプランを選ぶ。
- **Layout**:
  - 外側フレーム：白カード。`max-width: 1240px`、`border-radius: 20px`、`padding: 44px 40px`、`box-shadow: 0 18px 50px rgba(16,41,77,.10)`。薄いグレー背景（`#EEF1F5`）の中央に配置。
  - テーブル本体：**5等幅カラムの CSS Grid**（`grid-template-columns: repeat(5, 1fr); gap: 0;`）。全体を1px罫線（`#E6ECF3`）で囲み `border-radius: 14px; overflow: hidden;`。
  - 各カラム（カード）間は**縦の区切り罫線**（`border-right: 1px solid #E6ECF3`、最終カラムのみ無し）。
  - カード内は**縦並び・中央揃え**（`display:flex; flex-direction:column; align-items:center; text-align:center;`）。`padding: 24px 16px;`
  - カード内の縦順序：① アイコン → ② プラン名 → ③ 料金 → ④ スペックブロック → ⑤ 説明文。
  - **説明文は上段揃え**（スペックブロックの直下に配置。`margin-top:16px`。下端に押し下げない＝`margin:auto`は使わない）。

- **Components**（カード内、上から）:

  1. **アイコン**
     - サイズ：直径 `42px` の円（`border-radius:50%`）。中の線アイコンは `22px`、`stroke-width:1.7`、`stroke-linecap/linejoin:round`、塗りなし（稲妻のみ塗り）。
     - 通常プラン：背景 `#EEF4FD`、線色 `#3B6FD4`。
     - オンデマンド：背景 `#F6E6BD`、線色 `#A1761F`。
     - アイコン種別：お試し=フラスコ / 通常=時計 / 特急=ゲージ / 超特急=ナビ矢印 / オンデマンド=稲妻。
     - 下マージン `13px`。

  2. **プラン名**（`お試し` 等）
     - `font-weight:700; font-size:17px; letter-spacing:.02em; color:#27445E;`、下マージン `9px`。

  3. **料金**（例：`50` + `円/枚`）
     - 行：`display:flex; align-items:baseline; gap:3px;`、下マージン `13px`。
     - 数字：`font-size:34px; font-weight:900; color:#10294D;`（オンデマンドは `#A1761F`）。
     - 単位 `円/枚`：`font-size:13px; font-weight:500; color:#5C6B7A;`。
     - **重要**：オンデマンドでも「プレミアム」バッジを料金の上に置かない（置くと他プランと縦位置がズレるため）。料金行は5プランで必ず同じ高さに揃える。

  4. **スペックブロック**（`text-align:left; width:100%;`）
     - 行1：`データ化項目数：<b>{項目数}</b>`
       - 文字：`font-size:13px; line-height:1.6; font-weight:500; color:#27445E;`
       - **太字対象は値のみ**（`<b>` は `font-weight:800`）。ラベル「データ化項目数：」は通常ウェイト。
     - 区切り罫線：`height:1px; background:#E6ECF3; margin:10px 0;`
     - 行2：
       - 通常プラン：`会期終了から<b>{N}営業日</b>でデータ化`
       - オンデマンド：`<b>会期当日</b>にデータ化`
       - **太字対象は値のみ**（`{N}営業日` または `会期当日`）。オンデマンドの `<b>` 色は `#A1761F`。
     - オンデマンドのみ：行2の下に **`プレミアムOnly`** ラベル
       - `display:inline-block; margin-top:10px; font-size:11px; font-weight:800; color:#A1761F; background:#FBF1D8; border:1px solid #EAD295; border-radius:999px; padding:3px 11px; letter-spacing:.02em;`

  5. **説明文**（適したケース）
     - `font-size:12px; line-height:1.7; color:#5C6B7A; text-align:left; margin-top:16px;`。`text-wrap:pretty` 推奨。

- **プレミアム（オンデマンド）カードの差分**：背景 `#FDF6E8`（クリーム）。ただし当デザインでは d5 テーブル枠内のため、カード単体の枠線・影は持たず背景色のみで差別化。

## Interactions & Behavior
- **静的表示**。クリック/ホバー/アニメーションは無し（純粋な情報提示セクション）。
- もしCTA（申込ボタン等）を後で足す場合も、本デザインには含まれない（既存の指示では文言・要素を増やさない方針）。
- **レスポンシブ**：本プロトタイプは固定の5カラム。実装側のブレークポイントに応じて、
  - タブレット以下：横スクロール、または 2〜3 列で折り返し、
  - モバイル：1列の縦積み（各プランをカード/行として縦に並べる）
  を推奨。いずれの場合も「料金行の高さ揃え」「値のみ太字」「オンデマンドの `プレミアムOnly` 表示」は維持すること。

## State Management
- 状態管理は不要（静的データのレンダリングのみ）。
- データはプラン配列（下記）をループ描画。バックエンド連携する場合は同形のデータ構造を用意。

## Assets
- 画像アセットは無し。**アイコンはインライン SVG の線アイコン**（lucide 系のフラスコ/時計/ゲージ/ナビ矢印/稲妻に相当）。
  - 実装側に既存アイコンライブラリ（lucide / Material / SF Symbols 等）があればそれで代替可。`plans-data.jsx` 内 `Icon` のパスを参照。
- フォント：Google Fonts「Noto Sans JP」。

## Files
- `料金カード改修案 比較テーブル型.html` … 本デザインのプロトタイプ本体（このカラム表を描画）。
- `styles.css` … 基本カード（`.c` 系）と比較テーブル風テーマ（`.d5`）、スペック共通スタイル。本デザインで使うのは基本カード + `.d5` + 末尾のページ内 `<style>` 上書き。
- `plans-data.jsx` … プランデータ配列（`PLANS`）と `Icon` コンポーネント。**コピーの「データ部分」とアイコンSVGパスはここを参照**。
  - 各プラン：`{ key, name, icon, price, items, days, desc, premium, tone, ... }`
  - 行2の表示は「会期終了から `${days}営業日` でデータ化」、premium は「会期当日にデータ化」。

> プロトタイプ HTML は `styles.css` と `plans-data.jsx` を相対パスで読み込みます。3ファイルを同じ階層に置いてブラウザで開くと確認できます（React/Babel は CDN 読み込み）。

## 実装時のチェックリスト
- [ ] 5カラムが等幅で、間に縦罫線、全体が角丸の枠で囲まれている
- [ ] 料金行が5プランで同じ縦位置（オンデマンドに料金上バッジを置かない）
- [ ] スペックは「値のみ太字」（ラベルは通常）
- [ ] オンデマンドのみ、行2の下に `プレミアムOnly` バッジ（ゴールド）
- [ ] 説明文は上段揃え（スペック直下）
- [ ] オンデマンドカードはクリーム背景＋ゴールド系アクセント
- [ ] 「フリー」「両プラン」ラベルは存在しない
