# プレミアム会員向けダッシュボード（契約状況） 要件定義書

## 1. 概要
本ドキュメントは、既存のプレミアムプラン契約者が「プレミアム登録ページ (`premium_signup_new.html`)」にアクセスした際に表示される、**「契約状況ダッシュボード」**の機能要件およびUI要件を定義する。

## 2. 目的・背景
*   **以前の課題:** 既存会員がプレミアムLPにアクセスした際、既に契約済みであるにも関わらず「申し込む」ボタンやマーケティング訴求が表示され、混乱を招いていた。
*   **解決策:** 既存会員（`is_premium_member = true`）を検知した場合、マーケティングコンテンツを非表示にし、自身の契約状況を確認・管理できる専用ビュー（ダッシュボード）に差し替える。

## 3. 画面仕様 (`premium_signup_new.html` の動的切替)

### 3.1. 表示条件
*   ログインユーザーの契約情報を参照し、`planTier` が `premium` の場合に本ビューを適用する。

### 3.2. ヒーローエリア（契約ステータス）
*   **デザイン:** プレミアム感のあるグラデーション背景（青〜紫）を維持しつつ、情報はパーソナライズされたものに変更。
*   **表示項目:**
    *   **ユーザー名:** 「こんにちは、〇〇 様」
    *   **ステータスバッジ:** 「Status: Active (利用中)」等の明確な状態表示。
    *   **契約プラン名:** 「Premium Plan」 (ダイヤモンドアイコン等で装飾)。
    *   **次回更新日:** 
        *   **仕様:** 登録日に関わらず、**「現在日時の当月末日」** を自動算出して表示する（月末締め仕様に基づく）。
        *   表記例: 「次回更新予定日: 2026年3月31日」
    *   **有効機能リスト:** 「データ無期限保存」「画像・名刺データDL」「Excelレポート出力」「SPEEDレビュー利用可能」など、現在享受しているメリットを箇条書きで可視化。

### 3.3. クイックアクションエリア
*   頻繁に行う契約管理操作へのショートカットを配置する。
*   **請求書確認:**
    *   ラベル: 「請求書を確認する」
    *   遷移先: 請求書一覧ページ (`invoiceList.html`)
    *   説明: 「過去の請求書やお支払い履歴を確認」
*   **解約手続き:**
    *   ラベル: 「解約手続きを行う」
    *   遷移先: 解約手続きページ (`premium_cancel.html`)
    *   説明: 「解約のお手続きはこちら」

### 3.4. サポートエリア
*   お困りの際の導線を明確にする。
*   リンク先: `bug-report.html` (またはお問い合わせフォーム)。

### 3.5. 除外する要素
*   既存会員には不要な以下のマーケティング要素は**非表示**とする（DOM要素ごと除去または置換）。
    *   「今すぐ申し込む」CTAボタン
    *   プラン料金表
    *   機能比較表
    *   導入事例（利用者の声）
    *   よくある質問（FAQ）※ただし契約・解約に関するFAQは残す検討余地あり（現状は全削除でシンプル化）

## 4. UI/UX要件
*   **レイアウト:** 
    *   情報は**中央寄せ (Center Align)** を基本とし、PC/SP両方でバランス良く見えるよう配置する。
    *   項目のバラつきを防ぎ、視線誘導をスムーズにする。
*   **レスポンシブ:** モバイル表示時もカードデザインが崩れないよう調整する。
