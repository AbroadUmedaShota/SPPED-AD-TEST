# 実績管理画面（Performance Management） 要件定義書

## 1. 概要
管理者（Admin）がオペレーター（作業員）の実績を「品質（Quality）」「生産性（Productivity）」「コスト（Cost）」の観点で管理・評価するための専用画面。

## 2. 画面基本情報
- **パス:** `03_admin/performance-management.html`
- **対象ユーザー:** システム管理者

## 3. 画面構成

### 3.1. ヘッダーエリア
既存の実績管理画面を踏襲しつつ、全体のサマリー情報を表示する。

#### 表示項目
1.  **今月の支払い確定額**
    - 今月の報酬支払総額（および前月比 %）
    - 今月の総処理数（枚数）
    - 平均データ精度（%）
    ※上記3点は全グループ総合値

2.  **絞り込み機能（フィルター）**
    - **期間:** 開始日 ～ 終了日
    - **展示会（アンケート）:** 展示会単位での絞り込み
    - **指標:** 処理件数 / 平均処理時間 / 一致率 / エラー率 などの閾値フィルタ（任意）

### 3.2. メインエリア
「個人単位」と「グループ単位」をタブで切り替えて表示する。

#### 【タブA：個人単位】
個々のオペレーターの能力と貢献度を可視化する。
「誰が優秀か」「誰に指導が必要か」を一目で判断可能にする。

**一覧テーブル項目:**
| 項目名 | 説明 | 備考 |
| :--- | :--- | :--- |
| **オペレーター名 / 所属** | オペレーターの氏名と所属グループ名 | |
| **入力完了数** | エラーも含めた総入力枚数 | 生産性指標 (量) |
| **エラー件数** | 修正が必要だった枚数 | 品質指標 (負) |
| **有効作業数** | `入力完了数 - エラー件数` | 支払い対象枚数 |
| **作業時間** | 合計作業時間 / 1枚あたりの平均処理時間 | 生産性指標 (速さ) |
| **推定報酬額** | `有効作業数 × 単価` | コスト指標 |
| **評価** | 指標設定に基づく達成状況（例: 達成=青、未達=赤） | 視覚的インジケータ |

**詳細モーダル（個人）:**
行をクリックすることで表示。
- **複合グラフ（Combo Chart）:**
    - **X軸:** 日付（期間指定に基づく）
    - **Y軸（左）:** 処理枚数（棒グラフ）
    - **Y軸（右）:** エラー率（折れ線グラフ）
- **レーダーチャート:**
    - **軸:** 氏名 / 会社名 / 住所 / 電話番号 / メールアドレス
    - **意図:** 入力項目ごとの苦手分野を可視化し、指導に役立てる。

#### 【タブB：グループ単位】
組織ごとの処理能力（キャパシティ）とコストパフォーマンスを可視化する。
「どの会社に発注すべきか」「緊急時にどれだけ捌けるか」を判断可能にする。

**一覧テーブル項目:**
| 項目名 | 説明 | 備考 |
| :--- | :--- | :--- |
| **グループ名** | 会社名やチーム名 | |
| **稼働人数 / 登録人数** | 実働人数と登録総数 | 「いざという時の動員力」 |
| **最大処理能力 (日)** | `平均処理数(人/日) × 稼働人数` | 1日で捌ける推定枚数 |
| **品質レベル** | グループ全体の平均処理速度 / 品質ばらつき（標準偏差など） | 組織の安定性 |
| **コスパ (1円あたりの処理数)** | `有効作業数 ÷ 総支払額` | **※提案事項**<br>高いほど費用対効果が良い |
| **納期遅延発生数** | 納期を遵守できなかった件数 | **※提案事項**<br>遵守率ではなく「発生数」でアラート |

**詳細モーダル（グループ）:**
行をクリックすることで表示。
- **散布図（Scatter Plot）:**
    - **プロット:** 所属オペレーター全員
    - **X軸:** 作業スピード（速い ⇔ 遅い）
    - **Y軸:** 正確さ（高い ⇔ 低い）
    - **エリア定義:**
        - 右上: 優秀層（Star）
        - 右下: 早いが雑（Review Needed）
        - 左上: 遅いが丁寧（Training Needed）
        - 左下: 課題あり（Alert）

### 3.3. 指標設定（Target Settings）
画面上部、または設定モーダルにて、管理者が「目標値」を設定可能にする。
- 設定項目例: 目標処理枚数、許容エラー率、目標平均処理時間
- 反映: 一覧テーブルにて、目標未達の数値を赤字/赤背景で強調表示する。

## 4. 検討事項と提案

### 4.1. コストパフォーマンスの計算式
- **現状の課題:** 良い計算式が未定。
- **提案:** 単純な「単価」だけでなく、「修正にかかったコスト」も加味すべきですが、まずはシンプルに **「1円あたり何枚の有効データを作れたか（CPY: Cards Per Yen）」** を指標とすることを提案します。
    - 式: `有効作業総数 ÷ 総報酬支払額`
    - これにより、単価が高いベテランと、単価は安いがミスが多い新人の「最終的なコスパ」を比較可能です。

### 4.2. 納期遵守率の扱い
- **現状の課題:** オンデマンド作業で納期遵守は前提のため、指標として必要か？
- **提案:** 「遵守率 100%」が並ぶだけでは情報価値が低いため、**「遅延発生件数」** のみをカウントし、0件以外の場合に警告表示する形式を提案します。
