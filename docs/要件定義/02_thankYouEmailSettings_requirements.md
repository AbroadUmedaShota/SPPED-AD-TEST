## 1. 概要

本文書は、アンケート回答者に対するお礼メール送信機能に関する画面の要件を定義し、実装に必要な設計詳細を記述するものである。  
UIのレイアウト構造・状態遷移（シナリオ別）・JavaScriptによるロジック要件・非機能要件を網羅する。

### 1.1. 背景と目的

展示会・イベントにおいて取得した名刺情報・アンケート回答者に対し、担当者が任意のお礼メールを一括送信することを目的とする。
本機能は以下の業務フローにおける「お礼メール送信」ステップを担う。

```
[展示会開催] → [アンケート回答] → [名刺データ化完了] → [お礼メール設定・送信] → [送信完了確認]
```

### 1.2. 用語定義

| 用語 | 定義 |
| :--- | :--- |
| お礼メール | 展示会・イベント参加者（アンケート回答者）に対して送信する感謝・フォローアップを目的としたメール |
| 送信対象者 | 特定のアンケートに回答し、かつ担当者が送信チェックをONにした受信者 |
| 差し込み変数 | メール本文中に `{{会社名}}` のような形式で記述し、個別データで置換されるプレースホルダー |
| 送信可能期間 | 名刺データ化**完了日を起算**として、**7日後の 23:59:59 まで**お礼メール送信が許可される期間（確定値．延長なし）|
| 会期 | アンケートの有効回答受付期間（展示会の開催期間に相当） |
| グループユーザー | 同一契約企業に属するシステム利用者全員。送信権限はグループ単位で共有される |

## 2. 画面基本レイアウト構造

画面の拡張性と操作性を確保するため、以下の2カラム（メインエリア＋サイドツールバー）構成とする。

### 2.1. メインエリア（左側／中央）

#### 2.1.1. アンケート基本情報

| 表示項目 | 表示内容 | 備考 |
| :--- | :--- | :--- |
| アンケート名 | アンケートの識別名 | ツールチップアイコン（ⓘ）を右側に配置 |
| アンケートID | システム付番のID | 同上 |
| 会期期間 | 開始日〜終了日 | 同上 |

#### 2.1.2. シナリオ切り替えコントロール（モック開発専用）

- 本番実装では非表示とする開発・確認用コントロール。
- 「①会期前・会期中」「②会期後・データ化処理中」「③会期後・送信待機」「④会期後・送信完了」「⑤送信可能期間外」を切り替えるプルダウンまたはボタングループを画面上部に配置する。

#### 2.1.3. メール内容設定ブロック

- **メールテンプレート**: 標準文面（定型文）が初期値としてセットされる。
- **件名入力欄**: テキスト入力。**初期値（デフォルト）は 「このたびはご来場いただきありがとうございました」** （将来確定依頼）。最大文字数は **255文字（仗）** を上限とする。上限超過時は入力をとめエラーメッセージを表示する。
- **差し込み変数UI**:
  - 本文入力欄の上部または右側に「変数一覧（会社名・氏名等）」をバッジ状のボタンとして配置する。
  - ボタンクリック時に、本文テキストエリアのカーソル位置へ `{{会社名}}` などのプレースホルダー文字列を挿入する。
- **本文入力欄**: 複数行テキストエリア。
- **リアルタイムプレビュー領域**: 本文中の差し込み変数（例：`{{会社名}}`）をダミーデータ（`mockRecipients[0]`）の値で置換した結果を表示する領域。送信前の最終確認を目的とする。

#### 2.1.4. 送信対象者リスト

| 列名 | 内容 |
| :--- | :--- |
| 送信チェックボックス | 一括チェック（ヘッダー）および個別チェック。サイドバーの件数・金額とリアルタイム連動 |
| 企業名 | 送信対象者の所属企業名 |
| 氏名 | 送信対象者の氏名 |
| メールアドレス | 送信先アドレス |
| 送信日時（状態④以降） | 送信完了日時。未送信・対象外は空欄。**監査ログ対応**（FR-13） |
| ステータス（状態②以降） | 送信結果（成功／対象外／エラー）を色付きバッジで表示 |

### 2.2. サイドツールバー（右側・スクロール追従）

#### 2.2.1. 送信オプション

- **手動送信（デフォルト）**: ユーザーが「お礼メールを送信する」ボタンを押下するまで送信されない。すべてのアンケートでの標準動作。
- **自動送信**: 名刺データ化処理の完了を検知次第、システムが自動的にメールを送信する。**オンデマンドプラン（当日データ化）選択時は、速度感重視のため自動送信を推奨設定とする。**
- **送信しない**: お礼メール自体を送らない。確認通知も送信されない。

#### 2.2.2. 送信対象件数および費用試算

| 表示項目 | 内容 | 補足 |
| :--- | :--- | :--- |
| 送信対象件数 | ●件 / 全●件 | チェックボックスのON/OFFに連動しリアルタイム更新 |
| 無料枠残数 | 100件まで無料 | 無料枠上限を明示 |
| ご請求見込み金額 | ●円（**税別**）| §4.3 のロジックで算出。画面上に「※税別」と注記する |
| **送信結果件数**（状態⑤以降） | 送信結果：●件 | 送信完了後に表示。実際に届いた件数を集計（対象外・失敗除く） |

#### 2.2.3. アクションボタン

| ボタン名 | 活性条件 | 押下時の動作 |
| :--- | :--- | :--- |
| キャンセル | 常に活性 | 直前の保存済み状態に戻る（または一覧画面へ遷移） |
| 設定を保存する | 常に活性 | 以下の設定を保存する：件名・本文・送信オプション（手動／自動／送信しない）・送信チェックボックスのON/OFFstate。再アクセス時に保存した状態を復元する |
| お礼メールを送信する | 状態③（送信待機）かつ送信可能期間内のみ活性 | 押下で確認ダイアログ（§4.5）を表示 |

## 3. 画面状態（シナリオ）と状態遷移

本画面はフロントエンドモックであり、JavaScript の `currentStatus` 変数による以下の5状態間で UI を再描画する。

### 状態遷移図

```
[会期前・会期中]
      ↓（会期終了）
[会期後・データ化処理中]
      ↓（データ化完了）
[会期後・送信待機]
      ↓（送信実行）
[会期後・送信完了]

※ 上記いずれの状態においても、送信可能期間（1週間）を超過した場合は
   [送信可能期間外] 状態へ遷移する。
```

### 3.1. 状態①：会期前・会期中

- **状態コード**: `before_or_during_event`
- **遷移条件**: 現在日時が会期終了日以前
- **UI仕様**:
  - 送信対象者リスト領域には「会期終了後、名刺データ化が完了すると対象者が表示されます」というプレースホルダーメッセージを表示し、リスト自体は非表示とする。
  - メール内容（件名・本文・変数）の編集および設定の保存は可能とする。
  - サイドバーのご請求見込み金額は「算出中」または「0円」を表示する。
  - 「お礼メールを送信する」ボタンは非活性（`disabled`）とする。

### 3.2. 状態②：会期後・名刺データ化処理中

- **状態コード**: `after_event_processing`
- **遷移条件**: 会期終了日を経過し、名刺データ化処理が未完了
- **UI仕様**:
  - 送信対象者リスト領域に「現在アンケートおよび名刺データ化を処理中です。完了次第、対象者が表示されます。」を表示する。
  - 送信に関わるすべてのアクション（送信ボタン含む）は非活性（シアーダウン）とする。
  - メール内容の編集および設定の保存は可能とする。

### 3.3. 状態③：会期後・送信待機（名刺データ化完了）

- **状態コード**: `after_event_ready`
- **遷移条件**: 名刺データ化処理が完了し、かつ送信可能期間内、かつ未送信
- **UI仕様**:
  - ダミーのアンケート回答者データを読み込み、送信対象者リストを表示する。**実運用上の最大件数として500件程度を想定する。**大量件数でもチェック操作・件数計算が100ms以内に反映されるよう実装すること（§6 参照）。
  - 各行の先頭に「送信チェックボックス」を配置し、初期値はON（送信対象）とする。
  - チェックのON/OFFに連動して、サイドバーの「送信対象件数」および「ご請求見込み金額」をリアルタイムに再計算する。
  - 「お礼メールを送信する」ボタンを活性化し、クリックで確認ダイアログを表示する（確認ダイアログの仕様は §4.5 参照）。

### 3.4. 状態④：会期後・送信完了

- **状態コード**: `after_event_sent`
- **遷移条件**: 送信処理が実行済み
- **再送禁止仕様**:
  - 同一アンケートの同一送信対象者への**お礼メールの再送信は禁止**とする。
  - 一度 `after_event_sent` 状態に遷移したアンケートに対しては、送信ボタンを再活性化しない（手動・自動問わず）。
  - 誤送信が発生した場合の対応は本システムの対象外とし、ユーザーが自社メーラーよりフォローアップを行う運用とする。
- **UI仕様**:
  - **全体ステータス通知（画面上部）**:
    - 全件成功時：「すべてのメールを送信しました」をトースト通知または固定アラートで表示する。
    - 部分成功時：「**一部のメールが送信できませんでした。リスト内のエラー内容をご確認ください。**」を固定アラートで表示する。
  - **送信結果件数表示**: 画面上部またはサイドバーに「送信結果：●件」を表示する。送信失敗・対象外を除いた実繋送信完了数を集計する（FR-15）。
  - 送信対象者リストのチェックボックスは非表示または非活性とする。
  - 各行のステータス列に以下の視覚的区別を適用する。

    | ステータス | 表示 | 背景色 |
    | :--- | :--- | :--- |
    | 送信成功 | 「送信済み」バッジ | 薄緑（`#e6f4ea` 相当） |
    | 空変数あり送信済み | 「送信済み」バッジ＋⚠️アイコン | 薄緑＋黄色ボーダー相当 |
    | 対象外（チェックOFF） | 「対象外」バッジ | グレー（`#f1f3f4` 相当） |
    | 送信失敗（エラー） | 「送信失敗」バッジ＋ⓘアイコン | 薄赤（`#fce8e6` 相当） |

  - エラー行のⓘアイコンにホバーまたはクリックで、エラー理由（例：「宛先メールボックスが満杯です」「ドメインが存在しません」）のポップアップを表示する。
  - **送信失敗の能動的通知は行わない**（メール通知・プッシュ通知等は実施しない）。ユーザーは本画面の一覧表示で結果を確認すること。
  - メール本文の編集エリアは読み取り専用（ReadOnly）とする。
  - サイドバーの金額表示を「ご請求見込み金額」から「確定金額」に変更する。
  - 「お礼メールを送信する」ボタンは非表示または送信完了を示すラベルに切り替える。

### 3.5. 状態⑤：送信可能期間外（期限切れ）

- **状態コード**: `period_expired`
- **遷移条件**: 名刺データ化**完了日を起算**として **7日後** が経過し、かつ未送信
- **送信可能期間の定義**:
  - 起算日：名刺データ化完了日（当日。完了時刻は問わない）
  - 期限：完了日から **7日後の 23:59:59**（固定。延長・例外対応は行わない）
  - 例：3/16完了 → 3/23 23:59:59 が期限
- **UI仕様**:
  - 送信に関するすべての入力項目およびボタンを非活性（グレーアウト）とする。
  - 画面上部に警告アラートとして以下のメッセージを表示し、代替手段へ誘導する。  
    > **送信可能期間（名刺データ化完了から1週間）を経過したため、システムからのお礼メール送信はできません。お手持ちのメーラー等よりご連絡ください。必要に応じてCSVデータ等をダウンロードしてください。**

### 3.6. 排他制御の考慮（将来のバックエンド実装向け）

- 送信権限はグループユーザー全体に帰属するため（FR-22）、複数ユーザーが同時に本画面を操作する可能性がある。
- **楽観的ロック方式**を前提とした設計とする。
  - 「設定を保存する」または「お礼メールを送信する」ボタン押下時に、他ユーザーによって先に更新・送信がなされていた場合は、以下のエラーダイアログまたはトースト通知を表示する。  
    > **他のユーザーによってデータが更新されました。画面を再読み込みしてください。**

## 4. ロジック要件（JavaScript 実装詳細）

### 4.1. モックデータ構造

```javascript
/**
 * アンケート回答者ダミーデータ
 * @typedef {Object} Recipient
 * @property {number}  id          - 内部識別ID
 * @property {string}  company     - 企業名
 * @property {string}  department  - 部署名（空文字の場合あり）
 * @property {string}  title       - 役職名（空文字の場合あり）
 * @property {string}  name        - 氏名
 * @property {string}  email       - メールアドレス
 * @property {boolean} sendEnabled - 送信チェックボックスの初期値
 * @property {string}  status      - 'pending' | 'sent' | 'sent_with_warning' | 'failed' | 'excluded'
 *                                    sent_with_warning: 空変数のままトリム送信した行（⚠️マーク表示対象）
 * @property {string}  [errorMsg]  - status が 'failed' の場合のエラー理由
 * @property {boolean} [hasEmptyVariable] - true の場合、差し込み変数に空値が含まれていた（⚠️表示の根拠）
 * @property {string}  [sentAt]    - 送信日時（ISO 8601。4桁文字列。未送信・対象外は undefined）監査ログ対応
 */
const mockRecipients = [
  {
    id: 1, company: '株式会社A', department: '営業部', title: '部長',
    name: '山田 太郎', email: 'yamada@example.com',
    sendEnabled: true, status: 'pending', hasEmptyVariable: false
  },
  {
    id: 2, company: 'B株式会社', department: '', title: '',
    name: '佐藤 花子', email: 'sato@example.com',
    sendEnabled: true, status: 'pending', hasEmptyVariable: true  // 部署名・役職が空
  },
  // 実運用想定は最大500件程度。仮想スクロール等の実装を検討すること（§6 パフォーマンス要件参照）
];
```

### 4.2. 差し込み変数の挿入ロジック

1. 本文テキストエリアにフォーカスがある状態で、変数ボタン（例：「会社名」）をクリックする。
2. JavaScript にて `textarea.selectionStart` および `textarea.selectionEnd` を取得し、カーソル位置に変数文字列（例：`{{会社名}}`）を挿入する。
3. 挿入後、即座にリアルタイムプレビュー領域を更新する（詳細は §4.2.1 参照）。

#### 4.2.1. リアルタイムプレビューの置換ロジック

- `mockRecipients[0]` のデータを用いて、本文中の各変数を対応する値に `.replace()` で置換しプレビューを表示する。
- 空文字の変数が存在する場合は、置換後に連続するスペースをトリム処理して表示する（例：`{{会社名}} {{部署名}} {{氏名}}` → `株式会社A 山田 太郎`）。

#### 4.2.2. 未設定変数の警告表示

- 送信対象者のいずれかに対し、本文に含まれる変数の値が空文字の場合、プレビュー領域に以下の警告アラート（黄色）を表示する。
  > **※ 注意：「〇〇」が未設定の対象者が●名含まれています。**
- 警告は送信をブロックしない（情報提供のみ）。

#### 4.2.3. 空変数の送信時挙動（手動・自動共通）

- 必須・任意を問わず、空文字の変数は**プレースホルダーを削除（空文字置換）**し、残余スペースをトリムしたうえで送信する。手動・自動送信のいずれの場合も同一の挙動とする。
  - 例：本文 `{{会社名}} {{部署名}} {{氏名}}` に対し、部署名が空の場合 → `株式会社A 山田 太郎` として送信する。
- 送信処理はエラーで止めず**全送信対象者に対して実行する**。送信のスキップは行わない。
- **自動送信時の設計根拠**: オンデマンドプランを選択するユーザーは速度感を最優先とする。空変数を理由に送信を止める・スキップすることは、そのニーズに反する。送信可否よりも「届けること」を優先する方針とする。
- **「必須」変数の定義について**: `{{会社名}}` および `{{氏名}}` を必須と定義しているが、この「必須」は **UIの警告表示・プレビューの注意喚起にのみ使用** するものであり、送信のブロック条件とはしない。
- **送信後のリスト表示**: 空変数のまま送信した行には ⚠️ マークを表示し、ユーザーが後から確認できるようにする。
- **既知リスク（記録）**: 会社名・氏名が空白のまま届いたメールはスパムフィルターに引っかかる可能性が僅かにある。名刺データ化の性質上このケースは稀であり、許容するリスクと判断する。

#### 4.2.4. 差し込み変数の定義一覧

本サービス（展示会・イベントでの名刺データ化・アンケートサービス）の性質上、最低限以下の差し込み変数を用意する。

| # | 変数文字列 | 説明 | 必須／任意 |
| :--- | :--- | :--- | :--- |
| 1 | `{{会社名}}` | 対象者の所属企業名 | 必須 |
| 2 | `{{部署名}}` | 対象者の所属部署名 | 任意 |
| 3 | `{{役職}}` | 対象者の役職名 | 任意 |
| 4 | `{{氏名}}` | 対象者の氏名（姓＋名） | 必須 |
| 5 | `{{アンケート名}}` | 対象アンケート（展示会・イベント）の名称 | 任意 |
| 6 | `{{自社担当者名}}` | 自社（システム利用側）の担当者名 | 任意 |

- 上記変数はUIの変数リストからワンクリックで挿入できるものとする。
- `{{会社名}}` および `{{氏名}}` の「必須」は**警告表示・注意喚起を目的とした分類**であり、送信のブロック条件とはしない。空値であっても送信は実行される（§4.2.3 参照）。

### 4.3. 費用試算ロジック

> 根拠ドキュメント: [`01_billing_and_invoicing_logic.md §1.3.2`](../プロダクト/要件/01_billing_and_invoicing_logic.md)

```javascript
const FREE_LIMIT  = 100;  // 無料枠（件）: 1アンケートにつき最初の100通まで無料
const UNIT_PRICE  = 1;    // 1件あたり単価（円・税別）: 101通目以降 @1円 ※確定値

/**
 * 送信対象件数から請求見込み金額を算出する。
 * @param {number} count - 送信チェックがONの件数
 * @returns {{ billableCount: number, estimatedCost: number }}
 */
function calcEstimatedCost(count) {
  const billableCount  = Math.max(0, count - FREE_LIMIT);
  const estimatedCost  = billableCount * UNIT_PRICE;
  return { billableCount, estimatedCost };
}
```

- 画面上の「送信対象件数」「ご請求見込み金額」は、チェックボックスの変更イベントに連動してリアルタイムで更新する。
- 表示金額は税別とし、画面上に「※表示金額は税別です」と注記する。
- **課金判定のUI制御について**: 本サービスの請求方式は「実送信件数を翌月末に請求する後払い方式（銀行振込）」であるため、送信ボタン押下時に事前決済・残高審査・ブロック処理は行わない。

### 4.4. 送信可能期間の判定ロジック

```javascript
/**
 * 送信可能期間の定義（確定値・変更不可）
 * - 起算: 名刺データ化完了日の翌日 00:00:00
 * - 期限: 起算日から7日目（7 * 24h）の 23:59:59
 * - 延長・例外対応: なし
 */
const SENDING_PERIOD_DAYS = 7;

/**
 * 現在日時が送信可能期間内かどうかを判定する。
 * @param {Date} dataCompletedAt - 名刺データ化完了日時
 * @returns {boolean}
 */
function isWithinSendingPeriod(dataCompletedAt) {
  // 起算日: 完了日の翌日 00:00:00
  const startOfNextDay = new Date(dataCompletedAt);
  startOfNextDay.setDate(startOfNextDay.getDate() + 1);
  startOfNextDay.setHours(0, 0, 0, 0);

  // 期限: 起算日から7日後の 23:59:59
  const deadlineMs = startOfNextDay.getTime() + SENDING_PERIOD_DAYS * 24 * 60 * 60 * 1000 - 1;
  return Date.now() <= deadlineMs;
}
```

- `isWithinSendingPeriod` が `false` の場合、画面全体を非活性化し §3.5 の警告メッセージを表示する。
- 期限切れ後の延長は**システム上・運用上ともに対応しない**。ユーザーが期限を過ぎた場合は自社メーラーでの対応を案内する。

### 4.5. 送信確認ダイアログ（FR-11）

- 「お礼メールを送信する」ボタン押下時に以下の確認ダイアログを表示する。
  - **タイトル**: 「お礼メールを送信しますか？」
  - **本文**: 「送信対象：●件 / ご請求見込み金額：●円（**税別**）。この操作は取り消せません。」
  - **アクション**: 「送信する」（危険系スタイル）/ 「キャンセル」
- 「送信する」押下後は送信処理を実行し、完了後に状態④（`after_event_sent`）へ遷移する。

## 5. UIテキストおよびツールチップ要件

画面上のツールチップは、`title` 属性または専用のツールチップコンポーネント（CSSホバー等）を用いて実装する。

| 項目名 | ツールチップ表示文 |
| :--- | :--- |
| アンケート名 | 御社内でアンケートを識別するための名称です。<br>回答者の画面には表示されません。 |
| アンケートタイトル | 回答画面の冒頭に表示されるタイトルです。 |
| アンケート説明文 | 回答画面において、アンケートタイトルと第1問の間に表示される説明文です。 |
| 展示会会期 | アンケートの有効回答受付期間です。<br>この期間より前の回答はサンプルデータとして格納されます。<br>この期間を過ぎるとアンケートへの回答ができなくなります。<br>開始日はアンケート新規作成日の翌日以降から設定できます。<br>終了日は開始日と同日〜30日後の範囲で設定できます。<br>会期開始後は、以下の変更ができなくなります。<br>・アンケート名および展示会会期の変更<br>・設問の変更またはアンケートの削除<br>・名刺データ化設定の変更 |
| メモ | 御社内用のメモ欄です。<br>回答者の画面には表示されません。 |
| お礼メールを送らない | 「送信しない」を選択すると、御礼メールの送信確認メールが届かなくなり、御礼メールは送信されません。 |

> [!NOTE]
> §5 のツールチップ定義は本画面（お礼メール設定画面）に表示される項目に限定している。「アンケートタイトル」「アンケート説明文」「展示会会期」「メモ」等の他画面共通ツールチップは `docs/画面設計/仕様/tooltip_spec.md` を参照のこと。
