## 1. 概要

本文書は、名刺データ化設定画面（`bizcardSettings.html`）のUI/UXを抜本的に見直し（ブラッシュアップ）、本格的な2カラム構成の採用と他画面（お礼メール設定・アンケート作成）とのクーポン連携ロジックの実装について定義する。

### 1.1. 既存画面の課題とブラッシュアップの目的
1. **入力と結果の分離**: 現行画面では「データ化プラン」はメインエリアに、「見込み枚数」はサイドバーにあるなど、密接に関連する入力項目が分離しており、ユーザーの視線移動が多い。
2. **情報のノイズ過多**: 画面下部に「ご請求に関する詳細」「名刺データ化の詳細」などの長文アコーディオンが鎮座しており、画面全体が重く見え、重要な設定項目に集中しづらい。
3. **クーポンのサイロ化**: クーポンが各画面で独立しており、アンケート全体を通した割引管理が可視化されていない。

**解決策**: まるでECサイトの「カート・決済確認画面」のように、左カラムを「設定入力」、右カラムを「結果確認と確定（サマリー）」と明確に役割分担させる。

### 1.2. 対象デバイス定義
本画面はBtoB利用を想定した管理画面であるため、**デスクトップPC環境（フルHD・一般的なノートPC画面サイズ等）での利用を前提**とする。スマートフォン等のモバイル端末操作に特化したレスポンシブ最適化（要素の縦積みレイアウト等）は、本要件の対象外とする。

## 2. 新画面UIレイアウト構造（ブラッシュアップ案）

### 2.1. 画面全体の分割方針（2カラム）
- **左カラム（メイン入力エリア・幅約65%）**: ユーザーが思考し、選択・入力を行うための領域。不必要な注意書きは排除し、アクションに集中させる。
- **右カラム（サマリー・アクションエリア・幅約35%）**: 左カラムの入力結果（金額・スケジュール）をリアルタイムに反映し、最終確定を行う領域。縦スクロールに追従（Sticky）とする。

### 2.2. 左カラム（メイン入力エリア）の再構築
現行の項目を「設定フロー」として再整理する。

#### ① アンケート基本情報
- 画面最上部にコンパクトなカードとして配置（アンケート名、ID、会期期間）。

#### ② 【変更】STEP1：データ化プランと見込み枚数の選択
- **見込み枚数の移行**: サイドバーにあった「見込み枚数」入力を、左カラムのプラン選択直下に移動する。
- **プラン選択のUI改善**: 長い説明文本文を削り、特徴を箇条書きでまとめたカード型セレクタへ変更する。
- **「名刺データ化の詳細」のモーダル化**: プランごとの詳細なデータ化項目や入力規則に関する長文は、付近に「詳細仕様を見る」リンクを配置し、クリックで**モーダルダイアログ**を開く形式とする。

#### ③ STEP2：プレミアムオプションの選択
- チェックボックスで追加オプションを選択する。（※仕様説明のツールチップアイコン等はプレースホルダーとして配置のみ行い、文言・内容は別途検討とする）

#### ④ STEP3：名刺データ化は行わない（スキップ設定） / メモ
- **スキップ時の連動アクション**: 「名刺のデータ化は行わない」トグルをONにした場合、左側のプラン・オプションなどのフォーム群は丸ごとグレーアウト（非活性化）する。同時に、右カラムの見積もり金額表示は「0円」となる。
- **メモ欄**: 社内用メモ欄（アコーディオン）を配置する。

### 2.3. 右カラム（サマリー・アクションエリア）の再構築
お礼メール設定画面（`thankYouEmailSettings.html`）の右カラムと全く同じUI構造・インタラクション（反応速度）を採用し、全体で統一感のあるサマリーとする。

- **ご請求見積もりサマリー**: プラン料金＋プレミアムオプション等を含めた「ご請求見込み金額（税別）」を表示。
- **データ化完了予定日**: 目立つように配置。
- **ご請求・お支払いに関する注意事項（モーダル化）**: 長文を廃止し、リンクからモーダルダイアログで表示する。
- **クーポン適用エリア**: （後述3.2参照。新規入力、適用済み表示、エラーハンドリングを行う）
- **アクションボタン**: 「キャンセル」「設定を保存して依頼を確定する」ボタンを右カラム最下部に固定配置する。キャンセル押下時は、アンケート作成画面（`surveyCreation.html`）または前のフローへの遷移とする。

## 3. クーポン連携とインタラクション要件

### 3.1. データ構造の共通化
・クーポン情報（`appliedCoupon`）を全画面共通のトップレベルオブジェクトとして保持する。「名刺」または「お礼メール」機能の設定をスキップ（行わない）にした場合でも、適用された共通クーポンデータ自体は消滅しない。
・割引対象は、クーポンを適用したアンケートに紐づく従量課金項目のみとし、プレミアム利用料や追加アカウント料金は対象外とする。
・当該アンケートにおいてお礼メール送信費用が発生する場合は、無料枠控除後の課金対象額に対してクーポン条件を適用する。

### 3.2. クーポンUIの状態とエラーハンドリング
- **新規適用時のバリデーション**:
  - 新たなクーポンコードを入力して「適用」を押下した際、検証（バックエンド通信・モック非同期処理）中であることを示す短いローディング状態（スピナー等）を提示する。
  - **エラー表示**: コードが存在しない、使用期限切れ等で無効な場合は、入力欄の直下に赤文字で「無効なクーポンコードです」等のバリデーションエラーを明確に表示する。
- **適用済み表示と解除**:
  - 他画面で既にクーポンが適用されている場合、あるいは本画面で適用が成功した場合は、入力フォームを消して「**【適用中のクーポン】 SUMMER2026** `[変更・解除する]`」などのステータスUIへ切り替える。
  - **解除時の警告**: 「変更・解除は当該アンケートに紐づく従量課金の見積もりに影響します」という確認（警告）モーダルを挟み、承認された場合のみクリアする。

### 3.3. 状態管理（未保存時の挙動・通信保護）
- **未保存アラート（Dirty状態の管理）**:
  画面上のフォームを一度でも変更（プラン選択、見込み枚数変更、クーポン適用等）した状態で「キャンセル」ボタン、またはブラウザの「戻る・閉じる」操作を行った場合、「**変更が保存されていません。破棄して前の画面に戻りますか？**」という破棄確認モーダル（またはBeforeUnloadアラート）を表示し、誤操作によるデータ消失を防ぐ。
- **保存アクション時の状態（通信待機とエラー保護）**:
  「設定を保存して依頼を確定する」ボタン押下時は、ボタン内をスピナー表示に切り替え、二重送信を防止するため画面全体のフォームとボタンを非活性化（Disabled状態）にする。保存に成功した場合はトースト通知（完了メッセージ）を出して画面遷移し、失敗時（通信エラー等）は非活性を解除して画面上部に異常系エラー通知を表示する。

## 4. 移行・開発ステップ（今後のタスク）
1. **要件設計完了・合意**: 本ドキュメントにて定義した抜本的なUIレイアウト構想とインタラクション（モーダル・未保存警告等）の仕様合意。
2. **UI/プロトタイプ（HTML/CSS）実装**:
   - スマホ用レスポンシブ要件を除外した、BtoBデスクトップ専用の2カラムレイアウトとしてコーディングし直す。
3. **JSロジックの実装**:
   - イベント共通のクーポンデータストア（`localStorage`等）を用いた連携ロジックを実装。
   - `bizcardSettings.js` のリファクタリング（モック非同期通信、バリデーション、Dirtyフラグ管理、ローディング仕様の追加）。
