# 画面設計ドキュメント概要

`docs/画面設計/` は、画面設計・画面仕様の shared 正本ディレクトリです。社内向けの企画・体制・価格・会議資料は shared repo に置きません。

## ディレクトリ構成
- `仕様/`: サービス画面/管理画面の機能要件・画面仕様。
- `画面/`: 画面単位のUI設計メモ。

## 運用ルール
- 画面仕様の新規・改訂は `docs/画面設計/仕様/` を正本として更新する。
- shared repo で必要な仕様判断は `docs/画面設計/仕様/` に明記し、社内向け企画検討資料への依存を持ち込まない。
- shared/private の境界判断は `docs/リファレンス/共有規約/02_SHARED_DOC_BOUNDARY_RULES.md` に従う。
