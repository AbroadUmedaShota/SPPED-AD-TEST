# 請求関連仕様マップ

## 1. 目的

- repo 内の請求関連文書を `正本` `補助` `テンプレート` `private候補` `参照のみ` に分類する単一入口です。
- 請求関連の仕様更新時は、まず本書で正本と境界を確認し、その後に個別文書を更新してください。
- shared/private の境界判断は `docs/リファレンス/共有規約/02_SHARED_DOC_BOUNDARY_RULES.md` を正本とします。

## 2. 正本の使い分け

### 2.1 利用者向け請求書

| 文書 | 分類 | 役割 |
| :--- | :--- | :--- |
| `docs/画面設計/仕様/04_invoice_screen.md` | 正本 | 利用者向け請求の画面横断要件 |
| `docs/画面設計/仕様/05_invoice_document.md` | 正本 | 帳票/PDF/印刷要件 |
| `docs/ハンドブック/テスト/invoice_manual_checklist.md` | 正本 | 手動検証の正本 |
| `docs/画面設計/仕様/04_invoice_detail_requirements.md` | 補助 | INV-002 固有の DOM・画面骨格・操作要件 |
| `docs/画面設計/仕様/請求書関連ページ_仕様共有資料.md` | 補助 | 早見表・リンク集 |

### 2.2 管理者向け請求管理

| 文書 | 分類 | 役割 |
| :--- | :--- | :--- |
| `docs/画面設計/仕様/admin/billing_management_requirements.md` | 正本 | 管理者向け請求管理の画面要件 |

利用者向け請求書（INV-001/002/003）と管理者向け請求管理（A-06）は別責務です。PDF/印刷の表示要件は利用者向け請求の正本を参照し、管理者向け文書では請求管理 UI の責務に限定します。

## 4. プレミアム関連と請求表現の境界

| 文書 | 分類 | 取り扱い |
| :--- | :--- | :--- |
| `docs/画面設計/仕様/11_plan_feature_restrictions.md` | 補助 | プラン機能制限の補助資料。価格/請求ポリシーの正本にはしない |
| `docs/画面設計/仕様/premium/premium_dashboard_requirements.md` | 補助 | 請求書ページへの導線、画面に表示する請求関連文言のみ shared で扱う |
| `docs/画面設計/仕様/premium/premium_registration_requirements.md` | private候補 | 請求ポリシー、契約/法務説明を含むため shared では UI 文言中心に縮退する対象 |
| `docs/画面設計/仕様/premium/premium_page_requirements.md` | private候補 | 価格・キャンペーン表現を含むため shared では UI 要素のみ残す対象 |
| `docs/画面設計/画面/premium_registration_screen.md` | 補助 | 画面文言と導線は shared、価格/契約ルールは private 候補 |
| `docs/画面設計/仕様/premium/00_プレミアム構造資料.md` | private候補 | 企画・価格・運用背景を含むため shared 正本にしない |

shared に残す請求情報は以下に限定します。
- 画面に表示する文言
- 請求書ページへの導線
- 実装直結の表示条件・遷移条件

shared から外す対象:
- 価格政策
- 初月無料の適用ルール
- 請求サイクルの業務判断
- 契約/法務の背景説明
- 社内判断、会議経緯

## 5. Canonical Rules

### 5.1 ID

- URL の `id` 正本は `INV-...` です。`AGG-YYYYMM-{GROUP|PERSONAL}` は内部互換用に残します。
- INV-001 から INV-002 への遷移は原則 `INV-...` を使用します。
- 画面表示用の請求書番号は `YY-ユーザーID5桁-連番3桁` です。

### 5.2 帳票

- `Invoice` / `InvoiceItem` の shared データ契約は利用者向け請求の正本へ集約します。
- `items[].category` と `items[].taxable` は帳票整合上の必須項目です。
- 品目ラベルは具体課金名で固定し、`従量課金` は請求書の表示ラベルに使いません。アンケート依存の行は品目1にアンケート名、品目2に具体課金名を表示します。
- 追加アカウント料金の `description` は `請求対象 Nアカウント` のような請求対象人数のみを表示します。無料アカウント数 `2` 固定と `3アカウント目以降` の課金開始条件は内部ルールとして保持し、請求書表示には出しません。
- 改ページは固定 `20/42` 行ではなく、現行実装準拠の動的改ページを正とします。
- 主導線の請求書は `1請求書 = 1accountId` を正とし、`ご請求明細` は常に単一請求先アカウントの請求として読める状態を維持します。
- `AGG-...` 時の `isSubtotal` 行は内部互換の詳細/PDF でのみ表示対象です。印刷との差分は `05_invoice_document.md` に 1 箇所で定義します。

## 6. 更新時の判断先

1. 利用者向け請求書の見た目/遷移/帳票を変える場合
   - `04_invoice_screen.md`
   - `05_invoice_document.md`
   - 必要に応じて `04_invoice_detail_requirements.md`
2. 管理者向け請求管理を変える場合
   - `admin/billing_management_requirements.md`
3. プレミアム導線内の請求表現を変える場合
   - shared には UI 文言と導線のみ反映
   - 価格/契約/請求ポリシーは private 管理へ切り分け
