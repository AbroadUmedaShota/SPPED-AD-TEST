# 管理者画面 オペレーター・権限・実績要件

## 1. 目的

入力、照合、確認を担当する作業者、権限、実績を管理する責務を定義する。

この領域は、名刺データ化の作業品質と処理量を支える管理機能である。

## 2. 対象外

- 人事、勤怠、契約、評価制度
- 報酬計算の確定仕様
- 本番認可ロールの詳細設計
- 作業者向け専用UIの細部

## 3. 判断根拠

| 種別 | 扱い |
| --- | --- |
| `01_admin_service_structure.md` | 管理対象と責務境界の根拠 |
| `03_admin/`、`BY-*` | 参考: 現行モック。仕様上の正解とは扱わない |
| `data/admin/operators.json`、`data/admin/performance.json` | 参考: モックデータ |

## 4. 管理対象

| 管理対象 | 管理者が見る情報 | 管理者が変更してよい情報 |
| --- | --- | --- |
| オペレーター | 氏名、連絡先、所属、状態、担当可能作業 | 招待、停止、所属、担当可能範囲 |
| 権限 | 管理者、作業者、確認者などの役割 | ロール、利用可能機能、作業範囲 |
| 実績 | 処理件数、エラー率、差戻し数、期間別集計 | 原則は参照中心。補正は要確認 |
| 稼働状態 | 稼働中、停止中、休止、割当可否 | 状態更新、メモ |

## 5. 責務別要件

| 責務 | 要件 |
| --- | --- |
| 参照 | オペレーター、権限、稼働状態、処理実績を確認できる |
| 登録 | オペレーターを招待・登録できる |
| 編集 | 所属、担当可能作業、連絡先、権限、メモを編集できる |
| 状態更新 | 稼働中、停止中、休止、割当不可を更新できる |
| 出力 | 作業者一覧、実績一覧、期間別集計を出力できる |
| 通知 | 招待、割当、重要連絡を送信できる |
| 例外対応 | 誤割当、権限誤り、作業停滞、品質問題を処理できる |

## 6. 画面化する時の要点

- オペレーター管理はアカウント管理と似ているが、作業割当と品質確認を主責務にする。
- 実績画面は分析過多にせず、処理数、差戻し、品質指標、CSV出力を中心にする。
- 権限設定は画面要件と本番認可設計を分離して記録する。

## 7. 画面外確認事項

- 本番ロール、権限、認可の実装方式。
- 実績の集計元、集計タイミング、補正可否。
- 報酬・評価情報をshared資料に含めるか。

## 8. 未確定事項

- オペレーター、確認者、管理者の正式なロール名。
- 実績に報酬額や評価指標を含めるか。
- 作業者専用画面と管理者画面の境界。
