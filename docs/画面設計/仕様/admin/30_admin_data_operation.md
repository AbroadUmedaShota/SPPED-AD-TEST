# 管理者画面 名刺データ化・入力・照合要件

## 1. 目的

名刺データ、入力作業、照合作業、エスカレーションを管理する責務を定義する。

この領域は、回答に紐づく名刺画像や入力結果を納品可能な状態へ整えるための社内運用である。

## 2. 対象外

- OCRエンジン、画像処理、入力補助ロジックの詳細
- 作業者向け専用画面の詳細UI
- 報酬計算、勤怠、契約管理
- 本番ストレージ、ファイル保管期間、監査ログの詳細

## 3. 判断根拠

| 種別 | 扱い |
| --- | --- |
| `01_admin_service_structure.md` | 管理対象と責務境界の根拠 |
| `03_admin/`、`BY-*` | 参考: 現行モック。仕様上の正解とは扱わない |
| `data/admin/data_entry_status.json`、`data/responses/business-cards.json` | 参考: モックデータ |

## 4. 管理対象

| 管理対象 | 管理者が見る情報 | 管理者が変更してよい情報 |
| --- | --- | --- |
| 名刺データ | 画像、回答ID、入力状態、照合状態、納品状態 | 入力内容、状態、再入力/再照合フラグ |
| 入力タスク | 担当者、件数、進捗、期限、エラー | 割当、優先度、状態、差戻し |
| 照合タスク | 一致/不一致、差異内容、ロック状態 | 照合結果、再割当、差戻し |
| エスカレーション | 種別、優先度、対象データ、対応履歴 | 担当、対応結果、クローズ |

## 5. 責務別要件

| 責務 | 要件 |
| --- | --- |
| 参照 | アンケート別、グループ別、作業者別に入力・照合状況を確認できる |
| 登録 | エスカレーション、再作業、手動タスクを登録できる |
| 編集 | 入力結果、照合結果、担当、優先度、メモを補正できる |
| 状態更新 | 未着手、入力中、入力完了、照合中、差戻し、完了、例外を更新できる |
| 出力 | 納品用データ、作業一覧、差異一覧、進捗CSVを出力できる |
| 通知 | 作業者への割当通知、利用者への確認依頼を送信できる |
| 例外対応 | 読取不能、重複、差異、画像不備、顧客確認待ちを処理できる |

## 6. 画面化する時の要点

- 全体進捗、作業キュー、個票作業、例外対応を分ける。
- 一覧は件数と状態把握を優先し、個票編集は詳細画面へ分離する。
- エスカレーションは独立画面にするか、照合詳細内の処理にするかを次工程で決める。

## 7. 画面外確認事項

- 名刺画像と入力結果の保存先。
- ロック、排他制御、同時編集制御。
- 納品データの形式、出力条件、再出力条件。

## 8. 未確定事項

- 作業者専用画面を管理者画面に含めるか。
