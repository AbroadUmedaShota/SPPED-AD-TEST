# 管理者画面 請求・クーポン・営業日要件

## 1. 目的

請求、クーポン、営業日、期限前提を管理する責務を定義する。

この領域は、アンケート案件の料金確定、割引条件、納品期限、対応期限の前提を支える。

## 2. 対象外

- 価格表、契約条件、個別商談条件の確定
- 会計システム連携、決済、入金消込の本番設計
- 法務文面、請求書原本の帳票設計
- 営業日計算API、バッチ処理の詳細

## 3. 判断根拠

| 種別 | 扱い |
| --- | --- |
| `01_admin_service_structure.md` | 管理対象と責務境界の根拠 |
| `03_admin/` | 参考: 現行モック。仕様上の正解とは扱わない |
| `data/core/invoices.json`、`data/admin/coupons.json`、`data/admin/holidays.json` | 参考: モックデータ |

## 4. 管理対象

| 管理対象 | 管理者が見る情報 | 管理者が変更してよい情報 |
| --- | --- | --- |
| 請求 | 請求対象、請求額、請求書番号、状態、入金状態 | 請求作成、編集、状態、入金確認、メモ |
| クーポン | コード、割引率、期限、対象、利用回数、状態 | 作成、編集、停止、再開、一括作成 |
| 営業日 | 休業日、振替営業日、期限影響、更新履歴 | 休業日、振替営業日、解除、メモ |
| 期限前提 | 納品予定、対応期限、DL期限 | 例外補正、期限確認、期限メモ |

## 5. 責務別要件

| 責務 | 要件 |
| --- | --- |
| 参照 | 請求、クーポン、営業日、期限前提を検索・確認できる |
| 登録 | 請求、クーポン、特別休業日、振替営業日を登録できる |
| 編集 | 請求内容、クーポン条件、営業日設定、期限メモを編集できる |
| 状態更新 | 未請求、請求済み、入金待ち、入金済み、停止、期限切れを更新できる |
| 出力 | 請求一覧、請求書、クーポン一覧、営業日設定を出力できる |
| 通知 | 請求連絡、入金リマインド、期限関連通知を送信できる |
| 例外対応 | 請求誤り、クーポン誤適用、営業日設定誤り、期限例外を処理できる |

## 6. 画面化する時の要点

- 請求管理と利用者向け請求書表示は別責務として扱う。
- クーポンと営業日は設定系画面として、日常作業画面から分離する。
- 期限前提はデータ化、納品、請求に影響するため、関連画面から参照できるようにする。

## 7. 画面外確認事項

- 請求金額の計算元、確定タイミング、再計算可否。
- クーポンの適用優先順位、重複適用可否。
- 営業日とSLA/期限計算の本番ロジック。

## 8. 未確定事項

- 請求書単位とアンケート単位の正規関係。
- クーポンを顧客単位、アンケート単位、全体単位のどこへ紐づけるか。
- 営業日例外が既存案件へ自動反映されるか。
