# 管理者画面 アンケート管理要件

## 1. 目的

アンケート、設問、公開状態、回答状況を管理する責務を定義する。

アンケートはSPEED ADの中心案件であり、回答収集、名刺データ化、分析、請求へ接続する。管理者は利用者の作成作業を前提に、運用状態の確認と例外時の補正を担う。

## 2. 対象外

- 利用者向けアンケート作成画面のUI仕様
- 回答者向け回答画面のUI仕様
- 分析グラフの詳細表示仕様
- 公開URL、QRコード、メール配信の本番実装方式

## 3. 判断根拠

| 種別 | 扱い |
| --- | --- |
| `01_admin_service_structure.md` | 管理対象と責務境界の根拠 |
| `03_admin/` | 参考: 現行モック。仕様上の正解とは扱わない |
| `data/core/surveys.json`、`data/responses/` | 参考: モックデータ |

## 4. 管理対象

| 管理対象 | 管理者が見る情報 | 管理者が変更してよい情報 |
| --- | --- | --- |
| アンケート | 名称、所有者、会期、公開状態、回答数、関連設定 | 状態補正、内部メモ、運用上必要な基本情報 |
| 設問・フォーム | 設問種別、順序、公開状態、回答済み有無 | 原則は参照中心。例外補正は要確認 |
| 回答 | 回答件数、回答内容、回答日時、回答者情報相当 | 原則は参照中心。削除・修正は例外扱い |
| 公開・完了導線 | 公開URL、回答完了、サンクス画面、お礼メール設定との接続 | 状態確認、無効化、例外対応 |

## 5. 責務別要件

| 責務 | 要件 |
| --- | --- |
| 参照 | アンケート一覧、詳細、会期、公開状態、回答数、関連データ化状態を確認できる |
| 登録 | 管理者が運用上必要なアンケート案件を登録できるかは未確定として分ける |
| 編集 | 基本情報、会期、内部メモ、運用補正項目を編集できる |
| 状態更新 | 下書き、公開中、停止、終了、データ化対象などの状態を更新できる |
| 出力 | 回答データ、アンケート一覧、運用確認用CSVを出力できる |
| 通知 | 利用者への確認依頼、公開状態連絡、回答関連連絡を送信できる |
| 例外対応 | 公開誤り、回答不整合、フォーム変更相談、回答削除相談を処理できる |

## 6. 画面化する時の要点

- 一覧はアンケート案件の検索、状態確認、主要KPIに絞る。
- 詳細は基本情報、フォーム、回答、名刺データ化、履歴/メモに分ける。
- 公開後の設問変更は危険操作として扱い、通常編集とは分離する。

## 7. 画面外確認事項

- 公開URL、回答保存、回答削除の本番仕様。
- 回答済みアンケートの設問変更可否。
- 管理者操作時の監査ログ、通知履歴。

## 8. 未確定事項

- 管理者によるアンケート新規作成を許可するか。
- 公開後フォーム変更を不可、版管理、例外補正のどれにするか。
- 回答データの修正・削除権限をどこに置くか。
