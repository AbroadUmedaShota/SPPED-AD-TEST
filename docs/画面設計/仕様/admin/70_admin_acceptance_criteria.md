# 管理者画面 開発着手条件・受け入れ観点

## 1. 目的

管理者画面要件定義が開発着手可能な状態かを判断するための条件と受け入れ観点を定義する。

この資料は、モック実装または詳細設計に入る前のチェックリストとして使う。

## 2. 対象外

- テストケースの詳細手順
- APIレスポンス、DB schema、認可実装の確定
- HTML/CSS/JSの具体実装
- 本番運用手順、監視、バックアップ

## 3. 判断根拠

| 種別 | 扱い |
| --- | --- |
| `01_admin_service_structure.md` | 全体責務の根拠 |
| `10_`〜`50_` の責務別要件 | 受け入れ観点の根拠 |
| `60_admin_screen_structure.md` | 画面構成の根拠 |
| `03_admin/` | 参考: 現行モック。仕様上の正解とは扱わない |

## 4. 開発着手条件

| 条件 | 判定 |
| --- | --- |
| 管理対象と責務が定義されている | `01_` と `10_`〜`50_` が揃っている |
| 画面構成が定義されている | `60_` で必要画面と所有責務が分かる |
| 主要操作が定義されている | 参照、登録、編集、状態更新、出力、通知、例外対応が整理されている |
| 未確定事項が分離されている | 各資料の `未確定事項` と本資料の `画面外確認事項` に分かれている |
| 参考情報が分離されている | `90_` に隔離され、本文要件に混ざっていない |

## 5. 画面別受け入れ観点

| 画面群 | 主要項目 | 主要操作 | 状態・エラー/空状態 |
| --- | --- | --- | --- |
| 管理者ホーム | 要対応件数、主要状態、導線 | 状態確認、各画面遷移 | 要対応なし、データ取得失敗 |
| 顧客・ユーザー | 顧客、ユーザー、グループ、請求先 | 検索、登録、編集、停止、通知 | 招待中、停止中、メール不達 |
| アンケート | 基本情報、公開状態、回答数、関連設定 | 検索、詳細確認、状態更新、出力 | 下書き、公開中、終了、回答なし |
| データ化 | 名刺データ、入力状態、照合状態、例外 | 割当、入力、照合、差戻し、出力 | 未着手、作業中、差異あり、読取不能 |
| 請求・設定 | 請求、クーポン、営業日、期限 | 作成、編集、状態更新、出力、通知 | 未請求、入金待ち、期限切れ、設定なし |
| 作業者・実績 | 作業者、権限、稼働状態、実績 | 招待、編集、停止、集計出力 | 休止中、割当不可、実績なし |

## 6. 画面外確認事項

| 領域 | 確認事項 |
| --- | --- |
| API | 保存、検索、出力、通知、状態更新のエンドポイント |
| DB | 管理対象の正規データ構造、履歴、削除/停止表現 |
| 認可 | 管理者、作業者、確認者、利用者の操作範囲 |
| 監査ログ | 代理編集、状態更新、削除、出力、通知の記録 |
| 通知 | メール送信、再送、送信履歴、不達時の扱い |
| 出力 | CSV、請求書、納品データの形式と再出力条件 |

## 7. 完了判定

- `00_` ルール、`01_` 全体構造、`10_`〜`50_` 責務別要件、`60_` 画面構成、`70_` 本資料が揃っている。
- 各資料にfront-matterと必須4セクションがある。
- 管理者全体、顧客/ユーザー、アンケート、データ化、請求/設定、作業者/実績が対象化されている。
- 未確定事項が本文要件と分離されている。
