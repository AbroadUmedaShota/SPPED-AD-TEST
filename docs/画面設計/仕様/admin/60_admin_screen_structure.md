# 管理者画面 画面構成要件

## 1. 目的

責務別要件をもとに、開発着手可能な画面構成を定義する。

この資料は画面構成の正本であり、既存AD画面や既存HTML構成を前提にしない。必要な画面種別、一覧/詳細/設定/作業画面の分離、モーダル所有元を整理する。

## 2. 対象外

- 画面デザイン、レイアウト、HTML/CSS/JS実装
- 既存 `03_admin/` の画面名・画面数の追認
- API、DB、権限、監査ログの詳細設計
- 作業者向け専用画面の実装詳細

## 3. 判断根拠

| 種別 | 扱い |
| --- | --- |
| `01_admin_service_structure.md` | 管理対象と責務境界の根拠 |
| `10_`〜`50_` の責務別要件 | 画面構成の根拠 |
| `03_admin/` | 参考: 現行モック。仕様上の正解とは扱わない |

## 4. 画面構成方針

| 種別 | 方針 |
| --- | --- |
| 一覧 | 検索、絞り込み、状態確認、主要操作の入口に限定する |
| 詳細 | 基本情報、関連情報、操作、履歴/メモを確認する |
| 設定 | クーポン、営業日、権限など、日常作業と分ける |
| 作業 | 入力、照合、例外対応など、個票処理に集中する |
| モーダル | 補助操作に限定し、主要責務は独立画面または詳細画面に置く |

## 5. 必要画面案

| 画面群 | 画面 | 主な責務 |
| --- | --- | --- |
| 全体 | 管理者ホーム | 主要状態、要対応件数、各責務への入口 |
| 顧客・ユーザー | 顧客一覧、顧客詳細、ユーザー詳細、グループ詳細 | 顧客/ユーザー/グループの参照、登録、編集、状態更新 |
| アンケート | アンケート一覧、アンケート詳細、回答確認 | アンケート、設問、回答、公開状態の確認と補正 |
| データ化 | データ化進捗、入力タスク一覧、入力個票、照合タスク一覧、照合個票、エスカレーション | 入力、照合、例外対応、納品状態確認 |
| 請求・設定 | 請求一覧、請求詳細、クーポン設定、営業日設定 | 請求、割引、期限前提の管理 |
| 作業者 | オペレーター一覧、オペレーター詳細、実績確認 | 作業者、権限、稼働状態、実績確認 |

## 6. モーダル所有方針

| モーダル種別 | 所有元 | 方針 |
| --- | --- | --- |
| 確認 | 各画面共通 | 汎用確認のみ。業務ロジックは持たせない |
| メール送信 | 顧客詳細、アンケート詳細、請求詳細 | 送信対象と履歴が分かる画面から起動する |
| クーポン作成/編集 | クーポン設定 | 設定画面の補助操作として扱う |
| グループ編集 | グループ詳細 | 利用者側グループ管理との責務差分を明記する |

## 7. 画面外確認事項

- 画面遷移、URL、ルーティング、認可の実装方式。
- API、DB、保存先、監査ログ、通知履歴。
- モバイル対応、アクセシビリティ、表示性能。

## 8. 未確定事項

- エスカレーションを独立画面にするか、入力/照合詳細内のタブにするか。
- 回答確認をアンケート詳細内に含めるか、独立画面にするか。
- 作業者専用画面を管理者画面群に含めるか、別ポータル扱いにするか。
