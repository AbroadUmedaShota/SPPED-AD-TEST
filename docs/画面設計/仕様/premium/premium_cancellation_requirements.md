# プレミアムプラン解約画面 要件定義書

## 1. 概要
本ドキュメントは、プレミアムプラン利用者が契約を解除するための「解約画面」および「解約完了画面」の機能要件、UX要件を定義する。

## 2. 解約フローの変更点（重要）
*   **契約満了解約（自動更新の停止）:** (2026-04-28 改定)
    *   解約手続き完了後も、**現在の契約期間満了日（通常は当月末日）まで**はプレミアムプランの全機能を引き続き利用可能とする。
    *   「即時解約」ではなく「次回更新の停止」として扱う。
    *   期間満了日の翌日に Standard へ移行する仕様とする。
    *   解約画面ではユーザーに対し「期間終了まで利用可能であること」を明確に伝え、白基調の1カードで落ち着いて案内する。

## 3. 画面仕様：解約手続き画面 (`premium_cancel.html`)

### 3.1. 画面構成
*   **ヘッダー:** シンプルなタイトル「プレミアムプラン解約手続き」
*   **契約状況エリア:**
    *   「現在のプラン有効期限（＝当月末日）」を表示。
    *   「解約手続き（自動更新の停止）を行っても、上記日付までは引き続きプレミアム機能をご利用いただけます」というメッセージを表示する。
*   **期間終了後の差分案内:**
    *   契約終了後に Standard へ移行すると変わる内容を、状態カード内の情報行で簡潔に示す。
    *   失う機能の列挙ではなく、保存期間や出力、入力補助などの差分を落ち着いた表現で案内する。
    *   近日対応機能は期間終了後の変更点として強調せず、順次提供予定の機能名のみを列挙する。
    *   優先サポート、運用相談は現状想定しないため表示しない。
    *   問い合わせフォームとサポートセンターへの導線は、状態カード内の補助リンクとして表示する。
*   **解約理由アンケート（任意）:**
    *   ユーザーの声を拾うためのアンケートを設置するが、**回答は任意**とする（解約の障壁にしない）。
    *   選択肢（ラジオボタン）：
        *   価格が高い
        *   機能が使いにくい
        *   利用頻度が減った
        *   他社サービスへ移行
        *   その他（テキスト入力表示）
*   **アクションボタン:**
    *   **「自動更新を停止する」ボタン:** 押下時に最終確認モーダルを表示。
    *   **「キャンセル（戻る）」ボタン:** ダッシュボードまたは前の画面へ戻る。

### 3.2. 最終確認モーダル
*   誤操作防止のため、ボタン押下後にモーダルを表示する。
*   「次回の更新は行われません」「現在の期間満了まで利用可能です」といった確認を行う。
*   「自動更新を停止する」ボタン押下で解約予約APIをコールし、完了画面へ遷移する。

## 4. 画面仕様：解約完了画面 (`premium_cancel_complete.html`)

### 4.1. 画面構成
*   **感謝メッセージ:** 「これまでのご利用、ありがとうございました」を主見出しとして表示する。
*   **完了メッセージ:** 「自動更新の停止手続きが完了しました」「XXXX年XX月XX日までは引き続きプレミアム機能をご利用いただけます」を同じ状態カード内で明確に表示。
*   **再開案内:** 利用への感謝を述べ、いつでも再開可能であることを伝える。
*   **再開導線:** 再開は `premium_signup_new.html` から行える前提で、必要最小限の案内を添える。

## 5. データ処理要件
*   **プラン変更:** ユーザーの `autoRenewal` フラグを `false` に更新（即時ダウングレードはしない）。
*   **データ保持:** 
    *   契約満了日までは変更なし。
    *   満了日の翌日にバッチ処理等で Standard へ移行し、保存期間（30日）等の制限を適用する。
*   **請求:**
    *   当月分の請求は確定済み。次回の請求は発生しない。
