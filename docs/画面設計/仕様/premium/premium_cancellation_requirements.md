# プレミアムプラン解約画面 要件定義書

## 1. 概要
本ドキュメントは、プレミアムプラン利用者が契約を解除するための「解約画面」および「解約完了画面」の機能要件、UX要件を定義する。

## 2. 解約フローの変更点（重要）
*   **契約満了解約（自動更新の停止）:** (2026-02-13 改定)
    *   解約手続き完了後も、**現在の契約期間満了日（通常は当月末日）まで**はプレミアムプランの全機能を引き続き利用可能とする。
    *   「即時解約」ではなく「次回更新の停止」として扱う。
    *   期間満了日の翌日にフリープラン（Standard）へダウングレードされる仕様とする。
    *   解約画面ではユーザーに対し「期間終了まで利用可能であること」を明確に伝え、安心感を与える。

## 3. 画面仕様：解約手続き画面 (`premium_cancel.html`)

### 3.1. 画面構成
*   **ヘッダー:** シンプルなタイトル「プレミアムプラン解約手続き」
*   **契約状況エリア:**
    *   「現在のプラン有効期限（＝当月末日）」を表示。
    *   「解約手続き（自動更新の停止）を行っても、上記日付までは引き続きプレミアム機能をご利用いただけます」というメッセージを表示し、早めの手続きを促す。
*   **機能制限の予告:**
    *   契約終了後に以下の6機能が利用不可になることをアイコン付きで明示する。
        1.  データ無期限保存
        2.  Excelレポート出力
        3.  SPEEDレビュー
        4.  高度な入力対応
        5.  外部システム連携
        6.  優先サポート
*   **解約理由アンケート（任意）:**
    *   ユーザーの声を拾うためのアンケートを設置するが、**回答は任意**とする（解約の障壁にしない）。
    *   選択肢（ラジオボタン）：
        *   価格が高い
        *   機能が使いにくい
        *   利用頻度が減った
        *   他社サービスへ移行
        *   その他（テキスト入力表示）
*   **アクションボタン:**
    *   **「自動更新を停止する」ボタン:** 押下時に最終確認モーダルを表示。
    *   **「キャンセル（戻る）」ボタン:** ダッシュボードまたは前の画面へ戻る。

### 3.2. 最終確認モーダル
*   誤操作防止のため、ボタン押下後にモーダルを表示する。
*   「次回の更新は行われません」「現在の期間満了まで利用可能です」といった確認を行う。
*   「自動更新を停止する」ボタン押下で解約予約APIをコールし、完了画面へ遷移する。

## 4. 画面仕様：解約完了画面 (`premium_cancel_complete.html`)

### 4.1. 画面構成
*   **完了メッセージ:** 「自動更新を停止しました」「XXXX年XX月XX日までは引き続きプレミアム機能をご利用いただけます」を明確に表示。
*   **感謝の言葉:** 利用への感謝を述べ、いつでも再開可能であることを伝える。

## 5. データ処理要件
*   **プラン変更:** ユーザーの `autoRenewal` フラグを `false` に更新（即時ダウングレードはしない）。
*   **データ保持:** 
    *   契約満了日までは変更なし。
    *   満了日の翌日にバッチ処理等でフリープランへ移行し、保存期間（30日）等の制限を適用する。
*   **請求:**
    *   当月分の請求は確定済み。次回の請求は発生しない。
