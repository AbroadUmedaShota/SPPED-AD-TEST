# 公開「お客様のお声」ページ仕様

## 1. 概要

本書は、ログイン前トップ (`index.html`) から遷移する公開導線として追加する「お客様のお声」ページ群の表示仕様・導線・データ管理方針を定義する。

対象ページは以下の 3 画面とする。

- `customer-voices/index.html`: お客様のお声一覧ページ
- `customer-voices/company-monitor.html`: 先行利用モニター企業の詳細ページ
- `customer-voices/university-survey.html`: アンケート機能を利用した大学の詳細ページ

## 2. 目的

- ログイン前トップで不足している信頼補強を行う
- `無料ではじめる` 導線へ自然に復帰できる公開導線を用意する
- 「フル活用」と「アンケート単機能活用」の両方を見せ、利用イメージの幅を伝える

## 3. 導線

### 3.1. ログイン前トップからの入口

`index.html` には以下の 3 導線を配置する。

1. ファーストビュー内補助 CTA: `お客様のお声を見る`
2. 機能紹介後のティザーセクション: 2 事例の要約カード + `詳細を見る`
3. フッター常設リンク: `お客様のお声`

いずれの導線も、遷移先は `customer-voices/index.html` とする。

### 3.2. ページ間導線

- 一覧ページから各詳細ページへ遷移できること
- 各詳細ページ下部に `他の事例を見る` を配置し、一覧へ戻れること
- 一覧ページおよび詳細ページの主 CTA は `無料ではじめる` とし、遷移先は `../index.html?intent=signup#top` とする
- 副 CTA は `ログインはこちら` とし、遷移先は `../index.html#top` とする

### 3.3. ログイン前トップ復帰時の挙動

- `index.html` は `intent=signup` を受け取った場合、既存の新規アカウント作成モーダルを自動表示する
- モーダル表示後は URL から `intent` を取り除き、再読込時の意図しない再表示を避ける

## 4. 画面仕様

### 4.1. 一覧ページ

- 上部にヒーロー領域を表示し、「公開事例の入口」であることを明示する
- 中央に公開中の事例一覧を表示する
- 各事例カードには以下を表示する
  - 区分ラベル (`organizationType`)
  - 事例名 (`label`)
  - 要約 (`listingSummary`)
  - 利用機能 (`usedFeatures`)
- 下部に `無料ではじめる` / `ログインはこちら` を再掲する

### 4.2. 詳細ページ

詳細ページは同一テンプレートで統一し、以下の順で表示する。

1. 概要
2. 導入前の課題
3. 利用した機能
4. 運用イメージ
5. 導入後の変化
6. 担当者コメント
7. 次の行動

補足:

- 企業事例は「先行導入で得た価値」を主軸にする
- 大学事例は「アンケート単機能でも成立する手軽さ」を主軸にする
- 事例本文は匿名・要約表現を基本とし、実名・ロゴ・具体数値は承認済みの場合のみ掲載する

## 5. データ仕様

### 5.1. データソース

- 事例本文の正本は `data/customer-voices.json` とする
- 一覧ページ、詳細ページ、ログイン前トップのティザーは同一 JSON を参照する

### 5.2. 最低限の項目

各事例には最低限以下を持たせる。

- `slug`
- `label`
- `organizationType`
- `listingSummary`
- `usedFeatures`
- `challenge`
- `outcome`
- `quote`
- `ctaLabel`
- `publishStatus`

### 5.3. 任意項目

以下は任意項目として扱う。

- `overview`
- `operationImage`
- `heroImage`
- `accent`
- `accentStrong`
- `quote.role`

任意項目が未入力の場合、対応する表示ブロックは非表示とし、空セクションを出さない。

## 6. 実装メモ

- 画面追加先はルート配下の `customer-voices/`
- スタイルは `customer-voices/styles.css`
- 共通ロジックは `customer-voices/shared.js`
- 一覧描画は `customer-voices/index.js`
- 詳細描画は `customer-voices/detail.js`

## 7. 確認項目

- `index.html` の 3 導線から一覧ページへ到達できること
- 一覧ページの各カードから正しい詳細ページへ遷移できること
- `../index.html?intent=signup#top` でサインアップモーダルが自動表示されること
- `data/customer-voices.json` の任意項目が欠けてもレイアウト崩れや空セクションがないこと
- デスクトップ / タブレット / モバイルで表示崩れがないこと
- コンソールエラー、リンク切れ、画像切れがないこと
