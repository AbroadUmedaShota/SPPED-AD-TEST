# ログイン前トップ 仕様

## 1. 概要

本書は、ルート `index.html` に配置するログイン前トップの表示要件を定義する。対象は、展示会向けサービスの価値訴求、既存ユーザーログイン、無料利用開始、NEWS、公開事例、関連サービスへの遷移を両立する刷新済みトップである。

## 2. 目的

- 初回訪問時の迷いを減らし、`無料ではじめる` を最優先導線として提示する
- 既存ユーザーがファーストビューからログインできる状態を維持する
- 旧ログイン前ページの `お知らせ` 読み込みロジックを NEWS として継承する
- `お客様のお声` で信頼補強し、公開事例ページへ自然に接続する
- 旧ログイン前ページの `当社で提供するその他のサービス` を関連サービスとして掲載する

## 3. 主要要件

### 3.1. ファーストビュー

- ファーストビュー内に `無料ではじめる` を配置する
- 同じファーストビュー内に `お客様のお声を見る` を配置する
- ファーストビュー右側に既存ユーザー向けログインフォームを配置する
- ログインフォームはメールアドレス、パスワード、パスワード表示切替、規約同意、アカウント記憶、パスワード再設定、Googleログイン、テスト用アカウントを含む

### 3.2. コンテンツ構成

- NEWS はヒーロー直後に表示する
- FEATURES は `WEBアンケート作成`、`名刺情報のデータ化`、`御礼メールの送信` の3件を表示する
- FLOW は `準備`、`QR回答`、`名刺撮影`、`追客` の4ステップを表示する
- REASONS は `選ばれる理由` として6項目を表示する
- REASONS はステータスバッジを出さず、現在の提供・整備状況を説明文で表現する
- 機能紹介の後段に、公開事例の要約ティザーを配置する
- RELATED SERVICES は旧ログイン前ページの6件を表示する
- フッターに `お客様のお声` を常設する

### 3.3. 遷移先

- `無料ではじめる`: `?intent=signup#top`
- `お客様のお声を見る` およびティザーの詳細リンク: `customer-voices/index.html`
- ログイン成功モック: `user` は `02_dashboard/index.html`、`new` は `04_first-login/index.html`、`admin` は `03_admin/index.html`
- フッターの `お客様のお声`: `customer-voices/index.html`

### 3.4. NEWS 表示

- NEWS は `data/news.json` を参照する
- `items` のうち `title` を持つ項目を最大3件表示する
- `displayDate`、`date`、`publishedAt` の順に表示日として利用する
- `url` が http/https でない場合は `data-news-detail-url` を利用する
- 取得失敗または表示対象0件の場合は NEWS セクションを非表示にする

### 3.5. サインアップ挙動

- `intent=signup` を受け取った場合、既存の新規アカウント作成モーダルを自動表示する
- モーダル表示後は URL から `intent` を除去する
- モーダルは Escape、背景クリック、閉じるボタンで閉じられる
- モーダルを閉じた後は開いた要素へフォーカスを戻す

### 3.6. 公開事例ティザー

- ティザーは `data/customer-voices.json` を参照する
- 表示文言は `label` / `listingSummary` を基本とする
- `customer-voices/` 配下で使う `voicePageLabel` / `voicePageSummary` は詳細ページ専用とし、トップでは汎用表現を維持する

## 4. 受入条件

- ファーストビューの `無料ではじめる` からモーダルが開く
- ファーストビューの `お客様のお声を見る` から一覧へ遷移する
- `data/news.json` の上位3件が NEWS に表示される
- NEWS 取得失敗時にセクションが非表示になり、画面全体が崩れない
- `user`、`new`、`admin` のテストアカウントを選択するとメールアドレス欄へ反映され、既存の遷移先へ移動する
- `アカウント記憶` を有効にしたログインではメールアドレス欄の再入力を省ける
- 規約未同意、未入力、短いパスワードでエラーが表示される
- パスワード表示切替を操作しても入力値とログイン検証が維持される
- `選ばれる理由` が6項目表示され、ステータスバッジなしで説明文が表示される
- キーボードだけでログイン、CTA、モーダル開閉、フォーカス復帰ができる
- フッターの `お客様のお声` から一覧へ遷移する
- `intent=signup` 付き URL で再読込してもモーダルの再表示が暴れない
- トップのティザーが欠けてもレイアウト崩れや空ブロックが出ない
- 2560 / 1440 / 1280 / 1024 / 768 / 640 / 390px でテキストやUIが重ならない
- 1440pxを超える幅では、NEWS、FEATURES、FLOW、REASONS、VOICE、RELATED SERVICES の主コンテンツが中央最大幅に収まり、横方向に間延びしない
