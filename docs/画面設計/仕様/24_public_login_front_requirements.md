# ログイン前トップ 仕様

## 1. 概要

本書は、ルート `index.html` に配置するログイン前トップの表示要件を定義する。対象は、展示会向けサービスの価値訴求、既存ユーザーログイン、無料利用開始、お知らせ、導入事例、関連サービスへの遷移を両立する刷新済みトップである。

## 2. 目的

- 初回訪問時の迷いを減らし、`無料ではじめる` を最優先導線として提示する
- 既存ユーザーがファーストビューからログインできる状態を維持する
- 旧ログイン前ページの `お知らせ` 読み込みロジックを お知らせとして継承する
- `導入事例` で信頼補強し、導入事例ページへ自然に接続する
- 旧ログイン前ページの `当社で提供するその他のサービス` を関連サービスとして掲載する

## 3. 主要要件

### 3.1. ファーストビュー

- ヒーロー見出しは `スピードをアドバンテージに。` とし、36px・明朝系フォントで表示する
- ヒーローリード文は `展示会やイベントのリード獲得フォローを最適化する、` / `次世代WEBアンケート作成ツール。` の2行構成で、16px・現行本文フォントで表示する
- ファーストビュー内に `無料ではじめる` を配置する
- ファーストビュー右側に既存ユーザー向けログインフォームを配置する
- ログインフォームはメールアドレス、パスワード、パスワード表示切替、アカウント記憶、パスワード再設定、ログインボタン直下のGoogleログイン、Googleログイン下の新規アカウント作成ボタン、テスト用アカウントを含む
- 規約同意は新規アカウント作成モーダルでのみ求める

### 3.2. コンテンツ構成

- お知らせはヒーロー直後に表示する
- FEATURES は `WEBアンケート作成`、`名刺情報のデータ化`、`御礼メールの送信` の3件を表示する
- FLOW は `準備`、`QR回答`、`名刺撮影`、`追客` の4ステップを表示する
- REASONS は `選ばれる理由` として9項目を表示する
- REASONS の9項目は `無料プランからすぐ始められる`、`QRアンケートを手軽に作れる`、`チュートリアルで迷わず使える`、`業界最安値でコストダウンが期待できる`、`最短当日で追客までつなげられる`、`回答状況をタイムリーに確認できる`、`日本語以外の言語にも対応できる`、`多機能なアンケートを作れる`、`AIと人の検品でデータ化を支える` とする
- REASONS はステータスバッジを出さず、現在の提供・整備状況を説明文で表現する
- REASONS の `業界最安値でコストダウンが期待できる` には料金プランページへの導線を表示する
- REASONS の `チュートリアルで迷わず使える` にはチュートリアルページへの導線を表示する
- 機能紹介の後段に、導入事例の要約ティザーを配置する
- RELATED SERVICES は旧ログイン前ページの6件を表示する
- フッターに `導入事例` を常設する

### 3.3. 遷移先

- `無料ではじめる`: `?intent=signup#top`
- 導入事例ティザーの詳細リンク: `https://support.speed-ad.com/customer-voices/`
- 料金プランリンク: `05_support/plans/index.html`
- チュートリアルリンク: `05_support/tutorial/index.html`
- ログイン成功モック: `user` は `02_dashboard/index.html`、`new` は `04_first-login/index.html`、`admin` は `03_admin/index.html`
- フッターの `導入事例`: `https://support.speed-ad.com/customer-voices/`

### 3.4. お知らせ表示

- お知らせは `data/news.json` を参照する
- `items` のうち `title` を持つ項目を最大3件表示する
- `displayDate`、`date`、`publishedAt` の順に表示日として利用する
- `url` が http/https でない場合は `data-news-detail-url` を利用する
- 取得失敗または表示対象0件の場合は お知らせセクションを非表示にする

### 3.5. サインアップ挙動

- `intent=signup` を受け取った場合、既存の新規アカウント作成モーダルを自動表示する
- モーダル表示後は URL から `intent` を除去する
- モーダルは Escape、背景クリック、閉じるボタンで閉じられる
- モーダルを閉じた後は開いた要素へフォーカスを戻す

### 3.6. 導入事例ティザー

- ティザーは `data/customer-voices.json` を参照する
- 表示文言は `label` / `listingSummary` を基本とする
- `https://support.speed-ad.com/customer-voices/` 配下で使う `voicePageLabel` / `voicePageSummary` は詳細ページ専用とし、トップでは汎用表現を維持する

### 3.7. 制作補助エディタ

- `tools/login-front-reasons-editor.html` は、`選ばれる理由` の文言微調整用の制作補助画面である
- 制作補助エディタは `index.html` を複製プレビューとして読み込み、REASONS の9項目だけを画面上で直接編集できる
- 編集対象は見出し、説明文、リンク表示文言に限定し、項目数、表示順、アイコン、リンクURLは変更しない
- 編集内容はブラウザの `localStorage` に下書き保存し、本番HTMLファイルへ直接書き込まない
- 編集結果は `JSONをコピー` から差分JSONとして出力し、Codexによる確認・反映作業で本体へ適用する

## 4. 受入条件

- ファーストビューの `無料ではじめる` からモーダルが開く
- `data/news.json` の上位3件が お知らせに表示される
- お知らせ取得失敗時にセクションが非表示になり、画面全体が崩れない
- `user`、`new`、`admin` のテストアカウントを選択するとメールアドレス欄へ反映され、既存の遷移先へ移動する
- `アカウント記憶` を有効にしたログインではメールアドレス欄の再入力を省ける
- 新規アカウント作成時の規約未同意、未入力、短いパスワードでエラーが表示される
- パスワード表示切替を操作しても入力値とログイン検証が維持される
- `選ばれる理由` が9項目表示され、ステータスバッジなしで説明文が表示される
- `業界最安値でコストダウンが期待できる` の `料金プランを見る` から料金プラン比較ページへ遷移できる
- `チュートリアルで迷わず使える` の `チュートリアルを見る` からチュートリアルページへ遷移できる
- キーボードだけでログイン、CTA、モーダル開閉、フォーカス復帰ができる
- フッターの `導入事例` から一覧へ遷移する
- PC幅のフルページホイールナビでは、ページ最下部から上方向に1回スクロールした時に `導入事例` セクションで停止する
- `intent=signup` 付き URL で再読込してもモーダルの再表示が暴れない
- トップのティザーが欠けてもレイアウト崩れや空ブロックが出ない
- 2560 / 1440 / 1280 / 1024 / 768 / 640 / 390px でテキストやUIが重ならない
- 1440pxを超える幅では、お知らせ、FEATURES、FLOW、REASONS、VOICE、RELATED SERVICES の主コンテンツが中央最大幅に収まり、横方向に間延びしない
- `tools/login-front-reasons-editor.html` で `選ばれる理由` の見出し、説明文、リンク表示文言を複製画面上で直接編集できる
- 制作補助エディタで保存した下書きは再読込後も復元され、リセットで現在の `index.html` の文言に戻せる
- 制作補助エディタの差分JSONでは変更済み項目が `changed: true` で出力される
- 本番 `index.html` を開いた場合、制作補助エディタのツールバーや編集UIは表示されない
