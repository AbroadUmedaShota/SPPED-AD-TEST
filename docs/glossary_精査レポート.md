# glossary.csv 精査レポート ＋ サービス全体 言語統一の方針

作成日: 2026-05-20
対象: `docs/glossary.csv`（T-001〜T-350、350行）
ご依頼: 「両方の画面で言語を統一したいので、サービス全体で使用する語句をリストアップしたい。洗い出した情報を精査し、用語が意図と合っているかチェックしてほしい」

---

## 0. 結論（先に要点）

**現状の `glossary.csv` は、ご依頼の意図（サービス全体の言語統一用語集）とは合致していません。**

- 現状CSVの実体は「**admin側3画面の用語インベントリ**」です。サービス全体（ユーザー向け約35画面＋admin向け十数画面）に対し、カバー率は約4%です。
- 「両方の画面で統一」というご要望（＝ユーザー側とadmin側の両方）に対し、**ユーザー向け画面の用語が1つも入っていません**。
- 構造も「1出現＝1行」のインベントリ型で、言語統一に必要な「1概念＝1行・別表記を集約・統一語を決める」形になっていません。

→ 土台としては作り直しが必要です。本レポートと同時に、意図に沿った **統一用語集のたたき台 `docs/glossary_統一用語集_draft.csv`** を新規作成しました。

## 2. 精査結果 — 現状CSVの7つの問題点

### 2.1 カバー範囲が不足（最重要）

現状CSVが出典としているのは、実質 **admin画面3枚＋モーダル1枚** のみです。

| 出典 | 行範囲 |
|---|---|
| ユーザー管理（AD-02 仕様書＋ user-management.html / user-detail.html） | T-001〜T-170 |
| オペレーター管理（AD-09 仕様書＋ operator-management.html / groupManagementModal.html） | T-171〜T-278 |
| 実績管理（performance-management.html） | T-279〜T-350 |

**未カバー（ご要望の「サービス全体」に対する欠落）:**

- **ユーザー向け画面 = 収録ゼロ**。`02_dashboard/` の約35画面（ダッシュボード／アンケート作成・回答／グラフ／名刺設定／お礼メール・サンクス画面／請求書一覧・詳細・印刷／プレミアム加入・解約／FAQ・ヘルプ／不具合報告／パスワード再設定・変更／法的ページ 等）と、対応する `docs/リライト版仕様書/user/` 配下31本が一切参照されていません。
- **admin向け画面 = 3画面のみ**。請求管理・クーポン管理・営業日カレンダー管理・エスカレーション・アンケート管理／詳細・データ入力／照合 などが未カバーです。

### 2.2 構造が言語統一に不向き

現状は「（用語×出現箇所）の1出現＝1行」のインベントリ構造です。同じ概念が出現ごとにバラバラの行へ分散し、**統一語を決める欄も、別表記を集約する欄もありません**。

例：「パスワード再設定」という1概念が4行に散在 — T-047「パスワードリセットメール送信」／T-119「初期化メール再送」／T-144「パスワード初期化リンクを送信」／T-185「パスワード初期化メール」。これらが同義だと分かる列がありません。

### 2.3 出典参照が陳腐化（重大）

CSVが出典に記載する仕様書2本が **リポジトリに実在しません**。

| CSV記載の出典 | 実在 | 実際の正本 |
|---|---|---|
| `27_admin_user_management_requirements.md`（T-001〜T-106 で参照） | なし | `docs/画面設計/仕様/admin/10_admin_account_management.md` |
| `28_admin_operator_management_requirements.md`（T-171〜T-246 で参照） | なし | `docs/画面設計/仕様/admin/50_admin_operator_management.md` |

「AD-02」「AD-09」という画面ID体系、`§2.1` 等の章番号もこの存在しないファイルに基づくもので、**全体の52%（182行）の出典参照が検証不能**です。CSVは廃止された旧仕様体系に基づいて作られた可能性が高いです。

### 2.4 粒度が混在

「用語」列に、統一対象として粒度の異なる3種が無分類で同居しています。

- (A) 業務概念（本来の統一対象）：契約プラン、招待メール 等
- (B) 画面固有のUI部品名（約30行）：ソートボタン(T-265)、選択チェックボックス(T-250)、ツールボックス(T-257)、表示件数セレクト(T-266)、グループ管理ボタン(T-247)、グループ名入力欄(T-270) 等 — これらは「言語統一」ではなく個別画面のUIインベントリです。
- (C) enum値（状態の選択肢）：有効／一時停止／解約、招待中／登録済み／期限切れ／取消済み 等 — 1値1行では細かすぎ、「状態セット」単位でまとめるべきです。

### 2.5 重複

完全同一表記の用語が別IDで重複登録されています。例：

- 「新規ユーザー招待」… T-052 と T-107
- 「CSVエクスポート」… T-030 と T-109
- 「再開」… T-042 と T-116
- 「ログイン履歴」… T-076 と T-209（しかもカテゴリも不一致）

### 2.6 カテゴリ列が分類軸として機能していない

同種概念が異なるカテゴリに割り振られています。例：

- 「ログイン履歴」T-076=データマスタ／T-209=業務概念
- 「管理者操作ログ」T-044=データマスタ／「操作ログ」T-208=業務概念
- ロール概念が「ロール権限」(T-005,T-134) と「データマスタ」(T-126 権限ロール) に分裂

### 2.7 表記ゆれの検出が部分的・属人的

備考列に「ワードチョイス違い」「同義」等の表記ゆれメモがありますが、記録があるのは主に mockup 行（T-107以降）のみで、仕様書行どうし・仕様書とmockup横断のゆれはほぼ未記録です。検出基準の用語（ワードチョイス違い／同義／同一表記／別名）も不統一です。

## 3. ご質問への直接の回答 — 「洗い出した用語は意図と合っているか」

**部分的にしか合っていません。**

- 合っている点：用語ごとに「文脈での意味」を書いており定義のたたき台になる。一部の表記ゆれ（T-110, T-112, T-280 等）を備考で指摘しており着眼点は正しい。
- 合っていない点：(1) サービス全体ではなく admin 3画面に限定、(2) ユーザー側が完全欠落、(3) 構造が統一作業に使えない、(4) 出典が無効、(5) UI部品名・enum値が混在、(6) 重複・カテゴリ不整合。

「サービス全体で使用する語句」という観点では、**最も統一が必要な「画面をまたいで同じ概念が違う語で呼ばれている箇所」を体系的に拾えていません**。現状CSVが拾った350語の多くは1画面でしか使われないUI部品名で、統一対象としては優先度が低いものです。

## 5. 成果物と推奨アクション

### 5.1 作成した成果物

**`docs/glossary_統一用語集.csv`** — 意図（サービス全体の言語統一）に沿った構造で再設計した**確定版**用語集（75概念）。

- 「1概念＝1行」。列：`概念ID / カテゴリ / 概念名 / 統一表記（正） / 統一前の表記ゆれ（廃止） / 出現範囲 / 文脈での意味 / 補足・注意`
- 画面固有のUI部品名・個別enum値は除外し、**画面をまたぐ語句**に絞って収録。
- たたき台にあった `(要決定)` 項目は §6 の方針で確定済み（プラン体系・アンケートステータス体系・設問キー・対応言語など）。
- ※ たたき台 `docs/glossary_統一用語集_draft.csv` は本確定版に置き換わったため不要です。

### 5.2 推奨アクション

1. 確定版CSVの **§6 の意思決定（特に D-01 プラン体系）を関係者レビュー**し、必要なら微修正する。
2. 確定後、各画面のHTML／仕様書へ統一表記を反映（`02_dashboard/`・`03_admin/` 約50ファイル）。
3. 現状 `glossary.csv` は **「UI部品インベントリ」として用途を分けて残す**か、出典を現行正本（`docs/画面設計/仕様/admin/` 配下）へ張り直して整理する。言語統一の正としては `glossary_統一用語集.csv` を採用します。

## 6. 確定版で行った主要な意思決定（旧 `(要決定)` 項目）

たたき台で体系判断が必要だった項目を、以下の方針で確定しました。

| 概念ID | 項目 | 確定内容と根拠 |
|---|---|---|
| D-01 / D-02 | 契約プラン体系 | **無料プラン／プレミアムプランの2区分**に統一。内部値 free / premium。admin側「Pro」「Pro (年間)」は契約期間差でプラン名は「プレミアムプラン」。**※ 3区分(Free/Standard/Premium)表記・Enterprise の実在可否のみ経営確認が必要** |
| F-02 | アンケートのステータス | 3系統を**草稿→準備中→会期中→会期終了→データ化中→データ化完了→終了（＋削除済み）**の1系統に統合。旧表記の対応を補足列に明記。20_spec「停止」の要否のみ要確認 |
| F-07 | マトリックス表記 | 設問の正本（survey作成仕様）に準拠し「**マトリックス**」 |
| F-08 | 設問タイプの内部キー | survey作成仕様の**11キーを正**とし、graph-page等の別名キーを廃止。image_upload の作成画面採用のみ要確認 |
| F-09 | 対応言語 | 言語マスタの**5言語（日本語/英語/中国語簡体/中国語繁体/ベトナム語）**を正。韓国語はマスタ外のため要確認 |
| F-10 | サンクス画面 | 設定・表示とも「**サンクス画面**」（設定値「サンクスメッセージ」と整合） |
| E-02 / E-07 / H-03 | 管理画面の命名 | admin画面は「**〜管理**」で統一（請求管理・クーポン管理・照合管理）。仕様書の「請求一覧」等は画面内ビュー名扱い |
| K-02 | データ出力操作 | 動詞を「**出力**」に統一し形式名を冠する（CSV出力／Excel出力）。エクスポート・ダウンロード・レポートは不使用 |
| K-08 / L-01 / L-02 | 用途別に複数語を定義 | ログ＝操作ログ/更新ログ/ログイン履歴、期限＝納品期限/対応期限/ダウンロード期限/請求締切、記述欄＝内部メモ/理由 |
| I-02 / J-03 / L-03 | 同音異義の衝突 | 統合せず**文脈で区別**。「担当グループ」は「担当部署」へリネーム。「通知設定」は admin側を「エスカレーション通知設定」に。「有効」はカテゴリ併記を必須化 |
| その他 | カナ長音・スペース | サマリー（長音保持）、オペレーターID（スペース無し）等に統一 |
