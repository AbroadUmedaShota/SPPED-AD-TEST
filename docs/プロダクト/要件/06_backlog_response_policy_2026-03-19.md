# SPEEDAD 2026 Backlog返答方針メモ

作成日: 2026-03-19

## 1. 目的

担当中の `SPEEDAD 2026` 課題 12 件について、Backlog本文・コメント履歴・ローカル要件書を突き合わせ、返答文作成前の全体方針を固定する。

本メモは、次ターンで Backlog コメント文面を起こすための下書きであり、ここではコード変更や Backlog 更新は行わない。

## 2. 全体方針

- 仕様の最終根拠は、コメント断片ではなく要件書または Backlog wiki に寄せる。
- 決定済み事項は再議論せず、どの資料を正本にするかを返答で明示する。
- 未確定事項は、`判断待ちの論点` と `現時点の暫定運用` を分けて返す。
- 4月スコープ外や別基盤で進んでいるものは、画面実装論ではなく `扱い整理` として返す。
- 返答タイプは `確定回答 / 暫定整理 / 提案付き確認依頼 / スコープ調整 / 運用整理` のいずれかで整理する。
- 請求・課金系は、`docs/プロダクト/要件/01_billing_and_invoicing_logic.md` と Backlog wiki 集約方針を優先する。
- 多言語系は、`docs/プロダクト/要件/multilingual_requirements.md` を基準に、優先言語・未翻訳時フォールバック・対象画面を整理して返す。
- ヘルプ/チュートリアル系は、画面単位のモック論ではなく、独立運用・スコープ調整・別課題化の観点で返す。

### 2.1. 返答タイプの定義

- `確定回答`:
  - 要件書または運用方針文書で裏取りできる内容を、そのまま結論として返す。
- `暫定整理`:
  - Backlog コメント上では方向性が出ているが、wiki や要件書への正本化が未了のため、現時点の整理として返す。
- `提案付き確認依頼`:
  - Abroad 側の推奨案は提示するが、最終判断が相手側に残る内容として返す。
- `運用整理`:
  - 画面仕様そのものより、正本化先、運用導線、別課題化の整理を優先して返す。
- `スコープ調整`:
  - 対象外、後続対応、別導線への切替を明示して返す。
