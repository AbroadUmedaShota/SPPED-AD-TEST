# 請求・課金ロジック定義書

## 1. 料金体系の概要

本システムの課金体系は、利用実績に応じて発生する「従量課金」を基本とし、必要に応じて上位機能を利用するための月額オプションとして「プレミアムプラン」を追加できる構成とする。

従量課金とプレミアムプランは別軸で管理するが、実際の請求では同一請求内で合算して扱うことがある。組織利用時の「追加アカウント料金」も、この請求体系に含まれる。

## 2. プレミアムプラン仕様

プレミアムプランは、基本サービスに追加して契約する月額オプションであり、高度な分析機能や多言語対応などの上位機能を利用可能にする。

プレミアムプランは「上位機能の利用権」を提供するものであり、明示的に内包と定義したものを除き、従量課金そのものを免除するものではない。

### 2.1. 基本料金
*   **月額料金:** 10,000円（税別）
*   **契約単位:** 個人アカウント、またはグループ単位。

### 2.2. 特典および条件
*   **初月無料:** 新規契約した最初の月（契約月）の基本料金は免除される。
*   **日割り計算なし:** 契約・解約日に関わらず、月額固定料金が発生する。

### 2.3. プランに含まれる機能（※一部抜粋）
プレミアム基本料金に内包されるのは、上位機能の利用権である。以下の機能を利用しても、別途定義された従量課金ルール自体は継続する。
*   **多言語対応（名刺データ化）:** 日本語以外の名刺解析・データ化を可能にする機能。名刺データ化件数に対する従量課金は別途適用する。
*   **高度な分析（SPEEDレビュー）:** グラフ表示およびExcelレポート出力。
*   ※将来的には上位プランを追加し、プレミアムプラン2（仮）で独自ドメイン/お礼メール、プレミアムプラン3（仮）で外部連携を提供する構想がある。

### 2.4. 従量課金との境界
*   **基本原則:** プレミアム契約中であっても、名刺データ化費用、お礼メール送信費用などの従量課金は、明示的な例外がない限り継続する。
*   **オンデマンドプラン:** オンデマンドプランはプレミアム契約を利用条件とするが、料金はプレミアム月額に内包せず、名刺データ化費用として従量課金する。
*   **お礼メール送信費用:** プレミアム契約中であっても、無料枠を超える送信分は従量課金する。

### 2.5. 将来的な上位プラン構想
*   **プレミアム2:** 独自ドメイン/お礼メール機能を含む上位プランとして構想する。
*   **プレミアム3:** 外部連携機能を含む上位プランとして構想する。
*   **位置づけ:** いずれも現時点では構想段階であり、料金・適用条件・従量課金との境界は別途定義する。

## 3. グループ利用料金

組織（グループ）でサービスを利用する場合、プレミアム契約の有無にかかわらず、所属人数に応じて以下の追加アカウント料金が発生する。

### 3.1. 追加アカウント料
*   **2名まで（管理者含む）:** 無料。
*   **3名以上:** 1名追加につき、月額 300円（税別）。

### 3.2. 計算例
*   グループ機能を利用し、1グループ5名で利用する場合：
    *   2名分は無料 + 追加3名分 (300円 × 3) = **合計 900円/月**

## 4. 従量課金ロジック（利用実績課金）

利用実績に応じて、毎月の請求時に加算される。

### 4.1. 名刺データ化費用
単価は選択された「スピードプラン」に準ずる。
*   **通常プラン:** 50円/枚
*   **特急プラン:** 100円/枚
*   **超特急プラン:** 150円/枚
*   **オンデマンドプラン:** 200円/枚
*   **最低請求ルール:** 実際の作業件数が、設定した「想定件数」の50%に満たない場合は、想定件数の50%分を請求する。

### 4.2. お礼メール送信費用
*   **無料枠:** 1アンケート（または月間）につき最初の100通。
*   **超過料金:** 101通目以降、1通につき 1円（税別）。

## 5. クーポン仕様

本システムにおけるクーポンは、利用料金の割引に加え、販売代理店の管理や流入経路の特定を主な目的として運用される。

### 5.1. 目的と役割
*   **新規獲得施策:** 初回利用時の割引、期間限定キャンペーン、展示会や紹介施策向けの申込促進に使用する。
*   **販路識別:** どの代理店・紹介先・営業施策・広告導線から流入した案件かを識別するための管理キーとして使用する。
*   **見積・請求調整:** 従量課金項目に対する値引き条件を事前に定義し、概算見積もりと最終請求に同じ条件を反映するために使用する。
*   **代理店成果管理:** 代理店ごとの送客件数、申込件数、利用開始件数、請求発生件数を把握し、手数料計算や評価の基礎データとして使用する。
*   **個別救済対応:** 営業判断や障害対応、既存顧客への調整などで、特定案件のみ例外的な値引きを行う際の管理手段として使用する。

### 5.2. 割引の基本ルール
*   **定額割引:** 請求総額、または特定の従量課金項目（名刺データ化費用等）から一定額を差し引く。
*   **定率割引:** 対象項目の料金に対して一定の割合（%）を割引く。
*   **適用単位:** クーポンは請求全体ではなく、`アンケート単位` で適用する。
*   **適用数の制限:** 1アンケートに対して適用できるクーポンは1種類のみとする。
*   **適用対象:** クーポンは原則として従量課金項目に適用し、プレミアム利用料および追加アカウント料金には適用しない。これらへ適用する場合は別途明示する。
*   **対象範囲の考え方:** クーポンを適用したアンケートの課金項目からのみ割引を行う。たとえば `プレミアムプラン + アンケート1 + アンケート2` の請求で、クーポンをアンケート1に適用した場合は、アンケート1由来の課金項目のみが値引き対象となり、アンケート2由来の課金項目やプレミアム利用料には影響しない。
*   **月跨ぎ利用時の扱い:** 名刺データ化やお礼メール送信など、同一アンケートに紐づく従量課金が翌月以降にも発生する場合は、そのアンケートにクーポンが適用されている限り、翌月の利用料にも同じ割引条件を適用する。

### 5.3. 代理店管理と手数料運用
*   **代理店グループ:** 管理画面上で「代理店」に相当するグループを作成し、その配下でクーポンをバルク発行（一括生成）する。
*   **発行単位:** クーポンは、代理店単位、代理店配下の営業担当単位、または個別案件単位で発行できるようにし、どの粒度で成果を計測するかを明確にする。
*   **識別子としての役割:** クーポンコード自体を「値引き条件」と「販路識別子」の両方として扱い、申込時点でどの代理店経由かを確定できるようにする。
*   **成果トラッキング:** どのクーポンがどのユーザーによって使用されたかを記録し、代理店ごとの成約件数や手数料計算の基礎データとして活用する。
*   **追跡対象:** 少なくとも `発行元代理店` `発行担当者` `使用ユーザー` `対象アンケートまたは案件` `適用金額` `請求発生日` `入金有無` を追跡対象とする。
*   **手数料計算との関係:** 代理店手数料は、クーポン利用アンケートを起点に算定する。現時点の前提では `請求発生ベース` で確定し、算定母数は金額ではなく `請求対象件数` とする。
*   **手数料対象の範囲:** 代理店手数料の対象は、クーポンが適用されたアンケートに紐づく従量課金件数のみとし、プレミアム利用料および追加アカウント料金は対象外とする。
*   **手数料台帳の管理単位:** 代理店手数料は、請求書単位ではなく `アンケート × 請求月` 単位で履歴化し、未払・支払済・調整中を判別できる状態で管理する。
*   **再精算への備え:** 請求取消、件数修正、クーポン誤適用、値引き追加が発生した場合に差額再計算できるよう、再計算対象フラグと変更履歴を保持する。
*   **運用上の留意点:** 大型展示会のように件数が大きい案件では代理店手数料も増加するため、必要に応じて月次上限、案件上限、段階逓減、監査ルールを別途設ける。
*   **代理店評価との接続:** 同一代理店が発行したクーポンの利用件数、請求発生件数、実入金件数を集計し、代理店ごとの成果比較や継続取引判断に利用する。

### 5.4. 運用ルール
*   **有効期限:** クーポンごとに設定された期限内のみ有効。
*   **適用タイミング:** アンケート作成時、または名刺データ化依頼時にコードを入力することで、当該案件の概算見積もりおよび最終請求に反映される。
*   **案件との紐付け:** クーポンは利用時点でアンケートまたは案件に紐付け、その後の請求処理でも同一アンケートに対して同じ条件を引き継ぐ。
*   **請求書上の扱い:** 値引きが発生した場合は、対象課金項目に対する減額として明細上で判別できる状態を維持する。
*   **失効時の扱い:** 有効期限を過ぎたクーポンは新規適用不可とするが、適用済み案件に対する請求反映ルールは、適用時点の条件を優先する。
*   **クーポン外値引きの扱い:** クーポンを使わない値引きは後から販路判定や成果集計ができなくなるため、代理店経由案件は原則としてクーポン経由で管理する。

## 6. 請求・支払い仕様

### 6.1. 請求単位と宛先
*   **個人契約:** 個人アカウントへ請求。
*   **グループ契約:** **グループ管理者（オーナー）**へ、グループ内の全費用を一括請求する。
*   **グループ契約時の請求内訳:** プレミアム利用料（契約している場合）、追加アカウント料金、従量課金は、同一請求書内で合算しつつ、内訳は別行で表示する。

### 6.2. 請求サイクル
*   **締め日:** 毎月末日
*   **支払日:** 翌月末払い
*   **支払い方法:** 銀行振込

### 6.3. 請求書上の表示ルール
*   **請求明細の表示単位:** 従量課金とプレミアム利用料は、請求書上で別行として表示する。
*   **追加アカウント料金の表示:** グループ利用に伴う追加アカウント料金も、必要に応じて独立した明細行として表示する。
*   **表示上の考え方:** 従量課金、プレミアム利用料（契約している場合）、追加アカウント料金は別軸の課金要素として扱い、請求書上でも内訳が判別できる状態を維持する。
*   **品目1 / 品目2 の記載原則:** 請求書の明細欄では、`品目1` に主となる請求概要、`品目2` にその詳細を記載する。
*   **アンケート非依存の請求:** プレミアム利用料や追加アカウント料金のようにアンケートへ紐づかない請求は、`品目1` に課金種別を記載する。
*   **アンケート依存の請求:** 名刺データ化費用、お礼メール送信費用、クーポン値引きのようにアンケート単位で発生する請求は、`品目1` にアンケート名を記載する。
*   **品目2 の役割:** `品目2` には、課金種別、対象月、適用プラン、無料枠超過、値引き理由などの計算根拠や補足情報を記載する。数量と単価は数量欄・単価欄で表現するため、原則として `品目2` には重複記載しない。
*   **小計の切り方:** 通常請求では、`プレミアム利用料` と `追加アカウント料金` を契約系としてまとめて小計を出し、従量課金はアンケートごとに小計を出す。
*   **集計請求時の小計:** `AGG-...` の集計請求では、契約系小計・アンケート小計に加えて、元請求書単位の `請求書小計` を残す。
*   **A4帳票での改ページ:** 請求書はA4印刷を前提とし、`品目1` `品目2` の折り返しを許容する。明細件数は固定20件を保証せず、1ページ目は14行相当を安全側の目安として、実際の表示件数は各明細の表示高さに応じて可変とする。
*   **未記載行の扱い:** 実明細が少ないページでは、残余高さに応じて未記載行を補完し、帳票下端まで罫線を維持する。未記載行は見た目調整用であり、計算対象には含めない。
*   **代表例（アンケート非依存）:** `プレミアムプラン利用料 / 2025年8月分`
*   **代表例（アンケート非依存）:** `追加アカウント料金 / 営業部グループ 全5アカウント中 無料2アカウントを除く追加3アカウント分`
*   **代表例（アンケート依存）:** `アンケート名A / 名刺データ化費用 / 特急プラン`
*   **代表例（アンケート依存）:** `アンケート名A / お礼メール送信費用 / 無料枠100通超過分`
*   **代表例（アンケート依存）:** `アンケート名A / クーポン値引き / 展示会来場特典クーポン適用`
*   **代表例（小計）:** `プレミアムプラン・グループアカウント料金 / 小計`
*   **代表例（小計）:** `アンケート名A / 小計`

## 7. 共通ルール・ポリシー

### 7.1. 返金ポリシー
*   サービスの性質上、支払い済みの料金および確定した従量課金分の返金は行わない。

### 7.2. 免責事項（外部要因）
*   AWS障害、外部API（Google OCR等）の不具合、通信環境に起因する遅延や不具合については、請求の免除や損害賠償の対象外とする。
