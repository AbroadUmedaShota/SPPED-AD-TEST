# SPDAD2026 モック確認会レジュメ

## 1. 目的

本レジュメは、`SPDAD2026` の担当課題について、現状モックを見ながら仕様差分と未確定事項を確認し、企画判断が必要な論点をその場で整理するための会議用資料である。

Backlog の原文を追うことよりも、`何を見るか` `何が未確定か` `今回何を決めるか` を短時間で把握できる構成を優先する。

作成日: 2026-03-17

## 2. この会議での見方

### 2.1. 見る順番
1. モックを開いて現状の見え方を確認する。
2. Backlog 上の要件・最新コメント要点を確認する。
3. 差分 / 未確定点を整理する。
4. 会議で決めることをその場で確定する。
5. 決定後の次アクションを決める。

### 2.2. 今日決めたいこと
* 請求書の正式フォーマットと出力ルール。
* プレミアム機能と多言語機能の公開前運用ルール。
* 名刺撮影モーダルの最終 UX 方針。
* お礼メール設定画面の状態別画面仕様。
* チュートリアル / ヘルプ系のスコープと実装方針。

* 対象課題: `SPDAD2026-26`

* 決定後の次アクション: 請求書仕様を文書化し、表示項目と計算ロジックを要件へ反映する。

### SPDAD2026-63 データ化項目プラン：土日祝「当日中」は有効か

* 対象課題: `SPDAD2026-63`
* 確認するモック: `02_dashboard/bizcardSettings.html`
* 現状モックの見え方: プランや納期条件は画面内で設定・確認できるが、「当日中」の条件文言までは明示度が低い可能性がある。
* Backlog上の要件・最新情報: オンデマンドプランは営業日関係なく対応、その他プランは営業日ベースという整理あり。一方で既存資料には「18時まで入稿で当日中納品可能」とあり、条件表示要否の確認が入っている。

* 会議で決めること: 18 時条件の採用有無。条件文言の表示場所。将来のオンデマンド専用営業カレンダー前提を持つか。
* 決定後の次アクション: 名刺データ化設定画面とヘルプ文言の要件を更新する。
