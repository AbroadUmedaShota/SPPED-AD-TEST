# 製品ドキュメント概要

`docs/プロダクト/` は、サービス企画・アーキテクチャ・運用プロセスをまとめた製品ドキュメントの正本ディレクトリです。

## ディレクトリ構成
- `概要/`: プロジェクト全体像、要件インデックス、画面遷移。
- `アーキテクチャ/`: アーキテクチャ方針、データモデル、データインベントリ。
- `サービス企画/`: サービス企画の検討資料（機能棚卸し、プラン比較、方針ドラフト）。
- `プロセス/`: 業務プロセス、データフロー、KPI フロー。
- `要件/`: 製品横断の要件整理。請求・課金ロジックや多言語要件などを管理。
- `UIデザイン/`: デザインガイド、UI メッセージ、コンポーネント統合手順。
- `標準・規準/`: コーディング規約、レビュー観点。

## 運用ルール
- 仕様を更新した場合は、関連する `プロセス/` と `アーキテクチャ/` の参照整合を確認する。
- 実装差分を反映した場合は、必要に応じて `docs/変更履歴/CHANGELOG.md` に追記する。
- デザインシステム更新時は、関連文書だけでなく README 導線（`docs/README.md` と本ファイル）も更新対象に含める。
