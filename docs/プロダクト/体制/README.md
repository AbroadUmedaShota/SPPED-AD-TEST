# 体制ドキュメント

本ディレクトリは、SPEED AD の開発体制およびサービス運用体制に関する正本ドキュメントを管理します。社内および委託先の関係者が、責任分界と連携経路を共通理解できる状態を目的とします。

## ドキュメント一覧

1. **[01_development_team_structure.md](./01_development_team_structure.md)**  
   アブロードアウトソーシング株式会社と Rep 株式会社の関係、意思決定ライン、開発上の責任分界を整理します。
2. **[02_service_operation_team_structure.md](./02_service_operation_team_structure.md)**  
   実運用に関与する組織一覧、主責務、最小限の連携ルールを整理します。

## 運用方針

- 会社名・役職名・責任範囲に変更があった場合は、関連文書を同一更新で見直します。
- 実運用フローの詳細手順は `docs/プロダクト/プロセス/` または `docs/ハンドブック/` に分離し、本ディレクトリでは責務と連携経路の定義を優先します。
- 個人名や役職名が未確定の委託先情報は、断定せず会社単位の責務表現に留めます。
