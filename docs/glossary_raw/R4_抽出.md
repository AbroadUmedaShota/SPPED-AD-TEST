## 気づいた表記ゆれ・同音異義

### 1. サービス名称の表記ゆれ（最重要）

同一サービスに3種類の表記が混在している。

- **`SPEED AD`**（スペース有り） — `index.html` ヒーロー、ログインパネル、フッターブランド、メタタイトル等
- **`SPEED-AD`**（ハイフン有り） — `common/header.html`・`common/footer.html`・`04_first-login/index.html`・`02_dashboard/faq.html`・`02_dashboard/help.html`・`bug-report.html`・`05_support`全画面のナビ、タイトルタグ等
- **`SpeedAd`**（キャメルケース） — `password_change.html`・`password-change-complete.html`・`terms-of-service.html`・`specified-commercial-transactions.html` の `<title>` タグ

### 2. パスワード再設定・変更の呼称ゆれ

「ログイン前にパスワードを再設定する動線」に対して複数の呼称が使われている。

- **`パスワードをお忘れですか？`** — forgot-password.html の画面タイトル（疑問形）
- **`パスワードの再設定`** — reset-password.html の画面タイトル
- **`再設定完了`** — reset-password-complete.html の完了見出し
- **`パスワード再設定完了`** — reset-password-complete.html の `<title>` タグ
- **`パスワードリセット`** — 仕様書01・10の用語集見出し、パンくず文言等
- **`パスワードを忘れた方`** — index.html ログインパネルのリンクテキスト（ログイン前トップ）
- **`パスワードを忘れた場合`** — password_change.html step1内のリンクテキスト（ログイン後）

「ログイン後パスワード変更」との区別も呼称が統一されていない（「再設定」「リセット」「変更」が文脈混在）。

### 3. ヘルプ・FAQ・サポートの呼称ゆれ

利用者サポート機能の呼び方が画面・仕様書間でブレている。

- **`ヘルプセンター`** — `help.html`・`help-content.html`・`common/footer.html` の画面タイトル・ナビリンク
- **`ヘルプ`** — `05_support/common/header.html`・`footer.html` のナビリンクテキスト（単語のみ）
- **`SPEED-AD Help Center`** — `help.html` ヒーローのeyebrowテキスト（英字）
- **`よくある質問`** — `faq.html` 画面タイトル・`common/footer.html`・`05_support/faq/index.html`
- **`FAQ`** — `05_support/common/header.html`・`footer.html` のナビリンク（英字略称）
- **`SPEED-AD FAQ`** — `faq.html` ヒーローのeyebrowテキスト（英字）
- **`お問い合わせ`** — `common/footer.html`・`05_support/common/header.html`・`footer.html`・各CTAボタン
- **`不具合報告`** — `bug-report.html` のページタイトル・仕様書15番のタイトル
- **`SPEED AD 不具合報告フォーム`** — `bug-report.html` のフォーム内h1テキスト
- **`サポート`** — `index.html`（ログイン前フッター）のリンクラベル、`common/footer.html` フッターセクション見出し

「ヘルプ」と「ヘルプセンター」が同一概念を指すかどうか（ナビリンク短縮形 vs. 正式名称）不明確。

### 4. 御礼メール・お礼メールの表記ゆれ

- **`御礼メールの送信`** — `index.html` FEATURESカード03見出し・FLOW「追客」説明文
- **`お礼メール`** — `index.html` REASONSセクション06説明文（「ヘルプ記事やFAQから〜**お礼メール**の使い方を確認できます。」）

同じ機能を指すと思われるが、「御礼メール」と「お礼メール」が同一ページ内に混在している。

### 5. 「不具合報告」と「お問い合わせ」が同一ページを指す問題

- `05_support/common/header.html` ・ `05_support/common/footer.html` のナビリンク → **`お問い合わせ`** （`/bug-report/` へリンク）
- `02_dashboard/bug-report.html` の画面タイトル → **`不具合報告`**（h1）／**`SPEED AD 不具合報告フォーム`**（フォーム内h1）
- `02_dashboard/faq.html` のCTAボタン → **`お問い合わせ`**（`bug-report.html` へリンク）
- `02_dashboard/help.html` のCTAボタン → **`お問い合わせ`**（`bug-report.html` へリンク）
- 仕様書15番のタイトル → **`不具合報告フォーム`**

同一画面（`bug-report.html`）の呼称として「不具合報告」と「お問い合わせ」が混用されており、ナビからは「お問い合わせ」、ページ内では「不具合報告」と乖離している。
