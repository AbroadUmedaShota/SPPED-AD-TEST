## 1. ひと言サマリ（30 秒で読む）

- **プロダクト**: SPEED AD（展示会向け名刺データ化 / アンケート / リード活用 SaaS）
- **対象画面**: ダッシュボード内の「プレミアムプラン加入促進ページ」（`premium_signup_new.html`）
- **役割**: 月額 10,000 円（税別）プランの料金・特典・プラン比較・FAQ を 1 ページで案内し、申込フォームへ送る。
- **遷移先**: 「申し込み」CTA → `premium_registration_spa.html`（3 ステップ SPA、本書のスコープ外）
- **特徴**: 同一 URL でユーザー状態（5 種類）に応じてヒーロー・CTA・キャンペーン表示が切替わる。
- **ゴール（成功定義）**: 新規／再加入ユーザーが料金と機能差を理解し、「申し込み」CTA を押す確率を最大化する。既存会員は契約状況・請求書確認・解約導線へ最短で到達できる。

## 2. プロダクト前提（背景）

- SPEED AD は **展示会後のリード入力作業をゼロに近づける** ことを訴求する BtoB SaaS。
- 利用者は主に **マーケ／営業企画／経理** のクライアント担当者。意思決定者と起票者が分かれているケースが多い。
- 無料の Standard プランと、月額 10,000 円（税別）の Premium プランの 2 段階構成。Premium は **保存期間延長・出力・連携・ブランド管理** が中心の差分。
- 課金は **請求書払い（月末締め翌月末払い）**。クレジットカード入力は本フローに含まれない。
- 本ページはダッシュボード内に存在し、共通ヘッダー／サイドバー／フッター（`src/main.js` で挿入）を継承する。**メインコンテンツ領域のみ** を新規デザインの対象とする。

## 3. ターゲットユーザーと利用文脈

### 3.1 メインターゲット
- **新規ユーザー**: 未課金の Standard 会員。料金・機能を比較検討中。
- **再加入ユーザー**: 過去に解約済み。再契約条件（即日課金）を確認したい。

### 3.2 サブターゲット（同 URL に来訪する）
- **無料トライアル中ユーザー**: 期限と請求開始日を確認したい。
- **既存プレミアム会員（Active）**: 請求書確認・解約導線が必要。
- **解約予約中ユーザー**: 自動更新を再開できる導線が必要。

### 3.3 使われ方
- ダッシュボード左サイドバーやアップグレード導線から流入。
- スマホ閲覧（営業先の隙間時間）と PC 閲覧（社内決裁前）が両方ある。
- **判断材料**として読まれるため、「料金」「保存期間」「比較表」が早く見つかることが重要。

## 7. インタラクション仕様

- **FAQ アコーディオン**: クリックで `aria-expanded` トグル、回答領域を開閉（Q1 のみ初期 open）。
- **比較表 SP 表示**: `<details>` 形式で「詳細比較9項目を見る」サマリーをタップで開閉。PC では常時展開。
- **モーダル 3 種**:
  - `contactModal`: お問い合わせフォーム（氏名 / メール / 件名 / 本文）。現状リンクで `bug-report.html` 遷移なので、デザイン上は同モーダルの存在を踏まえて UX を設計（モーダル統一推奨）。
  - `resumeModal`: 解約予約中ユーザーの「自動更新を再開する」確認モーダル。
  - `resumeSuccessModal`: 再開完了モーダル。次回更新日を表示し、「ダッシュボードに戻る」で閉じる。
- 全モーダルに `role="dialog"` / `aria-modal="true"` / `aria-labelledby` を備える前提でデザイン。

## 8. デザイン要件・トーン

### 8.1 トーン & マナー
- **業務向けの落ち着き** ＋ **プレミアムらしい質感** のバランス。派手すぎない。
- 主役は「料金」と「機能差」。装飾はそれを邪魔しない強度に抑える。
- 不安を煽らない、過度な希少性訴求や赤系の警告色は使わない。
- 既存実装は「白背景 × 青アクセント × ダーク最終 CTA」の 3 層で構成。この骨格は維持してほしい。

### 8.2 共通レイアウト方針
- 各セクションは **角丸 (`rounded-2xl`)・白背景・薄い境界線・控えめなシャドウ** のカードで統一。
- ラベル → 見出し → 補足 → コンテンツの 4 段構成を踏襲。
- 縦リズムは 8px グリッド。セクション間は 32〜40px。
- 最終 CTA セクションだけ **ダーク背景（slate-900 系）** で視覚的アンカーにする。

## 11. アクセシビリティ要件

- 全インタラクティブ要素はキーボード操作可能。
- カラーコントラスト 4.5:1 以上を目標。
- FAQ の開閉は `aria-expanded` を切替え、回答に `role="region"` を付与。
- 装飾アイコンは `aria-hidden="true"`。
- モーダルは `role="dialog"` / `aria-modal="true"` / `aria-labelledby` を備え、初期フォーカスをタイトル付近に置く。
- 動的に挿入されるユーザー名は `textContent` 系で扱う想定（XSS 対策）。

## 14. 参考: 既存実装の見るべき点

- **良い点（このまま踏襲してほしい）**
  - 白基調＋青アクセントの落ち着いたトーン
  - 各セクション頭の小ラベル（`PRICE` / `ISSUES` / `PLAN COMPARISON` ほか）の統一感

  - キャンペーン枠 (`integrated-campaign-container`) のタイムライン表現が文字中心なので、**月次のビジュアルタイムライン化**（当月¥0 → 翌月¥10,000）を提案してほしい。
