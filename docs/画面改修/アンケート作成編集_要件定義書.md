# アンケート作成・編集画面 要件定義書（改修版）

## 1. 概要

本ドキュメントは、`surveyCreation.html`（アンケート作成・編集画面）の改修に伴う要件を定義する。
現行画面の課題（機能増加によるUI肥大化、設問グループの複雑さ）を解消し、設定と設問編集を物理的に分離した3エリア構成に再設計する。

### 1.1 改修の背景

- 機能追加に伴い、1カラム＋サイドパネル構成では縦スクロール量が過大になった
- 設問グループ化機能は運用上ほぼ使われておらず、UIを複雑にしている
- サンクス画面設定が別ページに分離されているが、アンケート作成フローから離脱する動線になっている
- 多言語設定の配置が基本情報に埋もれており、連携機能としての位置づけが不明瞭

### 1.2 改修方針

| 項目 | 方針 |
|------|------|
| 設問グループ化機能 | **廃止** — 設問はフラットリストに変更 |
| 画面レイアウト | 1カラム＋サイドパネル → **3エリア構成**（設定カラム / 設問カラム / アウトラインパネル） |
| サンクス画面設定 | 別ページ → **アンケート作成画面内にインライン統合** |
| 多言語設定 | 基本情報セクション → **連携設定セクションに移動・集約** |
| 言語タブの並び順 | **ドラッグで入れ替え可能**、最左タブ＝第一言語（回答画面のデフォルト表示言語） |
| 既存機能 | 上記以外の全機能は**維持** |

### 1.3 設問グループ廃止の方針

- 現行はモック段階のため、既存データのマイグレーションは不要
- 設問グループ関連のコード・UIを削除し、フラットリスト構成で新規実装する

### 1.4 サンクス画面設定ページの廃止

- 現行はモック段階のため、段階的な移行は不要
- `thankYouScreenSettings.html` を削除し、サンクス画面設定はアンケート作成画面内にインライン統合する
- サイドバー等からの旧ページへのリンクも合わせて除去する

## 2. 画面レイアウト

### 2.1 レイアウト構成

メインコンテンツを3つのエリアに分割する。
設定系（入力頻度:低）と設問編集（入力頻度:高）を物理的に分離し、作業効率を向上させる。

基本情報・メモは上部に横長カードとして配置し、その下に設定カラム／設問カラム／アウトラインパネルの3エリア構成を展開する。

| [Header] (Fixed, h-16)  SPEED-AD ロゴ / ユーザー情報                               |
+-----------------------------------------------------------------------------------+
|          |                                                                        |
| [Sidebar]|  [メインコンテンツ]                                                     |
| (w-64)   |                                                                        |
|          |  [Breadcrumb] ダッシュボード > アンケート作成・編集                        |
|          |  [Title] アンケート作成・編集                                            |
|          |                                                                        |
|          |  +-------------------------------------------------------------------+ |
|          |  | [基本情報カード]  (md:col-span-12 / lg:col-span-12)                | |
|          |  |  左(col-span-3): アンケート名・表示タイトル・説明                  | |
|          |  |  右(col-span-2): アンケートID・期間・メモ（2カラム分割）           | |

### 2.2 グリッド仕様

| エリア | グリッド幅 | 役割 | スクロール |
|--------|-----------|------|-----------|
| 基本情報カード（上部） | `md:col-span-12 / lg:col-span-12` | アンケート名・表示タイトル・説明・期間・ID・メモ | ページ全体と連動 |
| 設定カラム | `md:col-span-4 / lg:col-span-3` (~25%) | オプション機能・関連設定 | `sticky top-24`, カラム内で独立スクロール（`max-h-[calc(100vh-6rem)] overflow-y-auto`） |
| 設問カラム | `md:col-span-8 / lg:col-span-9 / xl:col-span-6` (~50%@xl) | 設問リストの編集・サンクス画面カード | ページ全体と連動 |
| アウトラインパネル | `xl:col-span-3` (~25%@xl) | 目次＋アクションボタン | `sticky top-[5rem]`, カラム内で独立スクロール。lg〜xl未満はドロワー（後述 §2.5 参照） |

### 2.3 設定カラムの縦スクロール対策

設定カラムには連携設定（多言語含む）・追加設定が集約されるため、全セクション展開時にカラム高さがビューポートを超える可能性がある。以下の対策を組み合わせる。

| 対策 | 詳細 |
|------|------|
| 独立スクロール | 設定カラム自体を `sticky` ＋ `overflow-y-auto` とし、設問カラムとは独立してスクロールできるようにする |
| アコーディオン初期状態 | 追加設定内の「名刺データ化設定」「お礼メール設定」は**デフォルト折り畳み**。連携設定の多言語UIも `multilingualEnabledToggle = OFF` 時は非表示 |
| セクション間の余白圧縮 | 設定カラム内はカード間 `gap-5` で余白を節約 |

**高さ見積もり（最大展開時）:**

| セクション | 概算高さ |
|-----------|---------|
| オプション機能（名刺＋多言語ON＋言語選択＋タブ＋連続回答） | ~360px |
| 関連設定（2設定すべて展開） | ~400px |
| **合計** | **~760px** |

ビューポート高さ（1080px想定）の範囲内に収まる場合が多いが、独立スクロールにより操作に支障なし。

### 2.4 レスポンシブ対応

| ブレークポイント | レイアウト |
|-----------------|-----------|
| `xl` (1280px~) | 3エリア構成（設定カラム / 設問カラム / アウトラインパネルがグリッド内に並列） |
| `lg` (1024px~1279px) | 設定カラム＋設問カラムの2カラム。アウトラインパネルはドロワーとして右端に固定（トグルボタンで開閉） |
| `md` (768px~1023px) | 設定カラム＋設問カラムの2カラム。アウトラインパネル非表示、`fixed bottom-0` のアクションバーを表示 |
| `sm` (~767px) | 1カラム（設定→設問の縦積み）。アクションバーは `fixed bottom-0` |

### 2.5 アウトラインパネルの表示モード

アウトラインパネルはブレークポイントに応じて表示モードが変わる。

| ブレークポイント | モード | 挙動 |
|-----------------|--------|------|
| `xl` (1280px~) | **グリッド内 sticky** | `xl:col-span-3` としてグリッドに静的配置。`sticky top-[5rem]` で固定 |
| `lg` (1024px~1279px) | **ドロワー（引き出し）** | `fixed right-0` で画面右端に固定配置。デフォルトは `translate-x-full`（非表示）。画面右端の「MAP」トグルボタン（`#outline-map-toggle-btn`）で開閉する |
| `md` 以下 | **非表示** | アウトラインパネル非表示。代わりにモバイルアクションバー（§2.4）を表示 |

- ドロワートグルボタン（`#outline-map-toggle-btn`）は `lg:flex xl:hidden` で lg〜xl 未満のみ表示
- ドロワーが開いている間は `chevron_left` → `chevron_right` アイコンに切り替わる

#### `md` / `sm` 時のアクションバー構成

```
+------------------------------------------------------------------+
| [プレビュー]  [QRコード]  [キャンセル]  [アンケート保存]           |
+------------------------------------------------------------------+
  fixed bottom-0, bg-surface, shadow-top, h-16, z-50
```

| ボタン | ID | 表示 | 状態制御 |
|--------|-----|------|---------|
| プレビュー表示 | `showPreviewBtnMobile` | アイコン＋テキスト | 常に有効 |
| QRコード | `openQrModalBtnMobile` | アイコン＋テキスト | アンケートID未発行時は `disabled` |
| 設問追加 | `addQuestionMobileBtn` | アイコン＋テキスト | 常に有効。タップでモバイル用設問タイプメニュー（`#mobileQuestionTypeMenu`）を表示 |
| アンケート保存 | `createSurveyBtnMobile` | `bg-primary` 強調表示（`flex-[2]`で幅広） | バリデーション通過後に有効化 |

> **注**: `md` / `sm` 時のアクションバーには「キャンセル」ボタンは**含まれない**。キャンセルはデスクトップのアウトラインパネル内のみに配置（`#cancelCreateSurvey`）。

## 3. 機能要件 — 基本情報カード・設定カラム

### 3.1 基本情報セクション

| 項目 | ID | タイプ | 必須 | バリデーション | 備考 |
|------|-----|--------|------|---------------|------|
| アンケート名（管理用） | `surveyName` | テキスト | ○ | 空禁止、最大50文字 | ヘルプアイコン付き |
| 表示タイトル | `displayTitle` | テキスト | ○ | 空禁止、最大100文字 | 回答者に表示される |
| 説明 | `description` | テキストエリア | - | 最大500文字 | |
| アンケート期間 | `periodRange` | 日付範囲 | ○ | 下記参照 | flatpickrで入力 |

**アンケート期間のバリデーションルール:**

| 状態 | 開始日 | 終了日 |
|------|--------|--------|
| 新規作成時 | 翌日以降（当日は不可） | 開始日以降 |
| 編集時・開始日が未来 | 変更可能（翌日以降） | 開始日以降 |
| 編集時・開始日が過去（回答受付中） | **変更不可**（readonly表示、変更理由を説明するツールチップ付き） | 延長のみ可能（現在の終了日以降に変更可。短縮は不可） |
| 編集時・終了日が過去（回答受付終了） | 変更不可 | 変更不可（再開する場合は新規アンケートを作成する運用） |

- 基本情報カードは画面上部に横長カードとして配置する（設定カラムではなく上部エリア）
- カード内は2カラム分割（`md:grid-cols-5`）: 左（`col-span-3`：アンケート名・表示タイトル・説明）/ 右（`col-span-2`：アンケートID・期間・メモ）
- 多言語対応ON時、表示タイトル・説明は各言語タブで入力可能になる（オプション機能の言語タブと連動）
- アンケート名（管理用）は管理画面でのみ使用するため、**多言語対象外**（常に単一文字列）

### 3.2 オプション機能セクション

連携に関わるON/OFF系機能と、多言語対応の設定をまとめるセクション。

#### 3.2.1 名刺画像添付

| 項目 | ID | タイプ | デフォルト | 備考 |
|------|-----|--------|-----------|------|
| 名刺画像添付を有効にする | `bizcardEnabled` | トグルスイッチ | OFF | OFFにすると「名刺データ化設定」「お礼メール設定」が無効化される |

#### 3.2.2 連続回答の許可

| 項目 | ID | タイプ | デフォルト | 備考 |
|------|-----|--------|-----------|------|
| 連続回答の許可 | `allowContinuousAnswer` | トグルスイッチ | **ON** | オプション機能セクション内に配置。同端末で続けて回答を受け付ける |

#### 3.2.3 多言語対応

| 項目 | ID | タイプ | デフォルト | 備考 |
|------|-----|--------|-----------|------|
| 多言語機能の有効化 | `multilingualEnabledToggle` | トグルスイッチ | OFF | プレミアムプラン限定。ONで以下の設定UIが展開される |

**ONにした場合に表示される設定:**

| 項目 | 詳細 |
|------|------|
| 追加言語の選択 | 対応言語リストからチェックで選択（`#languageSelectionPanel`）。最大2言語まで追加可能（日本語含め最大3言語構成） |

**初期対応言語リスト（確定）:**

| 言語 | コード | 表示名 |
|------|--------|--------|
| 日本語 | `ja` | 日本語 |
| 英語 | `en` | English |
| 中国語（簡体字） | `zh-CN` | 中文(简体) |
| 中国語（繁体字） | `zh-TW` | 中文(繁體) |
| ベトナム語 | `vi` | Tiếng Việt |

- 上記5言語を初期ラインナップとする。言語の追加は設定ファイル（`resources/i18n/supported_languages.json`）で管理し、コード変更なしで拡張可能とする
| 入力言語タブ | 選択された言語ごとにタブを生成。グリッド上部にスティッキー表示（`#mainAreaMultilingualTabs`）され、基本情報・設問文・選択肢が言語タブごとに入力可能になる |
| 言語タブの並び順 | **ドラッグ&ドロップで入れ替え可能**。最左のタブが第一言語となる |

#### 3.2.4 第一言語の決定ルール

```
+------------------------------------------------------------------+
|  言語タブ（ドラッグで並べ替え可能）                                  |
|                                                                    |
|  +---------+  +---------+  +---------+                             |
|  | English |  | 日本語   |  | 中文    |   ← ドラッグハンドル付き    |
|  +---------+  +---------+  +---------+                             |
|  ▲ 最左 = 第一言語                                                 |
|                                                                    |
|  回答画面を開いた際に、第一言語で表示される。                         |
|  回答者は回答画面上で言語切替が可能。                                |
+------------------------------------------------------------------+
```

- **第一言語**: タブの最左に配置された言語。回答画面のデフォルト表示言語となる
- **日本語の固定なし**: 日本語も含めて全言語の並び順を入れ替え可能
- **タブのドラッグ操作**: 各タブにドラッグハンドル（`drag_indicator`アイコン）を付与。Sortable.jsで並び替えを実装
- **並び順の保存**: 保存時に言語タブの順序を配列として保持する（例: `["en", "ja", "zh-TW"]`）
- **回答画面との連携**: 保存された配列の先頭要素が回答画面のデフォルト言語となる

### 3.3 関連設定セクション

名刺データ化・お礼メールの設定へのリンクをまとめるセクション。
各設定は専用の外部ページへ遷移する。

#### 3.3.1 名刺データ化設定

| 項目 | 詳細 |
|------|------|
| 開き方 | 設定カラム内のリンクカードをクリックして専用ページ（`bizcardSettings.html`）へ遷移 |
| 依存 | `bizcardEnabled = ON` の場合のみ有効 |
| 設定内容 | 既存の名刺データ化設定ページの内容を維持 |

#### 3.3.2 お礼メール設定

| 項目 | 詳細 |
|------|------|
| 開き方 | 設定カラム内のリンクカードをクリックして専用ページ（`thankYouEmailSettings.html`）へ遷移 |
| 依存 | `bizcardEnabled = ON` の場合のみ有効 |
| 設定内容 | 既存のお礼メール設定ページの内容を維持 |

#### 3.3.3 サンクス画面設定（インライン統合）

現行の別ページ（`thankYouScreenSettings.html`）の機能をアンケート作成画面内に統合する。
サンクス画面カードは設問カラムの末尾（設問リストの下）に配置する。

> **注**: サンクス画面設定はHTMLの構造上は設問カラム（`#questions-column`）内に配置されており、設定カラムの「関連設定」セクションには含まれない。

```
+--------------------------------------------+
| サンクス画面 (回答完了メッセージ)            |
|--------------------------------------------|
|                                            |
|  完了メッセージ (回答後に表示)              |
|  +--------------------------------------+  |
|  |                                      |  |
|  |                                      |  |
|  +--------------------------------------+  |
|                                            |
+--------------------------------------------+
```

| 項目 | ID | タイプ | デフォルト | バリデーション |
|------|-----|--------|-----------|---------------|
| サンクスメッセージ | `thankYouMessage` | テキストエリア | （空欄） | 最大500文字 |

- 多言語ON時、サンクスメッセージも言語タブごとに入力可能
- 名刺撮影の依頼有無は「オプション機能 > 名刺画像添付」トグルに統合済みのため、ここでは表示しない
- 連続回答の許可トグルは「オプション機能 > 連続回答の許可」（§3.2.2）に統合済みのため、ここでは表示しない

#### 3.3.4 クーポン適用状態の表示

| 項目 | 詳細 |
|------|------|
| 表示条件 | クーポンコードが適用されている場合のみ表示 |
| 表示内容 | クーポンコード、適用中であることのインジケーター |
| 操作 | 読み取り専用 |

### 3.4 メモ

| 項目 | ID | タイプ | 必須 | バリデーション |
|------|-----|--------|------|---------------|
| メモ | `memo` | テキストエリア | - | 最大1000文字 |

- 管理者のみが参照する内部メモ。回答者には非表示。
- 基本情報カード内の右カラムに配置する（設定カラムではない）。

## 4. 機能要件 — 設問カラム

### 4.1 設問リスト（フラット構成）

設問グループ機能を廃止し、設問をフラットなリストとして管理する。

```
+--------------------------------------------------+
|  設問設定                                         |
|--------------------------------------------------|
|                                                  |
|  +--------------------------------------------+  |
|  | ≡  Q1  自由記述  「ご意見をお聞かせ...」  ▼ |  |  ← Collapsed
|  +--------------------------------------------+  |
|                                                  |
|  +--------------------------------------------+  |
|  | ≡  Q2  単一選択  「満足度は？」      [▲]   |  |  ← Expanded
|  |--------------------------------------------|  |
|  | 基本設定                                    |  |
|  |   設問文: [________________]                |  |
|  |   ☑ 必須回答にする                         |  |
|  |                                            |  |
|  | 回答内容設定                                |  |
|  |   [選択肢リスト...]                        |  |
|  |                                            |  |
|  | 高度設定                                    |  |
|  |   ▶ 条件分岐を設定する                     |  |
|  +--------------------------------------------+  |
|                                                  |
|  +--------------------------------------------+  |
|  | ≡  Q3  複数選択  「設問文を入力して...」 ▼  |  |  ← Collapsed
|  +--------------------------------------------+  |
|                                                  |
|  [+ 設問を追加]                                   |
|                                                  |
+--------------------------------------------------+
```

### 4.2 設問カードの状態

| 状態 | 表示内容 |
|------|----------|
| Collapsed（折り畳み） | ドラッグハンドル / Q番号 / 設問タイプラベル / 設問文（truncate）/ 必須バッジ / タイプ変更ドロップダウン / 複製・削除ボタン / 開閉トグル |
| Expanded（展開） | 上記 ＋ 基本設定（設問文入力・必須チェック）/ 回答内容設定（タイプ別）/ 高度設定（条件分岐） |

### 4.3 設問の操作

| 操作 | 方法 |
|------|------|
| 追加 | リスト末尾の「+ 設問を追加」ボタン、またはFABメニュー |
| 並び替え | ドラッグ&ドロップ（Sortable.js） |
| 複製 | 設問カード内の複製ボタン |
| 削除 | 設問カード内の削除ボタン |
| タイプ変更 | 設問カード内のドロップダウン（下記ルール参照） |
| 番号 | Q1, Q2, Q3... 並び替え・追加・削除時に自動振り直し |

#### 設問タイプ変更時のデータ保持ルール

タイプ変更時は**確認ダイアログ**を表示し、ユーザーの明示的な承認を得てから実行する。

```
「設問タイプを変更すると、現在の回答設定が失われる場合があります。変更しますか？」
[キャンセル] [変更する]
```

| データ種別 | 変更後の扱い |
|-----------|-------------|
| 設問文（`text`） | **常に保持** — 全タイプ共通フィールドのため |
| 必須設定（`required`） | **常に保持** |
| 選択肢（`options`） | 選択肢系→選択肢系（例: single→multi）: **保持**。それ以外: **破棄** |
| タイプ固有設定（`validation`） | **破棄** — min/max/step等は新タイプのデフォルト値にリセット |
| マトリックス行・列 | マトリックス系→マトリックス系: **保持**。それ以外: **破棄** |
| 条件分岐（`jumpConfig`） | **保持** — タイプに依存しないため |
| 手書き設定 | handwriting→handwriting以外: **破棄** |

### 4.4 ドラッグ&ドロップの競合制御

同一画面内に3種類のドラッグ&ドロップ領域が存在するため、Sortable.jsのインスタンスを厳密に分離する。

| ドラッグ対象 | コンテナ | ハンドル | グループ名 |
|-------------|---------|---------|-----------|
| 設問カード | `#questionListContainer` | `.question-builder-handle` | `questions` |
| 言語タブ | `#languageEditorTabs` | `.lang-tab-handle` | `language-tabs` |
| 選択肢 | `.options-container`（設問カードごと） | `.option-handle` | `options-{questionId}` |

**競合防止ルール:**
- 各Sortable.jsインスタンスは `handle` オプションで専用のハンドル要素を指定し、カード全体のドラッグを防止
- 選択肢のドラッグイベントには `e.stopPropagation()` を設定し、親（設問カード）のドラッグに伝播しない
- 言語タブのドラッグ領域は設定カラム内に閉じており、設問カラムのドラッグとは物理的に分離されている
- ドラッグ中は対象要素に `.dragging-item` クラスを付与し、視覚的フィードバックを提供

## 7. 多言語対応の仕様

### 7.1 言語構成

| 項目 | 仕様 |
|------|------|
| 基本言語 | なし（固定言語なし、並び順で第一言語を決定） |
| 追加可能言語 | 英語(`en`)、中国語簡体字(`zh-CN`)、中国語繁体字(`zh-TW`)、ベトナム語(`vi`) — §3.2.2 の初期対応言語リスト参照 |
| 最大言語数 | 3言語（タブ3つまで） |
| 第一言語決定 | タブの最左が第一言語 |
| 対象プラン | プレミアムプラン限定 |

### 7.2 多言語入力の対象フィールド

| フィールド | セクション |
|-----------|-----------|
| 表示タイトル | 基本情報 |
| 説明 | 基本情報 |
| 設問文 | 設問カード |
| 選択肢テキスト | 設問カード（選択肢系タイプ） |
| マトリックス行・列ラベル | 設問カード（マトリックス系タイプ） |
| 説明カード本文 | 設問カード（説明カード） |
| サンクスメッセージ | 追加設定 |

### 7.3 言語タブUI仕様

```
+------------------------------------------------------------------+
|  [グリッド上部 sticky: #mainAreaMultilingualTabs, top-16, z-40]   |
|                                                                    |
|  ╔═══════════╗  ┌───────────┐  ┌───────────┐                      |
|  ║ ≡ 日本語  ║  │ ≡ English │  │ ≡ 中文⚠  │   ← ⚠ 未入力警告   |
|  ╚═══════════╝  └───────────┘  └───────────┘                      |
|  ▲ 選択中         ↕ ドラッグ可    ↕ ドラッグ可                      |
|  ▲ 最左＝第一言語                                                  |
|                                                                    |
|  タブの下部にグラデーションライン（青→インジゴ）                    |
+------------------------------------------------------------------+
```

- 言語タブは設定カラム内ではなく、3エリアグリッドの**上部にグローバル sticky** として配置される（`id="mainAreaMultilingualTabs"`）
- 多言語OFF時は `visibility: hidden` で非表示（スペースを占有しない）
- 多言語ON時は `visible` になり、ヘッダーの直下（`top-16`）に固定される
- `body.has-multi-lang-tabs` クラスが付与され、設定カラムの sticky top 位置が連動して調整される（`top: 8rem`）

- 各タブにドラッグハンドル（`≡`）を付与
- 未入力フィールドが残る言語タブには警告アイコン（`⚠`）を表示
- タブ切替時は入力内容が即座に保持される（タブ切替で入力値が消えない）

### 7.4 保存時の多言語バリデーション

| ルール | 動作 |
|--------|------|
| 第一言語の必須フィールドが空 | **ブロッカー**（保存不可） |
| 第二・第三言語の未入力 | **警告モーダル**表示（「未翻訳の項目があります」）、保存続行/中断を選択 |

### 7.5 回答画面との連携

| 項目 | 仕様 |
|------|------|
| デフォルト表示言語 | 言語タブ配列の先頭（第一言語） |
| 言語切替 | 回答画面上で回答者が切替可能 |
| 未翻訳フォールバック | 第一言語のテキストを表示し「未翻訳」バッジを付与 |

### 11.1 自動保存・下書き機能

3エリア構成により入力量が増加するため、入力中断時のデータ消失リスクに対応する。

| 項目 | 仕様 |
|------|------|
| 自動保存先 | `localStorage`（キー: `survey_draft_{surveyId}` / 新規時: `survey_draft_new`） |
| 保存トリガー | 入力フィールドの `change` / `input` イベント発火後、**debounce 5秒**で自動保存 |
| 保存対象 | 全入力値（基本情報・連携設定・追加設定・メモ・設問リスト） |
| 復元 | ページ読み込み時に下書きデータが存在する場合、「未保存の下書きがあります。復元しますか？」ダイアログを表示。復元 / 破棄を選択 |
| 下書きのクリア | サーバーへの保存成功時に自動削除 |
| `beforeunload` 警告 | 未保存の変更がある場合（サーバーに保存されていない変更）、ブラウザの離脱警告を表示 |
| サイズ制限 | `localStorage` の容量制約（5MB目安）を考慮し、手書きデータ（Base64）は下書き保存の対象外とする |
