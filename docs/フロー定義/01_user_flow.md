# ユーザー視点 エンドツーエンドフロー

| 項目 | 値 |
| --- | --- |
| 対象範囲 | アンケート作成 → 回答収集 → 名刺データ化 → お礼メール送信 |
| 主な登場人物 | 企業ユーザー・来場者（回答者）・事務局・システム |
| owner | product |
| last_reviewed | 2026-05-11 |

## 1. 登場人物

| 呼称 | 説明 |
| --- | --- |
| **企業ユーザー** | SPEED AD に契約している展示会出展企業の担当者（マーケター等） |
| **来場者** | 展示会に訪れ、QR コードでアンケートに回答するエンドユーザー（匿名） |
| **事務局** | SPEED AD 運営側のオペレーション担当（土地氏・清水氏） |
| **システム** | 人が介在せず自動で動く処理 |

---

## 2. 全体フロー概要

```
【企業ユーザー】アンケート作成・設定
        ↓
【企業ユーザー】QR コード取得・展示会で配布
        ↓
【来場者】QR 読取 → アンケート回答 → 名刺撮影 → 送信
        ↓
【システム】回答・名刺画像を受付
        ↓
【システム / 事務局】名刺データ化（OCR・入力・照合）
        ↓
【システム】お礼メール送信
        ↓
【企業ユーザー】データダウンロード
```

### Phase 2｜アンケート回答・名刺撮影（来場者）

| # | 操作 | 実行者 | 備考 |

| 2-1 | QR 読取または URL 直接アクセス | 来場者 | `survey-answer.html?surveyId={id}` |

| 2-3 | 名刺を撮影（名刺取得が有効な場合） | 来場者 | カメラ起動 → 撮影 → プレビュー確認 |

**要確認事項（Phase 2）**

### Phase 3｜名刺データ化（システム・事務局・データ入力パートナー）

| # | 処理 | 実行者 | 画面 / 手段 | 備考 |
| --- | --- | --- | --- | --- |
| 3-1 | 名刺画像・回答データの受付確認 | 事務局 | `03_admin/survey-management.html` | |

**要確認事項（Phase 3）**
- OCR の自動処理範囲（どこまで自動化、どこから手入力）

- データ化の SLA・納品期日の計算ルール（営業日カレンダーとの連動）
- DL 期限のデフォルト日数とユーザーへの通知タイミング
- 多言語名刺（英語・中国語等）の処理フロー
- データ化対象外の名刺（破損・白紙等）の扱い

## 4. ステータス遷移まとめ（アンケート）

```
草稿
  ↓ 公開操作
準備中（会期前）
  ↓ 会期開始日到達（自動）
会期中
  ↓ 会期終了日到達（自動）
会期終了
  ↓ データ化着手
会期終了（データ化中）
  ↓ データ化完了
会期終了（データ化完了）
  ↓ DL 期限到達（自動）
会期終了（DL 期限終了）
```

**要確認事項**
- 「草稿」→「準備中」のステータス変更は企業ユーザー自身が行うか、事務局が承認するか
- 会期終了の自動遷移は日付ベースか、手動操作か
