# アクションタスク管理（2025-10）

## 検証フロー
1. 仕様・受入条件を確認し、必要なデータセットや検証ユーザーを事前に準備する。
2. ブラウザ検証では表示崩れ・挙動・コンソールエラーを確認し、下記テンプレートで証跡を残す。
3. CL/Docs タスクは `rg` やリンクチェックの結果を取得し、証跡用スペースに貼り付ける。
4. 証跡を添付したらタスク管理ボードにステータス更新とレビュー依頼を行う。

### ブラウザ確認用 証跡テンプレート（必須）
```
[ブラウザ検証証跡]
- 画面キャプチャ: media/verification/2025-10/<task-id>/<page>_<yyyyMMddHHmm>.png
- コンソールログ: 有/無（保存先: media/verification/2025-10/<task-id>/<page>_console.log）
- 確認日時・環境: yyyy/MM/dd HH:mm / <Browser name + version> / <OS>
```

### FEタスク用 証跡テンプレート
```
[FE検証証跡]
- 動作確認GIF/スクショの保存パス: media/verification/2025-10/<task-id>/gif_or_img/<file>
- ブラウザ検証証跡: 上記テンプレートを流用（キャプチャとコンソールログを添付）
- 確認者署名欄: Reviewed by <Name / Date>
```

### CL/Docsタスク用 証跡テンプレート
```
[CL/Docs検証証跡]
- `rg`結果スクショ貼付: media/verification/2025-10/<task-id>/search/<file>.png
- リンクチェック結果ログ貼付: media/verification/2025-10/<task-id>/links/<file>.txt または同等のログ貼付欄
- 確認日時・環境: yyyy/MM/dd HH:mm / <Tool version> / <OS>
```
