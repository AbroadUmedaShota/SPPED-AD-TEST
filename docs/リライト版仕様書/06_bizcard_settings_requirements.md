# 名刺データ化設定画面 要件定義書

| 項目 | 値 |
| --- | --- |
| 対象画面 | `02_dashboard/bizcardSettings.html` |
| 正本区分 | 本書を正本とする |
| owner | product |
| last_reviewed | 2026-05-01 |

## 1. 文書目的

- 本書は、名刺データ化設定画面の要件を定義する。
- 本書を正本とする。
- 旧「本書が正」方針は破棄済み。

## 2. 画面概要

- アンケート単位で名刺データ化の実施有無、依頼条件、見込み枚数、追加オプション、クーポン適用、社内メモを設定する画面。
- 画面は左側の設定エリアと、右側の見積・確定エリアで構成する。
- アンケート基本情報、現在の設定値、最新の見積、クーポン適用結果をサーバーから取得して初期表示する。
- 更新はアンケート単位で永続化され、他ユーザー操作との競合を考慮する。

## 3. 画面要件

### 3.1 上部情報
- パンくずと画面タイトルを表示する。
- アンケート基本情報を表示する: アンケート名、アンケートID、会期期間。
- アンケート名と会期期間には補助説明導線（ヘルプポップオーバー）を設ける。

### 3.2 左カラム
- **データ化有効/無効トグル**「名刺のデータ化を行う」を設ける。OFF 時は設定フォームを非活性化し、画面別小計は 0 円表示とする（アンケート全体ブロックの挙動は §4.2 を参照）。
- **データ化プラン選択**: 単一選択。各プランは名称、概要、単価、納期目安を表示する。プラン詳細は補助モーダルで参照できる。
  - プランは **trial / normal / express / superExpress / onDemand** の 5 種。単価はプラン別（現在の画面は 0 / 50 / 100 / 150 / 200 円/枚）。
- **見込み枚数入力**: 数値入力と ±1 / ±10 / ±100 の増減操作を提供する。0 以上の整数のみ受け付ける。
- **プレミアムオプション**: 多言語対応などの追加オプションを複数選択可能とする。金額加算対象と説明のみの項目は明確に区別する。
- **社内管理用メモ**: 任意入力、利用者向け公開情報には含めない。

### 3.3 右カラム
右カラムは上から下に以下の順で構成する。

1. **名刺データ化の小計（税抜）**（`#estimatedAmount`）
   - 当画面の機能別金額のみ。**クーポン値引きは反映しない**（純粋な税抜小計）。
   - 「名刺機能分のみ」のラベルを併記し、当画面が部分小計であることを示す。
2. **完了予定日**（`#estimatedCompletionDate`）
   - 起算日 + 納期日数を補助表示する。
3. **詳細内訳**（`#estimateBreakdown`）
   - 想定件数、単価 × 枚数、プレミアム加算、選択オプション、最低請求注記、画面別小計（税抜）。
   - **クーポン値引行はここには出さない**（出現は §3.3.4 のアンケート全体ブロック配下のみ）。
4. **アンケート全体ブロック**（`.estimate-block-survey-total`、両画面で共通構造）
   - 見出し「アンケート全体料金（このアンケートの請求見込み）」。
   - 内訳: 「名刺データ化分 ¥X」「お礼メール分 ¥Y」を常時 2 行表示する。片方が未設定の場合は金額の代わりに「未設定（後ほど反映されます）」を表示する。
   - 集計: `税抜小計` → `クーポン値引き`（適用時のみ表示） → `税抜割引後`（適用時のみ表示） → `消費税（10%）` → `税込請求見込み`（最重要、ハイライト表示）。
   - 注記: 「※クーポン値引きはアンケート全体の料金に対して 1 回適用されます」を明示する。
   - 注記の直下に「ご請求・お支払いに関する注意事項」リンクを置き、押下で支払注意事項モーダルを開く。
5. **クーポン適用エリア**（`#couponSectionWrapper`）
   - アンケート全体ブロックの直下に配置する。
   - 未適用時はコード入力と検証導線、適用中はクーポン内容と解除導線を表示する。
   - 名刺データ化 OFF 時はクーポンも操作不可にする。
6. **対応言語**: 日本語 / 英語 / 中国語（繁体字・簡体字）／韓国語の表示。
7. **アクションボタン**: `キャンセル` / `設定を保存して依頼を確定`。

### 3.4 モーダル・補助導線
- プラン詳細モーダル
- 請求・支払い注意事項モーダル
- 未保存離脱確認モーダル
- クーポン解除確認モーダル

## 4. 状態と挙動

### 4.1 初期表示
- 画面表示時に以下を取得する。
  - アンケート基本情報
  - 既存の名刺データ化設定
  - 適用中クーポン
  - 現在の見積・完了予定日
  - 楽観ロック用 `version`
- 取得失敗時はエラーメッセージと再試行導線を表示する。
- クーポンは画面間共有キー（`sharedCoupon_{surveyId}`）を参照して復元する。

### 4.2 名刺データ化 OFF
- 入力欄は非活性化する。
- クーポン入力・検証・解除は不可とする。
- 画面別小計は 0 円とする。
- アンケート全体ブロックは引き続き表示し、お礼メール側小計のみで全体料金を計算する（名刺分は ¥0 として加算）。
- 完了予定日は「データ化を実施しない」と表示する。

### 4.3 入力変更時
- プラン、見込み枚数、プレミアムオプション、クーポン変更時は画面別小計と全体料金を再計算する。
- 入力変更後は `setSurveySubtotal(surveyId, 'bizcard', subtotal)` で localStorage に書き戻し、姉妹画面（お礼メール）からも同じ全体料金が見えるようにする。
- 再計算中は右カラムに処理中表示を行う。
- 再計算失敗時は最後に確定した値を保持しつつエラーを表示する。

### 4.4 クーポン
- 1 アンケートにつき 1 クーポンのみ適用可能。
- クーポンは **アンケート全体料金（名刺データ化小計 + お礼メール小計）に対して 1 回のみ適用** する。画面別小計には反映しない。
- 値引種別は **`discount`（金額値引き）のみ** をスコープとする。納期短縮型（旧 `speedBoost`）は本書のスコープ外であり、プロダクトとして対応しない。
- クーポンコード検証はサーバーに問い合わせる。
  - 成功時: `discountType: 'discount'`、割引値、メッセージ、適用可否を返す。
  - 失敗時: 入力欄に紐づくエラーを表示する。
- 解除時は確認後にサーバー側の適用状態を更新する。
- 状態としては `state.appliedCoupon` に **単一オブジェクト** で保持する（配列は禁止）。共有キー `sharedCoupon_{surveyId}` で姉妹画面と同じクーポンを共有する。
- クーポン値引行および「税抜割引後」行は `.estimate-block-survey-total` 配下にのみ出現させる。画面別小計エリア・詳細内訳には決して表示しない（表示層ガード）。

### 4.5 未保存離脱
- 変更がある場合、画面離脱前に確認する。
- `キャンセル` 押下時も同様に確認する。

### 4.6 保存
- 保存時は画面を一時的に操作不可とし、二重送信を防止する。
- 保存成功時は最新状態を反映し、呼出元画面（`surveyCreation.html` / `surveyCreation-v2.html`）へ戻る。
- 保存失敗時は原因に応じたメッセージを表示し、入力値は保持する。

## 5. 見積要件

### 5.1 見積入力
- 画面別小計（名刺データ化分）の計算に必要な入力:
  - `bizcardEnabled`
  - `bizcardRequest`（見込み枚数）
  - `dataConversionPlan`
  - `premiumOptions`
  - アンケート会期情報
- アンケート全体料金の計算に必要な入力:
  - 当画面の `subtotal`
  - お礼メール側の `subtotal`（localStorage `surveyEstimate_{surveyId}` 経由）
  - 適用クーポン（単一オブジェクト、なければ `null`）
  - `taxRate`（既定 `0.1`）

### 5.2 計算経路（クライアント側責務分離）
- **画面別計算（`bizcardCalculator.calculateEstimate`）**: 当画面の **税抜・クーポン未適用** の小計のみを返す。
  - 入力は §5.1 の画面別計算入力。`appliedCoupon` 引数は受け取らない（旧シグネチャから削除済み）。
  - 戻り値の `couponAmount` / `couponPercent` は互換目的で常に `0` を返す（実体は集計層）。
- **全体集計（`surveyEstimateAggregator.computeSurveyTotalEstimate`）**: 名刺・お礼メール双方の小計、クーポン、税率を受け取り、全体料金（税抜小計 → 値引き → 税抜割引後 → 消費税 → 税込）を返す純粋関数。
  - クーポン値引きはこの関数でのみ実施する（計算層ガード）。
  - 完了予定日は責務外。

### 5.3 見積結果
- **画面別小計の返却項目**（右カラム上部 / 詳細内訳で利用）:
  - 税抜小計 `subtotal`（= 互換用 `amount`）
  - 単価、見込み枚数、オプション加算額
  - 最低請求条件に基づく注記（`ceil(枚数 × 0.5) × 単価`）
  - 完了予定日 / 起算日 / 納期日数
- **アンケート全体料金の返却項目**（全体ブロックで利用）:
  - `bizcardSubtotal` / `thankYouSubtotal`
  - `subtotalAll`（税抜小計合算）
  - `couponAmount` / `couponPercent` / `couponApplied`
  - `afterCoupon`（税抜割引後）
  - `tax`（消費税、`floor(afterCoupon * taxRate)`）
  - `totalWithTax`（税込請求見込み）
  - `isPartial` / `missing`（片方未設定の検知）
- 完了予定日はバックエンド算出値を正とする。納期計算は業務日・休業日・プラン条件を考慮できること。

## 6. 連携要件（参考）

### 6.1 初期表示に必要な情報
- アンケート基本情報（名称・ID・会期期間）
- 現在の名刺データ化設定（有効/無効、枚数、プラン、オプション、社内メモ）
- クーポン適用状態
- 見積結果・完了予定日
- 競合検知用識別情報

### 6.2 保存時に必要な情報
- データ化有効/無効、見込み枚数、プラン、オプション、社内メモ、競合制御情報

### 6.3 クーポン検証で必要な情報
- クーポンコード、適用可否、割引種別（`discount` のみ）、割引値、メッセージ、適用後の全体料金プレビュー
- `speedBoost` 等の納期短縮型はスコープ外。サーバ側でも本画面向けには返さない想定とする。

### 6.4 アンケート全体料金共有のためのクライアント永続化（暫定）
- 共有キー: `surveyEstimate_{surveyId || 'temp'}`（localStorage）
  - 形式: `{ bizcard: { subtotal: number, updatedAt: ISO } | null, thankYou: { subtotal: number, updatedAt: ISO } | null, schemaVersion: 1 }`
  - 名刺データ化設定保存・編集時に `bizcard` を更新する。お礼メール画面は `thankYou` を更新する。
  - 当面はクライアント間連携用。本番化時はサーバ管理に移行する（§11.4 参照）。
- クーポン共有キー `sharedCoupon_{surveyId}` は引き続き利用する（本番化時はサーバ管理へ）。

### 6.5 設計上の論点
- エンドポイント命名、HTTP メソッド、競合制御方式、エラー応答形式
- 見積を都度再計算するか、保存時に再評価するか
- 完了予定日を見積系レスポンスに含めるか別責務に切り出すか
- アンケート全体料金（`surveyEstimate_*`）のサーバ移管時の取り扱い方針

## 7. 失敗系要件

- **バリデーションエラー**: 見込み枚数未入力/範囲外、プラン未選択
- **競合更新**: 他ユーザー更新を検知し、再読込または再取得導線を表示
- **認可エラー**: 編集権限がない場合は入力不可とし、理由を表示
- **一時障害**: 取得・保存・クーポン検証失敗時に再試行可能とする

## 8. 非機能要件

- 入力変更に対する見積更新は利用者を待たせない応答時間で行う。
- 二重送信防止。
- 画面再読込後も保存済み設定が正しく再現される。
- 楽観ロックにより他ユーザー更新との競合を検知できる。
- プラン連動（契約プランによる表示制限）に対応する。

## 9. 手動確認観点

- 初期表示で既存設定と見積が正しく表示される。
- ON/OFF 切替で入力可否・画面別小計・全体料金が正しく変わる。
- 見込み枚数・プラン・オプション変更で画面別小計・全体料金・完了予定日が更新される。
- 画面別小計（`#estimatedAmount` および詳細内訳）には **クーポン値引きが反映されない** ことを確認する。
- アンケート全体ブロックでのみ「クーポン値引き」「税抜割引後」行が出現することを確認する。
- お礼メール側設定が完了している場合は「お礼メール分」金額が、未完了の場合は「未設定（後ほど反映されます）」が表示される。
- クーポン適用/解除がサーバー応答に応じて全体料金にのみ反映される。
- 保存成功/失敗、競合更新、認可エラーが正しく表示される。
- 他画面（お礼メール設定等）と共有中のクーポンおよび小計が正しく再現される。

## 10. 本番実装で対応すること

画面外の確認事項は以下のとおり。

- **プラン連動（`plan-capabilities.json`）が未適用** — 契約プラン別の表示制限・上限が効かない
- **保存処理** — 保存時はサーバーへ送信し、成功/失敗を画面に表示する
- **クーポン検証** — サーバー側で有効性を確認する。`discountType` は `discount` のみを返却対象とする
- **プレミアム多言語の単価が未定義** — 料金加算ロジックが実質無効
- **追加項目の料金加算ロジックなし** — 選択肢表示のみで金額に反映されない
- **クーポン共有キー（`sharedCoupon_*`）がクライアント localStorage** — サーバ側管理へ移行
- **アンケート全体料金共有キー（`surveyEstimate_*`）がクライアント localStorage** — サーバ側管理（中間テーブル等）へ移行
- **楽観ロック画面外の確認事項** — 他ユーザー更新との競合検知なし

## 11. 本番化要件

本セクションでは、完成済み画面を本番環境へ移す際に、画面からは直接確定できないため開発会社と確認する事項をまとめる。API・DB・ログ・権限などの詳細は、画面上の確定仕様ではなく実装時の確認観点である。

### 11.1 API 契約

以下は確認用のたたき台であり、画面上の確定仕様ではない。詳細スキーマ・パスは開発会社と合意して確定する。

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/surveys/{surveyId}/bizcard-settings` | 初期表示（設定・画面別小計・全体料金・クーポン・version を一括取得） | — | 200 `{ settings, estimate, surveyTotal, coupon, version }` ／ 403 認可エラー ／ 404 |
| `POST /api/surveys/{surveyId}/bizcard-settings/estimate` | 画面別小計（名刺分）の再計算 | `{ bizcardEnabled, bizcardRequest, dataConversionPlan, premiumOptions }` | 200 画面別小計（§5.3 参照）／ 400 バリデーション |
| `GET /api/surveys/{surveyId}/total-estimate` | アンケート全体料金プレビュー | — | 200 `{ bizcardSubtotal, thankYouSubtotal, subtotalAll, couponAmount, afterCoupon, tax, totalWithTax, missing[] }` |
| `PUT /api/surveys/{surveyId}/bizcard-settings` | 設定保存 | `{ ...設定, version }` | 200 `{ settings, version }` ／ 409 競合 ／ 400 ／ 403 |
| `POST /api/surveys/{surveyId}/coupons/validate` | クーポン検証（`discountType: 'discount'` のみ） | `{ code }` | 200 `{ valid, discountType, discountValue, message, surveyTotalPreview }` ／ 400 |
| `DELETE /api/surveys/{surveyId}/coupons` | クーポン解除 | — | 200 `{ surveyTotalPreview }` |

- 画面別小計はクーポン未適用の税抜額のみを返却する。クーポン値引きは `total-estimate` / `coupons/validate` の応答にのみ含める。
- 見積と保存のトランザクション境界は別とし、保存時にサーバ側で最終全体料金を再評価する。
- プラン単価マスタ（`trial/normal/express/superExpress/onDemand`）はサーバ側管理とし、クライアントはマスタ参照 API 経由で取得する想定。
- クーポン種別 `speedBoost`（納期短縮）は本画面のスコープ外とし、API 応答に含めない方針。

### 11.2 権限・認可

| 利用者種別 | 取扱 |
| --- | --- |
| アンケート所有者・編集権限保持者 | 本画面の全操作可 |
| 閲覧のみ権限 | 表示のみ可、入力欄・保存ボタンは非活性（§7 認可エラー準拠） |
| 未ログイン | アクセス不可（ダッシュボード共通の認可で弾く） |
| 契約プラン非対応の機能 | プラン連動により入力欄を非活性化（`plan-capabilities.json` 参照） |

### 11.3 業務エラー時の挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| 初期取得失敗（5xx / ネットワーク） | エラーメッセージ＋再試行導線 | フォーム非表示、再試行で再取得 |
| 見積再計算失敗 | 右カラムにエラー表示、最後に成功した見積を保持 | 自動リトライなし、入力値は保持 |
| クーポン検証失敗（400） | 入力欄直下にサーバ返却メッセージを表示 | フォーム保持 |
| 保存バリデーションエラー（400） | 該当項目にエラー表示 | 入力値保持 |
| 楽観ロック競合（409） | 競合通知＋再読込導線 | 入力値を一時退避し再取得を促す |
| 認可エラー（403） | 「この画面を編集する権限がありません」 | フォーム非活性化 |

### 11.4 データ永続化

- 既存アンケートテーブルに名刺データ化設定項目を追加（`bizcard_enabled` / `bizcard_request_count` / `data_conversion_plan` / `premium_options`（JSON 配列）／`internal_memo` / `version`）。
- プラン単価・プレミアムオプション単価は独立マスタテーブルで管理（`plan_pricing_master` 等、想定）。
- クーポン適用状態はアンケートとクーポンの中間テーブル（1 アンケート 1 クーポン制約）で管理し、サーバ側を正とする（現行 `sharedCoupon_*` localStorage は廃止）。
- アンケート全体料金（名刺・お礼メール双方の小計）はサーバ側で算出する責務とし、クライアントの `surveyEstimate_*` localStorage は本番化時に廃止する。
- 画面別小計・全体料金ともキャッシュは持たず、都度算出を原則とする。

### 11.5 ログ・監査

- 操作ログ: 設定保存、クーポン適用・解除、プラン変更（誰が／いつ／どのアンケート／前後値）。
- 監査ログ: プラン連動制限に抵触した操作試行、認可エラー発生。
- 失敗ログ: 見積 API・保存 API・クーポン検証 API の 5xx 応答を記録。
- 保管期間は運用方針に従う（開発会社と確認）。

### 11.6 想定データ量

- 1 アンケートあたり名刺数は数十〜数千枚を想定。大規模キャンペーンで最大数万枚規模。
- 設定画面の同時編集者は同一アンケートに対して 1〜数名程度。楽観ロックで十分。
- 見積 API は入力中頻発するためデバウンス（数百ms）を前提とする。

### 11.7 対応ブラウザ・デバイス

- 想定対応: 最新版 Chrome / Edge / Safari / Firefox（直近 1 年以内のメジャーバージョン）。
- レスポンシブ対応: PC（横幅 ≥ 1024px）／タブレット／スマートフォン（横幅 ≥ 375px）。
- IE および非 Evergreen ブラウザは対象外。

### 11.8 受け入れ基準

- [ ] 初期表示で既存設定・クーポン・画面別小計・アンケート全体料金・完了予定日が一括取得され、再描画ちらつきがない
- [ ] プラン・枚数・オプション変更で画面別小計が再計算され、最低請求条件の注記が反映される
- [ ] **画面別小計（`#estimatedAmount` および詳細内訳）にはクーポン値引きが反映されない**（純粋な税抜小計）
- [ ] **クーポン値引行および「税抜割引後」行はアンケート全体ブロック（`.estimate-block-survey-total`）配下にのみ出現する**
- [ ] クーポン適用/解除でアンケート全体料金（税抜割引後・税込請求見込み）のみが変動する
- [ ] お礼メール側の設定が完了している場合、全体ブロックに該当小計が表示される。未完了の場合「未設定（後ほど反映されます）」が表示される
- [ ] 完了予定日はサーバ算出値が表示される（クライアントで再計算しない）
- [ ] 保存後の再読込で保存内容が完全再現される（`surveyEstimate_*` 経由で姉妹画面とも整合）
- [ ] 他ユーザーが先に保存した場合、409 を受けて競合通知＋再読込導線が出る
- [ ] 非プレミアムプランでプレミアム機能が入力不可になる
- [ ] 名刺データ化 OFF 時は入力欄・クーポン操作が非活性、画面別小計が 0 円固定になる（全体料金はお礼メール側のみで算出される）
- [ ] 保存 / クーポン検証で二重送信が発生しない
- [ ] `state.appliedCoupon` は単一オブジェクト（配列にならない）であることが実装上保証される
