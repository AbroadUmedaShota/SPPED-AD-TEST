## 1. 文書目的

- SPEED AD の名刺データ化業務における営業日カレンダーの管理と、各営業日に発生するアンケート案件への担当会社アサイン操作の要件を定義する。
- 本画面は **アサインの正本** であり、日付軸での俯瞰と一括アサインを単一画面で完結させる。個別案件単位のアサイン専用画面は新設しない。
- オペレーターの稼働実績・KPI 等の分析機能は本書の対象外とし、AD-10 実績管理の責務とする。
- オペレーター個人の情報管理は本書の対象外とし、AD-09 オペレーター管理の責務とする。

---

## 2. 画面概要

### 2.1 営業日カレンダー（AD-06）

管理者が営業日ルールの設定と、各日付に発生するアンケート案件への担当会社アサイン操作を行う画面。本画面は AD 配下のカレンダー管理および担当会社アサインの単一エントリポイントであり、個別日付の操作・案件アサインは 2.2 のフォーカスエリアで行う。

画面構成:
- ヘッダー: 画面タイトル「営業日カレンダー管理」、カレンダー CSV 出力ボタン、休業日 CSV 取込ボタン、祝日 API 同期ボタン。
- カレンダー本体: 月表示・年表示の切替トグル、前後ナビ、「今日」ボタン、凡例、フィルタパネル、日付セル群。
- 下部 2 ペイン: 左に「休業日の個別設定」テーブル、右に「更新ログ」テーブル。
- 削除確認モーダル: 個別設定削除時に表示。

### 2.2 フォーカスエリア（右パネル）

特定日付（単日または範囲選択）の詳細確認と操作を行う右側スライドインパネル。カレンダーの日付セルクリック、ドラッグ範囲選択、個別設定テーブルの編集ボタン押下のいずれからも開く。

タブ構成:
- アンケート詳細タブ: 当該日（または期間内）のアンケート一覧を表示。
- アサインタブ: 当該日のアンケートに対する担当会社アサイン操作。
- 営業日設定タブ: 当該日（または期間）を特別休業日／振替営業日／設定解除のいずれかに変更。

## 3. 機能要件

### 3.1 カレンダー表示・ナビゲーション

#### 3.1.1 表示モード

- 月表示・年表示の 2 モードを提供する。デフォルトは月表示。
- 縦表示モードは提供しない。
- 表示モード切替時は現在の対象月／年を維持する。

#### 3.1.2 月／年の移動

- 前月／翌月ボタンで 1 ヶ月ずつ移動する（月表示時）。
- 前年／翌年ボタンで 1 年ずつ移動する（年表示時）。
- 「今日」ボタンで当月へ即時ジャンプし、本日セルへスクロールする。

#### 3.1.3 日付セルの表示要素

各日付セルには以下を表示する。

- 日付数字（日曜は赤、土曜は青で着色）
- 日種別ラベル（営業日 / 休業日 / 振替営業日）
- 当該日に該当期間を持つアンケート一覧（最大 2 件、超過時は「他 N 件」表示）
- アンケート種別バッジ（⭐通常 / ⚡オンデマンド）
- 募集状況による背景色（未アサイン: ピンク系 / アサイン済: 緑系）
- メモアイコン📌（メモが登録されている日）

#### 3.1.4 凡例

カレンダー上部に以下の凡例を表示する。
- ⚡: オンデマンド
- ⭐: 通常
- 📌: メモあり
- ピンク背景: 未アサイン
- 緑背景: アサイン済

#### 3.1.5 ドラッグ範囲選択

- カレンダー上で複数日をドラッグして範囲選択できる。
- 範囲選択完了時にフォーカスエリアが開き、選択範囲に対する一括操作（営業日設定・メモ・アサイン）が可能となる。

#### 3.1.6 フィルタ

カレンダー全体に対する絞り込みを行う。複数条件の AND 検索とする。

- 種別: 通常 / オンデマンド / 両方
- 募集状況: 追加募集中のみ / 未アサインのみ / 自社アサイン済のみ
- 担当会社: 単一選択（**Lv4 / Lv3 のみ表示**。Lv2 は自社固定でフィルタ項目非表示）

フィルタ条件のリセット機能を提供する。

#### 3.1.7 ホバー時の追加表示

セルにマウスホバーした際の追加ツールチップは提供しない。詳細は日付クリックでフォーカスエリアから確認する。

---

### 3.2 個別休業日設定

#### 3.2.1 単日設定

- カレンダー日付セルをクリックするとフォーカスエリアの「営業日設定」タブが開く。
- 種別ラジオ（特別休業日 / 振替営業日 / 設定解除）と理由を入力して保存する。
- 「特別休業日」「振替営業日」を選択した場合、理由入力を必須とする。
- 「設定解除」を選択した場合、既存の個別設定があれば削除する。理由入力は不要。
- 保存実行は Lv4 / Lv3 のみ可能とする。Lv2 は閲覧のみ（保存ボタン非表示）。

#### 3.2.2 範囲設定

- ドラッグ範囲選択で複数日を選択した状態でフォーカスエリアを開き、種別と理由を一括適用する。
- 範囲内の既存個別設定は新しい設定で上書きする。
- 保存実行時は範囲内の全日付分の管理者操作ログを記録する。

#### 3.2.3 編集

- 個別設定一覧テーブルの「編集」ボタンを押下するとフォーカスエリアの「営業日設定」タブが開く。
- 編集ボタンは Lv4 / Lv3 のみ表示する。Lv2 は閲覧のみ。

#### 3.2.4 削除

- 個別設定一覧テーブルの「削除」ボタンを押下すると削除確認モーダルを表示する。
- 削除実行は Lv4 / Lv3 のみ可能とする。
- 削除結果は管理者操作ログに記録する。

#### 3.2.5 全社共通

- 特別休業日・振替営業日は全社共通の設定として管理し、会社ごとに異なる設定は持たない。
- 営業日ルールの追加・編集・削除は Lv4 / Lv3 のみ実行できる。

---

### 3.3 アサイン

#### 3.3.1 アサインタブの構成

フォーカスエリアの「アサインタブ」に以下を配置する。

| 要素 | 内容 | 表示条件 |
| ---- | ---- | -------- |
| 担当会社プルダウン | アサイン対象の担当会社を選択 | Lv4 / Lv3 のみ |
| 担当会社ラベル | 自社名を固定表示 | Lv2 のみ |
| 必要枚数 / アサイン済枚数 / 自社割当枚数入力 | 「必要 N 枚 / アサイン済 N 枚 / 自社割当 [入力] 枚」 | 全ロール |
| 募集状況バッジ | 未アサイン / 追加募集中 / 募集完了 のいずれか | 全ロール |
| 募集締切ボタン | 手動で募集完了に切替 | Lv4 / Lv3 のみ |
| 再募集ボタン | 募集完了から追加募集中に戻す | Lv4 / Lv3 のみ |
| 保存ボタン | アサイン内容を確定 | 全ロール（Lv2 は自社の割当のみ） |

#### 3.3.2 アサインの保存

- 担当会社と自社割当枚数を入力し、保存ボタン押下でアサインを確定する。
- 保存時にサーバー側で `Σ(各社割当枚数) ≤ 案件の必要枚数（expected_cards）` を再検証する。
- 同一案件への既存アサインが存在する場合、当該会社の割当枚数を上書き保存する。
- 自社割当枚数を 0 にして保存することでアサイン解除となる。
- 保存結果は管理者操作ログに記録する。

#### 3.3.3 範囲アサイン

- ドラッグ範囲選択で複数日を選択した状態でアサインタブを開いた場合、選択範囲内の全アンケートに対して一括アサインを実行する。
- 範囲内に複数のアンケートが含まれる場合、対象アンケート一覧を表示し、ユーザーが一括 / 個別を選択する。
- 範囲アサイン保存時は対象アンケート分の管理者操作ログを記録する。

#### 3.3.4 アサイン解除

- アサイン済状態のアサインタブで「アサイン解除」ボタンを表示する。
- アサイン解除は当該会社のアサイン分のみが対象。他社のアサインには影響しない。
- 解除実行は当該会社のアサイン操作権限を持つロール（Lv4 / Lv3 / 自社の Lv2）が行える。

#### 3.3.5 オンデマンド × 休業日

- オンデマンドアンケートが休業日（土日・祝日・特別休業日）に該当期間として乗っている場合、当該日のカレンダー表示は「営業日」に上書きする。
- 上書きは表示のみで、営業日マスタの正本データは変更しない。
- ⚡オンデマンドバッジで識別可能とする。

---

### 3.4 メモ

#### 3.4.1 単日メモ

- カレンダー日付セルをクリックするとフォーカスエリアの「アンケート詳細」タブにメモ入力欄が表示される。
- メモは自由記述（最大 500 文字）とし、保存ボタンで確定する。
- メモが登録された日付セルには📌アイコンを表示する。

#### 3.4.2 範囲メモ

- ドラッグ範囲選択時、選択範囲の全日付に同一メモを一括保存する。
- 既存メモは新しいメモで上書きする。

#### 3.4.3 メモの権限

- メモの登録・編集・削除は Lv4 / Lv3 / Lv2 が実行できる。
- Lv2 は自社の業務に関するメモのみ登録可能（テナント分離は §4.3 参照）。

---

### 3.5 個別設定一覧

#### 3.5.1 表示

- ページネーション付きのテーブル形式で個別休業日・振替営業日設定を一覧表示する。
- 1 ページあたりの表示件数は 9 件とする。
- 各行に表示する情報: 日付・種別（特別休業日 / 振替営業日）・理由・操作（編集 / 削除）。
- ソート順は登録日時の降順（新しい順）とする。

#### 3.5.2 操作

- 編集ボタン押下時はフォーカスエリアの「営業日設定」タブが開く（§3.2.3 参照）。
- 削除ボタン押下時は削除確認モーダルを表示する（§3.2.4 参照）。
- Lv2 は編集・削除ボタンを非表示とし、閲覧のみ可能とする。

---

### 3.6 更新ログ

#### 3.6.1 表示

- ページネーション付きのテーブル形式で操作ログを一覧表示する。
- 1 ページあたりの表示件数は 7 件とする。
- 各行に表示する情報: 日時・ユーザー・操作内容。
- ソート順は操作日時の降順（新しい順）とする。

#### 3.6.2 記録対象操作

- 個別休業日・振替営業日の登録・編集・削除
- アサイン保存・解除・募集締切・再募集
- メモの登録・編集・削除
- 祝日 API 同期実行
- CSV 出力・取込

#### 3.6.3 Lv2 のスコープ

- Lv2 がアクセスした場合は自社操作に関するログのみ表示する（§4.3 参照）。

---

### 3.7 CSV 出力・取込

#### 3.7.1 カレンダー CSV 出力

- ヘッダーの「カレンダー CSV 出力」ボタンから実行する。
- 出力対象は対象月（カレンダーの現在表示月）の全日付。
- 出力カラム: `date`, `day_of_week`, `status`, `status_label`, `source`, `reason`, `memo`。
  - `date`: YYYY-MM-DD 形式の日付
  - `day_of_week`: 日 / 月 / 火 / 水 / 木 / 金 / 土
  - `status`: `WORKDAY` / `HOLIDAY` / `SUBSTITUTE_WORKDAY`
  - `status_label`: 営業日 / 休業日 / 振替営業日
  - `source`: `DEFAULT`（曜日由来） / `NATIONAL_HOLIDAY`（祝日 API 由来） / `MANUAL`（個別設定由来）
  - `reason`: 個別設定の理由（MANUAL 時のみ。それ以外は空欄）
  - `memo`: 当該日のメモ（存在しなければ空欄）
- ファイル形式: UTF-8 BOM 付き / CRLF 改行 / カンマ区切り / ヘッダー行あり。
- Lv4 / Lv3 / Lv2 が実行可能。Lv2 は自社操作範囲のメモのみ出力対象に含める。
- 実行結果は管理者操作ログに記録する。

#### 3.7.2 休業日 CSV 取込

- ヘッダーの「休業日 CSV 取込」ボタンから実行する。Lv4 / Lv3 のみ実行可能。Lv2 は閲覧のみ。
- 取込対象は個別設定（特別休業日 / 振替営業日）すべて。
- 取込カラム: `date`, `event_type`, `reason`（全て必須）。
  - `date`: YYYY-MM-DD 形式
  - `event_type`: `HOLIDAY` / `SUBSTITUTE_WORKDAY`
  - `reason`: 最大 200 文字
- ファイル形式: UTF-8 BOM 付き or UTF-8 BOM 無し / CRLF または LF 改行 / カンマ区切り / ヘッダー行必須（列順はヘッダー名でマッピング）。

#### 3.7.3 取込プレビューと重複処理

- 取込ファイルをアップロードすると確認画面を表示する。
- プレビュー画面に取込予定行を表示し、既存設定と日付が重複する行をハイライト表示する。
- 重複行の処理モードをユーザーが選択する。
  - **上書き**: 既存設定を新 CSV の値で置換する。
  - **スキップ**: 重複行は取込対象から除外する。
- 確認ボタン押下で取込を実行する。

#### 3.7.4 バリデーションとエラー処理

- 行単位でバリデーションを実施する。
  - 日付形式（YYYY-MM-DD 以外はエラー）
  - `event_type` の値（`HOLIDAY` / `SUBSTITUTE_WORKDAY` 以外はエラー）
  - `reason` の文字数（200 文字超はエラー）
- エラー行のみ取込対象から除外し、正常行は取込を実行する（部分取込可能）。
- エラー行を行番号付きで一覧表示するエラーレポート画面を表示する。
- エラーレポート画面から「エラー行のみ CSV ダウンロード」を提供し、修正後の再取込を支援する。

#### 3.7.5 テンプレートダウンロード

- 取込画面に「テンプレート CSV ダウンロード」ボタンを配置する。
- ヘッダー行のみの空テンプレートを提供する（`date`, `event_type`, `reason`）。

#### 3.7.6 取込ログ

- 取込実行ごとに管理者操作ログへ記録する。
- 記録項目: 実行者・実行日時・取込件数（成功 / 失敗 / スキップ）・処理モード（上書き / スキップ）。

---

### 3.8 祝日 API 同期

#### 3.8.1 同期実行

- ヘッダーの「祝日 API 同期」ボタンから手動実行する。Lv4 / Lv3 のみ実行可能。
- 外部祝日 API から日本国民の祝日情報を取得し、`holidays` マスタに反映する。
- 取得対象期間は当年と翌年（必要に応じて運用で前年も含む）とする。
- 既存の祝日データと重複する場合は上書きする。

#### 3.8.2 同期結果

- 同期完了後に取得件数（追加 / 更新 / 削除）を表示する。
- 同期実行結果は管理者操作ログに記録する。

#### 3.8.3 自動同期

- 自動同期スケジュール（バッチ実行）の要件は本書では定義しない。本番化前に確定する。

## 4. 状態と挙動

### 4.1 日種別の判定優先順位

日付の表示種別は以下の優先順位で判定する。

1. 個別設定（特別休業日 / 振替営業日 / 設定解除）が存在する場合は個別設定の `event_type` を優先。
2. 祝日 API 取得結果に該当する場合は `HOLIDAY` とする。
3. 土曜・日曜は `HOLIDAY` とする。
4. 上記いずれにも該当しない平日は `WORKDAY` とする。

オンデマンドアンケートが該当期間として乗っている `HOLIDAY` は、カレンダー表示上のみ `WORKDAY` として上書き表示する（§3.3.5）。営業日マスタの正本値は変更しない。

### 4.2 案件の募集ステータス遷移

案件（アンケート）の募集ステータスは以下の 3 状態で管理する。

| ステータス | 条件 | カレンダーでの表示範囲 |
| ---------- | ---- | ---------------------- |
| 未アサイン | アサイン済の会社が 0 件 | 全ロール（Lv2 含む） |
| 追加募集中 | アサイン済の会社が 1 件以上 かつ `Σ(各社割当枚数) < 必要枚数` | 全ロール（Lv2 含む） |
| 募集完了   | `Σ(各社割当枚数) ≥ 必要枚数` または 募集締切ボタンによる手動切替 | アサイン済の会社のロール のみ表示。他社の Lv2 には非表示 |

ステータス遷移:
- **自動遷移（枚数ベース）**: 各社の割当枚数の保存・解除時に `Σ(各社割当枚数) ≥ 必要枚数` を検知して `募集完了` へ自動遷移する。割当枚数が必要枚数を下回った場合は `追加募集中` へ自動復帰する。
- **手動遷移**: Lv4 / Lv3 は「募集締切」ボタンで `追加募集中` から `募集完了` へ手動切替できる。「再募集」ボタンで `募集完了` から `追加募集中` へ戻せる。
- Lv2 は手動遷移操作を実行できない。

### 4.3 テナント分離

ロールごとのアクセス範囲。

| ロール | カレンダー上のアンケート表示 | アサイン操作 | 個別休業日設定 | メモ | 更新ログ |
| ------ | ---------------------------- | ------------ | -------------- | ---- | -------- |
| Lv4 アブロードマネージャー | 全社案件を表示 | 全社へのアサイン操作可 | 編集可 | 編集可 | 全社ログを表示 |
| Lv3 アブロードスタッフ | 全社案件を表示 | 全社へのアサイン操作可 | 編集可 | 編集可 | 全社ログを表示 |
| Lv2 オペレーター管理者 | 未アサイン / 追加募集中 案件は全社可視。募集完了は自社アサイン分のみ可視 | **自社のアサインのみ操作可** | 閲覧のみ | 自社業務分のみ編集可 | 自社操作分のみ表示 |
| Lv1 オペレーター | **アクセス不可（メニュー非表示・URL 拒否）** | – | – | – | – |

Lv2 がアクセスした場合のサーバー側挙動:
- アンケート一覧 API は「未アサイン または 追加募集中」案件は全社を返却、「募集完了」案件は `WHERE assignment.company_id = request.user.company_id` で絞り込む。
- アサイン保存 API は `request.user.company_id` を強制し、自社以外の `company_id` を指定したリクエストを 403 で拒否する。
- 個別休業日設定の更新・削除 API は Lv2 からのリクエストを 403 で拒否する。
- 募集締切・再募集 API は Lv2 からのリクエストを 403 で拒否する。

## 5. 失敗系要件

| ケース | ユーザー表示 | 挙動 |
| ------ | ------------ | ---- |
| アサイン保存: 自社割当枚数が `必要枚数 - Σ(他社割当枚数)` を超過 | 「自社割当枚数が必要枚数を超過しています」 | 保存を中止し入力欄にエラー表示 |
| アサイン保存: 競合更新（他管理者が同時にアサイン変更） | 「他の管理者により更新されています。最新情報を確認してください」 | 保存を中止し再読み込み導線を表示 |
| アサイン保存: Lv2 が自社以外を指定 | 「この操作を実行する権限がありません」 | API 側で 403 拒否し操作ログに記録 |
| 募集締切: Lv2 が実行 | 「この操作を実行する権限がありません」 | API 側で 403 拒否 |
| 個別設定保存: 理由未入力（特別休業日 / 振替営業日） | 「理由を入力してください」 | 保存を中止し入力欄にエラー表示 |
| 個別設定削除: Lv2 が実行 | 「この操作を実行する権限がありません」 | API 側で 403 拒否 |
| CSV 取込: ヘッダー不正 | 「必須列（date, event_type, reason）が見つかりません」 | 取込を中止しエラー表示 |
| CSV 取込: 行単位バリデーションエラー | 「N 件のエラー行を除外して取り込みました」＋ エラーレポート | 正常行のみ取込（部分取込）、エラー行をレポートに記録 |
| CSV 取込: 重複行検出 | 「N 件の重複があります。処理モードを選択してください」 | プレビュー画面で上書き / スキップを選択 |
| CSV 取込: Lv2 が実行 | 「この操作を実行する権限がありません」 | ボタン非表示 / API 側で 403 拒否 |
| 祝日 API 同期: API 接続失敗 | 「祝日 API への接続に失敗しました。時間をおいて再試行してください」 | 同期を中止し再試行ボタンを表示 |
| 祝日 API 同期: Lv2 が実行 | 「この操作を実行する権限がありません」 | ボタン非表示 / API 側で 403 拒否 |
| メモ保存: 500 文字超過 | 「メモは 500 文字以内で入力してください」 | 保存を中止し入力欄にエラー表示 |
| 範囲選択: 開始日 > 終了日 | （自動で開始日と終了日を入替えて処理） | 入替後の範囲で処理を継続 |
| フィルタ: 0 件ヒット | 「条件に一致する案件が見つかりませんでした」 | 空のカレンダーを表示 |

## 6. 非機能要件

- カレンダーの月切替レスポンスは 1 秒以内を目標とする。
- 個別設定一覧・更新ログのページネーションは各テーブルで独立とする。
- 更新・削除・送信など副作用を伴う API は Idempotency-Key により多重実行を防止する。
- CSV 取込は同期的に処理する場合、最大 1,000 行までを上限とする。それを超える場合はバックグラウンド処理とし、完了通知を行う。
- 祝日 API 同期は同期的に処理する。

## 7. 本番化要件

### 7.1 API（確認観点）

以下はたたき台であり、詳細スキーマ・パスは開発会社と合意して確定する。UI による非表示や導線制御は補助であり、保存・認可・状態遷移・テナント分離は API 側で成立させる。

| エンドポイント | 用途 | 主要レスポンス |
| -------------- | ---- | -------------- |
| `GET /api/admin/calendar/days?from={date}&to={date}` | 期間内の日種別・募集状況一覧取得 | `{ days[] }` |
| `GET /api/admin/calendar/manual-settings` | 個別休業日・振替営業日一覧（ページネーション） | `{ settings[], total, page }` |
| `POST /api/admin/calendar/manual-settings` | 個別休業日・振替営業日の登録（単日 / 範囲） | 201 / 400 理由未入力 / 403 権限なし |
| `PATCH /api/admin/calendar/manual-settings/{id}` | 個別設定の編集 | 200 / 403 権限なし / 409 競合 |
| `DELETE /api/admin/calendar/manual-settings/{id}` | 個別設定の削除 | 200 / 403 権限なし |
| `GET /api/admin/calendar/memos?date={date}` | メモ取得 | `{ memo }` |
| `POST /api/admin/calendar/memos` | メモ登録・編集（単日 / 範囲） | 200 / 400 文字数超過 |
| `DELETE /api/admin/calendar/memos/{date}` | メモ削除 | 200 |
| `GET /api/admin/calendar/surveys?from={date}&to={date}` | 期間内のアンケート一覧（テナント分離適用） | `{ surveys[] }` |
| `POST /api/admin/calendar/assignments` | 担当会社アサイン保存（単日 / 範囲 / 単一案件 / 一括） | 200 `{ succeeded, failed[] }` / 403 / 409 |
| `DELETE /api/admin/calendar/assignments/{id}` | アサイン解除 | 200 / 403 |
| `POST /api/admin/calendar/surveys/{id}/recruitment/close` | 募集締切（手動遷移） | 200 / 403 / 400 既に募集完了 |
| `POST /api/admin/calendar/surveys/{id}/recruitment/reopen` | 再募集（手動遷移） | 200 / 403 / 400 既に追加募集中 |
| `POST /api/admin/calendar/holidays/sync` | 祝日 API 同期実行 | 200 `{ added, updated, deleted }` / 502 外部 API 接続失敗 |
| `GET /api/admin/calendar/export?from={date}&to={date}` | カレンダー CSV 出力 | CSV ストリーム |
| `POST /api/admin/calendar/import` | 休業日 CSV 取込（プレビューモード対応） | 200 `{ preview[], duplicates[], errors[] }` / 201 確定実行 |
| `GET /api/admin/calendar/import/template` | 取込テンプレート CSV ダウンロード | CSV ストリーム |
| `GET /api/admin/calendar/companies` | 担当会社一覧（プルダウン用） | `{ companies[] }` |
| `GET /api/admin/calendar/operation-logs` | 更新ログ取得（テナント分離適用） | `{ logs[] }` |

### 7.2 権限・認可

AD-06 で扱うロールは以下とする。

| ロール | AD-06 での扱い |
| ------ | -------------- |
| Lv4 アブロードマネージャー（authority=4） | フルアクセス、全社横断 |
| Lv3 アブロードスタッフ（authority=3） | フルアクセス、全社横断 |
| Lv2 オペレーター管理者（authority=2） | 自社アサイン操作のみ可。個別休業日設定・募集締切判断は不可（§4.3 参照） |
| Lv1 オペレーター（authority=1） | **AD-06 アクセス不可（メニュー非表示・URL 直叩き拒否）** |

| 操作 | Lv4 | Lv3 | Lv2 | 備考 |
| ---- | --- | --- | --- | ---- |
| カレンダー閲覧 | 可 | 可 | 自社可視範囲のみ | テナント分離 §4.3 |
| アンケート閲覧 | 可 | 可 | 未アサイン / 追加募集中 は全社可視。募集完了は自社アサイン分のみ | テナント分離 §4.3 |
| アサイン保存・解除 | 可 | 可 | **自社へのアサインのみ** | API 側で `company_id` 強制 |
| 個別休業日設定（登録・編集・削除） | 可 | 可 | 不可（閲覧のみ） | 全社共通設定のため |
| 募集締切・再募集 | 可 | 可 | **不可** | Lv2 は判断不可 |
| メモ登録・編集・削除 | 可 | 可 | 自社業務分のみ可 | テナント分離 §4.3 |
| 祝日 API 同期 | 可 | 可 | 不可 | – |
| カレンダー CSV 出力 | 可 | 可 | 可（自社範囲） | – |
| 休業日 CSV 取込 | 可 | 可 | 不可 | 全社共通設定の変更を伴うため |
| 更新ログ閲覧 | 可 | 可 | 自社操作分のみ | テナント分離 §4.3 |
| 更新ログの削除・編集 | 不可 | 不可 | 不可 | 全管理者に対して禁止 |

権限判定は画面表示だけでなく API 側でも再検証し、直接 URL アクセスや API 直叩きによる操作を拒否する。テナント分離も API 側で強制する。

### 7.3 データ永続化

- 営業日マスタは曜日由来の `WORKDAY` / `HOLIDAY` を都度算出し、永続化しない（曜日ロジックのみ）。
- 祝日マスタ（`holidays`）は祝日 API 同期で取得した日付・名称・年を保持する。
- 個別設定（`manual_calendar_settings`）は日付・種別・理由・登録者・登録日時を保持する。日付の一意性制約あり。
- メモ（`calendar_memos`）は日付・内容・登録者・更新日時を保持する。日付の一意性制約あり。
- 案件アサイン（`survey_assignments`）は案件 ID・担当会社 ID・割当枚数・募集状況フラグ・登録者・登録日時・更新日時を保持する。案件 ID × 担当会社 ID の一意性制約あり。
- 募集状況の手動遷移（募集締切・再募集）は専用フラグで保持し、自動遷移ロジックの判定と独立して評価できるようにする。
- 管理者操作ログ（`calendar_operation_logs`）は専用テーブルで管理し、外部キー制約で操作対象のレコード ID と操作者 ID に紐付ける。

### 7.4 ログ・監査

- 以下の操作はすべて管理者操作ログテーブルに記録する。
  - 個別休業日・振替営業日の登録・編集・削除
  - アサイン保存・解除
  - 募集締切・再募集
  - メモの登録・編集・削除
  - 祝日 API 同期
  - CSV 出力・取込
- ログの記録項目: 操作種別・対象日付（または対象案件 ID）・操作者 ID・操作者ロール・操作日時・変更前値・変更後値・理由（入力された場合）。
- 一括操作（範囲設定・範囲アサイン・CSV 取込）は対象件数分のレコードを生成する。
- 管理者操作ログは改ざん不可とし、編集・削除不可とする。
- 管理者操作ログの保持期間は、監査要件と個人情報保護方針に基づき本番化前に確定する。

### 7.5 想定データ量

- 個別設定の年間件数: 数十件規模を想定（祝日 API + 特別休業日 + 振替営業日）。
- 案件アサインの年間件数: 数千件規模を想定。
- CSV 取込の 1 ファイルあたり行数: 最大 1,000 行を上限とする。
- 更新ログの保持期間: 直近 1 年分を表示対象とし、それ以上はアーカイブ。

### 7.6 受け入れ基準

- [ ] 月表示・年表示の 2 モードが切替可能で、縦表示は提供されない
- [ ] 「今日」ボタンで当月へジャンプし、本日セルへスクロールする
- [ ] ホバー時のツールチップは提供されず、詳細は日付クリックでフォーカスエリアから確認する
- [ ] フィルタは種別・募集状況・担当会社の 3 軸で AND 検索される
- [ ] 担当会社フィルタは Lv4 / Lv3 のみに表示され、Lv2 では非表示となる
- [ ] ドラッグ範囲選択で複数日に対する一括操作（営業日設定・メモ・アサイン）が可能となる
- [ ] 日種別は「個別設定 > 祝日 API > 曜日（土日 = 休業日）> 平日 = 営業日」の優先順位で判定される
- [ ] オンデマンドアンケートが乗った休業日は、表示上のみ「営業日」に上書きされ、営業日マスタの正本値は変更されない
- [ ] アサイン保存時に `Σ(各社割当枚数) ≤ 必要枚数` がサーバー側で再検証される
- [ ] 案件の募集ステータスは「未アサイン / 追加募集中 / 募集完了」の 3 状態で管理され、枚数による自動遷移と募集締切・再募集ボタンによる手動遷移が共存する
- [ ] 募集完了案件は、アサイン済の会社の Lv2 のみに表示され、他社の Lv2 には表示されない
- [ ] 未アサイン・追加募集中案件は、全 Lv2 に表示される
- [ ] アサイン保存・解除は Lv4 / Lv3 が全社操作可能で、Lv2 は自社のアサインのみ操作可能となる
- [ ] 募集締切・再募集は Lv4 / Lv3 のみ実行可能で、Lv2 では API レベルで 403 拒否される
- [ ] 個別休業日・振替営業日の登録・編集・削除は Lv4 / Lv3 のみ実行可能で、Lv2 は閲覧のみとなる
- [ ] 個別休業日設定は全社共通として管理され、会社ごとに異なる設定を持たない
- [ ] CSV 出力は対象月の 7 列（date, day_of_week, status, status_label, source, reason, memo）を UTF-8 BOM 付き / CRLF / ヘッダー行ありの形式で出力する
- [ ] CSV 取込はプレビュー画面を経由し、重複行に対して上書き / スキップをユーザーが選択する
- [ ] CSV 取込のバリデーションは行単位で実施され、エラー行のみ除外して正常行は取込される（部分取込可能）
- [ ] エラー行は行番号付きエラーレポート画面で確認でき、「エラー行のみ CSV ダウンロード」が提供される
- [ ] 取込テンプレート CSV のダウンロードボタンが取込画面に提供される
- [ ] 祝日 API 同期は Lv4 / Lv3 のみ実行可能で、取得結果（追加 / 更新 / 削除件数）が表示される
- [ ] Lv1 は AD-06 メニューに表示されず、URL 直叩きも API 側で拒否される
- [ ] AD-09 / AD-10 への明示的な導線リンクは AD-06 に設置されない
- [ ] 全管理者操作がログに記録され、変更前後の値を確認できる
- [ ] 管理者操作ログが管理者自身でも削除・編集できない

## 8. 範囲外

本書の対象に含めないもの。

- オペレーター個人へのアサイン（担当会社単位のアサインのみを本書で扱う。個人単位のアサインは BY-231 業務用オペレーター管理または別途定義する）
- オペレーターの稼働実績・KPI・報酬計算・グラフ表示（AD-10 実績管理の責務）
- オペレーター個人の情報管理（氏名・メール・対応可能言語・パスワード等は AD-09 オペレーター管理の責務）
- 担当会社マスタの管理画面（会社の新規登録・編集・削除は別画面で提供。本書ではアサイン時のプルダウン選択のみ扱う）
- 祝日 API の自動同期スケジュール（バッチ実行の要件は本番化前に確定）
- スマートフォン・タブレット表示の最適化（PC 表示を前提とする）
