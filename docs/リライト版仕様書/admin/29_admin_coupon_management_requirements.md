## 1. 文書目的

- SPEED AD のクーポン（割引コード）を発行・管理する画面の要件を定義する。
- 既存モックの UI/UX に依存せず、`docs/画面設計/仕様/admin/` 配下の正本へ統合するためのリライト版として整理する。
- クーポンの**定義管理**（発行・編集・停止・削除・一括作成・利用ログ参照）を対象とする。クーポンの**配信**（メール送付・キャンペーンプッシュ通知等）は本書のスコープ外とする（§8 参照）。
- 既存モック（`03_admin/coupon-management.html` および関連モーダル）と本書の記述が矛盾する場合、本書を優先する。

## 2. 画面概要

### 2.1 クーポン管理一覧（AD-05）

管理者がクーポンの検索・絞り込み・一覧閲覧を行う起点となる画面。本画面が AD 配下のクーポン管理の単一エントリポイントであり、個別クーポンの詳細確認は 2.2 のクーポン詳細モーダル、新規発行・編集・複製は 2.3 のモーダル、量産は 2.4 の一括作成モーダルで行う。

### 2.2 クーポン詳細モーダル

特定クーポンの基本情報・適用条件・利用統計・利用ログを確認するモーダル。
- AD-05 の一覧行クリックで同画面内にオーバーレイ表示し、閉じると一覧の検索条件・ページ・スクロール位置を保持して戻る。
- モーダル内はセクション分割で構成し、AD-05 内で扱う機能へアクセスできる。
- 基本情報エリアにはコード・名称・割引種別と値・有効期間・利用回数・対象企業/グループ・適用条件・ステータス・メモを配置する。
- 利用統計エリアには消化率（usageCount / usageLimit）・残数・最終利用日時・ユニーク利用者数を配置する。
- 利用ログエリアには当該クーポンの利用履歴（利用日時・利用者ID・注文ID）を表示し、CSV エクスポート導線を持つ。
- 編集・複製・削除の導線をフッターに配置する。
- ステータスの停止・再開は編集モーダル経由で行う（詳細モーダルからの直接操作は提供しない）。

### 2.3 クーポン新規作成・編集・複製モーダル

クーポンを新規発行・既存編集・既存からの複製発行するためのモーダル。
- AD-05 上部の「新規作成」、一覧行の「編集」、詳細モーダルまたは一覧行の「複製」から開く。
- 同一フォーム構成を共有し、モード（新規 / 編集 / 複製）によって編集可否と初期値の挿入元が異なる。
- 編集モードではクーポンコード・割引種別・割引値は変更不可とする（既に利用された可能性があり、過去の利用ログとの整合のため）。
- 複製モードでは既存クーポンのフィールド値をプリフィルし、コードは自動生成された新コードを初期値とする。

### 2.4 一括作成モーダル

複数のクーポンを一度に発行するモーダル。AD-05 上部の「一括作成」から開く。
- 件数指定モードと CSV インポートモードの2モードを提供する。
- いずれのモードでも、共通設定（割引・有効期限・利用回数上限・適用条件・対象企業）→ 個別プレビュー編集 → 作成実行 → 結果表示 + CSV ダウンロードの4段階フローとする。

## 3. 機能要件

### 3.1 一覧・検索

#### 3.1.1 クーポン一覧表示

- ページネーション付きのテーブル形式でクーポンを一覧表示する。
- 1ページあたりの表示件数は 25 / 50 / 100 件から切り替え可能とする。
- 各行に表示する情報: 選択チェックボックス・コード・名称・割引種別と値・有効期限・利用回数（使用/上限）・対象企業/グループ・ステータス・行操作（詳細 / 編集 / 複製 / 削除）。
- ステータス（有効 / 停止）はラベル（色付きバッジ）で視覚的に区別する。
- 派生ラベル（利用上限到達・期限切れ）はステータスとは別にサブラベルとして表示する。表示優先順位は §4.2 に従う。
- 利用回数上限が無制限の場合は「N / ∞」形式で表示する。
- 割引値は割引種別に応じて「20%」または「3,000円」のいずれかの形式で表示する。
- 論理削除済みクーポンは通常一覧から除外する。

#### 3.1.2 検索・フィルタリング

- 以下の条件で絞り込みを行う。複数条件の AND 検索とする。
  - キーワード（コード・名称の部分一致）
  - ステータス（有効 / 停止 の複数チェック）
  - 派生状態（利用上限到達 / 期限切れ の複数チェック）
  - 有効期限の範囲（開始日 〜 終了日）
  - 割引種別（率 / 固定額）
  - 対象企業/グループ（複数選択）
  - 適用プラン（プレミアム限定 / 全プラン 等）
- 検索条件のリセット機能を提供する。
- 検索条件エリアはアコーディオンで開閉可能とする。

#### 3.1.3 ソート

- 以下の列でソート可能とする。昇順 / 降順 / 解除の3状態を切り替える。
  - コード / 名称 / 割引値 / 有効期限 / 利用回数 / 作成日時
- ステータスはバッジ表示のみとし、ソート対象に含めない（派生ラベルとの混在により直感に反するため）。

---

### 3.2 クーポン操作

#### 3.2.1 詳細表示

- 一覧の行から対象クーポンを選択し、同画面上にクーポン詳細モーダルを開く。別画面遷移は伴わない。
- モーダルを閉じると一覧の検索条件・ページ・スクロール位置を保持したまま戻る。
- モーダル表示中もブラウザの戻る/進む操作で開閉できる URL 同期（クエリパラメータ等）を提供する。
- 詳細モーダルには以下のセクションを表示する。
  - 基本情報（コード・名称・割引種別と値・有効期間・対象企業/グループ・適用条件・ステータス・メモ）
  - 利用統計（消化率・残数・最終利用日時・ユニーク利用者数）
  - 利用ログ（利用日時・利用者ID・注文ID、CSV エクスポート可）
  - 操作ログ（当該クーポンに対する管理者操作履歴。詳細は §3.5.2）

#### 3.2.2 新規作成

- 「新規作成」モーダルから新規クーポンを発行する。
- 入力項目:
  - **クーポンコード**: 手動入力欄と「自動生成」ボタンの併用。自動生成ボタン押下時はサーバー側で一意性を保証したコードを発行する。
  - **クーポン名称**
  - **割引種別**: 率 / 固定額 の択一
  - **割引値**: 種別に応じた数値（率は 1〜50 の整数 [%]、固定額は 1 以上の整数 [円]）
  - **有効期限**（開始日・終了日。終了日 ≧ 開始日）
  - **利用回数上限**: 数値 + 無制限チェックボックス
  - **適用条件**: 対象プラン（複数選択）/ 最低利用金額（任意）/ 初回利用限定（任意）/ 既存ユーザー限定（任意）
  - **対象企業/グループ**: 複数選択（未選択時は全企業対象）
  - **メモ**（任意、500字以内、管理用内部メモ）
- 同一コードの重複はサーバー側で検証し、重複時は拒否する。
- 連続発行の利便性のため、新規作成後にモーダルを再度開いた際は、前回入力値（コード以外）をプリフィルする（セッション内のみ保持、リロードで破棄）。
- 新規作成は §7.2 のロール別権限に従う。
- 実行結果は管理者操作ログに記録する。

#### 3.2.3 編集

- 一覧の行から対象クーポンを選択し、編集モーダルを開く。
- **編集可能項目**: クーポン名称・有効期限・利用回数上限・適用条件・対象企業/グループ・メモ・ステータス（有効 / 停止）。
- **編集不可項目**: クーポンコード・割引種別・割引値。これらは過去の利用ログとの整合のため、発行後の変更を許可しない。変更が必要な場合は新規発行または複製で対応する。
- 有効期限の終了日を過去日に変更する場合、確認ダイアログを表示する（誤操作防止）。
- 保存時はサーバー側で入力値、権限、競合状態を再検証する。
- 実行結果は管理者操作ログに変更前/変更後の差分とともに記録する。

#### 3.2.4 削除

- 一覧の行から削除を実行する。
- 削除は **Lv4 のみ実行できる**（§7.2 参照）。
- 物理削除ではなく論理削除（`deleted_at`）とし、過去の利用ログとの外部キーは維持する。
- 実行前に「この操作は取り消せません」の確認ダイアログを表示する。
- 利用回数が1以上のクーポンを削除する場合、追加の警告メッセージを表示する。
- 実行結果は管理者操作ログに記録する。
- 論理削除済みクーポンは通常一覧から除外し、復元操作は AD-05 では提供しない。

#### 3.2.5 ステータス管理（停止・再開）

- クーポンを「有効」から「停止」へ、または「停止」から「有効」へ変更する。
- ステータス変更は編集モーダル経由、または §3.4.2 の一括操作経由で行う。
- 停止中のクーポンはユーザー側で利用不可となる（受付 API 側で `status = 有効` を検証する）。
- 利用上限到達・期限切れは派生状態（§4.2）であり、本操作の対象ではない。
- 実行前に確認ダイアログを表示する。
- 実行結果は管理者操作ログに記録する。

#### 3.2.6 クーポン複製

- 既存クーポンを雛形として新規クーポンを発行する。
- 詳細モーダルおよび一覧の行操作から「複製」を実行する。
- 複製モーダル（§2.3）は新規作成モーダルと同一構成で、以下を初期値とする。
  - コード: 自動生成された新コード（既存コードはコピーしない）
  - その他の項目: 既存クーポンの値をコピー
- 名称・有効期限・対象企業など、必要な項目は複製時に編集できる。
- 利用回数（usageCount）・利用ログは複製しない（新規クーポンは利用回数0からスタート）。
- 複製の実行結果は新規作成と同じく管理者操作ログに記録する。「複製元クーポンID」をログに含める。

---

### 3.3 一括作成

#### 3.3.1 件数指定モード

- 「一括作成」モーダルの件数指定タブで実行する。
- 4段階フロー:
  1. **共通設定**: 作成件数・割引種別と値・有効期限・利用回数上限・適用条件・対象企業/グループ・メモを入力。
  2. **コードプレビュー・編集**: サーバー側で件数分の一意なコードを自動生成し、テーブル形式で表示する。各行のコードと名称は個別編集可能。
  3. **作成実行**: 確認ダイアログののち、サーバーへ一括送信。
  4. **完了表示**: 成功件数を表示し、発行されたクーポンコード一覧を CSV ダウンロードできる。
- 一括作成は §7.2 のロール別権限に従う。
- 上限件数は §6 に従い、超過時はバックグラウンド処理とする。
- 実行結果は作成件数分の管理者操作ログを記録する。

#### 3.3.2 CSV インポートモード

- 「一括作成」モーダルの CSV インポートタブで実行する。
- 入力 CSV の必須列: `code`, `name`。任意列は本番化時に確定する。
- 4段階フロー:
  1. **共通設定**: 割引種別と値・有効期限・利用回数上限・適用条件・対象企業/グループ・メモを入力（コードと名称は CSV から取得するため除外）。
  2. **CSV アップロードとプレビュー**: アップロードした CSV をパースし、サーバー側でコード重複チェック・形式バリデーションを行い、結果をプレビュー表示する。エラー行は赤色で表示し、ユーザーが修正後に再アップロードできる。
  3. **作成実行**: バリデーション通過行のみを対象に一括送信。
  4. **完了表示**: 成功件数・失敗件数を表示し、結果 CSV をダウンロードできる。
- CSV のエンコーディングは UTF-8（BOM 有無問わず受け入れ）とする。
- 1ファイルあたりの上限件数は §6 に従う。

---

### 3.4 一括操作

#### 3.4.1 一括選択

- 一覧テーブルのチェックボックスで複数クーポンを選択できる。
- 全選択 / 全解除を提供する。
- 1件以上選択された状態で「一括操作」エリアが表示される。
- 一括操作の対象範囲は現在の検索・フィルタ条件に合致する全件ではなく、**ユーザーがチェックボックスで明示的に選択した行のみ**とする。

#### 3.4.2 一括ステータス変更（一括停止・一括再開）

- 選択されたクーポンのステータス（有効 / 停止）を一括変更する。
- 実行前に「対象件数・変更後ステータス」を表示する確認ダイアログを出す。
- 実行結果は対象件数分の管理者操作ログを記録する。

#### 3.4.3 一括削除

- 選択されたクーポンを一括で論理削除する。
- 一括削除は **Lv4 のみ実行できる**。
- 実行前に「対象件数」を表示し、文字列入力（例: 「削除」と入力）による二重確認を求める。
- 利用回数が1以上のクーポンが対象に含まれる場合、当該件数を警告表示する。
- 実行結果は対象件数分の管理者操作ログを記録する。

### 3.5 履歴・ログ

#### 3.5.1 更新ログ（画面横断・サイドバー）

- AD-05 上で実行された全管理者操作を時系列でタイムライン表示する。
- 表示項目: 操作種別・操作者・対象クーポン名/コード・メッセージ・日時。
- 種別: 新規作成・編集・削除・ステータス変更（停止/再開）・複製・一括作成・一括ステータス変更・一括削除。
- 種別フィルタを提供する。
- ログ自体の編集・削除は不可とする。

#### 3.5.2 操作ログ（クーポン詳細モーダル内）

- クーポン詳細モーダル内に、当該クーポンに対して管理者が行った操作の履歴を表示する。
- 表示項目: 操作日時・操作種別・操作者（管理者名）・変更前の値・変更後の値。
- 操作種別と期間で絞り込めるフィルタを提供する。
- §3.5.1 と同一の管理者操作ログテーブルを対象クーポン ID で絞り込んで取得する。
- ログ自体の編集・削除は不可とする。

#### 3.5.3 利用ログ（クーポン詳細モーダル内）

- 当該クーポンの利用履歴を表示する。
- 表示項目: 利用日時・利用者ID・注文ID。
- 最新50件を表示し、それ以上はページネーションまたは期間フィルタで参照できる。
- CSV エクスポート導線を提供する（§3.6.2）。
- 利用ログの編集・削除は不可とする。

#### 3.5.4 利用統計（クーポン詳細モーダル内）

- 当該クーポンの利用状況の集計値を表示する。
- 表示項目:
  - 消化率（usageCount / usageLimit、無制限の場合は「N 件利用」表示）
  - 残数（無制限の場合は「∞」表示）
  - 最終利用日時
  - ユニーク利用者数（distinct userId）
- 集計値は利用ログから派生し、DB に独立して持たない。

---

### 3.6 CSV エクスポート

#### 3.6.1 一覧 CSV エクスポート

- 現在の検索条件・ソート順に従って絞り込まれた一覧を CSV としてダウンロードする。
- 出力列: コード・名称・割引種別・割引値・有効期限（開始）・有効期限（終了）・利用回数・利用回数上限・対象企業/グループ・適用条件・ステータス・派生状態・作成日時・更新日時。
- 上限件数は §6 に従い、超過時は条件を絞るよう案内する。
- エンコーディングは UTF-8 BOM 付きとする（Excel 表示互換）。

#### 3.6.2 利用ログ CSV エクスポート

- クーポン詳細モーダルから当該クーポンの利用ログを CSV としてダウンロードする。
- 出力列: 利用日時・利用者ID・注文ID。
- 期間フィルタが適用されている場合は、フィルタ後の利用ログを出力する。
- エンコーディングは UTF-8 BOM 付きとする。

## 4. 状態と挙動

### 4.1 ステータスに応じた操作制限

AD-05 では以下の状態を扱う。

- ステータス（DB 値）: 有効 / 停止
- 派生状態（計算値）: 利用上限到達 / 期限切れ（§4.2）
- 削除状態: 未削除 / 論理削除済み

| ステータス | 利用可能な操作                                | 不可の操作 |
| ---------- | --------------------------------------------- | ---------- |
| 有効       | 詳細表示・編集・停止・複製・削除              | -          |
| 停止       | 詳細表示・編集・再開（=有効化）・複製・削除   | -          |

上表はステータス上の操作可否を示す。実行可否は §7.2 のロール別権限も合わせて判定する。

ステータスはサーバー側で再検証し、直接 URL アクセスや API 直叩きによる不可操作を拒否する。

### 4.2 派生状態と表示優先順位

派生状態は DB に独立して持たず、以下の計算式から導出する。

| 派生状態     | 判定式                                 |
| ------------ | -------------------------------------- |
| 期限切れ     | `今日 > validTo`                       |
| 利用上限到達 | `usageLimit > 0` かつ `usageCount >= usageLimit` |

一覧での表示優先順位（高い順）:

1. 停止（DB 値）
2. 期限切れ（派生）
3. 利用上限到達（派生）
4. 有効（DB 値）

クーポン受付 API 側では、利用時に以下を順に検証して可否を判定する。
1. `status = 有効`
2. `validFrom <= 今日 <= validTo`
3. `usageLimit < 0` または `usageCount < usageLimit`
4. §4.3 の適用条件をすべて満たす

いずれかが不成立の場合、利用を拒否する。

### 4.3 適用条件の評価ルール

クーポン利用時、以下の適用条件を AND で評価する。未設定の項目は条件なしとして扱う。

| 適用条件         | 評価内容                                                              |
| ---------------- | --------------------------------------------------------------------- |
| 対象プラン       | 利用者の現プランが指定プラン群に含まれること                          |
| 最低利用金額     | 利用対象の決済金額が指定値以上であること                              |
| 初回利用限定     | 利用者の課金履歴が0件であること                                       |
| 既存ユーザー限定 | 利用者の課金履歴が1件以上であること（初回利用限定との同時指定は不可） |
| 対象企業/グループ | 利用者の所属企業が指定企業群に含まれること（未指定時は全企業対象）    |

「初回利用限定」と「既存ユーザー限定」を同時に有効化することはできない（フォーム側および API 側で排他検証する）。

### 4.4 アクセス権限（テナント境界）

AD-05 はアブロード（運営側）の管理機能であり、クーポン管理は単一テナント内で完結する。AD-09 のような「自グループ内のみ」というテナント分離は適用しない。

クーポンの「対象企業/グループ」属性は、**利用対象を限定する条件**であり、AD-05 の操作可視範囲を限定する属性ではない。

権限制御は §7.2 のロール別権限のみで行う。

## 6. 非機能要件

- 一覧の検索レスポンスは2秒以内を目標とする。
- ページネーションは1ページ最大100件とし、全件数を表示する。
- 更新・削除・送信など副作用を伴う API は Idempotency-Key により多重実行を防止する。
- 一括作成・一括操作は同期的に処理する場合、最大100件までを上限とする。それを超える場合はバックグラウンド処理とし、完了通知を行う。
- CSV エクスポートは同期処理で最大10,000件までを上限とし、超過時はバックグラウンド処理とする。
- CSV インポートは1ファイルあたり最大1,000件までを上限とする。

## 7. 本番化要件

### 7.1 API（確認観点）

以下はたたき台であり、詳細スキーマ・パスは開発会社と合意して確定する。UI による非表示や導線制御は補助であり、保存・認可・状態遷移・利用可否判定は API 側で成立させる。

| エンドポイント                                     | 用途                                       | 主要レスポンス                                          |
| -------------------------------------------------- | ------------------------------------------ | ------------------------------------------------------- |
| `GET /api/admin/coupons`                           | クーポン一覧・検索                         | `{ coupons[], total, page }`                            |
| `GET /api/admin/coupons/{id}`                      | クーポン詳細取得                           | `{ coupon, usageStats, usageLogs, operationLogs }`      |
| `POST /api/admin/coupons`                          | 新規クーポン作成                           | 201 / 409 コード重複 / 400 形式不正                     |
| `POST /api/admin/coupons/{id}/duplicate`           | クーポン複製（既存からの新規作成）         | 201 新クーポンレスポンス                                |
| `PATCH /api/admin/coupons/{id}`                    | クーポン情報更新（編集可能項目のみ）       | 200 / 403 権限なし / 409 競合 / 400 編集不可項目を含む  |
| `DELETE /api/admin/coupons/{id}`                   | クーポン論理削除                           | 200 / 403 権限なし                                      |
| `POST /api/admin/coupons/{id}/code-generate`       | クーポンコードの自動生成（一意性保証付き） | `{ code }`                                              |
| `POST /api/admin/coupons/bulk-create`              | 一括作成（件数指定モード）                 | 200 `{ succeeded, failed[] }`                           |
| `POST /api/admin/coupons/bulk-import`              | 一括作成（CSV インポートモード）           | 200 `{ succeeded, failed[] }` / 400 形式不正            |
| `POST /api/admin/coupons/bulk`                     | 一括操作（ステータス変更・削除）の共通エンドポイント | 200 `{ succeeded, failed[] }` / 403 権限なし   |
| `GET /api/admin/coupons/{id}/usage-logs`           | クーポン利用ログ取得                       | `{ usageLogs[] }`                                       |
| `GET /api/admin/coupons/{id}/usage-logs.csv`       | 利用ログ CSV ダウンロード                  | text/csv                                                |
| `GET /api/admin/coupons.csv`                       | 一覧 CSV ダウンロード                      | text/csv                                                |

| `GET /api/admin/coupons/targets/companies`         | 対象企業/グループの選択肢取得              | `{ companies[] }`                                       |
| `GET /api/admin/coupons/targets/plans`             | 適用プランの選択肢取得                     | `{ plans[] }`                                           |
| `POST /api/coupons/{code}/validate`                | クーポン受付 API（ユーザー側決済フローで使用） | 200 適用可 / 400 適用不可（理由つき）                |

### 7.2 権限・認可

AD-05 で扱うロールは以下とする。

| ロール                                | AD-05 での扱い                                                                  |
| ------------------------------------- | ------------------------------------------------------------------------------- |
| Lv4 アブロードマネージャー（authority=4） | フルアクセス                                                                    |
| Lv3 アブロードスタッフ（authority=3）   | 閲覧・編集・ステータス変更（停止/再開）・複製・一括ステータス変更・CSV エクスポート可。**新規作成・削除・一括作成・一括削除は不可** |
| Lv2 オペレーター管理者（authority=2）  | AD-05 アクセス不可                                                              |
| Lv1 オペレーター（authority=1）       | AD-05 アクセス不可                                                              |

> **注記**: クーポンは金銭に直結するため、発行・削除系の操作を Lv4 限定としている。Lv3 の運用上の不足（例: Lv3 が新規発行できないと運用が回らない）が判明した場合、レビュー時に再確認する。

| 操作                                | Lv4 | Lv3 | 備考 |
| ----------------------------------- | --- | --- | ---- |
| クーポン一覧・詳細の閲覧            | 可  | 可  |  |
| 利用ログ・利用統計の閲覧            | 可  | 可  |  |
| 管理者操作ログの閲覧                | 可  | 可  | 横断・個別とも |
| 新規クーポン作成                    | 可  | 不可 | 発行は Lv4 のみ |
| クーポン編集（コード・割引以外）    | 可  | 可  |  |
| クーポン複製                        | 可  | 不可 | 実質的な新規発行のため Lv4 のみ |
| ステータス変更（停止・再開）        | 可  | 可  |  |
| クーポン削除                        | 可  | 不可 | 不可逆操作（論理削除）のため Lv4 のみ |
| 一括作成（件数指定モード）          | 可  | 不可 |  |
| 一括作成（CSV インポートモード）    | 可  | 不可 |  |
| 一括ステータス変更                  | 可  | 可  |  |
| 一括削除                            | 可  | 不可 | 二重確認必須 |
| 一覧 CSV エクスポート               | 可  | 可  |  |
| 利用ログ CSV エクスポート           | 可  | 可  |  |
| 管理者操作ログの削除・編集          | 不可 | 不可 | 全管理者に対して禁止 |

権限判定は画面表示だけでなく API 側でも再検証し、直接 URL アクセスや API 直叩きによる操作を拒否する。

### 7.3 データ永続化

- クーポンテーブルに論理削除フラグ（`deleted_at`）を持つ。
- クーポンテーブルは以下の主要カラムを持つ:
  - `id`, `code`(unique), `name`, `discount_type`(`PERCENT` / `FIXED_AMOUNT`), `discount_value`, `valid_from`, `valid_to`, `usage_limit`(`-1`で無制限), `status`(`ACTIVE` / `INACTIVE`), `memo`, `created_by`, `created_at`, `updated_at`, `deleted_at`
- 適用条件は専用テーブルまたは JSON カラムで保持し、対象プラン・最低利用金額・初回限定・既存ユーザー限定の各属性を管理する。
- 対象企業/グループは中間テーブルでクーポンと企業を多対多関連で管理する。空集合は「全企業対象」を意味する。
- クーポン利用ログは専用テーブルで管理し、外部キー制約でクーポンテーブル・ユーザーテーブル・注文テーブルと紐付ける。
- 管理者操作ログは専用テーブルで管理し、外部キー制約でクーポンテーブル・操作者（管理者）テーブルと紐付ける。
- 複製で作成されたクーポンは、操作ログに複製元クーポン ID を保持する（系譜追跡用）。
- 利用上限到達・期限切れ等の派生状態は DB に保持せず、参照時に計算する。

### 7.4 ログ・監査

- 以下の操作はすべて管理者操作ログテーブルに記録する。
  - 新規作成・編集・複製・削除
  - ステータス変更（停止・再開）
  - 一括作成（件数指定・CSV インポート）
  - 一括ステータス変更・一括削除
- ログの記録項目: 操作種別・対象クーポン ID・操作者 ID・操作日時・変更前値・変更後値・補助情報（複製元 ID 等）。
- 一括操作は対象件数分のレコードを生成する（バルクID で同一操作として紐付け可能にする）。
- 管理者操作ログは改ざん不可とし、編集・削除不可とする。
- 管理者操作ログの保持期間は、監査要件と個人情報保護方針に基づき本番化前に確定する。
- 利用ログ（ユーザー側の利用履歴）は管理者操作ログと別系統で永続化する。

### 7.5 想定データ量

- 管理対象クーポン数: 数千〜数万件規模を想定（キャンペーン単位の一括発行を含む）。
- 1クーポンあたりの利用ログ件数: 通常1〜数百件、上限なしの汎用クーポンで数万件規模を想定。
- 利用ログの保持期間: 直近2年分を表示対象とし、それ以上はアーカイブ。

### 7.6 受け入れ基準

- [ ] 一覧テーブルは選択チェックボックス・コード・名称・割引種別と値・有効期限・利用回数・対象企業/グループ・ステータス・操作の9列で表示される
- [ ] 表示件数の切替は 25 / 50 / 100 件のみとなる
- [ ] ステータスは「有効」「停止」の2値のみで、「利用上限到達」「期限切れ」はステータスではなく派生ラベルとして表示される
- [ ] 派生ラベルの表示優先順位が §4.2 の通り（停止 > 期限切れ > 利用上限到達 > 有効）になる
- [ ] 検索条件の AND 組み合わせで正しく絞り込まれる
- [ ] 割引種別を「率」と「固定額」のいずれかで選択でき、種別に応じて入力バリデーション（率は 1〜50% 整数、固定額は 1 以上の整数）が切り替わる
- [ ] 編集モーダルでクーポンコード・割引種別・割引値が変更不可になる
- [ ] 編集モーダルでステータスを「有効」「停止」の間で切り替えられる
- [ ] 複製操作で既存クーポンの値がプリフィルされ、コードは自動生成された新コードが入る
- [ ] 複製で作成されたクーポンの操作ログに「複製元クーポンID」が記録される
- [ ] 新規発行・削除・複製・一括作成・一括削除は Lv4 のみ実行できる
- [ ] 編集・ステータス変更・一括ステータス変更・CSV エクスポートは Lv4/Lv3 が実行できる
- [ ] Lv2/Lv1 は AD-05 にアクセスできない
- [ ] 削除実行時に確認ダイアログが表示され、利用回数が1以上のクーポンの場合は追加警告が表示される
- [ ] 一括削除実行時に文字列入力による二重確認が表示される
- [ ] 一括作成（件数指定）でサーバー側がコードの一意性を保証し、重複時は再生成導線が表示される
- [ ] 一括作成（CSV インポート）で必須列欠落・形式エラーが行単位で検出され、エラー行は赤色表示される
- [ ] 適用条件の「初回利用限定」と「既存ユーザー限定」を同時に指定すると API が拒否する
- [ ] クーポン受付 API（`POST /api/coupons/{code}/validate`）が §4.2 の順序で利用可否を判定する
- [ ] 利用ログ・利用統計はクーポン詳細モーダル内で参照でき、CSV エクスポートできる
- [ ] 派生状態（利用上限到達・期限切れ）が DB に独立して保持されず、参照時に計算で導出される
- [ ] 全管理者操作がログに記録され、変更前後の値を確認できる
- [ ] 管理者操作ログが管理者自身でも削除・編集できない
- [ ] 更新ログ（サイドバー）と操作ログ（詳細モーダル）が同一データソースから取得され、フィルタ条件のみが異なる
- [ ] 論理削除済みクーポンは通常一覧から除外され、復元操作は AD-05 で提供されない
