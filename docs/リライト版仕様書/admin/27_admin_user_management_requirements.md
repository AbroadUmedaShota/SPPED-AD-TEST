## 1. 文書目的

- 管理者がサービス利用者のアカウントを管理するための画面（一覧・詳細）の要件を定義する。
- 既存モックの UI/UX に依存せず、`docs/画面設計/仕様/admin/` 配下の正本へ統合するためのリライト版として整理する。
- 対象ユーザーは SPEED AD 社内のオペレーション担当者であり、エンドユーザー（展示会出展企業の担当者）は対象外。

---

## 2. 画面概要

### 2.1 ユーザー管理一覧（AD-02）

管理者が利用者アカウントの検索・絞り込み・一覧閲覧を行う起点となる画面。本画面が AD 配下のユーザー管理の単一エントリポイントであり、個別ユーザーの詳細確認と通常のアカウント操作は 2.2 のユーザー詳細モーダルで行う。アンケート本体、請求運用など、ユーザー管理以外の業務操作は該当画面または専用フローへ遷移する。

### 2.2 ユーザー詳細モーダル

特定ユーザーの基本情報・請求情報・サービス設定・内部メモ・契約状態・履歴・紐付きアンケートを確認し、通常のアカウント操作を行うモーダル。
- AD-02 の一覧行クリックで同画面内にオーバーレイ表示し、閉じると一覧の検索条件・ページ・スクロール位置を保持して戻る。
- モーダル内はタブまたは折りたたみセクションで領域を切り替え、AD-02 内で扱う機能へアクセスできる構成とする。
- 基本情報エリアには氏名・カナ・メールアドレス・電話・会社名・部署役職・内部メモを配置する。
- 内部メモは追記履歴として保持し、直近のメモと時系列のメモ履歴を確認できるようにする。
- 詳細設定エリアには請求情報、サービス設定、変更履歴、請求状態サマリを折りたたみ表示する。
- 履歴エリアにはログイン履歴、メール送信履歴、直近変更サマリを配置する。
- 紐付きアンケートは一覧とサマリのみを表示し、編集・削除・移管・回答データ削除は AD-03 / AD-12 側で行う。

## 3. 機能要件

### 3.1 一覧・検索

#### 3.1.1 ユーザー一覧表示

- ページネーション付きのテーブル形式で利用者アカウントを一覧表示する。
- 1ページあたりの表示件数は切り替え可能とする。
- 各行に表示する情報: アカウントID・会社名・部署・役職・氏名・メールアドレス・契約ステータス・契約プラン・管理フラグ・最終ログイン日時。
- 契約ステータス・管理フラグはラベル（色付きバッジ）で視覚的に区別する。

#### 3.1.2 検索・フィルタリング

- 以下の条件で絞り込みを行う。複数条件の AND 検索とする。
  - キーワード（氏名・メールアドレス・アカウントID の部分一致）
  - 会社名（部分一致）
  - 契約ステータス（有効 / 一時停止 / 解約）
  - 契約プラン（無料 / プレミアム）
  - 管理フラグ（重要 / 要注意 / 関係者）
- 詳細フィルタとして以下を追加提供する（通常は折りたたみ状態）。
  - 部署（部分一致）
  - 役職（部分一致）
  - アカウント作成日（開始〜終了の範囲指定）
  - 最終ログイン（7日以内 / 30日以内 / 90日以上未ログイン）
- 検索条件一式を名称付きで保存し、保存済み条件として再適用できる。
- 検索条件のリセット機能を提供する。

#### 3.1.3 CSVエクスポート

- 現在の検索・フィルタ条件に合致するユーザー全件をCSV形式でエクスポートする。
- 出力項目: アカウントID・会社名・部署・役職・氏名・メールアドレス・契約ステータス・契約プラン・管理フラグ・アカウント作成日・最終ログイン日時。
- CSVエクスポートはアブロードマネージャー（authority=4）のみ実行できる。
- エクスポート実行は管理者操作ログに記録する（操作者・日時・検索条件・出力件数）。

---

### 3.2 ユーザー操作

#### 3.2.1 詳細表示

- 一覧の行から対象ユーザーを選択し、同画面上にユーザー詳細モーダルを開く。別画面遷移は伴わない。
- モーダルを閉じると一覧の検索条件・ページ・スクロール位置を保持したまま戻る。
- モーダル表示中もブラウザの戻る/進む操作で開閉できる URL 同期（クエリパラメータ等）を提供する。

#### 3.2.2 基本情報・請求情報・サービス設定の保存

- ユーザー詳細モーダルの「変更を保存」押下時、基本情報・請求情報・サービス設定の変更と内部メモの追記をまとめて API 送信する。
- 基本情報は氏名・カナ・電話・会社名・部署・役職を対象とする。メールアドレス変更は §3.2.6 の専用操作として扱い、通常の保存操作には含めない。
- 請求情報は請求書宛名・請求先メール・郵便番号・住所を対象とする。
- サービス設定は権限ロール・利用モジュール・通知設定・MFA 必須・IP 制限を対象とする。
- 基本情報・請求情報・内部メモの更新はアブロードマネージャー / アブロードスタッフが実行できる。
- サービス設定の更新はアブロードマネージャー（authority=4）のみ実行できる。
- 保存時はサーバー側で入力値、権限、競合状態を再検証し、成功時に管理者操作ログへ差分を記録する。

#### 3.2.3 アカウント一時停止・再開

- 契約ステータスが「有効」のユーザーを「一時停止」に変更する。
- 契約ステータスが「一時停止」のユーザーを「有効」に戻す。
- 本操作は §3.4.1 の契約ステータス管理のうち高頻度操作を個別アクション化したものであり、同一の API、権限、監査ログ要件を適用する。
- 実行前に確認ダイアログを表示する。
- 実行結果は管理者操作ログに記録する（操作者・日時・理由）。
- 契約ステータス変更はサーバー側で現在状態と必要権限を再検証し、UI の表示/非表示だけに依存しない。
- 一時停止時は対象アカウントの既存セッションと refresh token を失効させ、再開後も再認証を必須とする。
- 不正利用・問い合わせ対応などでログイン禁止が必要な場合も一時停止として扱い、AD-02 では別の強制ロック状態を設けない。

#### 3.2.4 ユーザー削除

- 解約済みユーザーに対してのみ削除操作を提供する。有効・一時停止中ユーザーへの削除は不可とする。
- ユーザー削除はアブロードマネージャー（authority=4）のみ実行できる。
- 削除は物理削除ではなく論理削除（削除フラグ）とし、個人データは別途データ管理ポリシーに従って消去する。
- 実行前に「この操作は取り消せません」の確認ダイアログを表示し、ユーザー名の入力による二重確認を求める。
- 実行結果は管理者操作ログに記録する。
- 削除可否は契約ステータスだけでなく、依存データ有無とサーバー側の整合条件で判定する。
- 削除実行時は対象アカウントの既存セッション、未使用トークン、関連通知をサーバー側で失効・停止する。

#### 3.2.5 パスワードリセットメール送信

- 対象ユーザーのメールアドレス宛にパスワード再設定メールを送信する。
- 送信完了後に管理者操作ログおよびメール送信履歴に記録する。

#### 3.2.6 メールアドレスの管理者変更

- アブロードマネージャー（authority=4）はユーザーのメールアドレスを直接変更できる。
- ユーザー自身がメールアドレスを変更できない場合（退職・旧アドレスへのアクセス不能など）の対応用途。
- メールアドレス変更は通常の基本情報保存とは分離し、専用操作として実行する。
- 変更後のアドレスが既存アカウントと重複しないことをサーバー側でバリデーションする。
- 変更完了後、新メールアドレス宛に変更通知メールを送信する。
- 実行結果は管理者操作ログに記録する（変更前・変更後のメールアドレスを含む）。

#### 3.2.7 内部メモ・運用サマリ

- ユーザー詳細モーダルに、直近の管理者操作、最終ログイン、契約ステータス、直近のメール送信履歴を要約表示する。
- 内部メモは追記形式で登録し、登録日時・登録者・本文を時系列で表示する。
- 登録済みメモの編集・削除は原則不可とし、訂正が必要な場合は新しいメモとして追記する。
- 直近変更サマリには、最新の変更日時・操作者・操作種別・変更対象を表示する。

---

### 3.3 招待管理

#### 3.3.1 新規ユーザー招待

- メールアドレスを入力して招待メールを送信する。
- 新規ユーザー招待はアブロードマネージャー（authority=4）のみ実行できる。
- 招待メールには本登録用の有効期限付きURL（72時間有効）を含める。
- 招待と同時に仮アカウントレコードを生成し、「招待中」ステータスで管理する。
- 招待状態は「招待中 / 登録済み / 期限切れ / 取消済み」で管理する。
- 同一メールアドレスの未登録招待が存在する場合、新規招待は作成せず、既存招待の再送導線へ誘導する。
- 招待URLから本登録が完了した時点で仮アカウントを正規アカウントへ昇格し、招待状態を「登録済み」に変更する。
- 招待URLはワンタイム性を持ち、登録完了後または失効後の再利用を許可しない。

#### 3.3.2 招待管理

- 招待中・期限切れ・取消済みの未登録招待を一覧表示する。
- 表示項目: 招待先メールアドレス・招待日時・有効期限・ステータス（招待中 / 期限切れ / 取消済み）。
- 72時間経過した招待は自動的に「期限切れ」へ遷移する。
- アブロードマネージャー / アブロードスタッフは、個別の未登録招待に対して招待メールを再送できる。再送時は旧トークンを即時失効し、新しいトークンと有効期限を発行する。
- 招待キャンセルはアブロードマネージャー（authority=4）のみ実行できる。取消済みの旧URLは利用できない。
- 登録済みの招待は管理一覧の通常表示から除外し、監査用途の履歴としてのみ参照する。

---

### 3.4 契約・プラン管理

#### 3.4.1 契約ステータス管理

- ユーザー詳細モーダルで契約ステータスを変更する（有効 / 一時停止 / 解約）。
- 契約ステータス変更の実行起点は本節とし、§3.2.3 の一時停止・再開アクションも本節と同一の状態遷移として扱う。
- 一時停止・再開はアブロードマネージャー / アブロードスタッフが実行できる。
- 解約はアブロードマネージャー（authority=4）のみ実行できる。
- 「解約」への変更は有効または一時停止からのみ実行でき、不可逆であることを確認ダイアログで明示する。
- 解約実行時は理由入力とユーザー名入力による二重確認を必須とする。
- 解約実行時は対象アカウントの既存セッション、未使用トークン、関連通知をサーバー側で失効・停止する。
- ステータス変更は管理者操作ログに記録する（変更前・変更後・理由）。

#### 3.4.2 プラン管理

- ユーザー詳細モーダルで現在の契約プランを表示する（無料 / プレミアム）。
- アブロードマネージャー（authority=4）がプランを強制変更できる（ユーザー操作に関わらず即時反映）。
- 変更理由を必須入力とし、管理者操作ログに記録する（変更前プラン・変更後プラン・理由・操作者・日時）。
- プラン変更により利用可能機能、表示制御、ダウンロード可否が変わる場合は、変更前に影響範囲を表示する。

---

### 3.5 データエクスポート

#### 3.5.1 個人データエクスポート

- 特定ユーザーに紐付く開示対象の個人データを一括でエクスポートする。
- 個人情報エクスポートは社内最上位ロール（アブロードマネージャー、authority=4）のみ実行できる。
- 出力対象: アカウント基本情報・ログイン履歴・アンケート回答データ・名刺データ・メール送信履歴。
- JSON形式またはCSV形式（種別ごとにファイル分割）でダウンロードする。
- 出力対象は権限上必要な個人データに限定し、秘匿トークン、認証情報、監査ログの改ざん防止情報は含めない。
- 実行時は理由入力を必須とし、管理者操作ログに記録する（対象ユーザーID・出力項目・出力日時・操作者・理由）。
- 実行前に出力対象項目とファイル形式を確認する画面を表示する。
- ダウンロードは期限付きの一時URLまたは同等の失効可能な手段で提供する。
- 期限切れ後の再ダウンロード可否は本番化時に定義し、再ダウンロードを許可する場合も社内最上位ロールと操作ログ記録を必須とする。

---

### 3.6 コミュニケーション

#### 3.6.1 管理者→ユーザーへのメール送信

- ユーザー詳細モーダルから対象ユーザーに直接メールを送信する。
- 入力項目: 件名・本文。
- 件名・本文を自由入力するメール送信は、社内最上位ロール（アブロードマネージャー、authority=4）のみ実行できる。
- パスワードリセット、招待再送、メールアドレス変更通知など、本文を自由入力しない定型通知は本制限の対象外とする。
- 送信前にプレビューを表示する。
- 送信時は理由入力を必須とする。
- 送信はメール送信履歴に記録する。

#### 3.6.2 メール送信履歴

- ユーザー詳細モーダルに、そのユーザーへ管理者が送信したメールの履歴を表示する。
- 表示項目: 送信日時・件名・送信者（管理者名）・送信先メールアドレス。
- メール本文の閲覧は社内最上位ロール（アブロードマネージャー、authority=4）のみ可能とする。
- 社内最上位ロール以外には本文を表示せず、送信日時・件名・送信者・送信先のみを表示する。

---

### 3.7 履歴・ログ

#### 3.7.1 ログイン履歴

- ユーザー詳細モーダルにそのユーザーのログイン履歴を表示する。
- 表示項目: ログイン日時・IPアドレス・デバイス種別（デスクトップ / モバイル）・ブラウザ・ログイン結果（成功 / 失敗）。
- 最新50件を表示し、それ以上はページネーションまたは期間フィルタで参照できる。

#### 3.7.2 管理者操作ログ

- ユーザー詳細モーダルに、そのユーザーに対して管理者が行った操作の履歴を表示する。
- 表示項目: 操作日時・操作種別・操作者（管理者名）・変更前の値・変更後の値・理由（入力された場合）。
- 操作種別の例: ステータス変更・プラン変更・メールアドレス変更・削除・パスワードリセット送信・データエクスポート・AD-03 / AD-12 側で実行されたアンケート移管や回答データ削除など。
- 操作種別と期間で絞り込めるフィルタを提供する。
- ログ自体の編集・削除は不可とする。

---

### 3.8 他画面連携

#### 3.8.1 請求状態サマリと請求書一覧への直リンク（AD-04）

- ユーザー詳細モーダルに、当該ユーザーの請求状態サマリを表示する。
- 表示項目: 最新請求ステータス・未入金有無・直近請求日・直近請求金額。
- ユーザー詳細モーダルに「このユーザーの請求一覧を見る」リンクを設置する。
- リンク先は AD-04 請求管理画面を当該ユーザーでフィルタした状態で開く。

---

### 3.9 紐付きアンケート参照

#### 3.9.1 アンケート一覧表示

- ユーザー詳細モーダルに、そのユーザーが所有するアンケートの一覧を表示する。
- 表示項目: アンケート名・ステータス・会期・データ化プラン・名刺処理進捗・請求予定金額。
- ステータスのサマリ（実施中 / 準備中 / 草稿 / 終了の件数）を一覧上部に表示する。

#### 3.9.2 アンケート詳細への導線

- 各アンケート行に AD-03 アンケート管理または AD-12 アンケート詳細へのリンクを配置する。
- AD-02 ではアンケート本体の編集、削除、ユーザー移管、データ化プラン・スピード変更、回答データの部分削除を提供しない。
- 上記操作は AD-03 / AD-12 側の要件として扱い、実行結果のみ管理者操作ログやアンケート一覧に反映する。

## 4. 状態と挙動

### 4.1 契約ステータスに応じた操作制限

AD-02 では以下の状態を扱う。

- 契約ステータス: 有効 / 一時停止 / 解約
- 招待状態: 招待中 / 登録済み / 期限切れ / 取消済み
- 削除状態: 未削除 / 論理削除済み

パスワード誤入力など認証機能側の一時ロックが必要な場合は、AD-02 の契約ステータスとは別に認証基盤側で扱う。

| 状態     | 利用可能な操作                                                                 | 不可の操作                                 |
| -------- | ------------------------------------------------------------------------------ | ------------------------------------------ |
| 有効     | 詳細表示・一時停止・解約・プラン変更・定型通知送信・基本情報編集・アンケート参照 | 削除・AD-02上でのアンケート編集             |
| 一時停止 | 詳細表示・再開・解約・プラン変更・定型通知送信・基本情報編集・アンケート参照     | 削除・AD-02上でのアンケート編集             |
| 解約     | 詳細表示・削除・データエクスポート（社内最上位ロールのみ）・アンケート参照     | 一時停止・再開・プラン変更・アンケート編集 |

上表は契約ステータス上の操作可否を示す。実行可否は §7.2 のロール別権限も合わせて判定する。

自由文メール送信、メール本文閲覧、個人情報エクスポートは、契約ステータスにかかわらず社内最上位ロール（アブロードマネージャー、authority=4）のみ実行できる。

契約ステータスはサーバー側で再検証し、直接URLアクセスや API 直叩きによる不可操作を拒否する。

### 4.2 招待状態に応じた操作制限

| 招待状態 | 利用可能な操作 | 不可の操作 |
| -------- | -------------- | ---------- |
| 招待中   | 招待再送・招待キャンセル・履歴参照 | 新規招待の重複作成 |
| 登録済み | 監査用途の履歴参照 | 招待再送・招待キャンセル |
| 期限切れ | 招待再送・履歴参照 | 旧URLの利用・同一メールでの新規招待重複作成 |
| 取消済み | 履歴参照 | 招待再送・旧URLの利用 |

上表は招待状態上の操作可否を示す。新規招待と招待キャンセルは §7.2 に従いアブロードマネージャーのみ実行できる。

同一メールアドレスに招待中または期限切れの未登録招待が存在する場合は、新規招待ではなく既存招待の再送またはキャンセルを案内する。

### 4.3 削除状態に応じた操作制限

| 削除状態 | 一覧表示 | 利用可能な操作 | 不可の操作 |
| -------- | -------- | -------------- | ---------- |
| 未削除   | 通常表示 | 契約ステータスに応じた操作 | なし |
| 論理削除済み | 通常一覧から除外し、監査用途の履歴でのみ参照 | 監査ログ参照 | 再開・編集・メール送信・招待・プラン変更 |

上表は削除状態上の操作可否を示す。削除実行そのものは §7.2 に従いアブロードマネージャーのみ実行できる。

論理削除済みユーザーを通常一覧へ戻す復元操作は AD-02 では提供しない。

### 4.4 プラン変更時のデータ扱い

- プラン変更は、利用可能機能、表示制御、ダウンロード可否を再判定する契機とする。
- AD-02 では、プラン変更だけを理由にアンケート本体または回答データを30日で削除するルールを定義しない。
- プレミアム解約・無料プランへの移行などでデータ保持ルールが変わる場合も、AD-02 では詳細な保持期限を扱わない。
- プラン変更による影響は管理者操作ログに記録する。

### 4.5 AD-03 / AD-12 でのアンケート移管後の表示

- アンケート移管は AD-03 / AD-12 側で実行する。
- 移管完了後、AD-02 の紐付きアンケート一覧では旧所有者側から対象アンケートを除外し、移管先ユーザー側に表示する。
- 移管後の編集可否・ダウンロード可否は、移管先ユーザーの契約状態と権限で再判定された結果を表示する。
- 移管操作だけを理由に、AD-02 上の紐付きアンケート表示以外のデータ管理状態は変更しない。

## 6. 非機能要件

- 一覧の検索レスポンスは2秒以内を目標とする。
- ページネーションは1ページ最大50件とし、全件数を表示する。
- CSVエクスポートは最大10,000件まで対応する。超過時はエラーを表示し、条件を絞るよう誘導する。
- 個人情報開示エクスポートは完了まで5分以内を目標とし、時間がかかる場合はバックグラウンド処理としてメール通知で完了を知らせる。
- 更新・削除・送信・エクスポートなど副作用を伴う API は Idempotency-Key により多重実行を防止する。

## 7. 本番化要件

### 7.1 API（確認観点）

以下はたたき台であり、詳細スキーマ・パスは開発会社と合意して確定する。UI による非表示や導線制御は補助であり、保存・認可・状態遷移・トークン失効は API 側で成立させる。

| エンドポイント                                          | 用途                                             | 主要レスポンス                                                              |
| ------------------------------------------------------- | ------------------------------------------------ | --------------------------------------------------------------------------- |
| `GET /api/admin/users`                                  | ユーザー一覧・検索                               | `{ users[], total, page }`                                                  |
| `GET /api/admin/users/export`                           | ユーザー一覧CSVエクスポート                      | csv / 400（件数上限超過） / 403 権限なし                                    |
| `GET /api/admin/users/{id}`                             | ユーザー詳細取得                                 | `{ user, loginHistory, operationLogs, surveys[] }`                          |
| `PATCH /api/admin/users/{id}`                           | ユーザー情報更新（メールアドレスを除く基本情報・ステータス・プラン等） | 200 更新後データ / 403 権限なし / 409 競合                                  |
| `PATCH /api/admin/users/{id}/email`                     | メールアドレス変更                               | 200 更新後データ / 403 権限なし / 409 重複・競合                            |
| `DELETE /api/admin/users/{id}`                          | ユーザー論理削除                                 | 200 / 400（解約済み以外） / 403 権限なし                                    |
| `POST /api/admin/users/{id}/password-reset`             | パスワードリセットメール送信                     | 200                                                                         |
| `POST /api/admin/invitations`                           | 招待メール送信                                   | 201 / 409 重複                                                              |
| `GET /api/admin/invitations`                            | 招待一覧                                         | `{ invitations[] }`                                                         |
| `POST /api/admin/invitations/{id}/resend`               | 招待再送                                         | 200 新トークン発行 / 400 登録済み                                           |
| `DELETE /api/admin/invitations/{id}`                    | 招待キャンセル                                   | 200 / 400 登録済み                                                          |
| `POST /api/admin/users/{id}/emails`                     | 管理者→ユーザーへの自由文メール送信（社内最上位ロールのみ） | 201 / 403 権限なし                                                          |
| `GET /api/admin/users/{id}/emails`                      | メール送信履歴（本文は社内最上位ロールのみ）     | `{ emails[] }`（アブロードスタッフには本文を返さない）                      |
| `POST /api/admin/users/{id}/export`                     | 個人情報エクスポート（社内最上位ロールのみ）     | zip or 202（非同期） / 403 権限なし                                         |

アンケート編集・削除・移管・回答データ部分削除は AD-03 / AD-12 側の API として定義し、AD-02 の API 確認観点には含めない。

### 7.2 権限・認可

既存資料で確認できる管理者ロールは以下とする。

| ロール | AD-02 での扱い |
| ------ | -------------- |
| アブロードマネージャー（authority=4） | ユーザー管理機能の対象。AD-02 における社内最上位ロール |
| アブロードスタッフ（authority=3） | ユーザー管理機能の対象。通常管理者ロール |
| オペレーター管理者（authority=2） | AD-02 / AD-11 の対象外 |
| オペレーター（authority=1） | AD-02 / AD-11 の対象外 |

権限設計の基本方針は以下とする。

- アブロードスタッフは日常問い合わせ対応、状況確認、軽微な顧客情報更新、定型通知を担当する。
- アブロードマネージャーはアカウント所有権、契約、請求影響、セキュリティ設定、一括出力、個人情報持ち出し、不可逆操作を担当する。
- オペレーター管理者 / オペレーターは AD-02 のユーザー管理機能へアクセスできない。

| 操作                                   | アブロードマネージャー（authority=4） | アブロードスタッフ（authority=3） | 備考 |
| -------------------------------------- | -------------------------------------- | ---------------------------------- | ---- |
| ユーザー一覧・詳細の閲覧               | 可                                     | 可                                 | スタッフにも問い合わせ対応に必要な範囲で表示する |
| 紐付きアンケートの参照                 | 可                                     | 可                                 | 編集・削除・移管は AD-03 / AD-12 側 |
| 請求状態サマリ・AD-04リンクの参照      | 可                                     | 可                                 | 請求操作は AD-04 側 |
| 管理者操作ログの閲覧                   | 可                                     | 可（本文・機微情報はマスク）       | メール本文、個人情報エクスポート理由などは表示制限する |
| 内部メモの追記                         | 可                                     | 可                                 | 登録済みメモの編集・削除は不可 |
| 基本情報の更新                         | 可                                     | 可                                 | 氏名・カナ・電話・会社名・部署・役職 |
| 請求情報の更新                         | 可                                     | 可                                 | 請求先の表示・連絡先変更まで。請求ステータス操作は AD-04 側 |
| サービス設定の更新                     | 可                                     | 不可                               | 権限ロール・利用モジュール・通知設定・MFA 必須・IP 制限 |
| 一時停止・再開                         | 可                                     | 可                                 | 理由入力・ログ必須 |
| 解約                                   | 可                                     | 不可                               | 理由入力・ユーザー名入力による二重確認必須 |
| プラン変更                             | 可                                     | 不可                               | 請求・利用可能機能へ影響するため |
| メールアドレス変更                     | 可                                     | 不可                               | アカウント所有権に関わるため専用操作 |
| ユーザー削除                           | 可                                     | 不可                               | 論理削除、要二重確認 |
| 新規ユーザー招待                       | 可                                     | 不可                               | アカウント発行に関わるため |
| 招待再送                               | 可                                     | 可                                 | 旧トークン失効・ログ必須 |
| 招待キャンセル                         | 可                                     | 不可                               | アカウント発行フローの停止に関わるため |
| パスワードリセットなどの定型通知       | 可                                     | 可                                 | 本文自由入力なし |
| 管理者→ユーザーへの自由文メール送信    | 可                                     | 不可                               | 理由入力・ログ必須 |
| メール送信履歴の本文閲覧               | 可                                     | 不可                               | スタッフには本文を返却・表示しない |
| CSVエクスポート                        | 可                                     | 不可                               | 一括個人情報出力のためログ必須 |

| アンケート編集・削除・移管・回答データ削除 | AD-02 では提供しない                  | AD-02 では提供しない              | AD-03 / AD-12 側の権限設計に従う |
| 管理者操作ログの削除・編集             | 不可                                   | 不可                               | 全管理者に対して禁止 |

アブロードマネージャーとアブロードスタッフの主な差分は、アカウント発行・所有権変更・契約変更・プラン変更・削除・一括出力・自由文メール・セキュリティ設定を社内最上位ロール限定とする点で定義する。

権限判定は画面表示だけでなく API 側でも再検証し、直接URLアクセスや API 直叩きによる操作を拒否する。

### 7.3 データ永続化

- ユーザーテーブルに論理削除フラグ（`deleted_at`）を持つ。
- 管理者操作ログは専用テーブルで管理し、外部キー制約でユーザーテーブルと紐付ける。
- メール送信履歴は専用テーブルで管理する（送信日時・件名・本文・送信先・送信者）。
- 招待レコードはトークン・有効期限・ステータス・失効日時を持つ専用テーブルで管理する。
- 招待再送時は旧トークンを失効済みとして保存し、最新トークンのみ有効にする。

### 7.4 ログ・監査

- 以下の操作はすべて管理者操作ログテーブルに記録する。
  - CSVエクスポート
  - ステータス変更・プラン変更・メールアドレス変更・削除
  - パスワードリセット送信・管理者からのメール送信
  - AD-03 / AD-12 側で実行されたアンケート編集・削除・移管・回答データ削除
  - データエクスポート
- ログの記録項目: 操作種別・対象ユーザーID・操作者ID・操作日時・変更前値・変更後値・理由。
- CSVエクスポートでは、検索条件、出力件数、出力日時、操作者IDを記録する。
- 管理者操作ログは改ざん不可とし、編集・削除不可とする。
- 管理者操作ログの保持期間は、監査要件と個人情報保護方針に基づき本番化前に確定する。
- メール送信履歴、ログイン履歴、招待履歴など他のログ種別は各データ種別の運用方針に従う。

### 7.5 想定データ量

- 管理対象ユーザー数: 数百〜数千件規模を想定。
- ログイン履歴の保持期間: 直近1年分を表示対象とし、それ以上はアーカイブ。

### 7.6 受け入れ基準

- [ ] 検索条件の AND 組み合わせで正しく絞り込まれる
- [ ] 部署・役職を含む詳細フィルタと保存済み条件が機能する
- [ ] CSVエクスポートはアブロードマネージャーのみ実行でき、検索条件・出力件数・操作者が管理者操作ログに記録される
- [ ] 解約済み以外のユーザーに削除ボタンが表示されない
- [ ] 一時停止時に既存セッションが失効し、再開後も再認証が必要になる
- [ ] アブロードスタッフは一時停止・再開を実行できるが、解約は実行できない
- [ ] 解約はアブロードマネージャーのみ有効または一時停止から実行でき、理由入力とユーザー名入力による二重確認が必要になる
- [ ] メールアドレス変更は通常の基本情報保存では実行されず、アブロードマネージャーの専用操作からのみ実行される
- [ ] メールアドレス変更後に変更通知メールが変更後アドレスへ送信される
- [ ] サービス設定、プラン変更、新規ユーザー招待、招待キャンセル、ユーザー削除はアブロードマネージャーのみ実行できる
- [ ] 同一メールの未登録招待がある場合、新規招待は作成されず再送またはキャンセル導線が表示される
- [ ] 招待再送後に旧URLが失効し、新URLのみ利用できる
- [ ] 論理削除済みユーザーは通常一覧から除外され、復元操作は AD-02 で提供されない
- [ ] 権限のない管理者ロールからの更新・削除・エクスポート API が拒否される
- [ ] 個人情報エクスポートはアブロードマネージャーのみ実行でき、アブロードスタッフでは拒否される
- [ ] 自由文メール送信はアブロードマネージャーのみ実行でき、アブロードスタッフでは拒否される
- [ ] メール送信履歴の本文はアブロードマネージャーのみ閲覧でき、アブロードスタッフ向け API レスポンスには本文が含まれない
- [ ] AD-02 ではアンケート編集・削除・移管・回答データ部分削除を直接行えない
- [ ] AD-03 / AD-12 側でアンケート移管後、AD-02 の旧所有者一覧から対象アンケートが消え、移管先ユーザー側に表示される
- [ ] 全管理者操作がログに記録され、変更前後の値を確認できる
- [ ] 管理者操作ログが管理者自身でも削除・編集できない
- [ ] データエクスポートに対象ユーザーの必要な個人データが含まれ、秘匿トークンや認証情報が含まれない
- [ ] 内部メモが追記履歴として表示され、登録済みメモを直接編集・削除できない
- [ ] CSVエクスポートが検索条件に一致する全件を出力する
