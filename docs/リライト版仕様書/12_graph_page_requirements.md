## 1. 文書目的

- 本書は、グラフ分析画面の要件を定義する。
- 本書を正本とする。
- 旧 `07_graph_analysis.md`（Insight Finder 構想）と旧 `12_graph_page_requirements.md` v1.x を本書に統合する。旧 `07_graph_analysis.md` は `_archive/` 退避対象。

## 2. 画面概要

- SPEED レビュー画面から遷移し、同一アンケート `surveyId` に紐づく回答データを設問別の集計グラフとして一覧する画面。
- 画面上部に期間フィルタと表示設定のコントロールバー、下部に設問別チャートカードの 2 列グリッドを配置する。
- 会期内の期間絞り込み、表示設定の切替、Excel レポートの一括出力を提供する。
- 想定利用者は広告運用担当と管理者（チーム共有用レポートを外部配布する役割）。

## 3. 画面要件

### 3.1 上部情報
- パンくず（アンケート一覧 > SPEED レビュー > グラフ分析）を表示する。
- 画面タイトルはアンケート名を含む形（例: 「グラフ分析: {アンケート名}」）で表示する。
- ヘッダ右端に `Excel 一括出力` ボタンを設ける。

### 3.2 分析コントロール（sticky ツールバー）
- **期間フィルタ**
  - セレクタは「会期全体」「N日目（会期内の各日）」「カスタム範囲」の 3 系統。会期範囲は回答データと会期設定から自動決定する。
  - 「カスタム範囲」選択時のみ、開始日時・終了日時の日付ピッカーを展開する。日本語ロケールとする。
  - リセット導線で「会期全体」に戻す。
- **表示設定**（全チャートに一括適用、`localStorage` に永続化）
  - サマリー（カード上のインサイトバッジ）
  - 数値ラベル（グラフ内数値表示）
  - 中央合計（ドーナツ中央の合計表示）
  - 詳細表（カード下部のデータ表）
  - グリッド線（棒グラフのみ）
- 期間変更・表示設定変更は即時に全カードを再描画する。

### 3.3 チャートカード
- 1 設問につき 1 カード（マトリクス設問はカード内のセレクタで行を切替）。
- カード構成: 設問番号 chip（`Q1` 等）、設問タイトル、有効回答件数、インサイトバッジ、グラフ、詳細データ表。
- インサイトバッジは「サマリー」ON かつ選択式・評価尺度のみ表示する（例: `Top {選択肢} {%}`、`Avg {平均} / {段階数}`）。

### 3.4 状態表示
- **読込中**: スケルトンカードを複数枚表示する。
- **エラー**: エラーメッセージと再試行のためのガイドを表示する。
- **設問 0 件**: 「このアンケートにグラフ化可能な質問がありません。」を表示する。

### 3.5 Excel 出力 UI
- `Excel 一括出力` ボタン押下で出力オプションモーダルを開く。
- モーダル項目（既定すべて ON）:
  - 目次シート
  - グラフ画像
  - マトリクス全項目
  - 対象外設問
- 実行中は画面右下に進捗オーバーレイ（アイコン・テキスト・進捗バー・パーセント）を表示する。
- 完了時は完了アイコン表示後、数秒でフェードアウトする。

### 3.6 レスポンシブ
- チャートグリッドは広幅で 2 列、狭幅で 1 列とする。

## 4. 状態と挙動

### 4.1 初期表示
- URL パラメータ `surveyId` を受け取り、該当アンケートの設問定義と回答データを取得する。
- 期間フィルタの既定は「会期全体」。
- 表示設定は `localStorage` に保存された値を復元する。未保存時は既定値を適用する。
- 取得失敗時はエラー表示と再試行導線を示す。

### 4.2 期間フィルタ
- フィルタ値の変更により対象回答を絞り込み、すべての設問カードを再集計・再描画する。
- 「カスタム範囲」は開始・終了の両方が揃った時点で反映する。

### 4.3 表示設定
- チェック変更は即時適用し、`localStorage` に保存する。
- 再読込後も選択状態が復元できること。

### 4.4 マトリクス行セレクタ
- マトリクス設問では、カード内セレクタで行（または列）を切り替え、グラフ・chip・有効回答件数・詳細表を同期更新する。

### 4.5 Excel 出力中の挙動
- 出力中は `Excel 一括出力` ボタンを操作不可とし、進捗オーバーレイを表示する。
- 出力中のページ離脱・タブ閉じはブラウザ標準の警告を発火する。
- 出力中も画面操作（スクロール等）が可能であること。

## 5. 機能要件

### 5.1 設問別集計・可視化
- 設問タイプに応じて集計方法と既定チャート種別を出し分ける。

| 分類 | 対象タイプ | 既定チャート | 集計方法 |
|------|------------|--------------|----------|
| 単一選択 | single_choice / dropdown / rating | ドーナツ | 選択肢ごとの件数。選択肢定義順で表示 |
| 複数選択 | multi_choice / ranking | 横棒 | 選択肢ごとの件数。**件数降順** で表示 |
| 評価尺度 | rating_scale | 横棒（赤→緑グラデ） | 段階ごとの件数・平均スコア |
| マトリクス SA | matrix_sa / matrix_single | ドーナツ（行セレクタで切替） | 行×列のクロス集計 |
| マトリクス MA | matrix_ma / matrix_multi | 横棒（行セレクタで切替） | 行×列のクロス集計 |
| 自由記述・添付 | text / free_text / number / date / datetime / time / handwriting / image / photo / file / file_upload / upload | 一覧（グラフなし） | 回答日時降順で一覧化、画像・添付はプレビュー導線付き |
| 説明文 | explanation | 出力対象外 | スキップ |

- 集計は画面側で全件処理する（サーバ集計は前提としない）。
- 設問 ID は `Q1`, `Q2`, ... の形式で chip に整形する。マトリクスは `[現在行 / 総行数]` を付与する。
- 対象外扱いの設問には「対象外」プレフィックス（例: `Ex_Q5`）を付与する。

### 5.2 チャート描画
- チャートライブラリは **ApexCharts** を使用する。
- ドーナツ・棒・横棒・積み上げ棒に対応する。
- 色はサービス共通パレットに従う。評価尺度（rating_scale）のみ赤→緑のグラデーションを動的生成する。
- アニメーション有効、ツールバー（ApexCharts ネイティブ PNG エクスポート）は非表示。
- マトリクス設問はラベル文字数に応じた凡例位置・グラフ高さの自動調整を行う。

### 5.3 詳細データ表
- 設問タイプに応じて 4 テンプレートを切り替える: マトリクス表、評価尺度表、一覧表、通常表（項目・件数・割合）。
- 通常表・評価尺度表は合計行を付与する。
- 自由記述・添付一覧はスクロール領域で表示し、画像・添付はプレビューモーダル起動ボタンを提供する。

### 5.4 Excel 一括出力
- **ライブラリ**: ExcelJS で `.xlsx` を生成。グラフ画像化は html2canvas を経由する。
- **ファイル名**: `{アンケート名}_分析レポート.xlsx`
- **シート構成**:
  - **目次シート**（オプション ON 時）: タイトル、アンケート名、出力日時、設問一覧表（`No. / 設問内容 / シート` の 3 列）。各行は該当シートへのハイパーリンクを持つ。
  - **設問別シート**: シート名は `Q1`, `Q2`, ...。対象外設問は `Ex_Q{番号}`。シート先頭に設問タイトル、次にデータ表、グラフ画像オプション ON のときは右側または下部にグラフ画像を配置する。
- **マトリクス設問**: 各行を個別のグラフとして一時描画し画像化して配置する（SA はドーナツ、MA は横棒）。
- **データ表**: 件数・割合の数値書式（例: `件`、パーセント）を適用する。
- **画像品質**: 高解像度スケールでキャプチャする。
- **処理中の UI**:
  - 進捗オーバーレイで「{総数}件中 {n}件目を処理中...」とバーを更新する。
  - 重い処理は適度にメインスレッドを解放し、画面操作を阻害しないようにする。
- **完了・エラー**:
  - 成功時はトーストとオーバーレイ完了表示。
  - 失敗時はエラートーストを表示し、オーバーレイを閉じる。

### 5.5 離脱警告
- Excel 出力中のみ、ページ離脱時に `beforeunload` 警告を発火する。通常操作時は警告しない。

### 5.6 URL パラメータ
- `surveyId`: データソース特定用。未指定時はフォールバック ID を使用する。
- 期間・表示設定の URL ハンドオフは対象外とする（将来計画）。

## 6. 連携要件（参考）

### 6.1 初期表示に必要な情報
- アンケート設問定義（ID、タイプ、タイトル、選択肢、マトリクス行/列、評価尺度設定、会期開始/終了）
- 回答データ（回答 ID、回答日時、設問ごとの回答値）

### 6.2 入力データ形式
- 設問定義と回答データはそれぞれ JSON で取得する。
- 回答値は設問タイプごとに配列・オブジェクト・スカラーが混在しうるため、正規化ロジックで吸収する（例: 単一選択でも配列化されている場合に対応）。
- 回答データは新形式（`{ surveyId, answers: [...] }`）と旧形式（配列直）の両方を許容する。

### 6.3 設計上の論点
- エンドポイント命名、集計のクライアント/サーバ分担、設問スキーマ拡張時の集計ロジック変更方針。

## 7. 失敗系要件

- **データ取得失敗**: 設問定義取得失敗はエラー表示、回答データ取得失敗は空配列として扱う。
- **回答 0 件**: 期間フィルタの各日オプションは抑止、「カスタム範囲」のみ表示する。
- **設問 0 件**: グラフ 0 件のメッセージを表示する。
- **Excel 出力失敗**: エラートーストを表示し、画面を通常状態へ復旧する。
- **マトリクス行 0 件**: 該当設問を空状態として扱う（対象外設問扱い）。

## 8. 非機能要件

- **パフォーマンス**: 期間・表示設定変更時の再描画が利用者を待たせない応答時間であること。設問数が多いアンケートでも UI 操作を阻害しない。
- **セキュリティ**: 設問テキスト・回答値は画面表示時にエスケープする（XSS 対策）。
- **アクセシビリティ**: ローディング・エラー・セレクタに適切な ARIA 属性を付与する。
- **ブラウザ対応**: 最新 2 バージョンの Chrome / Firefox / Safari / Edge。
- **ローカル永続化**: 表示設定のみ `localStorage` に保存する。PII は保存しない。

## 9. 手動確認観点

- パンくず・タイトルがアンケート名込みで正しく表示される。
- 期間フィルタの「会期全体」「N日目」「カスタム範囲」が期待通り動作し、リセットで全体に戻る。
- 表示設定 5 項目の変更が全カードに反映され、再読込後も復元される。
- 設問タイプ別に既定チャート種別が出し分けられる（単一選択=ドーナツ、複数選択=件数降順横棒、評価尺度=赤→緑グラデ棒、マトリクス=行セレクタ、自由記述=一覧）。
- マトリクス行セレクタで chip・グラフ・件数が同期更新される。
- Excel 一括出力が目次シート + 設問別シート + グラフ画像で期待通り生成される。
- Excel 出力中のページ離脱で標準警告が表示される。

## 10. 現行モックとの差分（要整合）

現行モックは以下の点で本番要件から未達のため、本番実装時に解消する:

- **認可ゲート欠落** — `graph-page.html` は URL 直打ちで誰でもアクセス可能。SPEED レビュー側の遷移時判定のみに依存しており、本画面自体に二重防御がない。SPA ルータ側またはサーバセッション確認の追加を要する。
- **属性フィルタ / ドリルダウン / クロス集計 / VS モード未実装** — 旧 `07_graph_analysis.md` が Premium 機能として規定していたが、UI もロジックも存在しない。
- **Premium Upsell Modal 未実装** — プラン判定に基づくアップセル導線が未実装。
- **PDF レポート未実装** — 現行出力は Excel のみ。PDF 化方式は未決定。
- **自動考察コメント未実装** — 旧 `07_graph_analysis.md` の構想のみで実装なし。
- **3-Pane Layout 未実装** — 旧 `07_graph_analysis.md` の左サイドバー + Canvas Masonry 構成は採用されず、単一メインカラム + sticky ツールバーで実装されている。
- **棒/円切替ボタン無効** — 内部フラグ `allowToggle` が常に false 扱いで、ユーザによるチャート種別切替が実質無効。
- **マトリクス MA の列セレクタ未実装** — マトリクス MA で「列で切替」する想定が残存するが、現行は行セレクタのみ。
- **期間・表示設定の URL ハンドオフ未実装** — 共有 URL からの再現ができない。
- **多言語対応未着手** — 画面文言が日本語ハードコード、設問タイトルも `survey.name.ja` 直参照。
- **Excel 出力の「マトリクス全項目」オプション未連動** — モーダルで選択できるが、現行は常に全行出力。
- **`surveyId` のホワイトリスト検証なし** — URL から取得した値をそのまま fetch パスに連結する。
