# 請求書一覧 要件定義書

| 項目 | 値 |

## 1. 文書目的

- 本書は、請求書一覧画面の要件を定義する。
- 本書を正本とする。
- 請求書詳細画面（`invoice-detail.html`）は別書の対象とし、本書では遷移先として扱う。

## 2. 画面概要

- 利用者が直近の請求状況を一覧で確認する画面。
- 請求月、請求対象期間、請求書番号、契約区分、請求額をカード形式で表示する。
- カード押下で請求書詳細画面へ遷移する。
- 共通ヘッダー、サイドバー、フッターは `src/main.js` により placeholder へ注入される。

## 3. 画面要件

### 3.1 ページタイトル

- 画面タイトル「請求書一覧」を表示する。
- 補足文「直近の請求状況を確認できます。」を表示する。
- パンくず領域を表示する。

### 3.2 件数表示

- 一覧上部に「N件表示中」を表示する。
- 初期表示時は `0件表示中` とし、データ取得後に件数を更新する。

### 3.3 ステータスメッセージ

- 読み込み中、データなし、取得失敗のメッセージを一覧上部に表示する。
- メッセージ領域は `role="status"` / `aria-live="polite"` を持つ。
- 取得失敗時は「再読み込み」ボタンを表示する。

### 3.4 請求書カード

- 請求書はカード形式で縦に表示する。
- カードごとに以下を表示する。

| 項目 | 表示内容 |
| --- | --- |
| 請求月 | `YYYY年MM月請求` |
| 請求対象期間 | `YYYY-MM-DD〜YYYY-MM-DD` |
| 請求書番号 | 表示用請求書番号 |
| 契約区分 | `個人` または `グループ` |
| 請求額 | 税込金額を `¥ 99,999` 形式で表示 |

- 請求ステータス（未入金 / 入金済 / 延滞 / 取消）は内部的に集約するが、現在のカード上には表示しない。
- 表示用請求書番号と、詳細遷移に使う `invoiceId` は別項目として扱う。
- カードはクリックまたは Enter / Space キーで選択できる。
- 選択時は `invoice-detail.html?id=<invoiceId>` へ遷移する。

### 3.5 表示順

- 請求書は発行日の新しい順に表示する。
- 同じ請求月・契約区分の請求は、一覧では1件に集約して表示する。

## 4. 状態と挙動

### 4.1 初期表示

- `core/invoices.json` から請求書データを取得する。
- `core/groups.json` を参照し、請求が個人契約かグループ契約かを判定する。
- データ取得中は「読み込み中です…」を表示する。

### 4.2 データなし

- 請求書が 0 件の場合、「対象の請求書がありません。」を表示する。
- 一覧は空表示とし、件数は `0件表示中` とする。

### 4.3 取得失敗

- 請求データ取得に失敗した場合、「請求データの取得に失敗しました。ページを再読み込みしてください。」を表示する。
- 「再読み込み」ボタン押下でページを再読み込みする。

### 4.4 請求ステータス集約

- 集約元の請求に `overdue` が含まれる場合、集約後の状態は延滞とする。
- `overdue` がなく `unpaid` が含まれる場合、未入金とする。
- すべて `canceled` の場合、取消とする。
- 上記以外は入金済とする。

## 5. 非機能要件

### 5.1 レスポンシブ

- PC、タブレット、スマートフォンでカード内容が見切れないこと。
- 請求額、請求月、契約区分が小画面でも読み取れること。

### 5.2 アクセシビリティ

- 請求書一覧は `aria-label="請求書一覧"` を持つ。
- 請求書カードはキーボードで選択できる。
- 読み込み・エラーなどの状態メッセージは支援技術へ通知する。

### 5.3 セキュリティ

- 本画面はログイン済みユーザー向け画面とする。
- 表示する請求書は、ログイン中ユーザーが閲覧できるものに限定する。
- URL 直打ちやデータ取得 API 側でも閲覧権限を確認する。

## 6. 画面フロー

- ダッシュボードまたはプレミアム案内画面の「請求書確認」導線 → 本画面。
- 本画面の請求書カード選択 → `invoice-detail.html?id=<invoiceId>`。

## 7. 既知の課題（実装と要件のギャップ）

| 区分 | 項目 | 内容 |
| --- | --- | --- |
| API | 請求書 API 未接続 | 現在は `data/core/invoices.json` を取得している。サーバ取得方式は画面外の確認事項。 |
| 権限 | 閲覧権限 | 現在の画面単体では、請求書ごとの閲覧権限を確認できない。 |
| 表示 | ステータスフィルタ | JS 側にフィルタ処理があるが、HTML 側に `invoice-status-filter` が存在しない。採用有無は画面外の確認事項。 |
| 集約 | 月次・契約区分ごとの集約 | 同一月・同一契約区分の請求を1件にまとめる。集約ルールは開発会社と確認する。 |

## 8. 本番化要件

本セクションでは、完成済み画面を本番環境へ移す際に、画面からは直接確定できないため開発会社と確認する事項をまとめる。API・DB・ログ・権限などの詳細は、画面上の確定仕様ではなく実装時の確認観点である。

### 8.1 API 契約

以下は確認用のたたき台であり、画面上の確定仕様ではない。詳細スキーマ・パスは開発会社と合意して確定する。

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/invoices` | 請求書一覧取得 | セッション Cookie | 200 `{ invoices: [...] }` ／ 401 未認証 |

- API はログイン済みセッション必須とする。
- サーバ側で閲覧可能な請求書のみ返却する。
- 一覧用に返す項目は、請求書 ID、発行日、請求対象期間、契約区分、請求額、ステータスを含める。

### 8.2 権限・認可

| 利用者種別 | 取扱 |
| --- | --- |
| 未ログイン | ログイン画面へリダイレクト |
| 個人アカウント | 自分の請求書のみ閲覧可 |
| グループ管理者 | 管理対象グループの請求書を閲覧可 |
| 一般メンバー | 請求書閲覧可否は画面外の確認事項 |

### 8.3 業務エラー時の挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| 一覧取得失敗 | 「請求データの取得に失敗しました。ページを再読み込みしてください。」 | 再読み込みボタンを表示 |
| セッション切れ | ログイン画面へ遷移 | 現在 URL を戻り先として保持 |

### 8.4 ログ・監査

- 操作ログ: 請求書一覧表示、請求書カード選択、再読み込み押下。
- 監査ログの要否は画面外の確認事項とする。
- 請求金額や請求先情報は不要にログへ出力しない。

### 8.5 受け入れ基準

- [ ] 請求書一覧が発行日の新しい順で表示される
- [ ] 請求月、請求対象期間、請求書番号、契約区分、請求額が表示される
- [ ] 請求ステータスは現在のカード上には表示されない
- [ ] 表示用請求書番号とは別に、詳細遷移用の `invoiceId` で詳細画面へ遷移する
- [ ] データなし時に「対象の請求書がありません。」が表示される
- [ ] 取得失敗時にエラーメッセージと再読み込み導線が表示される
- [ ] カードクリックで請求書詳細へ遷移する
- [ ] Enter / Space キーでカードを選択できる
- [ ] 未ログインユーザーは URL 直打ちしても画面に到達できない
