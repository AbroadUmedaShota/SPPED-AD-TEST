# プレミアムプラン解約手続き 要件定義書

| 項目 | 値 |

## 1. 文書目的

- 本書は、プレミアムプラン解約手続き画面の要件を定義する。
- 本書を正本とする。
- 解約完了画面（`premium_cancel_complete.html`）は別書の対象とし、本書では遷移先として扱う。

## 2. 画面概要

- プレミアム会員が、自動更新の停止手続きを行うための画面。
- 解約後も現在の契約期間終了日まではプレミアム機能を利用できることを案内する。
- 解約理由アンケートは任意入力とし、解約操作の前に確認モーダルを表示する。
- 共通ヘッダー、サイドバー、フッターは `src/main.js` により placeholder へ注入される。

## 3. 画面要件

### 3.1 ページタイトル

- 画面中央に「プレミアムプラン解約手続き」を表示する。

### 3.2 現在のプランステータス

- カード上部に「現在のプランステータス」を表示する。
- Premium Plan バッジを表示する。
- 有効期限を「YYYY年M月D日 まで」の形式で表示する。
- 現在の画面では閲覧日の当月末日を表示する。

### 3.3 解約後に利用できなくなる機能

- 見出し「解約すると以下の機能が利用できなくなります」を表示する。
- 以下 6 つの機能カードを表示する。
  - データ最長1年保存
  - Excelレポート出力
  - SPEEDレビュー
  - 高度な入力対応
  - 外部システム連携
  - 優先サポート
- 補足文として、解約手続きは自動更新の停止であり、現在の契約期間終了日まではプレミアム機能を利用できることを表示する。

### 3.4 解約理由アンケート

- セクション見出し「アンケート（任意）」を表示する。
- 補足文「今後のサービス改善のため、よろしければ理由をお聞かせください。」を表示する。
- 解約理由はラジオボタンで選択する。未選択でも解約操作は可能とする。

| value | 表示文言 |
| --- | --- |
| `usage_frequency` | 利用頻度が減った |
| `price` | 価格が見合わない |
| `competitor` | 他社サービスに切り替える |
| `other` | その他 |

- 「その他」を選択した場合のみ、具体理由入力欄を表示しフォーカスする。
- その他理由は任意入力とする。

### 3.5 アクション

- 「キャンセルして戻る」ボタンを表示し、押下で `index.html` へ遷移する。
- 「解約する」ボタンを表示し、押下で確認モーダルを開く。
- 「解約する」押下だけでは解約を確定しない。

### 3.6 解約確認モーダル

- モーダルタイトル「本当に解約しますか？」を表示する。
- 自動更新停止後も現在の契約期間満了まではプレミアム機能を利用できる旨を表示する。
- アクションは以下 2 つ。

| ボタン | 挙動 |
| --- | --- |
| 自動更新を停止する | 解約理由を取得し、解約完了画面へ遷移する |
| キャンセル（解約しない） | モーダルを閉じ、画面に戻る |

- モーダル外クリックでも閉じる。

## 4. 状態と挙動

### 4.1 初期表示

- ヘッダー、サイドバー、フッターを読み込む。
- 現在の契約終了日を表示する。
- 解約理由アンケートは未選択状態で表示する。
- その他理由入力欄は非表示で開始する。

### 4.2 その他理由入力欄の表示

- `cancelReason = other` 選択時、`other-reason-container` を表示する。
- `otherReasonDetail` にフォーカスする。
- 他の理由を選択した場合、その他理由入力欄を非表示にする。

### 4.3 解約確定

- 確認モーダルの「自動更新を停止する」押下で解約確定処理を行う。
- 解約理由が未選択の場合は `no_reason` として扱う。
- 「その他」選択時は `otherReasonDetail` の値を併せて扱う。
- 成功後は `premium_cancel_complete.html` へ遷移する。

## 5. 非機能要件

### 5.1 レスポンシブ

- PC、タブレット、スマートフォンで主要情報と操作ボタンが見切れないこと。
- 機能カードはスマートフォン 2 列、デスクトップ 3 列で表示する。

### 5.2 アクセシビリティ

- 解約理由のラジオボタンはラベルクリックで選択できる。
- 確認モーダルは `role="dialog"` / `aria-modal="true"` / `aria-labelledby` を備える。
- モーダル表示中はフォーカスをモーダル内に閉じ込める。
- Escape キーで閉じる挙動は画面外の確認事項とする。

### 5.3 セキュリティ

- 本画面はログイン済みのプレミアム会員または無料トライアル中ユーザーが利用する。
- 未ログイン、未契約、解約予約済みユーザーの扱いは画面外の確認事項とする。
- 解約確定はサーバ側の契約状態を確認したうえで行う。

## 6. 画面フロー

- `premium_signup_new.html` の既存会員ビュー「プラン解約」または「トライアル終了」導線 → 本画面。
- 本画面「キャンセルして戻る」 → `index.html`。
- 本画面「解約する」 → 解約確認モーダル。
- 解約確認モーダル「キャンセル（解約しない）」 → 本画面。
- 解約確認モーダル「自動更新を停止する」 → `premium_cancel_complete.html`。

## 7. 既知の課題（実装と要件のギャップ）

| 区分 | 項目 | 内容 |
| --- | --- | --- |
| API | 解約 API 未接続 | 現在は解約確定後に完了画面へ遷移するのみ。契約状態の更新は画面外の確認事項。 |
| データ | 契約終了日 | 現在は閲覧日の当月末日をクライアントで算出して表示している。契約終了日はサーバ値を正とする。 |
| データ | 解約理由 | 現在は console 出力のみ。保存先、利用範囲、保管期間は画面外の確認事項。 |
| 確認用処理 | `localStorage` 更新 | `simulationUserData` を更新する処理は確認用であり、契約状態の正本にはしない。 |
| アクセシビリティ | モーダル属性 | `role` / `aria-*` / フォーカストラップが未整備。 |

## 8. 本番化要件

本セクションでは、完成済み画面を本番環境へ移す際に、画面からは直接確定できないため開発会社と確認する事項をまとめる。API・DB・ログ・権限などの詳細は、画面上の確定仕様ではなく実装時の確認観点である。

### 8.1 API 契約

以下は確認用のたたき台であり、画面上の確定仕様ではない。詳細スキーマ・パスは開発会社と合意して確定する。

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/subscription/me` | 契約状態と契約終了日の取得 | セッション Cookie | 200 `{ plan, status, currentPeriodEnd }` ／ 401 未認証 |
| `POST /api/subscription/cancel` | 自動更新停止 | `{ reason, otherReason }` | 200 `{ ok: true, currentPeriodEnd }` ／ 400 バリデーション ／ 409 状態不正 |

- 全エンドポイントはログイン認証済みセッション必須。
- 解約 API は CSRF 対策と二重送信防止を備える。
- 解約理由は任意項目として扱う。

### 8.2 権限・認可

| 利用者種別 | 取扱 |
| --- | --- |
| 未ログイン | ログイン画面へリダイレクト |
| Standard ユーザー | アクセス不可 |
| アクティブなプレミアム会員 | 本画面を利用可 |
| 無料トライアル中ユーザー | トライアル終了導線として利用可 |
| 解約予約中ユーザー | 解約済み状態を案内し、自動更新再開導線へ誘導するかは画面外の確認事項 |

### 8.3 業務エラー時の挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| 契約状態取得失敗 | 「契約情報の取得に失敗しました。時間をおいて再度お試しください。」 | 解約ボタンを無効化 |
| 解約 API 失敗 | 「自動更新の停止に失敗しました。もう一度お試しください。」 | モーダル内または画面上部にエラー表示 |
| 既に解約予約済み | 「すでに自動更新は停止されています。」 | 完了画面またはプラン案内へ誘導 |
| セッション切れ | ログイン画面へ遷移 | 現在 URL を戻り先として保持 |

### 8.4 データ永続化

- 契約状態はサーバ側契約テーブルを正とする。
- 解約理由を保存する場合、契約 ID、ユーザー ID、理由、自由記述、送信日時を記録する。
- 解約理由の保管期間、閲覧権限、分析利用範囲は画面外の確認事項とする。

### 8.5 ログ・監査

- 操作ログ: 解約画面表示、解約確認モーダル表示、解約確定、解約失敗。
- 監査ログ: 自動更新停止はユーザー ID、契約 ID、実行時刻、停止前後状態を記録する。
- 個人情報や自由記述はアプリケーションログへ出力しない。

### 8.6 受け入れ基準

- [ ] プレミアム会員のみ本画面へ到達できる
- [ ] 契約終了日がサーバ値で表示される
- [ ] 解約理由は未選択でも解約確認へ進める
- [ ] 「その他」選択時のみ自由記述欄が表示される
- [ ] 「解約する」押下で確認モーダルが表示され、即時解約されない
- [ ] 「キャンセル（解約しない）」でモーダルが閉じる
- [ ] 「自動更新を停止する」成功後に `premium_cancel_complete.html` へ遷移する
- [ ] 解約 API 失敗時に入力内容を保持し、再試行できる
- [ ] モーダルに必要なアクセシビリティ属性が付与される

## 10. 用語集

| 用語 | 定義 |
| --- | --- |
| 解約 | プレミアムプランの自動更新を停止する操作。 |
| 契約期間終了日 | 現在のプレミアム機能を利用できる最終日。 |
| 解約予約中 | 自動更新は停止済みだが、契約期間終了日まではプレミアム機能を利用できる状態。 |
| Standard | プレミアム期間終了後に移行する通常プラン。 |
