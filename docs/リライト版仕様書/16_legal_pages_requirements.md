# 法務静的ページ群 要件定義書

## 1. 文書目的

- 本書は、法務 3 静的ページ（利用規約・個人情報保護方針・特定商取引法に基づく表示）の要件を定義する。
- 本書を正本とし、法令上の最小要件と運用規定をまとめる。
- 3 画面とも JavaScript ロジックを持たない完全静的ページであり、共通シェル（header / sidebar / footer）と Tailwind CSS のみで構成される。

## 2. 画面概要

- 日本国内の法令（消費者契約法・特定商取引法・個人情報保護法）および BtoB SaaS の慣行に沿って、運営者が登録ユーザー・回答ユーザー・潜在顧客に開示すべき法務情報を公開する。
- 3 画面は共通のレイアウト（ヘッダー・サイドバー・メインカード・フッター）で構成し、本文のみ各画面で異なる。
- 法務原文は HTML に直接ベタ書きとする。CMS / 多言語 / バージョン管理は本版対象外。
- premium 課金機能のリリース前提として、特定商取引法ページは必須公開とする。

### 2.1 想定利用者と典型シーン

- 登録ユーザー（運営担当者・管理者）: サービス利用前後にフッター経由で参照する。
- 潜在顧客: サービストップのフッターリンクから参照する。
- 回答ユーザー: アンケート回答画面のフッター経由で個人情報の取扱いを確認する。
- 監査・法務担当: 社内ステークホルダーとして内容確認を行う。

### 2.2 対象範囲

- 対象:
  - 利用規約（23 条構成）
  - 個人情報保護方針（7 セクション構成）
  - 特定商取引法に基づく表示（10 行のテーブル）
  - フッター「規約等」リンク群からの遷移
- 対象外:
  - 法務原文の文言レビュー（弁護士確認・業界横並び調査）
  - CMS 化・Markdown 管理・バージョン管理ワークフロー
  - 多言語化（現状は ja のみ）
  - 同意ログ取得（クリック時のタイムスタンプ記録等）

## 3. 共通仕様

### 3.1 レイアウト

- `<body>` 直下に以下 5 ブロックを構成する。
  1. モバイル用サイドバーオーバーレイ
  2. ヘッダー（共通シェルから動的注入）
  3. サイドバー（共通シェルから動的注入）
  4. メインコンテンツ（タイトル + コンテンツカード）
  5. フッター（共通シェルから動的注入）
- メインコンテンツ領域は最大幅を揃え、本文ラッパーはカード状に表示する。
- タイトル領域にはパンくず placeholder と `<h1>` を配置し、下罫線で本文と区切る。

### 3.2 共通動作

- 3 画面ともページ固有初期化ロジックを持たず、`main.js` の `switch (currentPage)` には該当ケースを追加しない。
- パンくずは `breadcrumb.js` の `breadcrumbPaths` に未登録とし、`#breadcrumb-container` は空のまま表示する。
- 共通シェルの注入完了後、テーマ切替・言語切替・Esc キーグローバルリスナーは全画面共通で動作する。

### 3.3 レスポンシブ

- モバイル幅（< 640px）: コンテンツカードのパディングを縮小し、サイドバーはドロワー化する。
- タブレット以上: カードパディングを拡大し、メイン領域に余白を確保する。
- デスクトップ（lg 以上）: サイドバーを常時表示する。

### 3.4 アクセシビリティ

- `<html lang="ja">` を設定する。
- 各画面に `<h1>` を 1 つだけ配置する。
- ランドマーク（`<main>` / `<header>` / `<footer>` / `<nav>`）を保持する。
- カラーコントラストは WCAG AA 水準を想定する。
- スキップリンクの追加は推奨対応とする。

### 3.5 SEO / メタ情報

- 各画面に独自の `<title>` を設定する。
- `<meta name="description">` の付与は推奨対応とする。
- ダッシュボード配下のページは noindex 方針を検討する。
- canonical URL は本番ドメイン確定後に追加する。
- 構造化データ（JSON-LD）・OGP は本版対象外とする。

## 4. 画面個別仕様

### 4.1 利用規約（`terms-of-service.html`）

- `<title>`: `SpeedAd - 利用規約`
- `<h1>`: 利用規約
- 構成: 第 1 条〜第 23 条（23 条構成）+ 末尾に制定日表記。
- 主要条文（MVP 必須）:
  - 第 1 条 適用 / 第 2 条 定義（9 用語）
  - 第 3 条 登録 / 第 4 条 登録事項の変更 / 第 5 条 パスワード及びユーザー ID の管理
  - 第 6 条 禁止事項（18 号）
  - 第 7 条 本サービスの停止等 / 第 8 条 権利帰属
  - 第 9 条 登録抹消等 / 第 10 条 退会
  - 第 11 条 本サービス料金 / 第 12 条 本サービスの内容の変更、終了
  - 第 13 条 保証の否認及び免責（AWS / Google OCR 等の外部依存免責を含む）
  - 第 14 条 データベースサービス
  - 第 15 条 個人情報保護 / 第 16 条 情報の利用目的
  - 第 17 条 事例の公開 / 第 18 条 委託
  - 第 19 条 本規約等の変更 / 第 20 条 連絡／通知
  - 第 21 条 サービス利用契約上の地位の譲渡等
  - 第 22 条 分離可能性 / 第 23 条 準拠法及び管轄裁判所
- 末尾表記: `YYYY年MM月DD日制定` または `YYYY年MM月DD日改訂` を必ず付与する（MVP 必須）。
- パンくず（規定）: アンケート一覧 → 利用規約。

### 4.2 個人情報保護方針（`personal-data-protection-policy.html`）

- `<title>`: `個人情報保護方針 - SPEED AD`
- `<h1>`: 個人情報保護方針
- 構成: 序文（法令遵守宣言）+ 7 セクション。
  1. 個人情報の取得（取得手段と取得項目）
  2. 個人情報の利用目的（運営・問い合わせ対応・案内メール・通知・不正利用者特定の 5 項目）
  3. 個人情報の第三者提供（法令・生命身体・公衆衛生・行政協力の 4 例外）
  4. 個人情報の管理（セキュリティ対策の宣言）
  5. 個人情報の開示・訂正・削除（請求手続き）
  6. 個人情報保護方針の変更（効力発生条件）
  7. お問い合わせ窓口（運営事務局・連絡先メール）
- 個人情報保護法 26 条（安全管理措置）を満たす記述とする。
- 末尾表記: `YYYY年MM月DD日制定` を必ず付与する（MVP 必須）。
- パンくず（規定）: アンケート一覧 → プライバシーポリシー。

### 4.3 特定商取引法に基づく表示（`specified-commercial-transactions.html`）

## 5. 法令準拠の最小要件

| 法令 | 必須掲示事項 | 対応画面 |
| --- | --- | --- |
| 民法 / 消費者契約法 | サービス提供条件、契約解除、損害賠償の制限 | 利用規約 |
| 個人情報保護法（26 条ほか） | 利用目的、第三者提供、開示・訂正・削除請求、お問い合わせ窓口、安全管理措置 | 個人情報保護方針 |
| 特定商取引法（通信販売） | 事業者名 / 代表者 / 所在地 / 電話 / メール / 販売価格 / 支払時期および方法 / 引渡時期 / 商品代金以外に必要な費用 / 返品条件 の 11 項目 | 特定商取引法に基づく表示 |

## 6. リンク連携

- フッター「規約等」セクションから 3 画面へ同一タブで遷移する。
- 会員登録・プレミアム申込画面の同意チェックボックスラベルから、`target="_blank"` で利用規約・個人情報保護方針を別タブ表示する（推奨）。
- 回答画面のフッターから個人情報保護方針を参照できるようにする。
- 利用規約第 15 条「個人情報保護」から個人情報保護方針への相互リンクは推奨対応とする。

## 7. コンテンツ管理プロセス

### 7.1 改訂フロー

1. 法務担当者が原文改訂版を作成する。
2. 開発者が該当 HTML を編集し、末尾に `YYYY年MM月DD日改訂` を追記する。
3. PR レビューで法務担当者とプロダクトマネージャーの両承認を必須とする。
4. 本番反映時、重大な変更は登録ユーザーへメール通知する。
5. 改訂内容と承認者を改訂履歴ドキュメントに記録する。

### 7.2 バージョン表記

- 利用規約・個人情報保護方針・特定商取引法に基づく表示のいずれも、末尾に制定日または最終改訂日を必ず明記する。

### 7.3 同意記録ポリシー

- 利用規約は第 19 条に基づき、改訂時にメール / アプリ内通知による周知を行う。
- 改正個人情報保護法対応として、同意取得時のバージョン ID・取得時刻の記録を将来的に整備する（§9 参照）。
- 現時点ではサーバー側同意ログは未実装とし、規約リンクへの導線整備のみをスコープとする。

## 8. 受入条件

### 8.1 機能受入

- [ ] フッターの「利用規約」「プライバシーポリシー」「特定商取引法に基づく表示」リンクから各画面へ正しく遷移する。
- [ ] 各画面の `<title>` と `<h1>` が正しい文言で表示される。
- [ ] 共通シェル（header / sidebar / footer）が動的注入される。
- [ ] 特定商取引法画面のテーブルが 10 行表示され、罫線とラベル列幅 30% が適用される。
- [ ] テーマ切替がフッターのアイコンで動作する。
- [ ] モバイル幅でカードパディングが縮小し、サイドバーがドロワー化する。

### 8.2 アクセシビリティ受入

- [ ] `<html lang="ja">` が設定されている。
- [ ] 各画面に `<h1>` が 1 つだけ存在する。
- [ ] 特商法画面のテーブル `<th>` が行ヘッダーとして機能する。
- [ ] フォーカス可能要素にフォーカスリングが表示される。
- [ ] テキストサイズを 200% に拡大しても本文が見切れない。

### 8.3 法務受入（MVP 公開前必須）

- [ ] 利用規約末尾の制定日が確定日付で記載されている。
- [ ] 個人情報保護方針に最終改訂日が明記されている。
- [ ] 誤字・表記揺れが修正されている。
- [ ] 連絡先メールアドレスのドメインが運用窓口と一致している。
- [ ] 特定商取引法画面の事業者情報（住所・電話・代表者名）が登記簿と一致している。
- [ ] 特定商取引法の必須 11 項目が全て記載されている。
- [ ] premium 課金リリース時に特定商取引法画面が公開済みである。

### 8.4 SEO / メタ受入（推奨）

- [ ] 各画面に `<meta name="description">` が設定されている。
- [ ] ダッシュボード配下のページの `robots` 方針が確定している。
- [ ] canonical URL が本番ドメインで設定されている。
