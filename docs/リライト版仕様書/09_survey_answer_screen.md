## 1. 文書目的

- 本書は、アンケート回答画面（エンドユーザー向け）の要件を定義する。
- 本書を正本とする。
- 旧「本書が正」方針は破棄済み。

## 2. 画面概要

- 配信 URL または QR コードから匿名でアクセスした回答者が、アンケートに回答し送信する画面。
- ログイン不要。モバイルファーストの単一カラム構成。
- 対象利用者は以下:
  - **エンドユーザー（来場者・被調査者）**: 実回答を行う。
  - **プレビュー利用者（アンケート作成者）**: 作成画面から「プレビュー」で起動し、見た目・挙動を検証する。
- 回答送信後はサンクス画面（`thankYouScreen.html`）へ遷移する。
- 設問タイプ・多言語設定はアンケート作成画面（`01_survey_creation_requirements`）の出力を受けて描画するのみで、本画面は制限ロジックを持たない。

## 3. 画面要件

### 3.1 画面構成
- 最大幅 768px 固定の単一カラム。PC・タブレット・モバイルで同一レイアウト。
- 上から順に以下を配置する:
  - 言語切替セレクタ
  - アンケートタイトル・説明
  - 設問リスト（縦一列、ページング無し）
  - 名刺プレビューエリア（撮影済み時のみ）
  - フッター（名刺撮影ボタン／送信ボタン／手入力リンク）

### 3.2 言語切替
- アンケート定義に含まれる利用可能言語を自動抽出し、ドロップダウンに列挙する。
- 利用可能言語が 1 件以下の場合は切替 UI ごと非表示にする。
- 切替時は全設問文・選択肢・UI ラベルを再描画し、入力済み値は保持する。
- 対応言語: 日本語 / 英語 / 中文（简体）/ 中文（繁體）/ Tiếng Việt の 5 種。

### 3.3 タイトルカード
- アンケート名と説明を表示する。多言語テキストは現在ロケール向けに解決する。
- タイトル未定義時は「アンケート」と表示する。

### 3.4 設問カード
- 全設問を縦一列に並べる。設問 0 件時は「設問がありません」を表示する。
- 各設問カードは次を含む:
  - 設問番号（Q.01 形式）
  - 必須バッジ（必須設問のみ）
  - 設問文
  - 回答コントロール（設問タイプ別、§4 参照）
- 必須設問には左縁ボーダーによる視覚的マーキングを行う。
- 設問カードはアコーディオン形式で開閉可能とする。

### 3.5 フッター
- 以下 3 要素を固定表示する:
  - **名刺撮影ボタン**（`bizcardSettings` ON 時のみ表示）: 初期「名刺を撮影」、撮影済み時「撮影済み」で非活性化。
  - **送信ボタン**: 押下で送信処理。
  - **手入力リンク**（`bizcardSettings` ON 時のみ表示）: 「名刺が手元に無い方」からの導線。

### 3.6 モーダル
- 送信中モーダル（キャンセル不可）
- 名刺アップロードモーダル（ストレージ選択 / カメラ起動 / プレビュー / 最終確認）
- 手入力モーダル（9 項目フォーム + 確認ステップ）
- 離脱確認モーダル（ブラウザバック・外部リンク時）
- ドラフト復元モーダル（ページロード時、ドラフト存在時のみ）
- 画像拡大モーダル（名刺サムネイルクリック時）

## 4. 設問タイプ別 回答 UI

アンケート作成画面で定義される 12 種の設問タイプをすべて描画する。

| タイプ | UI | 回答値 |
|--------|----|--------|
| `free_answer` | 複数行テキスト（自動リサイズ） | 文字列 |
| `single_answer` | ラジオボタン | 単一文字列 |
| `multi_answer` | チェックボックス | 文字列配列 |
| `dropdown` | プルダウン | 単一文字列 |
| `number_answer` | 数値入力 | 数値 |
| `date_time` | 日付・時刻入力（`date` / `time` / `datetime` 切替） | ISO 形式文字列 |
| `rating_scale` | N 段階評価ラジオ（左右＋任意中間ラベル） | 数値 |
| `matrix_sa` | 行×列のラジオ表（行ごとに 1 選択） | 行 → 列 ID のマップ |
| `matrix_ma` | 行×列のチェックボックス表 | 行 → 列 ID 配列のマップ |
| `handwriting_space` | 手書きキャンバス（ペン/消しゴム/色/太さ/undo/redo/クリア） | base64 画像データ URL |
| `explanation_card` | 説明文のみ（回答収集なし） | — |
| `image_upload` | 画像ファイル選択＋プレビュー | base64 画像データ URL |

### 4.1 設問タイプ別補足
- `free_answer`: 最小/最大文字数の警告をリアルタイム表示する。
- `multi_answer`: 選択数上限 `maxSelections` に到達したら未選択肢を非活性化する。
- `rating_scale`: 段階数（`points`）、左右ラベル、中間ラベル有無をアンケート定義から受ける。
- `handwriting_space`: 高解像度端末の DPI スケーリングに対応する。線幅は 1〜20px、初期値 5。
- `image_upload`: `<input type="file" accept="image/*">`。サイズ・形式の追加検証は未実装（§7 参照）。
- `matrix_*`: 行・列ラベルとも多言語テキスト解決の対象とする。

### 4.2 未対応タイプ
- `normalizeQuestionType()` が該当しない設問タイプは「未対応の設問タイプです」をインラインで表示する。

## 5. 状態と挙動

### 5.1 初期化
- URL パラメータ `preview` / `surveyId` / `answerLocale` を取得する。
- `surveyId` が無く、かつプレビューでもない場合はエラー表示する。
- データソースを以下で分岐する:
  - **プレビューモード (`?preview=1`)**: `localStorage.surveyPreviewData` を入力源とする。
  - **本番モード**: `data/surveys/{surveyId}.json` を取得する。
- ロケールは URL 指定 → アンケート既定 → `ja` の順でフォールバックする。
- 取得失敗時はエラー領域にメッセージを表示しフォームを非表示とする。

### 5.2 プレビュー/本番モード分岐

| 項目 | プレビュー | 本番 |
|------|-----------|------|
| データ入力源 | `localStorage.surveyPreviewData` | `data/surveys/{surveyId}.json` 取得 |
| 名刺撮影・手入力 | 押下でトースト「プレビューのため使用不可」 | 通常動作 |
| ドラフト自動保存 | 無効 | 有効 |
| 離脱警告 | 無効 | 有効 |
| 送信 | `simulateUpload()` 後、現在ページを完了画面で置換 | サンクス画面へ遷移 |
| `localStorage` 保存 | 現行は残るが本番同様に書き込まれる（整合要、§10） | 書き込み |

### 5.3 バリデーション
- **必須設問**: 空回答の場合、送信をブロックする。対象 fieldset を赤枠で強調し、トースト「必須項目を入力してください。」を表示する。
- 上記以外（文字数、数値範囲、選択上限未満、メール形式、マトリクス充填、画像サイズ・形式）は現行ではリアルタイム警告のみ、または未検証（§10 参照）。
- エラー表示は 3 秒で自動非表示のトースト方式。エラー項目へのフォーカス移動・スクロールは未実装。

### 5.4 回答送信
- 二重送信防止のため、送信中は送信ボタンを非活性化する。
- ペイロード構造:
  - `surveyId`
  - `submittedAt`（ISO 8601）
  - `answerLocale`（現在ロケール）
  - `answers`（`{questionId: 値}` + 特殊キー `bizcardImages` / `manualBizcardInfo`）
- 送信成功時の遷移:
  - **本番**: `thankYouScreenSettings.url || 'thankYouScreen.html'` に遷移する。クエリに `surveyId` / `answerLocale` を付与する。`plan === 'premium'` の場合のみ `continuous=true` を付与する。
  - **プレビュー**: 現在ページを完了画面（インライン HTML）で置換する。
- 送信失敗時はエラー領域に文言を表示し、送信ボタンを再活性化する。

### 5.5 ドラフト保存・復元
- 本番モードのみ 30 秒間隔で自動保存する（`hasUnsavedChanges` が立っている時のみ書込）。
- ドラフトキー: `survey_draft_{surveyId}_{sessionId}`（セッション ID はタブ単位で独立）。
- 画像系のうち `bizcardImages` はドラフト対象外（容量対策）。`handwriting_space` / `image_upload` はドラフトに含まれる（§10）。
- ページロード時にドラフトを検出したら復元モーダルを出し、「復元する」でフォームへ流し込む。
- 送信成功時はドラフトを削除する。

### 5.6 離脱警告
- 以下の 2 経路で離脱確認モーダルを表示する:
  - ブラウザバック（`popstate`）
  - ページ外へのリンククリック
- 「離れる」で遷移、「留まる」で現在ページに戻る。
- ブラウザ再読み込み・タブ閉じへの `beforeunload` 警告は未実装（§10）。

### 5.7 多言語解決
- UI ラベル（ボタン・プレースホルダ等）は静的辞書から取得する。
- アンケート定義内の設問文・選択肢・タイトル・説明は多言語テキストオブジェクトを現在ロケール向けに解決する。現在ロケール値が無ければ `ja` → 任意の最初のキー、の順でフォールバックする。
- `<html lang>` 属性を現在ロケールに追従する。

## 6. 名刺撮影/手入力フロー

`bizcardSettings` が ON のアンケートのみフッターに該当ボタンを表示する。

### 6.1 名刺撮影
- ステップ: 表面取得 → 表面プレビュー → 裏面取得（スキップ可）→ 裏面プレビュー → 最終確認 → 保存。
- 取得手段: ストレージ選択 または カメラ起動（`capture="environment"` で背面カメラ優先）。
- 取得済み画像はサムネイル表示。再撮影・削除・拡大表示導線を備える。
- 保存時に `state.answers.bizcardImages = {front, back|null}` として格納する。
- 撮影済み後のボタンは「撮影済み」に切替え非活性化する。

### 6.2 手入力
- 入力項目: 姓・名・メールアドレス・会社名・部署名・役職名・電話番号・郵便番号・住所・建物名の 10 欄（すべて任意）。
- 2 段階保存（入力 → 確認 → 保存）。確認画面からの「修正する」で入力画面に戻る。
- 保存時に `state.answers.manualBizcardInfo` に格納する。

## 7. 連携要件（参考）

### 7.1 入力
- アンケート定義 JSON（設問リスト、タイトル、説明、多言語設定、サンクス画面設定、プラン種別）

### 7.2 出力（送信ペイロード）
- `surveyId` / `submittedAt` / `answerLocale` / `answers`（各設問回答、名刺画像、手入力情報を含む）

### 7.3 設計上の論点
- 実送信 API のエンドポイント・認証ヘッダ・レスポンス形式・リトライ戦略
- 回答データの保管方式（現行は `localStorage` 平文保存の暫定実装）
- OCR 言語判別・名刺データ化連携の実装範囲
- サンクス画面遷移先 URL の既存クエリマージ方式

## 8. 失敗系要件

- **データ取得失敗**: エラー領域にメッセージを表示しフォームを非表示とする。
- **プレビューデータ不在**: `?preview=1` で `surveyPreviewData` が無い場合、その旨を表示する。
- **必須未入力送信**: トーストで通知し、送信をブロックする。
- **送信失敗**: エラー領域に表示し、送信ボタンを再活性化する（現行擬似プログレスでは到達不可）。
- **ドラフト復元失敗**: 新規回答として扱う。

## 9. 非機能要件

- **パフォーマンス**: 設問 50 問想定での初期描画を待たせない応答時間で行う。
- **二重送信防止**: 送信中は送信ボタンを非活性化する。
- **モバイル対応**: Pull-to-Refresh を抑止する（iOS Safari は限定的）。
- **アクセシビリティ**: `<label for>` 紐付け・セマンティック HTML・キーボード操作を基本対応とする（WCAG AA 検証は §10）。
- **ブラウザ対応**: 最新 2 バージョンの Chrome / Firefox / Safari / Edge（モバイル含む）。

## 10. 現行モックとの差分（要整合）

現行モックは以下の点で正式要件から未達のため、本番実装時に解消する。番号が小さいほど優先度が高い。

1. **実送信 API 未接続** — `simulateUpload()` は擬似プログレスのみで常に成功。実サーバ送信・認証・リトライ・失敗時 UI はすべて未実装。
2. **マトリクス設問の回答集約バグ（最重要）** — `matrix_sa` / `matrix_ma` は `{questionId}-{rowId}` の name 空間に分散するが、回答収集は `questionId` 単独で行っており、送信ペイロードから欠落する。**本番実装前に必ず解消**。
3. **`localStorage` への PII 平文保存** — 手入力の氏名・メール・電話・住所、手書き画像、アップロード画像がすべて `localStorage` に平文で保存される。本番運用前に `sessionStorage` / メモリ状態 / サーバ保管へ移行必須。
4. **`beforeunload` 警告未登録** — ブラウザ再読み込み・タブ閉じで未保存警告が出ない。
5. **名刺必須/任意の送信前確認未実装** — アンケート側の名刺必須フラグを送信処理は参照していない。
6. **OCR 言語判別未実装** — 「OCR言語: 日本語 (ダミー)」固定表示。
7. **連続回答判定が `plan === 'premium'` 単独** — アンケート作成画面の `allowContinuousAnswer` 設定が参照されていない。
8. **送信時バリデーションが `required` のみ** — 文字数・数値範囲・選択最小件数・メール形式・画像サイズ/形式は送信ブロックにならない。
9. **`dropdown` のプレースホルダ未生成** — 先頭空 `<option>` が無く、初期値が選択済み扱いになる可能性。
10. **`explanation_card` に `required: true` を付与すると常に失敗扱い** — 設問タイプ側で `required` 無効化すべき。
11. **ドラフトに画像系が含まれる** — `handwriting_space` / `image_upload` の base64 が `localStorage` 容量を圧迫する。
12. **送信済み `survey_response_*` の自動削除なし** — 同一端末で大量回答すると容量圧迫。
13. **プレビュー時も `localStorage.survey_response_*` が書かれる** — プレビュー分岐が保存処理の後にあるため。
14. **サンクス画面 URL の既存クエリマージ未対応** — `thankYouSettings.url` に既存クエリがあると連結で壊れる。
15. **エラー時のフォーカス自動移動・スクロール未実装**
16. **`t('survey.tapToCapture')` 未定義 i18n キー** — フォールバック文字列に依存。
17. **WCAG 2.1 AA 未検証** — `aria-required` / `aria-live` / フォーカストラップ / カラーコントラスト等。
18. **死コード残存** — `.header-unpinned` / `#survey-page-header` / `state.draftExists` / `state.idleTimer` など。

## 11. 本番化要件

本書はモックアップ作成段階の要件として作成したが、最終的に本番環境構築を担う開発会社が実装に着手するために必要な情報を本セクションに集約する。本画面は **未ログイン回答者向けの公開画面** であり、他の管理画面とはセキュリティ・性能要件が大きく異なる点に留意する。

### 11.1 API 契約

想定エンドポイントと入出力。詳細スキーマ・パスは実装着手時に確定する（以下パスは想定）。

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/public/surveys/{surveyId}` | アンケート定義取得（公開・未認証） | クエリ `answerLocale?` | 200 アンケート定義 JSON ／ 404 ／ 410 会期終了 |
| `POST /api/public/surveys/{surveyId}/responses` | 回答送信 | `{ submittedAt, answerLocale, answers, bizcardImages?, manualBizcardInfo?, csrfToken, answerSessionId }` | 201 `{ responseId, redirectUrl }` ／ 400 ／ 409 重複送信 ／ 410 会期終了 ／ 429 レート超過 |
| `POST /api/public/surveys/{surveyId}/bizcard-upload` | 名刺画像アップロード（多段分割送信） | `multipart/form-data`（`answerSessionId`, `side`, `file`） | 200 `{ uploadId }` ／ 413 サイズ超過 ／ 415 形式不正 |
| `POST /api/public/surveys/{surveyId}/answer-session` | 回答セッション開始（CSRF トークン・セッション ID 払い出し） | — | 200 `{ answerSessionId, csrfToken, expiresAt }` |

- 全エンドポイント HTTPS 必須。
- 回答送信は冪等キー（`answerSessionId`）で 1 セッション 1 回のみ受理し、同一セッションからの重複送信は 409 で拒否する。
- レート制限: 回答送信は同一 IP あたり N 回/分、名刺アップロードは同一セッションあたり M 回/分（具体値は実装時確定）。
- CSRF トークンはセッション開始 API で払い出し、送信 API は Double Submit Cookie 等で検証する想定。

### 11.2 権限・認可

- **全公開（未ログインアクセス）前提**。認可制御は行わず、誰でもアクセス可能。
- 代わりに以下で濫用を抑止する:
  - CSRF トークン検証
  - 回答セッション ID によるワンタイム化
  - レート制限（IP ベース）
  - bot 対策（reCAPTCHA v3 等の導入余地。導入要否は要確定）
  - アンケート公開ステータス・会期期間のサーバ側チェック（非公開・会期外は 404/410）
- プレビューモード（ダッシュボード利用者がアンケート作成画面から起動）のみ、認証済みセッションを用いて `localStorage.surveyPreviewData` を利用する。プレビュー時はサーバ書込を行わない。

### 11.3 業務エラー時の挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| アンケート取得失敗（404 / 410） | 「このアンケートは現在ご利用いただけません」 | フォーム非表示 |
| CSRF トークン検証失敗 | 「セッションの有効期限が切れました。ページを再読込してください。」 | セッション再発行を促す |
| 必須未入力（送信前クライアント検証） | トーストで通知、送信ブロック、該当項目にスクロール＋フォーカス | — |
| サーババリデーション（400） | 該当項目にエラー表示 | 入力値保持 |
| 重複送信（409） | 「このアンケートはすでに回答済みです。」＋サンクス画面遷移 | 冪等応答として扱う |
| 会期終了（410） | 「回答受付は終了しました」 | フォーム非活性化 |
| レート超過（429） | 「アクセスが集中しています。しばらく時間をおいて再度お試しください。」 | 自動リトライなし |
| 画像サイズ・形式不正（413 / 415） | 該当画像に個別エラー表示 | 他の入力値は保持 |
| ネットワーク切断 | 「ネットワークに接続できません。接続を確認してから再度お試しください。」 | 送信ボタン再活性化 |
| 送信 5xx | 「ただいま送信に失敗しました。時間をおいて再度お試しください。」 | 自動リトライなし |

### 11.4 データ永続化

- 回答テーブルを新設: `response_id`（主キー）／`survey_id`／`answer_session_id`（ユニーク）／`answer_locale`／`submitted_at`／`answers`（JSON）／`submitter_ip_hash`／`user_agent_hash`。
- 名刺画像・手書き画像はオブジェクトストレージ（S3 等）に保管し、DB には URL / オブジェクトキーのみ保持。
- 手入力名刺情報（氏名・メール・電話・住所など PII）はサーバ側で暗号化保管を検討（要確定）。
- ドラフトはクライアント側のみ（`sessionStorage` 等への移行を推奨）。サーバ側ドラフトは持たない。現行モックの `localStorage` 平文 PII 保存は廃止（§10 項 3）。
- `matrix_sa` / `matrix_ma` の回答は `{ rowId: colId }` / `{ rowId: [colId...] }` の形式で永続化する（§10 項 2 のバグを解消したうえで）。

### 11.5 ログ・監査

- 操作ログ: 回答送信（`response_id`・`survey_id`・タイムスタンプ・IP ハッシュ）。
- アクセスログ: IP / UA / アンケート ID / タイムスタンプ。bot 検知・不正検知の素材とする。
- 失敗ログ: 送信 API の 4xx / 5xx、レート超過、CSRF 失敗。
- PII（手入力名刺情報）は監査ログに出力しない。
- 保管期間は運用方針に従う（最低 90 日を想定、要確定）。

### 11.6 想定データ量

- キャンペーン規模に依存し、1 アンケートあたり **数千〜数十万** 件の回答想定。
- ピーク時（展示会初日など）の同時回答者数は **数百〜数千** を想定。
  - 回答送信 API は水平スケール可能な構成とし、DB は書込負荷への耐性を設計時確定。
  - 名刺画像アップロードは帯域・ストレージ書込の律速になりやすく、事前署名 URL 等による直接アップロードを検討。
- 初期描画は設問 50 問規模を想定。大規模時は LCP / CLS の監視を推奨。
- CDN 配信 / アンケート定義 JSON のキャッシュ（ETag / Cache-Control）で原本 API への負荷を抑制する。

### 11.7 対応ブラウザ・デバイス

- 想定対応: 最新版 Chrome / Edge / Safari / Firefox（直近 1 年以内のメジャーバージョン、モバイル含む）。
- レスポンシブ対応: 最大幅 768px 固定のモバイルファースト単一カラム（§3.1）。PC（横幅 ≥ 1024px）でも同一レイアウトで表示、スマートフォン（横幅 ≥ 375px）を主対象とする。
- IE および非 Evergreen ブラウザは対象外。
- カメラ起動（`capture="environment"`）は対応端末のみ動作。非対応時はストレージ選択にフォールバック。

### 11.8 受け入れ基準

- [ ] 未ログインで配信 URL からアクセスでき、回答送信までサーバへ到達する
- [ ] マトリクス設問の回答が `{ rowId: colId(s) }` 構造で送信ペイロードに含まれる（§10 項 2 のバグが解消される）
- [ ] CSRF トークン・回答セッション ID 検証が有効で、改ざん送信が 400/403 で拒否される
- [ ] 同一セッションからの二重送信が 409 で冪等応答される
- [ ] 会期終了・非公開アンケートへの回答が 404/410 で拒否される
- [ ] IP ベースのレート制限が動作し、429 がクライアントに適切に表示される
- [ ] 手入力 PII・名刺画像・手書き画像が `localStorage` 平文に残らない（ドラフトは `sessionStorage` / メモリ / 画像除外で構成）
- [ ] `beforeunload` による再読込・タブ閉じ警告が動作する（§10 項 4）
- [ ] 送信成功時にサンクス画面（10 番）へ遷移し、`allowContinuousAnswer` / `continuous=true` が 08 番の設定と整合する
- [ ] 設問 50 問・同時回答者数百人のピーク想定で初期描画・送信とも待たせない応答時間を維持する
- [ ] WCAG 2.1 AA（`aria-required` / `aria-live` / フォーカス管理 / カラーコントラスト）を満たす
