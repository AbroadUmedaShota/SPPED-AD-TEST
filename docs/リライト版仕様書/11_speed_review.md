## 1. 文書目的

- 本書は、SPEEDレビュー画面（回答一覧確認・インライン編集画面）の要件を定義する。
- 画面モックアップ（`02_dashboard/speed-review.html` およびペア実装）を正本とする。
- 旧「本書が正」方針は破棄済み。

## 2. 画面概要

- 展示会当日〜会期中の回答仕分けと、回答ごとの詳細確認・インライン編集を行う単一画面。
- アンケート回答と名刺データ化結果を 1 行 1 レコードに結合した一覧を中心に、上部にアナリティクスダッシュボード（KPI + 時間帯グラフ + 設問別グラフ）、右側にフィルタサイドバーを配置する。
- 想定利用者は広告運用担当・営業/マネージャー・管理者。管理者は詳細分析ページ（`graph-page.html`）への起点としても利用する。
- URL パラメータ `?surveyId=xxx` が必須。未指定時はエラー表示のみ。

## 3. 画面要件

### 3.1 上部情報
- パンくず「アンケート一覧 > SPEEDレビュー」を表示する。
- 画面タイトルとアンケート名（`アンケート名: {name}`）を表示する。

### 3.2 アナリティクスダッシュボード
- **KPI 行**: 回答総数カード、現在選択中の設問カード（設問選択モーダル起動 + 詳細分析ボタン `#graphButton`）。
- **時間帯別回答数グラフ**: 折れ線。X 軸モードを `自動`（投稿範囲 ±1h）/ `固定`（9:00-19:00）で切替可能。
- **回答内容の内訳グラフ**: 設問タイプに応じて Chart.js（単一選択・複数選択・rating_scale）または ApexCharts（matrix_sa / matrix_ma）で描画する。
- **集計データテーブル**: グラフと同じ設問の集計を手書きテーブルで併記する。
- 選択中設問がグラフ化不可のタイプ（`text` / `number` / `date` / `handwriting` / `image` / `explanation` など）の場合は、グラフ領域を非対応メッセージに差し替える。

### 3.3 メインテーブル
- 列: 展開ボタン / 回答ID / 回答日時 / 氏名 / 会社名 / 設問回答（動的カラム）の 6 列構成。
- 氏名・会社名は名刺データから取得する。動的カラム「設問回答」は選択中設問の回答を表示する（22 文字超は省略）。
- 名刺データ化が未完了の行はステータス `processing` と判定し、氏名・会社名列に「データ化進行中」アニメーションを表示する。
- 行クリックで回答詳細モーダル、展開ボタンでインライン展開行を表示する。両者は排他ではなく両立する。

### 3.4 インライン展開行
- 名刺画像（表裏タブ切替・回転・ホイールズーム）と名刺情報（会社名・氏名・部署/役職・Email・電話番号）を表示する。
- 再度展開ボタンを押下すると削除する。

### 3.5 回答詳細モーダル
- 左側に名刺情報（画像 + 名刺項目）、右側に設問・回答の一覧を表示する。
- 編集モード切替ボタン（`編集する` / `キャンセル` + `保存する`）でフォームに切替える。対応設問タイプは単一選択 / 複数選択 / matrix_sa / matrix_ma / テキスト。
- 画像操作（±90 度回転、0.5〜5.0 倍ズーム）はモーダル・インライン・拡大モーダル間で状態が同期する。

### 3.6 設問選択モーダル
- 現在のアンケートの全設問を一覧表示する。
- 各設問にグラフ化可否アイコン（`analytics` / `description`）を付与し、選択中設問をハイライトする。
- KPI カード内の設問カード／「設問を変更」リンクから起動する。

### 3.7 フィルタサイドバー（右側フローティング）
- **簡易検索**: 表示対象日セレクト（会期全体 / N 日目 / カスタム範囲）とステータスセレクト（全て / 完了 / 進行中）。
- **詳細検索**: 開始日時・終了日時（flatpickr、時刻指定可）とステータスセレクト。
- `custom` 選択で詳細検索に自動切替する。簡易と詳細のステータスは双方向同期する。
- リセットボタンで会期全体 + 全ステータスに復帰する。
- 初期表示後 2.5 秒で自動クローズ、手動操作でキャンセルする。

### 3.8 ページネーション
- 1 ページ表示件数: 25 / 50 / 100 / 200 から選択。初期値 25。
- 最大 5 ページ ウィンドウ + 先頭/末尾の省略記号補完。
- 前へ / 次へボタン。

### 3.9 UI 状態の永続化
- ソート・フィルタ・ページ・選択中設問を `localStorage` に `surveyId` スコープで保存し、再訪時に復元する。

## 4. 状態と挙動

### 4.1 初期化
- Skeleton 表示 → パンくず注入 → `surveyId` 取得 → UI 状態復元 → データ取得 → データ結合 → 画像 URL 非同期解決 → 初期描画の順で行う。
- `surveyId` 未指定時はエラー行「アンケートIDが指定されていません。」を表示する。

### 4.2 データソース
回答データはクライアントサイドで結合する。以下を並列取得し、主ソース不在時はフォールバックチェーンで再取得する。

| 用途 | 主ソース | フォールバック |
|------|----------|----------------|
| 設問定義 | `data/core/surveys.json` | — |
| アンケート詳細 | `data/demo/demo_surveys/{surveyId}.json` | `data/surveys/enquete/{surveyId}.json` |
| 回答データ | `data/demo/demo_answers/{surveyId}.json` | `data/responses/answers/{surveyId}.json`（`{answers:[...]}` ラップ対応） |
| 名刺データ | `data/demo/demo_business-cards/{surveyId}.json` | `data/responses/business-cards/{surveyId}.json` → `data/business-cards/{surveyId}.json` |

- サンプルデータの生成条件・件数規模は `14_speed_review_sample_data_requirements.md` に独立保持する。本書は消費側、14 番は生成側として責務分離。

### 4.3 ソート・フィルタ
- ソートキーは `answerId` / `answeredAt` / `fullName` / `companyName` / `dynamicQuestion` の 5 種、昇降切替。初期は `answeredAt desc`。
- 文字列ソートは日本語ロケール比較（`localeCompare`）で行う。
- フィルタは日付範囲（単日または範囲）とステータス（完了/進行中）の 2 軸。

### 4.4 編集保存（現行はモック）
- 編集モードで変更を検出した場合、結合済みデータ（`allCombinedData`）をメモリ上で上書きし、一覧を再描画、トースト「回答を更新しました。」を表示する。
- **サーバ送信は一切行わない。ページリロードで変更は消失する**（§9-2 課題）。
- 差分がない場合は「変更はありませんでした。」を表示する。

### 4.5 プレミアム機能ゲーティング
- `#graphButton`（詳細分析）押下時、プレミアムアカウントは `graph-page.html?surveyId=...` に遷移、フリーアカウントはプレミアム誘導モーダル（`modals/premiumFeatureModal.html`）を表示する。
- フリーアカウント判定は `getCurrentGroupAccountType() === 'free'` に加え、**`surveyId` が 8 件のハードコードリスト（`sv_0002_26001` 〜 `sv_0002_26008`）に含まれる場合に強制的に `true` に上書きされる**（§9-1 最重要課題）。

## 5. 連携要件（参考）

### 5.1 初期表示に必要な情報
- アンケート基本情報（名称・ID・会期期間）
- 設問定義（設問 ID / 設問文 / 設問タイプ / 選択肢 / マトリクス行列）
- 回答データ（回答 ID / 回答日時 / 設問回答配列）
- 名刺データ（氏名 / 会社名 / 連絡先 / 部署・役職 / 名刺画像 URL / データ化ステータス）
- 名刺画像（推定パス解決 + 失敗閾値付きの順次 HEAD/GET）

### 5.2 回答編集保存で必要な情報
- 回答 ID、差分が発生した設問の ID と新しい回答値、楽観ロック用識別子。
- 本番実装では、設問タイプごとの値形状（単一=string / 複数=string[] / matrix=行 ID キーの map）に従って送信する。

### 5.3 連動画面
- **名刺データ化設定** (`bizcardSettings.html`): 設定画面で確定した依頼内容に応じて `cardStatus` が `processing` / `completed` に推移し、本画面のステータス表示と整合する。
- **お礼メール設定** (`thankYouEmailSettings.html`): 同一 `surveyId` で設定を共有する兄弟画面。将来クーポン適用状態等を本画面で可視化する場合の連携先。
- **詳細分析ページ** (`graph-page.html`): `#graphButton` の遷移先。データ源泉は同一、本画面は「一覧 + 簡易ダッシュボード」、分析ページは「全設問分析 + Excel 出力」で分担。

### 5.4 設計上の論点
- 回答編集のサーバ保存エンドポイント、楽観ロック方式、エラー応答形式。
- フリーアカウント判定のプラン連動（`plan-capabilities.json`）への切替時期と経路。
- 名刺データ・画像のフォールバック経路（`data/responses/*` / `data/business-cards/*`）の統廃合。

## 6. 失敗系要件

- **データ取得失敗**: 主ソース不在はフォールバックで救済。全失敗時はテーブルに空メッセージと再試行導線を示す。
- **名刺画像失敗**: インライン `onerror` の 1 段フォールバック後に非表示。連続失敗が閾値（5 件）を超えたら以降の HEAD/GET を停止する。
- **編集保存失敗（将来）**: 本番実装では入力値を保持しつつサーバエラーを表示し、競合検知時は再読込導線を示す。
- **認可エラー**: 編集権限がない場合は編集モードを提供しない（将来実装）。

## 7. 非機能要件

- 900 件規模（展示会 1〜5 日 × 1 日 300 件想定）のフィルタ・ソート・ページネーションが実用速度で動作する。
- 画像 URL 解決は同時並列数とリトライ上限で制御し、失敗時のサーバ負荷を抑制する。
- UI 状態は `surveyId` スコープで永続化し、リロード後の利用再開を妨げない。
- レスポンシブ対応: モバイルでもテーブル横スクロール + グラフ縦積みで閲覧可能。

## 8. 手動確認観点

- `?surveyId=xxx` 付き URL で 初期描画され、初期ソートが `answeredAt desc` になる。
- 日付セレクト変更で flatpickr が自動追従し、`custom` で詳細検索が展開する。
- ステータスセレクト 2 種が双方向同期する。
- 行クリックで回答詳細モーダルが開き、編集モードで差分が検出・保存される（現状はメモリ内のみ）。
- 展開ボタンでインライン展開行が開閉し、画像回転・ズームがモーダル・インライン間で同期する。
- フリーアカウントでの詳細分析ボタンがプレミアム誘導モーダルを出し、プレミアムでは `graph-page.html` に遷移する。
- リロード後、ソート・フィルタ・ページ・選択中設問が復元される。

## 9. 現行モックとの差分（要整合）

現行モックは以下の点で正式要件から未達のため、本番実装時に解消する。

1. **フリーアカウント判定が `surveyId` 8 件ハードコード（最重要）** — プラン連動（`plan-capabilities.json`）に差し替える。現行は契約プランよりも `surveyId` 一致が優先される。
2. **回答編集保存がメモリ上書きのみ** — PUT/POST 経路・楽観ロック・競合検知いずれも未定義。リロードで変更が消失する。
3. **エクスポート機能が本画面に存在しない** — CSV / 画像 / 結合データは `index.html`（一覧）/ `graph-page.html`（分析）側に委譲。プラン別出力マトリクスはプラン機能制限仕様書側で管理する。
4. **フリーワード検索未実装** — フィルタは日付 + ステータスの 2 軸のみ。氏名・会社名・回答テキストの横断検索はなし。
5. **「未回答を末尾へ」ソート未実装** — 動的カラムの未回答行は `localeCompare` の既定順（空文字列は先頭寄り）。
6. **名刺画像拡大モーダル配線未完** — `showCardImagesModal` 関数は実装済みだが、現行 UI からの呼出経路が未配線。
7. **インライン展開行の XSS エスケープ不足** — 会社名・氏名・Email・電話番号等が `escapeHtml` を経由せずテンプレート文字列に埋め込まれている。静的 JSON のうちは無害だが実サーバ化時に必須対応。
8. **モーダルの `<dialog>` 化・アクセシビリティ補強未実施** — `role="button"` / `aria-labelledby` / フォーカストラップ / Esc 閉じ / 編集モード時の初期フォーカス移動が未対応。
9. **多言語対応未実装** — アンケート名は `.ja` 固定で参照、UI 文字列はすべて日本語ハードコード。
10. **名刺データの 2 段フォールバック** — ディレクトリ移行の残骸（`data/responses/business-cards/` → `data/business-cards/`）が残存。配置統一で 1 段化可能。
