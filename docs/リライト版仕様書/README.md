# リライト版仕様書の読み方

本フォルダの文書は、完成済み画面を本番環境へ移すための開発会社提出用要件定義です。

画面の見た目、導線、入力項目、状態ごとの表示、エラー時の扱いを中心に整理します。実装方式の細部、インフラ、社内運用の詳細、法務原文の確認は別資料で扱います。

## 提出対象

- 利用者向け画面の表示・操作・遷移
- 入力チェック、保存、送信、完了時の動き
- ログイン状態、契約状態、権限による表示制御
- 本番移行時に開発会社と確認する API や保存先の考え方
- 受け入れ時に確認する主な条件

## 今回の範囲外

- サーバー内部の詳細設計
- DB 物理設計
- インフラ構成の詳細
- 社内だけで扱う運用手順
- 法務文面そのもののレビュー
- 旧画面、サンプル画面、検証用画面

## 書き方の方針

- 「現在の画面」は完成済み画面の状態を指す。
- 画面で確認できる内容は本文要件として扱う。
- API、DB、権限、監査ログ、保管期間など画面から確定できない内容は「画面外の確認事項」として扱う。
- 「画面外の確認事項」は確定仕様ではなく、開発会社とのすり合わせ対象とする。
- 根拠がない将来案は本文要件にしない。
- 未決事項は本文に混ぜず、必要な場合だけ「画面外の確認事項」として分ける。
- 画面側の仕様をもとに、開発時に迷いやすい点だけを短く補足する。
