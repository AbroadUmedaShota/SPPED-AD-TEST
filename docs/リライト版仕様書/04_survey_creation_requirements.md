## 1. 文書目的

- 本書は、アンケート作成・編集画面の要件を定義する。
- 本書を正本とする。
- 設問グループ機能・サンクス画面別ページは廃止済みのため本書では扱わない。

## 2. 画面概要

- イベント・展示会等で回収するアンケートを、非エンジニアの運用担当者が GUI のみで作成・編集し、回答者向け URL/QR コードを発行するための画面。
- 画面上部に基本情報カード、下段は 3 エリア構成（設定 / 設問 / アウトライン）のグリッドで構成する。
- 新規作成、既存複製、多言語展開といった複数の利用シーンに対応する。
- 実装本体は `02_dashboard/src/surveyCreation-v2.js`（v2 系）。`surveyCreation.js` は同ファイルへの 1 行再エクスポート。

## 3. 画面要件

### 3.1 上部情報
- パンくずと画面タイトル（アンケート作成・編集）を表示する。
- 多言語 ON 時は言語タブを sticky 表示し、画面全体の編集対象言語を切り替える。

### 3.2 基本情報カード
- アンケート名（管理用、ja 固定、必須、最大 100 字）。
- 表示タイトル（多言語、必須、最大 100 字）。
- 説明（多言語、任意、最大 500 字）。
- アンケート ID（保存後に自動採番、読み取り専用）。
- アンケート期間（期間ピッカー、必須）。
- 主要フィールドには補助説明導線（ヘルプポップオーバー）を設ける。

### 3.3 設定カラム（左）
- **オプション機能**: アコーディオン式。
  - 名刺画像添付トグル（初期 ON）。OFF 時は「名刺データ化設定」「お礼メール設定」リンクをグレーアウトする。
  - 多言語機能トグル（初期 OFF）。ON で言語選択パネルと言語タブを有効化する。
- **関連設定（外部ページ遷移）**: 名刺データ化設定 / お礼メール設定 / サンクス画面設定。
- `sticky` で独立スクロールし、設問カラムとは独立して表示する。

### 3.4 設問カラム（中）
- 設問カードのフラットリスト。設問グループ機能はなし。
- 各カードは折り畳み / 展開の 2 状態を持つ。
  - 折り畳み: 番号、サマリ（第一言語テキスト）、タイプバッジ、必須バッジ、複製 / 削除。
  - 展開: 設問文（言語数分）、回答設定（タイプ別 UI）、高度設定（条件分岐、UI のみ）。
- カード間の「ここに追加」セパレータから任意の位置に設問を挿入できる。
- 末尾にはインライン追加ボタン、画面右下には FAB を配置し、どちらからでも全 11 タイプを選択できる。
- Q 番号は並び替え・追加・削除のたびに自動採番する。

### 3.5 アウトラインパネル（右）
- 基本情報 / オプション機能 / 関連設定の静的アンカーと、設問一覧の動的リストを表示する。
- 項目クリックで該当箇所にスムーズスクロールする。
- ドラッグで設問の並び替えが可能で、設問カラムと同期する。
- バリデーションエラー（必須未入力、選択肢不足等）や未翻訳箇所を赤ドット・バッジで可視化する。
- XL 以上はグリッド内 sticky、LG〜XL 未満は右端ドロワー、MD 以下は非表示（モバイルバーに置換）。

### 3.6 アクションボタン
- プレビュー表示（常時有効）、QR コード（保存後に有効化）、アンケート保存（バリデーション通過後に有効化）、キャンセル。
- デスクトップはアウトラインパネル内に配置、モバイルは画面下部の固定アクションバーに配置する（キャンセルはデスクトップのみ）。

### 3.7 モーダル
- 通知モーダル（情報・警告・エラー）。
- 確認モーダル（キャンセル / OK）。
- QR コードモーダル（外部 HTML 読込）。
- プレビューモーダル（iframe でプレビュー画面を表示、スマホ / タブレット枠切替）。

## 4. 状態と挙動

### 4.1 初期表示
- URL パラメータ（`surveyId` / `surveyName` / `displayTitle` / `periodStart` / `periodEnd`）を受理し、対応フィールドに復元する。
- `surveyId` 指定時は既存アンケート JSON を取得し、基本情報・設問詳細を復元する。
- 取得失敗時もエラーで止めず、空状態で編集可能な状態にする。

### 4.2 言語選択と第一言語
- 多言語 OFF 時は `ja` 固定で言語タブを非表示にする。
- 多言語 ON 時は言語選択パネル（ja / en / zh-CN / zh-TW / vi の 5 言語、§5 非該当のため §6 多言語参照）で使用言語を選ぶ。
- 選択中言語は **最大 3 言語、最小 1 言語**。第一言語は選択順の先頭で、回答画面のデフォルト表示言語となる。
- 言語タブはドラッグで並び替え可能。先頭のタブを入れ替えると第一言語が切り替わる。

### 4.3 設問タイプ変更
- CHOICE 系相互（`single_answer` ↔ `multi_answer` ↔ `dropdown`）は選択肢を引き継ぎ、確認なしで変更する。
- MATRIX 系相互（`matrix_sa` ↔ `matrix_ma`）は行・列を引き継ぎ、確認なしで変更する。
- 上記以外で選択肢 / 行列データを保持している場合は、ポップオーバー内に警告を表示して確認を取る。
- 変更確定時は `text` / `required` / `id` を保持し、タイプ依存のフィールド（選択肢・行列・config）を新タイプの既定値で置き換える。

### 4.4 複製・削除・並び替え
- 複製: 各言語の設問文に「 (コピー)」を付与し、直後に挿入する。
- 削除: 確認モーダル後に削除する。
- 並び替え: 設問カラム、言語タブ、アウトライン、各設問内の選択肢の 4 系統でドラッグ並び替え可能。

### 4.5 プレビュー
- 入力状態を正規化してブラウザ内に保持し、プレビューモーダル内の iframe で回答画面を表示する。
- モーダル内でスマホ枠 / タブレット枠の切替が可能。

### 4.6 保存
- 保存前に必須項目と設問構造のバリデーションを実行する。
- 二重送信を防止する。
- 成功時は保存完了を通知し、QR コード発行を有効化する。
- 失敗時はエラーメッセージを表示し、入力値は保持する。

### 4.7 離脱
- キャンセル操作・ブラウザ離脱時の未保存警告は、保存 API 接続と合わせて整備する（現状モックは離脱確認なし）。

## 5. 設問タイプ仕様（11 種）

全 11 タイプを FAB・インライン追加・セパレータ・モバイルメニューのいずれからでも追加できる。

| タイプキー | ラベル | 概要 | 主要フィールド |
|-----------|--------|------|---------------|
| `free_answer` | フリーアンサー | 自由記述。文字数上限を将来設定可 | `config.minLength` / `maxLength` |
| `single_answer` | シングルアンサー | 単一選択 | `options[]`（言語別テキスト） |
| `multi_answer` | マルチアンサー | 複数選択 | `options[]` |
| `dropdown` | ドロップダウン回答 | 単一選択（プルダウン） | `options[]` |
| `rating_scale` | 評定尺度 | N 段階評価 | `config.points`（3/4/5 固定または 6〜10）、最小 / 最大 / 中間ラベル |
| `number_answer` | 数値回答 | 数値入力 | `config.min` / `max` / `unit` / `step` |
| `matrix_sa` | マトリックス(SA) | 行×列の単一選択 | `matrixRows[]` / `matrixCols[]` |
| `matrix_ma` | マトリックス(MA) | 行×列の複数選択 | `matrixRows[]` / `matrixCols[]` |
| `date_time` | 日付/時間 | 日付・時刻入力 | `config.showDate` / `showTime` |
| `handwriting` | 手書きスペース | 手書き入力キャンバス | `config.height`（既定 200px） |
| `explanation_card` | 説明カード | 案内文のみ（非回答） | `config.description`（多言語） |

- CHOICE 系（`single_answer` / `multi_answer` / `dropdown`）は選択肢を共通 UI で編集する。最後の 1 件は削除不可。
- MATRIX 系（`matrix_sa` / `matrix_ma`）は行・列を共通 UI で編集する。最後の 1 件は削除不可。
- `rating_scale` のポイント数は 3・4・5 の固定チップとカスタム入力（6〜10、範囲外は自動クランプ）で指定する。中間ラベルは表示トグルで制御する。
- `number_answer` / `date_time` / `handwriting` / `explanation_card` の設定 UI はデータモデル側は整備済みだが、入力 UI の一部は未整備（§10）。

## 6. 多言語対応

- 対応言語: **ja / en / zh-CN / zh-TW / vi の 5 言語**。
- 同時選択は **最大 3 言語、最小 1 言語**。第一言語は選択順の先頭で、回答画面のデフォルト表示言語。
- 多言語フィールド（表示タイトル、説明、設問文、選択肢、行列ラベル、評定尺度ラベル、説明カード文）は言語コードをキーとする object として保持する。
- アンケート名（管理用）のみ ja 固定。
- 言語タブはドラッグで並び替え可能で、先頭入替により第一言語が変化する。
- 追加言語タブや設問カードには未入力件数のバッジを表示し、翻訳抜けを可視化する。
- 多言語 OFF に戻すと `ja` 固定にリセットする。

## 7. バリデーション

### 7.1 項目別
- アンケート名 / 表示タイトル / アンケート期間は必須。未入力でブロッカー。
- 設問数は 1 件以上必要（未満はブロッカー）。
- CHOICE 系の選択肢は 2 件以上必要（未満はブロッカー）。
- 第一言語の設問文未入力は警告（アウトラインに赤ドット）。
- 追加言語の翻訳未入力は警告（タブ・カードにバッジ表示）。
- 基本情報 textarea は 500 字、input は 100 字を上限とする。
- 評定尺度ポイント数（カスタム）は 6〜10 に自動クランプ。
- 選択肢・マトリックス行 / 列の最後の 1 件は削除不可。
- 言語数は 1〜3 で制限する。

### 7.2 エラー表示タイミング
- フィールドは一度入力があった後（touched）にエラー表示する。
- 保存ボタン押下時は必須フィールドを強制的に touched 化し、最初のエラー要素へスクロールする。
- 設問ゼロで保存時は警告モーダルを表示し、設問追加ボタンへスクロールする。

## 8. 非機能要件

- 設問追加・並び替え・タイプ変更・言語切替などの主要操作は体感的に軽快に動作すること。
- XSS 対策として、ユーザー入力は DOM ファクトリ経由（テキストノード / value 代入）で処理し、`innerHTML` には流さない。
- モーダルは `aria-modal` / フォーカストラップ / Escape 閉じ / 前フォーカス復帰を備える。
- 設問タイプバッジ・必須バッジ・評定尺度の中間ラベルトグルはキーボード操作（Enter / Space）に対応する。
- SP / Tablet / Desktop の各ブレークポイントで表示崩れなく利用できること。
- 対応ブラウザは Chrome / Firefox / Safari / Edge の最新 2 バージョンを目安とする。

## 9. 手動確認観点

- 空状態で保存すると「設問が必要」の警告が表示され、設問追加ボタンへスクロールする。
- 必須項目未入力のまま保存すると各フィールドにエラー表示され、最初のエラーへスクロールする。
- 多言語 ON → 言語追加 → タブ順入替で第一言語が変化する。
- CHOICE ↔ CHOICE / MATRIX ↔ MATRIX は確認なしでデータが引き継がれる。
- それ以外のタイプ変更は警告ポップオーバーで確認が出る。
- 設問を並び替えると Q 番号が即時に振り直される。
- URL パラメータを付けて開くと各フィールドが復元され、保存ボタンが有効化する。
- プレビューモーダルで iframe と端末枠切替が機能する。

## 10. 現行モックとの差分（要整合）

現行モックは以下の点で正式要件から未達のため、本番実装時に解消する:

- **保存 API 未接続** — 保存ボタンは成功トーストを表示するのみで、永続化・ナビゲーションは未実装。
- **プラン連動未適用** — `plan-capabilities.json` は v2 から未参照。多言語 ON / 最大言語数 / 最大設問数 / 条件分岐はプランに関係なく一律動作する。
- **条件分岐ロジック未実装** — 高度設定セクションは固定文言のプレースホルダーのみ。
- **既存アンケート読込パス不整合** — 詳細データは `data/surveys/enquete/{surveyId}.json` 側にあるが、現状は一覧メタ（`data/surveys/{surveyId}.json`）のみ取得するため設問詳細が復元されない。
- **設問グループ廃止済み** — 旧仕様書のグループ構造・旧レンダラ（`ui/surveyRenderer.js` 等）はデッドコードとして残存。本書の対象外。
- **数値 / 日付時間 / 手書き / 説明カードの回答設定 UI 一部未実装** — データモデルは保持するが、作成画面での入力 UI が未整備（プレビュー時には反映）。
- **`free_answer` の minLength / maxLength 入力欄未実装** — データモデルとプレビュー転写のみ。
- **下書き自動保存未接続** — `services/surveyService.js` にスキーマ定義はあるが未接続。
- **新規作成時の期間ピッカー minDate 未指定** — 過去日付が選択可能になっている。

## 11. 本番化要件

本書はモックアップ作成段階の要件として作成したが、最終的に本番環境構築を担う開発会社が実装に着手するために必要な情報を本セクションに集約する。

### 11.1 API 契約

想定エンドポイントと入出力。詳細スキーマは実装着手時に確定する。

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/surveys/{id}` | 既存アンケートの編集用ロード | — | 200 `Survey`（基本情報＋設問詳細）／ 404 未存在 |
| `POST /api/surveys` | 新規アンケート作成 | `{ surveyName, displayTitle, period, languages, questions, options }` | 201 `{ survey: Survey }`（`surveyId` 採番含む）／ 400 バリデーションエラー |
| `PUT /api/surveys/{id}` | 既存アンケートの保存 | 同上 | 200 `{ survey: Survey }` ／ 409 競合（他者編集） |
| `POST /api/surveys/{id}/draft` | 下書き自動保存（将来） | 同上の部分ペイロード | 200 `{ savedAt }` |
| `POST /api/surveys/{id}/publish` | 公開（会期前 → 会期中の明示遷移が必要な場合） | — | 200 `{ status }` |

- 全エンドポイントはログイン認証済みセッション必須。CSRF トークンを付与する。
- 保存は二重送信防止のため `Idempotency-Key` ヘッダーを付与することを推奨（要確定）。
- 多言語フィールドは言語コードをキーとする object で送受信する。
- 既存アンケートロードは §10 の「読込パス不整合」を解消し、設問詳細まで単一 API で取得する。

### 11.2 権限・認可

| 利用者種別 | 取扱 |
| --- | --- |
| グループ管理者 | 所属グループ内アンケートの新規作成・編集・保存が可能 |
| 一般メンバー（編集権限あり） | 所属グループ内アンケートの編集・保存が可能（新規作成可否は運用方針で確定、要確定） |
| 一般メンバー（閲覧のみ） | 本画面へアクセス不可 |
| 未ログイン | アクセス不可。ログイン画面へリダイレクト |
| プラン制限 | 多言語機能・最大言語数・最大設問数・条件分岐は `plan-capabilities.json` に従う（§10 参照） |

### 11.3 業務エラー時の挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| 保存バリデーションエラー（400） | 該当フィールド直下にメッセージ、最初のエラーへスクロール（§7.2） | 入力値を保持 |
| 既存ロード失敗（404 / 5xx） | 空状態で編集可能な状態を維持し、トーストで通知 | §4.1 に準拠、ブロックしない |
| 競合（409、他者が同一アンケート編集） | 「他のユーザーによって更新されました。再読込してください。」 | 再読込後に入力をマージする方針は要検討 |
| プラン超過（403） | 「現在のプランでは利用できない機能です。」 | 該当機能 UI を非活性化 |
| 通信失敗（5xx） | 「保存に失敗しました。時間をおいて再度お試しください。」 | 自動リトライなし、入力値保持 |
| ネットワーク切断 | 「ネットワークに接続できません。」 | 自動リトライなし、入力値保持 |

### 11.4 データ永続化

- アンケートテーブル: 基本情報（`survey_id` / `name` / `display_title`（多言語）／ `description`（多言語）／ `period_start` / `period_end` / `status` / `plan` / `group_id` / `created_by` / `updated_at`）。
- 設問テーブル（アンケート 1 : N 設問）: `question_id` / `survey_id` / `order` / `type`（§5 の 11 種）／ `required` / `text`（多言語）／ `config`（JSON）。
- 選択肢・行列テーブル（設問 1 : N）: `option_id` / `question_id` / `order` / `label`（多言語）。または設問の `config` 内に埋め込む形（要確定）。
- 多言語値は言語コードをキーとする JSON として格納する前提。
- 下書き保存用領域は別カラムまたは別テーブルで保持（`draft_payload`）。保存確定時に本データへ反映。

### 11.5 ログ・監査

- 操作ログ: 新規作成・保存・削除・プレビュー発行・QR コード発行。実行ユーザー・アンケート ID・グループ ID を記録。
- 監査ログ: 公開（会期前 → 会期中）遷移、条件分岐等のプラン制限関連操作。
- 失敗ログ: 保存失敗・既存ロード失敗の発生時刻・原因を記録。

### 11.6 想定データ量

- 1 アンケートあたりの設問数は数十件規模を想定。100 件超は UI 性能面で要検討（§8 非機能要件）。
- 多言語値を持つため、JSON ペイロードサイズが数百 KB に達するケースを想定。保存 API はサイズ上限を設ける（要確定）。
- 同時編集は基本的に 1 アンケートあたり 1 名を想定。競合は §11.3 で扱う。

### 11.7 対応ブラウザ・デバイス

- 想定対応: 最新版 Chrome / Edge / Safari / Firefox（直近 2 バージョン、§8 参照）。
- レスポンシブ対応: PC（横幅 ≥ 1024px）／タブレット／スマートフォン（横幅 ≥ 375px）。
- IE および非 Evergreen ブラウザは対象外。

### 11.8 受け入れ基準

- [ ] 既存アンケートロード時に設問詳細まで単一 API で復元される（§10 の読込パス不整合解消）
- [ ] 保存ボタン押下でサーバーへ送信され、成功時はナビゲーション（一覧 or 詳細）へ遷移する
- [ ] 保存中はボタンが無効化され、二重送信が発生しない（§4.6）
- [ ] バリデーションエラー時に最初のエラーへスクロールする（§7.2）
- [ ] 離脱時（キャンセル・ブラウザ戻る・リロード）に未保存確認が表示される（§4.7）
- [ ] 多言語フィールドが言語コードキーの object として正しく保存・復元される
- [ ] プラン制限により、多言語・最大言語数・最大設問数・条件分岐がプランに応じて制御される（§10 解消）
- [ ] `surveyId` がサーバー採番され、保存後に読み取り専用で表示される
