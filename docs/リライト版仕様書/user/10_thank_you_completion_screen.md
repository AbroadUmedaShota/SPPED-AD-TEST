## 1. 文書目的

- 本書は、アンケート回答完了画面（サンクス画面）の要件を定義する。
- 本書を正本とする。
- 旧「本書が正」方針は破棄済み。

## 2. 画面概要

- アンケート回答者が送信ボタンを押し切った直後に遷移するランディング画面。
- 送信完了の明示的フィードバックを返し、連続回答が許可されている場合は同じアンケートへ戻る導線を提示する。
- 管理画面の「サンクス画面設定」（08 番）で作成者が設定した本文を、回答者の表示ロケールに合わせて描画する。
- 単一カードのみのシンプルな構成で、ダッシュボード共通のヘッダー・サイドバー・フッターは持たない。

## 3. 想定利用者・利用前提

- **アンケート回答者（匿名）**: ログイン不要で到達。本画面のみを見る。
- 広告運用担当・管理者は本画面を直接操作しない（プレビューはアンケート回答画面側でインライン描画）。
- 個人情報・認証情報は扱わない。ブラウザストレージ（localStorage / sessionStorage / Cookie）への書込みも行わない。

## 4. 画面要件

### 4.1 レイアウト
- 画面中央に単一のカードを配置する。
- カード内は上から順に「SpeedAd ロゴ」「見出し」「本文」「連続回答ボタン（条件付き）」を縦並びで表示する。
- 外枠はビューポート全高を確保し、全デバイスで中央配置を維持する。

### 4.2 ロゴ・見出し・本文
- ロゴは SpeedAd の共通ロゴを中央表示する。
- 見出しは i18n 固定文言（例: 「ご回答ありがとうございました」）を表示する。作成者が設定した文字列は見出しには反映されない。
- 本文は作成者が 08 番画面で設定したサンクスメッセージを表示する。未設定時は i18n デフォルト文言にフォールバックする。

### 4.3 連続回答ボタン
- 条件を満たすときのみ「続けて回答する」ボタンを表示し、押下時は同一アンケートの回答画面へ戻る。
- 非表示条件下ではボタン自体を描画しない（スペースも残さない）。

### 4.4 レスポンシブ
- 全ビューポートで中央配置の単一カードを維持する。
- カード最大幅は 448px 相当。それ未満のビューポートでは横余白を段階的に詰める。
- モバイル専用レイアウトや PC/モバイルの分岐は持たない。

## 5. 表示コンテンツ・フォールバック

### 5.1 本文の優先順
- 本文は以下の優先順で決定する。
  1. `thankYouScreenSettings.thankYouMessage`（08 番で設定）
  2. `thankYouScreenSettings.description`（補助的な説明文）
  3. i18n デフォルト文言（`thankYouScreen.body`）
- いずれの候補もロケール別に解決される（§7 参照）。

### 5.2 i18n 固定部分
- 見出し・ドキュメントタイトル・連続回答ボタンのラベルは i18n 固定文言とし、作成者設定は反映しない。
- ロゴ・背景色・レイアウトも固定とする。

### 5.3 文字エスケープ
- 本文はプレーンテキストとして描画する。HTML タグが含まれていてもタグとしては解釈せず、文字としてそのまま表示する。

## 6. 連続回答モード

### 6.1 表示条件
- 以下すべてが成立したときに限り、連続回答ボタンを可視化する。
  - 連続回答モード指定あり（URL パラメータ `continuous=true`）
  - アンケート ID が特定可能
  - 画面の連続回答ボタン DOM が存在する
- 上記以外のケースではボタンを表示しない。

### 6.2 遷移仕様
- ボタン押下時は同一アンケートの回答画面へ通常のリンク遷移する。
- 遷移 URL にはアンケート ID と表示ロケールを引き継ぐ（URL エンコード済み）。
- 自動リダイレクトは行わない。遷移は必ずユーザー操作を伴う。

### 6.3 責務分界
- プラン判定（連続回答を許可するプランか否か）は本画面では行わない。呼び出し元の回答画面が `continuous=true` を付けるか否かを決定する。
- 本画面は URL に付与された値を機械的に解釈するのみとする。

## 7. 多言語対応

### 7.1 対応言語
- 現行対応は日本語 / 英語 / 中国語（簡体字） / 中国語（繁体字）の 4 言語。
- ベトナム語は画面外の確認事項（現在は辞書未整備のため日本語にフォールバックする）。

### 7.2 ロケール決定優先順
- 表示ロケールは以下の優先順で決定する。
  1. URL パラメータ `answerLocale`
  2. アンケート定義の `defaultAnswerLocale`
  3. アンケート定義の `editorLanguage`
  4. 日本語（最終フォールバック）

### 7.3 切替 UI
- 画面内に言語切替 UI は設けない。ロケールは到達時点で固定する。
- 回答者が言語を変えたい場合は、呼び出し元の回答画面側で切り替えて再送信する運用とする。

## 8. データモデル

### 8.1 入力

| 種別 | 項目 | 用途 |
| --- | --- | --- |
| URL パラメータ | `surveyId` | アンケート定義の特定。未指定時は本文デフォルト・連続回答ボタン非表示。 |
| URL パラメータ | `continuous` | 連続回答ボタンの表示条件。`true` のみ有効。 |
| URL パラメータ | `answerLocale` | 表示ロケール最優先指定。 |
| アンケート定義 | `thankYouScreenSettings.thankYouMessage` | 本文第 1 候補。多言語オブジェクト。 |
| アンケート定義 | `thankYouScreenSettings.description` | 本文第 2 候補。多言語オブジェクト。 |
| アンケート定義 | `defaultAnswerLocale` / `editorLanguage` | ロケール決定の後段フォールバック。 |

### 8.2 多言語オブジェクト形式
- `thankYouMessage` および `description` はロケールキー付きオブジェクトとして保持する（例: `{ "ja": "...", "en": "...", "zh-CN": "...", "zh-TW": "..." }`）。
- 表示時はロケール完全一致 → エイリアス一致 → フォールバックロケール → 日本語の順で文字列を解決する。

### 8.3 08 番仕様書とのデータモデル整合（最重要）
- 08 番仕様書（サンクス画面設定）は `Survey.thankYouMessage` を **フラット単言語の文字列** として定義している。
- 本画面（20 番）は `thankYouScreenSettings.thankYouMessage` を **ネスト構造 + 多言語オブジェクト** として読み取る。
- **整合方針**: 本書（20 番）の仕様を正とし、08 番側を後続で追従整合化する。具体的には以下を 08 番側で更新する。
  - `thankYouMessage` の格納位置を `thankYouScreenSettings` ネスト配下に変更。
  - 型を多言語オブジェクトに拡張。
  - `description` フィールドを追加（本文第 2 候補として）。
  - 管理画面の入力 UI に多言語対応入力欄を追加する。

## 9. エラー処理

- アンケート定義 JSON の取得に失敗した場合、画面はサイレントにフォールバックする。
  - 本文は i18n デフォルト文言を表示する。
  - 連続回答ボタンは強制的に非表示にする。
  - ユーザー向けエラー UI は表示しない（画面外の確認事項として対応）。
- JSON 未取得でも画面はブロックせず、読み込み完了状態で静止する。

## 10. 非機能要件

### 10.1 パフォーマンス
- 外部通信はアンケート定義 JSON の取得 1 回のみ。
- 本画面固有のパフォーマンス目標値は設定しない。

### 10.2 セキュリティ
- 個人情報・認証情報・トラッキング情報を扱わない。
- URL パラメータはサーバー送信・永続化を行わない。
- 本文はプレーンテキスト描画のためスクリプトインジェクション耐性を持つ。

### 10.3 アクセシビリティ
- 現時点はベストエフォート対応。
- 言語属性（`<html lang>`）を確定ロケールに同期する。
- ロゴ画像に代替テキストを付与する。
- セマンティック要素で構造化する。
- WCAG 2.1 Level AA 準拠検証は 画面外の確認事項として対応。

### 10.4 ブラウザ対応
- PC Chrome 最新 / iOS Safari 最新 / Android Chrome 最新を想定する。

## 11. Definition of Done

- 本文が「作成者設定（多言語対応）→ 補助説明文 → i18n デフォルト」の優先順で表示されること。
- 見出し・ボタンラベルは常に i18n 固定文言が表示されること。
- 連続回答ボタンが所定の条件下でのみ表示されること。
- ロケールが 4 段階優先順で正しく決定され、4 言語（ja / en / zh-CN / zh-TW）で文言が切り替わること。
- JSON 取得失敗時にサイレントフォールバック（デフォルト文言表示・連続回答ボタン非表示）が動作すること。
- 主要デバイスで表示崩れがないこと。

## 13. 本番化要件

本セクションでは、完成済み画面を本番環境へ移す際に、画面からは直接確定できないため開発会社と確認する事項をまとめる。API・DB・ログ・権限などの詳細は、画面上の確定仕様ではなく実装時の確認観点である。

### 13.1 API 契約

以下は確認用のたたき台であり、画面上の確定仕様ではない。詳細スキーマ・パスは開発会社と合意して確定する。

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/public/surveys/{surveyId}/thank-you` | 回答完了画面表示用の設定取得 | URL パス `surveyId`、クエリ `answerLocale` | 200 `{ thankYouScreenSettings: { thankYouMessage: {ja,en,zh-CN,zh-TW}, description: {...} }, defaultAnswerLocale, editorLanguage }` ／ 404 `surveyId` 未知 |

- 公開画面用途のため未認証で呼出可。CDN キャッシュ可（公開情報のみ）。
- レスポンスは PII を一切含まない（メッセージ本文と表示用ロケール情報のみ）。
- 具体パス・キャッシュ TTL は実装時確定（想定）。

### 13.2 権限・認可

本画面は **ログイン不要の完全公開画面**。アンケート回答者（匿名）が `surveyId` を知っていれば到達可能であり、認証・認可チェックは行わない。

| 利用者種別 | 取扱 |
| --- | --- |
| アンケート回答者（匿名） | 本画面の主対象。URL 直アクセスで到達可 |
| 管理者・運用担当 | 本画面は直接操作しない。プレビューは呼び出し元（13 番アンケート回答画面）のインライン描画で行う |
| 非対象サーベイ ID での到達 | `surveyId` 不明でもフォールバック（i18n デフォルト文言表示・連続回答ボタン非表示）で静止する（§9） |

### 13.3 業務エラー時の挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| アンケート定義 JSON 取得失敗（5xx / ネットワーク） | i18n デフォルト文言にフォールバック | ユーザー向けエラー UI は表示しない（§9、画面外の確認事項で検討） |
| `surveyId` 未指定 | i18n デフォルト文言表示 + 連続回答ボタン非表示 | エラー画面は出さない |
| 未知 `surveyId`（404） | 同上（サイレントフォールバック） | 連続回答ボタン強制非表示 |
| 本文が全ロケール空 | i18n デフォルト文言（`thankYouScreen.body`）を表示 | §5.1 優先順に従う |

### 13.4 データ永続化

- 本画面自身は **何も永続化しない**（ブラウザストレージ書込み禁止は §3 で規定済み）。
- 表示源泉は `Survey.thankYouScreenSettings` を正とする。08 番（サンクス画面設定）のフラット単言語構造から、本書のネスト＋多言語オブジェクト構造へスキーマ拡張が必要（§8.3 参照）。
- スキーマ拡張の主項目: `thankYouScreenSettings.thankYouMessage`（多言語オブジェクト）／`thankYouScreenSettings.description`（多言語オブジェクト）。既存 `Survey.thankYouMessage` は非推奨化し、読み込みは新旧両対応の移行期間を置く。

### 13.5 ログ・監査

- 本画面は公開匿名画面のため、サーバ送信ログは **記録しない** ことを原則とする。
- 例外: アンケート定義取得 API のアクセスログ（`surveyId`・ロケール・HTTP ステータスのみ）をサーバ側で保持。PII・IP 生値はマスクする。
- クライアント側 `console` ログは `console.debug` のみ。保管期間は運用方針に従う（開発会社と確認）。

### 13.6 想定データ量

- 同時表示数: アンケート回答ピーク時（展示会会期中）に **数百回/時間** を想定。
- ペイロード: アンケート設定 JSON 1 回取得のみ（数 KB）。画像・動画は含まない。
- CDN 配信と HTTP キャッシュで十分にスケールする前提。

### 13.7 対応ブラウザ・デバイス

- 対応: 最新版 Chrome / Edge / Safari / Firefox（直近 1 年以内のメジャーバージョン）。
- レスポンシブ対応: PC（横幅 ≥ 1024px）／タブレット／スマートフォン（横幅 ≥ 375px）。カード最大幅 448px、モバイル専用レイアウトは持たない（§4.4）。
- IE および非 Evergreen ブラウザは対象外。

### 13.8 受け入れ基準

- [ ] 本文が「作成者設定（多言語） → 補助説明文 → i18n デフォルト」の優先順で表示される（§5.1）
- [ ] 4 ロケール（ja / en / zh-CN / zh-TW）で文言が切り替わる
- [ ] 連続回答ボタンが §6.1 全条件成立時のみ可視化され、それ以外は DOM にスペースも残さない
- [ ] 本文に HTML タグが含まれても文字列として表示される（プレーンテキスト描画・XSS 耐性）
- [ ] アンケート定義 JSON 取得失敗時、エラー UI を出さずデフォルト文言で静止し連続回答ボタンが非表示になる
- [ ] 08 番の管理画面から多言語入力した値が本画面に反映される（§8.3 の 08 番側追従完了後）
- [ ] 本画面がブラウザストレージ（localStorage / sessionStorage / Cookie）に一切書込みを行わない
