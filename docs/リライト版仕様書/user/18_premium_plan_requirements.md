# プレミアムプラン 要件定義書（統合版）

| 対象画面 | ファイル名 |
| --- | --- |
| プレミアム加入促進ページ | `02_dashboard/premium_signup_new.html` |
| プレミアム登録SPA | `02_dashboard/premium_registration_spa.html` |
| プレミアム登録完了 | `02_dashboard/premium_registration_complete.html` |
| 解約手続き | `02_dashboard/premium_cancel.html` |
| 解約完了 | `02_dashboard/premium_cancel_complete.html` |

## 1. 全体フロー

```
ダッシュボード / サイドバー
    ↓
加入促進ページ (premium_signup_new.html)
    ├─ 新規 / 再加入ユーザー → 登録SPA (premium_registration_spa.html)
    │       └─ 登録完了 → プレミアム登録完了 (premium_registration_complete.html)
    │                              └─ ダッシュボード (index.html)
    ├─ 既存会員 Active → 解約手続き (premium_cancel.html)
    │       └─ 解約完了 (premium_cancel_complete.html)
    │                    └─ ダッシュボード (index.html)
    ├─ 既存会員 解約予約中 → 自動更新再開モーダル → ダッシュボード
    └─ 既存会員 / トライアル → invoiceList.html（請求書確認）
```

## 2. 共通要件

### 2.1 共通コンポーネント

- 共通ヘッダー・サイドバー・フッターは `src/main.js` により各 placeholder へ注入される。
- 全画面はログイン済みユーザー専用。未ログイン時はログイン画面へリダイレクトする。

### 2.2 ブラウザ・デバイス対応

- 対応ブラウザ: 最新版 Chrome / Edge / Safari / Firefox（直近 1 年以内のメジャーバージョン）。IE および非 Evergreen ブラウザは対象外。
- レスポンシブ対応: PC（横幅 ≥ 1024px）/ タブレット / スマートフォン（横幅 ≥ 375px）。

## 3. プレミアム加入促進ページ（premium_signup_new.html）

### 3.1 概要

SPEED AD ダッシュボード内で、プレミアムプランの料金・特典・プラン比較・FAQ を案内し、加入申込フォームへ誘導するページ。ユーザー状態に応じて同一 URL でヒーローセクション・主要導線・キャンペーン表示が動的に切り替わる。

想定利用者: Standard ユーザー（新規加入候補）/ 過去解約済みの再加入候補 / 既存プレミアム会員 / 無料トライアル中 / 解約予約中ユーザー。

### 3.2 画面要件

#### ヒーローセクション

- パンくず「ホーム → プレミアムプラン」を表示する。
- ラベル「PREMIUM PLAN」を見出し上部に表示する。
- ヒーロー見出し（デフォルト）「事務作業とデータ入力時間を、ゼロへ。」を表示する（ユーザー状態により差替え、§3.3）。
- 主要価値を示す 3 枚のアイコンカードを表示する。
  - 「データ最長1年保存」: 契約期間中、最長 1 年間保管
  - 「プロフェッショナル仕様」: 高度な設問設計と外部連携
  - 「高速処理」: 即日利用開始可能

#### 料金セクション

- 見出し「プラン料金」を表示する。
- 月額 `¥10,000`（税別）、補足「1 日あたり約 330 円」を表示する。
- 「✨ 今なら初月無料！」バッジ（ID: `first-month-free-message`）を表示する。
- 付帯条件: 初期費用 0 円 / 請求書払い（月末締め翌月末払い）。
- 右側に CTA ボタン「今すぐ申し込む」を配置し、`premium_registration_spa.html` へ遷移する。
- トラストバッジ「データ最長1年保存」「いつでも解約可能」を CTA 直下に表示する。
- 料金セクション下部に「初月無料キャンペーン」詳細コンテナ（ID: `integrated-campaign-container`）を配置する。新規ユーザーのみ JS で動的に描画する（§3.4）。

#### 特別機能カード

- 見出し「プレミアムプランだけの特別機能」を表示する。
- 6 枚のカードで機能を訴求する（SP 2 列 / Desktop 3 列）。
  1. 最長1年データ保存: 名刺データを最長 1 年間保存・活用
  2. 高度な入力対応: 手書き文字や画像のデータ化
  3. Excel レポート: 集計データを自由に加工・分析
  4. SPEED レビュー: 入力データと画像を並べて確認・修正
  5. 自社ブランディング: 自社ドメイン送信、ロゴ非表示
  6. 外部システム連携: Slack や SFA/CRM との自動連携

#### プラン比較表

- 見出し「プラン比較」を表示する。Standard（無料） / Premium の 2 列で 9 項目を比較する。SP では横スクロールで閲覧する。

| # | 機能 | Standard (無料) | Premium |
| --- | --- | --- | --- |
| 1 | 追加設問・機能 | × | 設問分岐・画像アップロード・手書き |
| 2 | アンケート構成 | 1 ページ完結 | 改ページ・進捗バー表示 |
| 3 | 多言語対応 | × | 作成・回答表示 |
| 4 | お礼メール | 制限あり | 自動配信・独自ドメイン・条件分岐 |
| 5 | データ化スピード | 通常 | オンデマンド利用可（※別途費用） |
| 6 | 分析・レポート | SPEED レビュー（閲覧のみ） | SPEED レビュー + Excel/CSV 出力 |
| 7 | 特殊データ DL | × | 名刺・画像結合データ出力 |
| 8 | データ保存期間 | 30 日 | 最長 1 年 |
| 9 | 外部連携 | Slack | Slack, SFA/CRM |

#### CTA セクション（FAQ 直前）

- 見出し「今すぐプレミアムプランを始めませんか？」を表示する。
- 補足「業務効率が劇的に向上します。最短 1 分で登録完了。」を表示する。
- CTA ボタン「今すぐ申し込む」で `premium_registration_spa.html` へ遷移する。
- トラストバッジ「データ最長1年保存」「いつでも解約可能」「ビジネス機能完備」を併記する。

#### FAQ セクション

- 見出し「よくある質問」を表示する。アコーディオン形式（初期状態: 開）。
  - Q. プレミアムプランの主な機能は何ですか？
  - A. データ保存期間が最長 1 年になるほか、手書き設問、画像アップロード、多言語対応、Excel/CSV ダウンロード、外部連携（Slack/CRM）、自社ドメイン送信、ロゴ非表示などの高度な機能が月額料金内で全て利用可能。
- 追加の疑問はサポートセンター（`https://support.speed-ad.com/help/`）への導線を提示する。
- 最下部にお問い合わせ導線（`bug-report.html`）を配置する。

#### モーダル群

| モーダル ID | 役割 | 表示条件 |
| --- | --- | --- |
| `contactModal` | お問い合わせフォーム（氏名・メール・件名・本文） | `open-support-modal` リンク押下 |
| `resumeModal` | 自動更新再開の確認モーダル | 解約予約中ユーザーが「自動更新を再開する」カードを押下 |
| `resumeSuccessModal` | 自動更新再開の完了モーダル（次回更新日を表示） | `resumeModal` の「再開する」押下後 |

- `resumeSuccessModal` の「ダッシュボードに戻る」押下でシナリオを `premium-member`（解約予約解除）へ更新し、ページをリロードする。

### 3.3 ユーザー状態別表示

#### 状態判定フラグと優先順位

| 優先 | フラグ | 意味 |
| --- | --- | --- |
| 1 | `is_premium_member`（`is_cancelled` 併存可） | 既存プレミアム会員 |
| 2 | `is_free_trial` | 無料トライアル中 |
| 3 | `is_rejoining_user` | 再加入候補 |
| 4 | （デフォルト） | 新規ユーザー |

#### 状態別表示内容

| ユーザー状態 | ヒーロー見出し | メイン領域 | キャンペーン | CTA |
| --- | --- | --- | --- | --- |
| 新規ユーザー | 「事務作業とデータ入力時間を、ゼロへ。」 | 料金・機能比較・FAQ 標準表示 | 初月無料キャンペーン表示 | 「今すぐ申し込む」→ `premium_registration_spa.html` |
| 再加入ユーザー | 「おかえりなさい！ 再びご利用ありがとうございます！」 | 再加入課金注意書き（当月分発生、契約期間を動的表示） | 非表示 | 「再加入する」→ `premium_registration_spa.html` |
| 既存プレミアム会員（Active） | 氏名表示＋Status: Active、現プラン・次回更新予定日（当月末） | 請求書確認 / プラン解約 のクイックアクション 2 枚＋サポート導線 | 非表示 | 表示しない |
| 既存プレミアム会員（解約予約中） | 氏名表示＋Status: Auto-Renewal Off、利用期限＋注意書き | 請求書確認 / 自動更新を再開する の 2 枚＋サポート導線 | 非表示 | 自動更新再開モーダル |
| 無料トライアル | 氏名表示＋Status: Free Trial、トライアル期限・初回請求予定日 | 請求書確認（空状態）/ トライアル終了 の 2 枚＋サポート導線 | 非表示 | 表示しない |

#### 日付計算ルール

- 「次回更新予定日」「利用期限」「トライアル期限」: 閲覧日の当月末日を動的算出する。
- 「初回請求予定日」: 閲覧日の翌月 1 日を算出する。
- 再加入者向け「契約期間」: 本日〜当月末日。
- サーバ連携時は API レスポンスの契約日付を正とする（クライアント計算は非採用）。

#### 初月無料キャンペーン描画条件

`integrated-campaign-container` は `is_rejoining_user = false` かつ `is_premium_member = false` の場合のみ描画する。内容: 登録月（当月、¥0）と翌月（¥10,000）のタイムライン、「現在のご登録なら `{当月末日}` まで無料」のサマリ。

#### FAQ アコーディオン

`.faq-question` クリックで `aria-expanded` をトグルし、`.faq-answer` の開閉クラス（`open`）を切り替える。

#### ユーザー状態の取得

- 本番ではユーザー状態 API のレスポンスを表示判定の基準とする。
- 状態取得に失敗した場合は、加入・再開など契約に関わる操作を行わせず、再読み込み導線を表示する。

### 3.4 非機能要件

- **レスポンシブ**: SP（≥ 375px）/ Tablet / Desktop（≥ 1024px）で表示崩れなし。プラン比較表は横スクロール対応。特別機能カードは SP 2 列 / Desktop 3 列。
- **アクセシビリティ**: FAQ アコーディオンは `aria-expanded` を状態に応じて切替える。装飾アイコンは `aria-hidden="true"` を付与する。モーダルは `role="dialog"` / `aria-modal="true"` / `aria-labelledby` を備える。コントラスト比 4.5:1 以上。
- **セキュリティ**: ヒーロー差替え時の氏名は `textContent` 等でエスケープして描画する（`innerHTML` 直接注入不可）。

### 3.5 API 仕様

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/subscription/me` | 課金状態取得（`is_premium_member` / `is_free_trial` / `is_cancelled` / `is_rejoining_user` / `next_renewal_at` / `trial_end_at` 等） | セッション Cookie | 200 `SubscriptionStatus` ／ 401 未認証 |
| `POST /api/subscription/resume` | 解約予約のキャンセル（自動更新再開） | — | 200 `{ next_renewal_at }` ／ 409 状態不正 |
| `POST /api/help/contact` | お問い合わせ送信 | `{ name, email, subject, message }` | 200 `{ ok: true }` ／ 400 バリデーション |

全エンドポイントはログイン認証済みセッション必須。

### 3.6 権限・認可

| 利用者種別 | 取扱 |
| --- | --- |
| 未ログイン | ログイン画面へリダイレクト |
| Standard（未課金）ユーザー | 新規ユーザービュー |
| 過去解約済みユーザー | 再加入ユーザービュー |
| 無料トライアル中ユーザー | トライアルビュー |
| アクティブなプレミアム会員 | 既存会員ビュー（Active） |
| 解約予約中ユーザー | 既存会員ビュー（自動更新停止中） |

### 3.7 エラー挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| 課金状態取得失敗（5xx） | 「プラン情報の取得に失敗しました。時間をおいて再度お試しください。」 | 自動リトライなし。再読込導線を提示 |
| 自動更新再開失敗（409 / 5xx） | 「自動更新の再開に失敗しました。サポートへお問い合わせください。」 | モーダル内でエラー表示、成功モーダルへは遷移しない |
| ネットワーク切断 | 「ネットワークに接続できません。」 | CTA・再開ボタンは無効化しない（再押下で再試行可能） |
| セッション切れ（401） | ログイン画面へ自動遷移 | 現在 URL を戻り先として保持 |

### 3.8 受け入れ基準

- [ ] ユーザー状態が `GET /api/subscription/me` から取得され、確認用データに依存しない
- [ ] シナリオ切替ボタン（`scenario-switcher-*`）が本番ビルドから除去される
- [ ] 「次回更新予定日」「利用期限」「初回請求予定日」が API レスポンス値で表示される
- [ ] 自動更新再開がサーバ API 経由で完了し、再開完了後に実データで更新日が表示される
- [ ] ヒーロー差替え時の氏名はエスケープ処理で描画される
- [ ] モーダル（`resumeModal` / `resumeSuccessModal` / `contactModal`）に `role="dialog"` / `aria-modal="true"` / `aria-labelledby` が付与される
- [ ] デッドコード（`#sticky-cta`、`open-support-modal` のモーダル起動残骸）が整理される
- [ ] データ保存期間の表記が「最長 1 年」で統一される

## 4. プレミアム登録SPA（premium_registration_spa.html）

### 4.1 概要

無償プラン利用者がプレミアムプラン（月額 10,000 円税別）への申込を自己完結で行う 3 ステップ SPA。同一ページ内でステップを切替え、ページ遷移は発生させない。

- **Step 1: 入力** — 請求情報フォーム
- **Step 2: 確認** — 入力内容の表示と規約同意
- **Step 3: 完了** — 登録完了通知とダッシュボード戻り

パンくず「ダッシュボード → プレミアムプラン → 登録設定」。想定利用者: SPEED AD クライアント担当者（主に会社の経理・管理部門）。

### 4.2 画面要件

#### ステップ進捗インジケーター

- 「1. 入力 → 2. 確認 → 3. 完了」を最上部に表示する。
- 各ステップ円の状態:
  - **active**: 現在ステップ（番号表示、強調色）
  - **completed**: 完了済（チェックアイコン、強調色）。完了済円クリックで対応ステップへ戻れる（Step 2 → Step 1 のみ）
  - **未到達**: 番号のみ表示
- 連結線（connector）も active 時に色付けする。

#### Step 1（入力）

- セクションタイトル「請求情報の入力」を表示する。
- 入力項目（必須は「必須」バッジを表示）:

| 項目 | 必須 | 備考 |
| --- | --- | --- |
| 氏名（姓） | 必須 | 例: 山田 |
| 氏名（名） | 必須 | 例: 太郎 |
| 会社名 | 必須 | 例: 株式会社SpeedAd |
| 部署名 | 任意 | 例: 営業部 |
| 役職名 | 任意 | 例: 部長 |
| 電話番号 | 必須 | ハイフンなし、例: 0312345678 |
| 郵便番号 | 必須 | 7 桁、`maxlength="7"`、「住所検索」ボタン付き |
| 住所 | 必須 | 例: 東京都港区六本木1-2-3 |
| 建物名 | 任意 | 例: 六本木ヒルズ XX階 |

- フォーム右上に sticky「会員情報を引用」ボタンを表示する（md 以上、かつ会員情報が有効な場合のみ）。
- 画面下部に「確認画面へ進む」ボタンを配置する。

#### Step 2（確認）

- セクションタイトル「入力内容の確認」を表示する。
- 注記「以下の内容で登録します。よろしければ『この内容で登録する』ボタンを押してください。修正する場合は『修正する』ボタンで戻ってください。」を表示する。
- 入力内容を `dl` で読取表示する（氏名 / 会社名 / 部署名 / 役職名 / 電話番号 / 郵便番号（`〒XXX-XXXX`）/ 住所 / 建物名）。任意項目が未入力の場合は「-」を表示する。
- 料金説明カードを表示する。
  - 月額 10,000 円（税別）
  - 月末締めで計算し、翌月第 1 営業日に管理画面「ご請求内容一覧」からダウンロード可
  - 課金開始日情報をユーザー区分に応じて挿入する（§4.4 登録送信 参照）
- 規約同意チェックボックス（`consent-checkbox`）。ラベルリンク先: 利用規約（`terms-of-service.html`、新規タブ）/ プライバシーポリシー（外部 URL、新規タブ）。
- アクション: 「修正する」（Step 1 に戻る）／「この内容で登録する」（チェック ON で活性化）。

#### Step 3（完了）

- 「Welcome to Premium」を表示する。バッジアイコンとパーティクル演出を伴う装飾的な表現とする。
- 説明文「プレミアムプランへの登録が完了しました。あなただけの特別な体験が、今ここから始まります。」を表示する。
- 「ダッシュボードへ戻る」ボタン（`index.html` へのリンク）を配置する。

#### モーダル群

- **会員情報確認モーダル（`account-check-modal`）**: 画面初期化時、会員情報が有効な場合に表示。
  - 表示項目: 氏名／会社名／部署名／役職名／電話番号／郵便番号／住所／建物名／メール
  - アクション: 「はい、この情報を使って登録へ進む」（自動入力 → Step 2 へ）／「いいえ、情報を入力する」（Step 1 へ）
- **離脱防止モーダル（`prevent-leave-modal`）**: Step 1 / Step 2 で未保存変更がある状態で離脱・戻る操作を検知したとき表示。
  - 文言: 「入力中の情報があります。このページを離れると、入力中の情報が失われますがよろしいですか？」
  - アクション: 「はい、ページを離れる」（遷移許可）／「いいえ、このページに留まる」（留まる）
- **エラーモーダル（`error-modal`）**: 登録処理失敗時に表示。文言「登録処理中にエラーが発生しました。もう一度お試しください。」。「もう一度試す」で閉じて Step 2 に留まる。オーバーレイクリックでも閉じる。

### 4.3 状態と挙動

#### ステップ遷移

- 初期ロード時に会員情報を評価する。
  - プレミアム会員判定時は Step 3 へ直接遷移する（重複登録防止）。
  - 会員情報が有効な場合は会員情報確認モーダルを表示する。
  - それ以外は Step 1 を表示する。
- Step 1 → Step 2: 「確認画面へ進む」押下 + Step 1 全バリデーション通過が条件。
- Step 2 → Step 1: 「修正する」押下またはステップ 1 円クリック。入力値は保持するが、規約同意チェックと送信ボタン活性化状態はリセットする。
- Step 2 → Step 3: 「この内容で登録する」押下 + 送信成功が条件。
- ステップ切替時は画面上端へスムーズスクロールする。

#### バリデーション

- フィールドの `blur` 時（リアルタイム検証）および「確認画面へ進む」押下時（全項目一括検証 `validateForm`）に実行する。
- 検証通過: フィールドに緑のチェックアイコン（`check_circle`）を表示し、枠線を緑に変える。
- 違反: 該当項目直下にエラーメッセージを表示し、枠線を赤に変える。

#### 住所検索

- 「住所検索」ボタン押下で `zip` 値から数字のみを抽出し、自社プロキシ経由で外部 API に照会する。
- 7 桁未満の場合は API を呼ばず「郵便番号を7桁で入力してください。」を表示する。
- 検索中はボタンを「検索中...」に差替え、`disabled` にする（二重押下防止）。
- 成功時は `address1 + address2 + address3` を結合して住所欄に自動入力し、建物名欄へフォーカス移動する。
- 該当なし: 「該当する住所が見つかりませんでした。」を表示する。
- API 失敗・ネットワーク切断時: 「住所の取得に失敗しました。」を表示する。

#### 会員情報引用

- 会員情報は認証済みユーザーのプロフィール API から取得する。有効性の判定条件: 「姓」「名」「会社名」の 3 点が埋まっていること。
- 会員情報確認モーダルから「この情報を使って登録へ進む」を選んだ場合、フォームに自動入力 → 全項目検証 → 全通過なら Step 2 に遷移する。通過しないときは Step 1 に留まる。
- Sticky「会員情報を引用」ボタンは Step 1 表示中のみ表示する（Step 2 / Step 3 では非表示）。Sticky から押した場合は自動入力のみ行い、Step 1 に留まる（自動遷移しない）。

#### 離脱防止

作動条件: Step 1 または Step 2 に滞在中、かつ `hasUnsavedChanges` フラグが true。

| 契機 | 挙動 |
| --- | --- |
| `beforeunload`（ブラウザ閉じ・リロード・URL 直打ち） | ブラウザ標準ダイアログを表示 |
| ページ内 `<a>` タグクリック | 離脱防止モーダルを表示 |
| `popstate`（戻る/進む） | 離脱防止モーダルを表示 |

#### 登録送信

- 「この内容で登録する」押下で送信ボタンを `disabled` 化し、スピナー＋「処理中...」に差替え（二重送信防止）。
- 成功時は Step 3 に遷移する。失敗時はエラーモーダルを表示し、Step 2 に留まる。送信ボタンは再度有効化する。
- **課金開始日メッセージ**（Step 2 料金カード内に動的挿入）:

| ユーザー区分 | 契約開始日 | 初月課金 | 課金開始月 |
| --- | --- | --- | --- |
| 新規（`is_rejoining_user=false`） | 申込日 | 無料 | 翌月 1 日 |
| 再加入（`is_rejoining_user=true`） | 申込日 | 満額（当月分）| 申込月 |

### 4.5 非機能要件

- **レスポンシブ**: PC（≥ 1024px）/ タブレット / スマートフォン（≥ 375px）で崩れなく利用できること。Sticky「会員情報を引用」ボタンは md 以上でのみ表示する（モバイルでは非表示）。

- **セキュリティ**: 未ログイン時はログイン画面へリダイレクトする。既にプレミアム会員のユーザーは Step 3 へ直接遷移させ、重複登録を防ぐ。登録ペイロードは HTTPS 必須、CSRF トークンを付与する。動的 DOM 挿入は固定文言化し、ユーザー入力を混入させない。

## 5. 解約手続き（premium_cancel.html）

### 5.1 概要

プレミアム会員が自動更新の停止手続きを行う画面。解約後も契約期間終了日まではプレミアム機能を継続利用できることを案内する。解約理由アンケートは任意。解約前に確認モーダルを表示する。

### 5.2 画面要件

- 画面中央に「プレミアムプラン解約手続き」を表示する。
- **現在のプランステータスカード**: Premium Plan バッジ / 有効期限「YYYY年M月D日 まで」（サーバ値）。
- **利用不可になる機能カード**（6 件、SP 2 列 / Desktop 3 列）: データ最長1年保存 / Excelレポート出力 / SPEEDレビュー / 高度な入力対応 / 外部システム連携 / 優先サポート。
  - 補足文: 解約は自動更新の停止であり、契約期間終了日まで利用可能である旨を表示する。
- **解約理由アンケート**（任意）:
  - 見出し「アンケート（任意）」、補足文「今後のサービス改善のため、よろしければ理由をお聞かせください。」
  - ラジオボタン選択肢:

    | value | 表示文言 |
    | --- | --- |
    | `usage_frequency` | 利用頻度が減った |
    | `price` | 価格が見合わない |
    | `competitor` | 他社サービスに切り替える |
    | `other` | その他 |

  - 「その他」選択時のみ自由記述欄（`otherReasonDetail`）を表示しフォーカスする。
- **アクション**: 「キャンセルして戻る」→ `index.html` へ遷移 / 「解約する」→ 確認モーダルを開く（この時点では解約確定しない）。
- **解約確認モーダル**:
  - タイトル「本当に解約しますか？」
  - 自動更新停止後も契約期間満了まで利用可の旨を表示する。

    | ボタン | 挙動 |
    | --- | --- |
    | 自動更新を停止する | 解約確定処理 → `premium_cancel_complete.html` へ遷移 |
    | キャンセル（解約しない） | モーダルを閉じ画面に戻る |

  - モーダル外クリックでも閉じる。

### 5.3 状態と挙動

- 初期表示: 解約理由は未選択、その他理由入力欄は非表示。
- `cancelReason = other` 選択時: `other-reason-container` を表示し `otherReasonDetail` にフォーカス。他の理由に切り替えた場合は非表示に戻す。
- 解約確定: 確認モーダルの「自動更新を停止する」押下で実行する。
  - 解約理由未選択時は `no_reason` として扱う。
  - 「その他」選択時は `otherReasonDetail` の値を併せて送信する。
  - 成功後は `premium_cancel_complete.html` へ遷移する。

### 5.4 非機能要件

- **レスポンシブ**: PC・タブレット・スマートフォンで主要情報と操作ボタンが見切れないこと。
- **アクセシビリティ**: ラジオボタンはラベルクリックで選択可。確認モーダルに `role="dialog"` / `aria-modal="true"` / `aria-labelledby` を付与する。モーダル表示中はフォーカスをモーダル内に閉じ込める。
- **セキュリティ**: ログイン済みのプレミアム会員または無料トライアル中ユーザーのみ利用可。解約確定はサーバ側の契約状態を確認したうえで行う。CSRF 対策と二重送信防止を実装する。

## 7. プレミアム登録完了（premium_registration_complete.html）

### 7.1 概要

プレミアム登録が完了したことを通知する画面。請求書案内が後日メールで届くこと、入金確認後にプレミアム機能が有効になることを案内する。本画面は `premium_registration_spa.html` の SPA 内 Step 3 とは別画面として存在する。どちらを採用するかは開発会社と確認する。

### 7.2 画面要件

- **完了アイコン**: 画面中央に緑色のチェックアイコン。初期表示時に `scale-in` アニメーションで表示する（装飾扱い）。
- **完了メッセージ**（`fade-in-up` アニメーションで順次表示）:
  - 見出し（`h1`）「プレミアム登録が完了しました!」
  - 説明文: 登録へのお礼 / 登録メールアドレスへ請求書案内を送ること / 入金確認後にプレミアム機能が有効になること / ダッシュボードで請求書を確認できること
- 「ダッシュボードへ戻る」ボタン → `index.html` へ遷移する。
- アニメーションは表示補助であり、操作可能になるタイミングを遅延させない。
- 本画面内に入力項目・再送信操作・モーダルは持たない。
- 請求書が未発行の場合も「後ほど案内を送る」文言を表示する。

### 7.3 非機能要件

- **レスポンシブ**: PC・タブレット・スマートフォンで完了メッセージと戻りボタンが見切れないこと。中央寄せの単一カラムで表示する。
- **アクセシビリティ**: 完了見出しを `h1` として扱う。チェックアイコンは装飾扱いとし、スクリーンリーダーで重複読み上げしない。「ダッシュボードへ戻る」ボタンはキーボード操作でフォーカスできる。`prefers-reduced-motion` 対応は画面外の確認事項とする。
- **セキュリティ**: ログイン済みユーザー向け画面。登録完了状態の正しさはサーバ側契約状態を正とする。請求書や契約 ID などの機微情報は URL に含めない。

### 7.4 API 仕様

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/subscription/me` | 登録完了後の契約状態確認 | セッション Cookie | 200 `{ plan, status, billingStatus }` ／ 401 未認証 |

登録処理そのものは `premium_registration_spa.html` の登録 API で扱う。本画面では契約状態の確認と表示制御のみを行う。

### 7.5 権限・認可

| 利用者種別 | 取扱 |
| --- | --- |
| 未ログイン | ログイン画面へリダイレクト |
| 登録直後ユーザー | 本画面を表示可 |
| 登録未完了ユーザー | プレミアム登録画面またはプラン紹介ページへ誘導 |
| 既存プレミアム会員 | ダッシュボードまたはプレミアム案内ページへ誘導 |

### 7.6 エラー挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| 契約状態取得失敗 | 「登録状態の確認に失敗しました。時間をおいて再度お試しください。」 | ダッシュボード戻り導線は表示 |
| 登録未完了 | 「登録が完了していません。お手続きを再開してください。」 | 登録画面への導線を表示 |
| セッション切れ | ログイン画面へ遷移 | 現在 URL を戻り先として保持 |

### 7.7 受け入れ基準

- [ ] 登録 API 成功後に本画面または SPA 内完了ステップへ遷移する
- [ ] 見出し「プレミアム登録が完了しました!」が表示される
- [ ] 請求書案内メールが送られる旨が表示される
- [ ] 入金確認後にプレミアム機能が有効になる旨が表示される
- [ ] 「ダッシュボードへ戻る」押下で `index.html` へ遷移する
- [ ] 未ログインユーザーは URL 直打ちしても画面に到達できない
- [ ] 直 URL アクセス時の登録状態確認方針が開発会社と合意されている
