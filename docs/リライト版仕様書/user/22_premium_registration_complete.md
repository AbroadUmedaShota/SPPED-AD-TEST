# プレミアム登録完了 要件定義書

| 項目 | 値 |

## 1. 文書目的

- 本書は、プレミアム登録完了画面の要件を定義する。
- 本書を正本とする。
- プレミアム登録 SPA（`premium_registration_spa.html`）の完了ステップとは別画面として扱う。

## 2. 画面概要

- プレミアム登録が完了したことを利用者へ通知する画面。
- 請求書案内が後日メールで送られること、入金確認後にプレミアム機能が有効になることを案内する。
- ダッシュボードへ戻る導線を提供する。
- 共通ヘッダー、フッターは `src/main.js` により placeholder へ注入される。

## 3. 画面要件

### 3.1 完了アイコン

- 画面中央に緑色のチェックアイコンを表示する。
- アイコンは初期表示時にスケールインアニメーションで表示する。

### 3.2 完了メッセージ

- 見出し「プレミアム登録が完了しました!」を表示する。
- 説明文として以下を表示する。
  - 登録へのお礼
  - 登録メールアドレスへ請求書に関する案内を送ること
  - 入金確認後にプレミアム機能が有効になること
  - ダッシュボードで請求書を確認できること

### 3.3 ダッシュボード戻り導線

- 「ダッシュボードへ戻る」ボタンを表示する。
- 押下で `index.html` へ遷移する。

### 3.4 アニメーション

- 完了アイコンは `scale-in` アニメーションで表示する。
- 見出し、説明文、ボタンは `fade-in-up` アニメーションで順次表示する。
- アニメーションは表示補助であり、操作可能になるタイミングを遅延させない。

## 4. 状態と挙動

- 初期表示時に完了メッセージを表示する。
- 本画面内に入力項目、再送信操作、モーダルは持たない。
- 登録完了状態の確認方法は画面外の確認事項とする。
- 請求書がまだ発行されていない場合も、本画面では「後ほど案内を送る」文言を表示する。

## 5. 非機能要件

### 5.1 レスポンシブ

- PC、タブレット、スマートフォンで完了メッセージと戻りボタンが見切れないこと。
- 中央寄せの単一カラムで表示する。

### 5.2 アクセシビリティ

- 完了見出しを `h1` として扱う。
- チェックアイコンは装飾扱いとし、スクリーンリーダーで重複読み上げしない。
- アニメーションを好まない利用者への配慮として、`prefers-reduced-motion` 対応は画面外の確認事項とする。
- 「ダッシュボードへ戻る」ボタンはキーボード操作でフォーカスできる。

### 5.3 セキュリティ

- 本画面はログイン済みユーザー向け画面とする。
- 登録完了状態の正しさはサーバ側契約状態を正とする。
- 請求書や契約 ID などの機微情報は URL に含めない。

## 6. 画面フロー

- `premium_registration_spa.html` の登録 API 成功後 → 本画面または SPA 内完了ステップ。
- 本画面「ダッシュボードへ戻る」 → `index.html`。
- 請求書確認はダッシュボードまたは `invoiceList.html` 側で扱う。

## 7. 既知の課題（実装と要件のギャップ）

| 区分 | 項目 | 内容 |
| --- | --- | --- |
| フロー | SPA 完了ステップとの整理 | 本画面は登録完了用の別画面として存在する。`premium_registration_spa.html` 内の Step 3 完了表示とどちらを採用するかは画面外の確認事項。 |
| 状態確認 | 完了状態の検証 | 現在の画面単体では、直 URL で到達した場合に登録完了済みか確認できない。 |
| 表示 | 請求書案内 | 請求書発行日、支払期限、請求書一覧への直接導線の表示要否は画面外の確認事項。 |
| アクセシビリティ | アニメーション抑制 | `prefers-reduced-motion` への対応が未整備。 |

## 8. 本番化要件

本セクションでは、完成済み画面を本番環境へ移す際に、画面からは直接確定できないため開発会社と確認する事項をまとめる。API・DB・ログ・権限などの詳細は、画面上の確定仕様ではなく実装時の確認観点である。

### 8.1 API 契約

以下は確認用のたたき台であり、画面上の確定仕様ではない。詳細スキーマ・パスは開発会社と合意して確定する。

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/subscription/me` | 登録完了後の契約状態確認 | セッション Cookie | 200 `{ plan, status, billingStatus }` ／ 401 未認証 |

- 登録処理そのものは `19_premium_registration_spa.md` の登録 API で扱う。
- 本画面では契約状態の確認と表示制御のみを扱う。

### 8.2 権限・認可

| 利用者種別 | 取扱 |
| --- | --- |
| 未ログイン | ログイン画面へリダイレクト |
| 登録直後ユーザー | 本画面を表示可 |
| 登録未完了ユーザー | プレミアム登録画面またはプラン紹介ページへ誘導 |
| 既存プレミアム会員 | ダッシュボードまたはプレミアム案内ページへ誘導 |

### 8.3 業務エラー時の挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| 契約状態取得失敗 | 「登録状態の確認に失敗しました。時間をおいて再度お試しください。」 | ダッシュボード戻り導線は表示 |
| 登録未完了 | 「登録が完了していません。お手続きを再開してください。」 | 登録画面への導線を表示 |
| セッション切れ | ログイン画面へ遷移 | 現在 URL を戻り先として保持 |

### 8.4 データ永続化

- 契約情報、請求先情報、規約同意ログは `19_premium_registration_spa.md` の本番化要件に従う。
- 本画面では新規保存処理を行わない。
- 画面表示に必要な契約状態はサーバ側契約テーブルを正とする。

### 8.5 ログ・監査

- 操作ログ: 登録完了画面表示、ダッシュボード戻り押下。
- 監査ログ: 登録完了・規約同意は登録 API 側で記録する。
- 本画面では個人情報や請求情報をログへ出力しない。

### 8.6 受け入れ基準

- [ ] 登録 API 成功後に本画面または SPA 内完了ステップへ遷移する
- [ ] 見出し「プレミアム登録が完了しました!」が表示される
- [ ] 請求書案内メールが送られる旨が表示される
- [ ] 入金確認後にプレミアム機能が有効になる旨が表示される
- [ ] 「ダッシュボードへ戻る」押下で `index.html` へ遷移する
- [ ] 未ログインユーザーは URL 直打ちしても画面に到達できない
- [ ] 直 URL アクセス時の登録状態確認方針が開発会社と合意されている
