# 請求書詳細 要件定義書

| 項目 | 値 |
| --- | --- |
| 対象画面 | `02_dashboard/invoice-detail.html` |
| 正本区分 | 本書を正本とする |
| owner | product |

## 1. 文書目的

- 本書は、請求書詳細画面の要件を定義する。
- 本書を正本とする。
- 印刷専用画面（`invoice-print.html`）は別書の対象とし、本書では関連画面として扱う。

## 2. 画面概要

- 利用者が請求書の宛先、発行日、請求書番号、請求金額、振込先、明細を確認する画面。
- 画面上に請求書帳票レイアウトを表示する。
- 「帳票ダウンロード」ボタンから PDF を生成する。
- 共通ヘッダー、サイドバー、フッターは `src/main.js` により placeholder へ注入される。

## 3. 画面要件

### 3.1 ページタイトル

- 画面タイトル「請求書詳細」を表示する。
- 補足文「請求内訳、明細、振込先を請求書形式で確認できます。」を表示する。
- パンくず領域を表示する。

### 3.2 帳票ダウンロード

- ヘッダー右側に「帳票ダウンロード」ボタンを表示する。
- 押下時、表示中の請求書帳票を PDF として保存する。
- PDF ファイル名は `invoice-<invoiceId>.pdf` とする。

### 3.3 読み込み・エラー表示

- 読み込み中はローディングオーバーレイを表示する。
- 請求書 ID が指定されていない場合、「請求書IDが指定されていません。」を表示する。
- 対象データがない場合、「対象の請求書が見つかりませんでした。」を表示する。
- データ取得に失敗した場合、「請求データの取得に失敗しました。」を表示する。

### 3.4 請求書ヘッダー

- 発行日を表示する。
- 請求書番号を表示する。
- タイトル「御 請 求 書」を表示する。

### 3.5 宛先・差出人

- 宛先会社名を「御中」付きで表示する。
- 担当者名を「様」付きで表示する。
- 差出人として SPEED AD 運営会社情報を表示する。
- 会社ロゴ、住所、電話番号、登録番号、社印画像を表示する。
- 差出人情報は、請求書詳細画面、PDF、印刷用画面で同じ正式値を使う。正式値は画面外の確認事項とする。

### 3.6 金額・振込先

- 件名として「SPEED AD利用料 YYYY年MM月分」を表示する。
- 小計（課税対象）、消費税等、小計（非課税）、合計請求金額を表示する。
- 振込先として金融機関名、支店名、口座種類、口座番号、口座名義、支払期日を表示する。
- 補足として、振込手数料負担と支払期日が休日の場合の案内を表示する。

### 3.7 請求明細

- 明細表の列は以下とする。

| 列 | 内容 |
| --- | --- |
| No. | 明細番号 |
| 品名1 | 項目名 |
| 品名2 | 内訳 |
| 数量 | 数量と単位 |
| 単価 | 単価 |
| 金額 | 金額 |

- 明細が複数ページにまたがる場合、2ページ目以降を追加する。
- ページ番号を `1/2` などの形式で表示する。
- プレミアム利用料、追加アカウント料金、名刺データ化費用、お礼メール送信費用、クーポン値引きは並び順を揃えて表示する。
- 契約関連、アンケート単位の小計行を表示する。

## 4. 状態と挙動

### 4.1 初期表示

- URL クエリ `id` から請求書 ID を取得する。
- 請求書 ID がない場合はエラーを表示し、帳票描画を行わない。
- 請求書データ、ユーザー情報、グループ情報を取得し、請求先を解決する。

### 4.2 請求先の決定

- 個人アカウントの場合、ユーザー情報から請求先会社名と担当者名を表示する。
- 請求先別指定がある場合は、請求先別指定の会社名・担当者名を優先する。
- グループアカウントの場合、グループ請求先または作成者情報をもとに請求先を表示する。
- 請求先が見つからない場合はエラー扱いとする。

### 4.3 集約請求

- `AGG-YYYYMM-GROUP` または `AGG-YYYYMM-PERSONAL` 形式の ID の場合、同月・同契約区分の請求をまとめて表示する。
- 集約時は明細と金額を合算し、請求書小計行を挿入する。
- ステータスは延滞、未入金、取消、入金済の順で判定する。

### 4.4 PDF ダウンロード

- PDF 生成中はローディングオーバーレイを表示する。
- 生成に失敗した場合は「PDFの生成に失敗しました。」を表示する。
- PDF 生成後は画面表示用スタイルを元に戻す。

## 5. 非機能要件

### 5.1 レスポンシブ

- 画面上では帳票を横スクロールまたは縮小なしで確認できることを目標とする。
- 明細表は長い品名・説明文でも折り返して表示する。

### 5.2 印刷・PDF

- A4 縦向きでの出力を前提とする。
- 明細が多い場合はページを分割し、各ページにページ番号を表示する。
- PDF 出力時は余白、影、ページ番号位置を出力用に調整する。

### 5.3 アクセシビリティ

- 主要見出しは `h1` として扱う。
- ダウンロードボタンはキーボード操作で押下できる。
- エラー表示は利用者が内容を確認できる位置に表示する。

### 5.4 セキュリティ

- 本画面はログイン済みユーザー向け画面とする。
- 請求書 ID の URL 直打ちでも、サーバ側で閲覧権限を確認する。
- 請求先情報や金額を不要にログへ出力しない。

## 6. 画面フロー

- `invoiceList.html` の請求書カード選択 → 本画面。
- 本画面「帳票ダウンロード」 → PDF 保存。
- 印刷専用表示を利用する場合 → `invoice-print.html?id=<invoiceId>`。

## 7. 既知の課題（実装と要件のギャップ）

| 区分 | 項目 | 内容 |
| --- | --- | --- |
| API | 請求書 API 未接続 | 現在は `data/core/invoices.json` などの静的 JSON を取得している。 |
| 権限 | 閲覧権限 | 現在の画面単体では、請求書 ID に対する閲覧権限を確認できない。 |
| PDF | クライアント PDF 生成 | `html2pdf.js` で画面から PDF を生成している。サーバ生成にするかは画面外の確認事項。 |
| 差出人 | 会社情報 | 請求書詳細画面、PDF、印刷用画面で使う会社名、住所、電話番号、登録番号、社印画像の正式値は画面外の確認事項。 |
| 旧実装 | 未使用描画関数 | `renderInvoiceDetail` には旧レイアウト用の要素更新が残っている。整理要否は画面外の確認事項。 |

## 8. 本番化要件

本セクションでは、完成済み画面を本番環境へ移す際に、画面からは直接確定できないため開発会社と確認する事項をまとめる。API・DB・ログ・権限などの詳細は、画面上の確定仕様ではなく実装時の確認観点である。

### 8.1 API 契約

以下は確認用のたたき台であり、画面上の確定仕様ではない。詳細スキーマ・パスは開発会社と合意して確定する。

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/invoices/{id}` | 請求書詳細取得 | セッション Cookie | 200 `InvoiceDetail` ／ 403 権限なし ／ 404 なし |
| `GET /api/invoices/{id}/pdf` | PDF 取得または生成 | セッション Cookie | 200 PDF ／ 403 権限なし ／ 404 なし |

- API はログイン済みセッション必須とする。
- 請求先、明細、金額、振込先、支払期限を返却する。
- 集約請求の扱いは開発会社と確認する。

### 8.2 権限・認可

| 利用者種別 | 取扱 |
| --- | --- |
| 未ログイン | ログイン画面へリダイレクト |
| 個人アカウント | 自分の請求書のみ閲覧可 |
| グループ管理者 | 管理対象グループの請求書を閲覧可 |
| 一般メンバー | 請求書閲覧可否は画面外の確認事項 |

### 8.3 業務エラー時の挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| ID 未指定 | 「請求書IDが指定されていません。」 | 帳票は表示しない |
| 請求書なし | 「対象の請求書が見つかりませんでした。」 | 帳票は表示しない |
| 権限なし | 「この請求書を表示する権限がありません。」 | 一覧へ戻る導線を表示 |
| 取得失敗 | 「請求データの取得に失敗しました。」 | 再読み込みを案内 |
| PDF 生成失敗 | 「PDFの生成に失敗しました。」 | 画面表示は維持 |

### 8.4 データ永続化

- 請求書情報、請求先、明細、入金状態はサーバ側の請求データを正とする。
- 画面側で請求情報を更新しない。
- 請求書 PDF の生成・保管方式は画面外の確認事項とする。

### 8.5 ログ・監査

- 操作ログ: 請求書詳細表示、PDF ダウンロード押下、取得失敗。
- 監査ログの要否は画面外の確認事項とする。
- 請求金額、請求先、口座情報は不要にログへ出力しない。

### 8.6 受け入れ基準

- [ ] `id` 指定ありで対象請求書が表示される
- [ ] `id` 未指定時にエラーメッセージが表示される
- [ ] 発行日、請求書番号、宛先、金額、振込先、支払期日が表示される
- [ ] 明細が品名、数量、単価、金額付きで表示される
- [ ] 明細が多い場合にページ分割される
- [ ] 「帳票ダウンロード」押下で PDF が保存される
- [ ] 権限のない請求書は URL 直打ちでも表示できない
