| 項目 | 値 |

| 起動方式 | クエリパラメータ `?tutorial=1[&step=N]` |

## 1. 文書目的

- 本書は、SPEED AD の初回操作チュートリアルの要件を定義する。
- 本書を正本とする。
- 新規アカウントの初回ログイン時に **自動起動される強制チュートリアル**として位置付ける。
- スキップは可能（吹き出し内のスキップボタンから確認モーダル経由）。スキップ確定または完了到達のいずれかで強制再生は終了する。
- 途中離脱（リロード・タブ閉じ・ブラウザ戻る等）には進行状態の保存で対応し、次回アクセス時に中断地点から再開する（§9）。
- ヘルプセンター等からの再閲覧導線は本書のスコープ外（次フェーズで定義）。
- 既存実装（`first-login-tutorial.*`, `surveyCreationTutorial.js`, `tutorial.js`, `app-tutorial.*`, `src/tutorial/`）は本書策定にあわせて廃止する（§12）。

## 2. 画面概要

- 初めて SPEED AD を利用する運用担当者が、アンケート一覧 → 新規作成 → 設問追加（基本＋特殊）→ プレビュー確認 → QR コード → 保存 までの基本フローを誘導付きで体験する。
- チュートリアルは support サイト配下の専用ディレクトリ **`05_support/tutorial/`** で動作する自己完結バンドルである。`02_dashboard/index.html`・`02_dashboard/surveyCreation.html` を基にしたチュートリアル用 2 ページと、その動作に必要なダッシュボードアプリ一式（`src/`・`common/`・`modals/`・`assets/`・`service-top-style.css`・`data/`）を同梱し、`02_dashboard/` を参照せず単独で動作する。
- 自己完結構成のため、`05_support/` が独立サブドメイン化される本番環境でも support サブドメイン単独でチュートリアルを配信できる。実 `02_dashboard/` 画面はチュートリアル用スクリプトの読み込み・強制リダイレクトを一切持たない。
- 起動判定は URL クエリパラメータ `?tutorial=1` で行う。本番／モック共通。
- チュートリアル中の操作は本番データに反映させない（フォーム送信・`localStorage` 書き込み等はガードでインターセプトする、§5.4）。
- ユーザーが「練習である」ことを誤認しないよう、**画面常時表示の練習モードバッジ**（§3.7）と**保存系アクションでの明示的な吹き出し文言**（§4.2 #8 / §4.4 #29）の 2 段構えで知らせる。
- 完了到達またはスキップ確定で完了フラグを立て、以降は強制再生しない。

### 2.1 採用デザインパターン

ゲームのチュートリアルで広く使われる以下の UI パターンを組み合わせて構成する。本書の各セクションはこのパターンを実現するための仕様として読む。

| パターン | 位置付け | 関連節 |
| --- | --- | --- |
| **スポットライト（Spotlight UI）** | 画面全体を暗くし、ハイライト対象だけを切り抜いて目立たせる | §3.2 / §8.1 |
| **コーチマーク（Coachmark）** | ハイライト要素に隣接する吹き出しで一言ずつ案内する | §3.2 / §8.2 |
| **ポインタアニメ（タップ誘導）** | ユーザー操作ステップで指差し / 矢印を対象上に表示する | §8.4 |
| **ロックステップ（Lock-step）** | 指定要素以外はクリック・キーボード操作を一切受け付けない | §5.1 |
| **ハンドホールディング（Hand-holding）** | テキスト入力は自動で済ませ、ユーザーには「次へ」「クリック」だけを求める | §3.6 |
| **強制チュートリアル** | 新規アカウント初回ログイン時に自動起動し、完了またはスキップ確定まで強制的に再生する | §1 / §7 |

## 3. 画面構成

### 3.1 追加ファイル

`05_support/tutorial/src/tutorial/` 配下にチュートリアル用モジュールを置く。

| ファイル | 役割 |
| --- | --- |
| `index.js` | 起動制御（`?tutorial=1` 検出、進行状態復元、各モジュール初期化） |
| `steps.js` | ステップ定義（対象要素・進行方式・吹き出し文言の配列） |
| `overlay.js` | マスク・ハイライト切り抜き・吹き出しの描画 |
| `state.js` | 進行状態の読み書き（§6） |
| `guards.js` | 本番モジュールへの差し込みフック（§5.4） |
| `tutorial.css` | マスク・吹き出し・進行バー・スキップモーダルのスタイル |

チュートリアル専用ディレクトリ `05_support/tutorial/` を自己完結バンドルとして新設する。`02_dashboard/` 配下から `src/`・`common/`・`modals/`・`assets/`・`service-top-style.css` と必要なデータ（`data/core/` 等）を複製して同梱し、`05_support/tutorial/index.html`（`02_dashboard/index.html` のコピー）・`05_support/tutorial/surveyCreation.html`（`02_dashboard/surveyCreation.html` のコピー）・`05_support/tutorial/survey-answer.html`（`02_dashboard/survey-answer.html` のコピー。ブロック D のプレビューで iframe 表示する回答画面）を置く。各ページは同梱した `src/tutorial/index.js` と `tutorial.css` を読み込み、アセットはすべてバンドル内のローカルパスで解決する（`02_dashboard/` への参照は持たない）。`05_support/tutorial/index.html` は `?tutorial=1` が無ければ自動付与してチュートリアルを起動する。

### 3.2 オーバーレイ
- 画面全体を半透明マスクで覆い、ハイライト対象要素のみ切り抜いて操作可能にする。
- ハイライト要素近傍に吹き出し（タイトル・本文・進行ボタン群）を表示する。
- 吹き出し配置は対象要素を基準に `top` / `bottom` / `left` / `right` / `center` を指定する。
- マスクは対象要素の位置・サイズに追従し、スクロール・リサイズで再計算する。

### 3.3 吹き出しの進行ボタン
- 「次へ」（手動進行ステップで表示）／「戻る」（先頭ステップ以外で表示）／「スキップ」（常時表示）の 3 種。
- 対象要素クリックで自動進行するステップでは「次へ」を出さず、案内文を「クリックして次へ」に置換する。
- 「スキップ」押下時はスキップ確認モーダル（§3.5）を表示する。

### 3.4 進行バー
- 画面最上部 sticky、高さ 40px。
- 左端に練習モードバッジ（§3.7）、中央に「ステップ N / 30」、右端に横長プログレスバーを配置する。
- 「ステップ N / 30」の右側に注記「練習モードのため、入力内容や作成したアンケートは保存されません。」を 12px の補助色で常時表示する。

### 3.5 スキップ確認モーダル
- 起動: 吹き出しの「スキップ」押下。
- 文言（暫定）: タイトル「チュートリアルを終了しますか？」／ 本文「終了するとアンケート一覧画面に移動します。途中まで入力した内容は破棄されます。」
- ボタン: 「続ける」（モーダルを閉じてチュートリアル継続）／「終了する」（完了フラグを立て、`?tutorial=1` を外して `02_dashboard/index.html` を再読み込み）。
- Escape キー・背景クリックは「続ける」と同等の挙動とする。

### 3.6 進行方式の 3 カテゴリ

各ステップは以下のいずれか 1 つに分類する。§4 の各ステップ表で「進行方式」列に明示する。

| 進行方式 | システム挙動 | ユーザー挙動 | 吹き出し本文の書き方（例） |
| --- | --- | --- | --- |
| **説明のみ** | 何もしない | 説明を読んで「次へ」を押下 | 「ここがアンケート一覧です。作成済みのアンケートが一覧で表示されます。」 |
| **自動入力** | 対象フィールドに値を自動セット（システム実行） | 入力内容を確認して「次へ」を押下 | 「アンケート名に『初めてのアンケート』を自動で入力しました。確認できたら『次へ』を押してください。」 |
| **ユーザー操作** | 何もしない（クリックを待つ） | 指示された要素を**自分でクリック** | 「右上の『アンケート新規作成』ボタンを押してください。」 |
| **ユーザー操作＋システム後続処理** | クリックを待つ。クリック検出後、本番ハンドラを §5.4 ガードでインターセプトし、ページ遷移・ボタン有効化等の後続処理をシステムが代行 | 指示された要素を**自分でクリック**（以降はシステムが進める） | 「『作成する』ボタンを押してください。」 |

- **テキスト入力は必ず自動入力で行う**。ユーザーにキーボード入力をさせない。
- **ボタンクリック・タイプ選択など導線操作は必ずユーザーに行わせる**。手順を体で覚えてもらう目的のため、システムからの自動クリックは行わない。
- 「ユーザー操作＋システム後続処理」は、保存・遷移など本番ハンドラの起動を伴うステップ専用。チュートリアル中は本番処理を実行せず、後続のチュートリアル進行（URL遷移・ボタン有効化等）に置き換える。
- 吹き出し本文は上表の調子に揃え、誰が何をする時点なのかを毎ステップ明示する。
- autofill ステップの吹き出し本文は「ここが**[フィールド名]**です。〔1行用途説明〕。練習のため『〔値〕』を自動入力しました。確認できたら『次へ』を押してください。」のテンプレートに従う。1 ステップ＝1 フィールドの粒度で構成する。

### 3.7 練習モードバッジ
- 視覚仕様は §8.5 進行バーに統合する。本節は配置・役割のみ記載する。
- チュートリアル起動中、画面最上部進行バーの左端に常時表示する。
- 表示テキスト: 「🎓 練習モード」（暫定）。
- 背景は警告系の差し色（例: amber 系）でダッシュボード本体カラーと明確に差を付ける。
- バッジ自体はクリック不可。役割は「いま操作している画面は練習用である」と一目で伝えること。
- 進行バーの注記「練習モードのため、入力内容や作成したアンケートは保存されません。」と合わせ、ユーザーが画面のどこを見ても練習中と分かる状態にする。

## 4. ステップ仕様

ステップは 4 ブロック・全 30 ステップとする。対象要素 ID は実画面で既に定義されている ID をそのまま使う。先頭にようこそ画面（§4.0）を挟む。

### 4.0 ようこそ画面（プレリュード）

初回ログイン直後（進行状態なし & 開始ステップ = 1）に限り、ブロック A 開始前にフルスクリーンの「ようこそ画面」を 1 枚挟む。中断再開・URL での `step` 指定で起動した場合は表示しない。

| 要素 | 内容 |
| --- | --- |
| アイブロウ | `SPEED AD`（モノスペース・トラッキング広め） |
| タイトル | `ようこそ`（36px・シアングラデ文字） |
| 本文 | 「アンケート作成から QR 発行までの基本フローを、数分で体験しましょう。」 |
| 主アクション | 「始める」（プライマリーボタン）→ ブロック A ステップ 1 へ遷移 |
| 副アクション | 「あとで」（控えめテキストボタン）→ チュートリアルを終了する（お試しモードは §7.1 のアカウント作成 CTA、それ以外は `02_dashboard/index.html` へ遷移） |

- スタイルは §8 視覚仕様に準拠（グラスパネル＋グラデ枠＋発光）。
- 表示中は進行バー・マスク・スポットライト・コーチマーク・ポインタを非表示にする。
- 「始める」押下後にそれらを表示してステップ 1 を開始する。
- このプレリュードはステップカウントに含めない（「ステップ 01 / 30」表記は変わらない）。

## 5. 操作制御

### 5.1 操作のロック
- マスク内（ハイライト対象外）の要素はクリック・キーボード操作を受け付けない。
- ハイライト対象要素のみフォーカス可能とし、Tab 移動も対象内に閉じる。
- 吹き出しの「次へ / 戻る / スキップ」のみ常時操作可能とする。
- **対象外要素がクリックされた場合は、ハイライト対象要素のスポットライト枠と吹き出しを 200ms 周期で 2 回パルス点滅させ、操作可能な箇所を視線誘導する**。無反応にすると「壊れた」と誤認されるため。
- ブラウザのリロード・タブ閉じ・ブラウザ戻る・URL 直打ち等での離脱は技術的に阻止しない。進行状態の保存と次回アクセス時の再開で吸収する（§9）。

### 5.2 入力ステップの扱い
- テキスト入力対象フィールド（アンケート名・表示タイトル・設問文・選択肢ラベル等）は、ステップ開始時にチュートリアル側が値を自動入力し `input` イベントを dispatch する。
- 自動入力値は本番バリデーションを通過する内容とする（必須・期間・最大文字数を満たす）。
- **自動入力ステップが有効な間、対象フィールドには `readonly` 属性を付与してユーザーが値を上書きできないようにする**。ステップ完了（「次へ」押下）後に `readonly` を解除する。これにより、ユーザーが値を空にしたままステップを進めて後段のバリデーションが破綻するケースを防ぐ。
- 設問タイプ選択・追加ボタン押下・QR ボタン押下等のクリック操作はユーザー自身が行う（自動クリックしない）。

### 5.3 ナビゲーション
- ページ遷移を伴うステップ（#8, #30）は、遷移前に進行状態を `localStorage` に書き出し、遷移先 URL にも `?tutorial=1&step=N` を付与する。
- 遷移先ページの `05_support/tutorial/src/tutorial/index.js` は、URL クエリと `localStorage` の進行状態を突き合わせて該当ステップから再開する。
- **再開描画前に、共通HTML（ヘッダー / サイドバー / フッター）の非同期注入完了を `loadCommonHtml` の Promise で `await` してからスポットライト位置計算・吹き出し描画を開始する**。注入完了を待たずに `getBoundingClientRect()` を呼ぶと、共通領域分のレイアウトが未確定でハイライトと吹き出しの位置がずれる。

### 5.4 本番モジュールへのガードと差し替え

実画面のフォーム送信・遷移処理が、チュートリアル中も本番動作してしまうのを防ぐ。あわせて、本番ハンドラが未実装の箇所はチュートリアル側で進行用の代替処理を行う。

- グローバル API: `window.SpeedAD.tutorial.shouldBlock(action)` を `guards.js` が提供する。`?tutorial=1` 時のみ `true` を返す。
- ガード対象モジュールへの差し込み行は `// tutorial-guard` コメントを付け、検索性を担保する。

| 本番モジュール / アクション | チュートリアル時の挙動 | 補足 |
| --- | --- | --- |
| `main.js` `createSurveyFromModalBtn` クリックハンドラ（アンケート新規作成リダイレクト） | 本来の遷移 URL を上書きし、`05_support/tutorial/surveyCreation.html?tutorial=1&step=9` 付きで遷移 | 既存ハンドラに `// tutorial-guard` で1〜2行差し込み。実 `02_dashboard/` 画面ではチュートリアル層が読み込まれず `window.SpeedAD.tutorial` が未定義のためガードは作動せず、通常動作する |
| `surveyCreation-v2.js#attemptSave`（「アンケートを保存する」ボタンハンドラ） | 現状は `showToast` のみで API・`localStorage` 書き込み無し。ガード差し込みではなく、チュートリアル側で完了ステップ（30）への進行を実行 | 本番化時は保存 API 呼び出しが追加されるため、その時点で改めてガード差し込みが必要（§13 関連） |
| `qrCodeModal.js` 起動（ステップ 27） | 通常どおり開く | ガード不要 |

> `surveyService.js#saveSurveyDataToLocalStorage` は現コードで呼び出し箇所が無い死コードのため、ガード対象から外す。整理は §12 廃止対象側で扱う。
> `main.js`「続きから再開」バナー処理（`showTutorialResumeBanner` / `speedad-tutorial-status` 周辺）は §12 で削除対象のため、ガード対象から外す。

### 5.5 全ステップで保証する不変条件

ステップ個別仕様（§4）に加えて、すべてのステップで以下の不変条件を満たす。実装側はステップ表に書かれていなくても本節の条件を常に適用する。

**1. 対象要素の到達性**
- ステップ仕様に `target` セレクタがある場合、要素が DOM 上に存在しなければ最大 4 秒間出現を待機する（MutationObserver）。
- ステップ定義で `waitForElement: false` と明示した場合のみ即時失敗する（既定はすべて待機する）。
- 対象要素がビューポート外にある場合は `scrollIntoView({ block: 'center', behavior: 'smooth' })` で自動スクロールしてから描画する。

**2. クリック検出の救済**
- ユーザー操作ステップで対象要素が表示されてから **6 秒経過**しても `click` イベントが検出されない場合、吹き出しフッターに「ボタンが反応しないときは『次へ』で先に進めます」のヒントを 12px / 控えめ色で追記表示する。同時に、user-action ステップで非表示にしていた「次へ」ボタンを一時的に表示し、ユーザーが現ステップを進められる脱出経路を提供する。
- ヒントと一時開放した「次へ」は現ステップが変わったら自動で消える。
- 救済の目的は「ユーザーが本人はクリックしたつもりだが進まない」「対象ボタンの本番ハンドラ側に未知のエッジケースがある」等の状況での脱出経路提供。自動進行はしない（あくまでユーザーの明示的なクリックで進む）。

**3. 吹き出しの画面内クランプ**
- 吹き出し（コーチマーク）の表示位置は計算後、`left` を `clamp(8px, raw, viewport - width - 8px)`、`top` を `clamp(8px, raw, viewport - height - 8px)` で必ず画面内に収める。
- 進行バー（高さ 48px）の下にもクランプ範囲を寄せる。

**4. 対象要素消失時の復帰**
- `MutationObserver` で `document.body` の subtree を監視し、`currentTargetEl` が DOM から取り除かれた場合：
  - ステップ仕様の `target` セレクタで再解決を試行する。
  - 再解決成功時はスポットライト・吹き出しの位置を再計算する。
  - 再解決失敗時はスポットライトを非表示にし、吹き出しを画面中央へフォールバック表示する。ユーザーは「スキップ」または「戻る」で脱出できる。

これらの不変条件は §4 の各ステップ表に明示しない（全ステップ共通の前提）。

| 種別 | キー | 保存先 | 内容 | 寿命 |
| --- | --- | --- | --- | --- |
| 進行状態 | `speedad-tutorial-progress` | `localStorage` | 現在のステップ番号、各入力ステップで自動入力した値 | 完了到達／スキップ確定で削除 |

  - 完了画面の「完了」押下（ステップ 30）
  - スキップ確認モーダルの「終了する」押下
- いずれの場合も進行状態（`speedad-tutorial-progress`）を破棄する。
- 本番化時、完了フラグはサーバー側の初回ログインフラグへ、進行状態はサーバー側ユーザー領域に置換する（§13）。

### 7.1 お試しユーザーのアカウント作成導線

- アカウント未保有のユーザーが、公開トップ（`index.html`）およびサポートサイト共通フッターの「チュートリアル」リンクからチュートリアルを体験できる。これらの導線リンクには `?trial=1` を付与する。

- お試しモードでは、完了（ブロック D 最終ステップ）・スキップ確定・ようこそ画面「あとで」のいずれの離脱でも `02_dashboard/index.html` へは遷移せず、全画面の「アカウント作成 CTA 画面」を表示する（§9 の既定の退出先を上書きする）。
- CTA 画面は 2 アクション: 「アカウントを作成」（公開トップのサインアップ `index.html?intent=signup` へ遷移）／「閉じる」（公開トップ `index.html` へ遷移）。
- 初回ログイン経由（アカウント保有者）のセッションは従来どおり、完了・スキップ時に `02_dashboard/index.html` へ遷移する。
- 本番では公開サイトとアプリ（`02_dashboard/`）がドメインを跨ぐため、CTA からサインアップへの遷移先は絶対 URL とする（§13.1 と同じ論点）。

## 8. 視覚仕様

全体方針はグラスモーフィズム＋ホログラフィック。ベタ塗りや Material 既定の影は使わず、半透明＋ぼかし＋シアン→紫の発光で奥行きと先進性を出す。

### 8.1 マスク（スポットライト）
- 背景: 中央が明るく外周が暗くなるラジアルグラデーション（中心 `rgba(11,18,32,0.45)` → 外周 `rgba(11,18,32,0.78)`）。
- 背景全体に `backdrop-filter: blur(2px)` を適用し、後ろの実画面を僅かにぼかす。
- ハイライト切り抜き: 対象要素の外側 6px の余白、角丸 8px。
- 切り抜き境界外周にシアン → 紫のグロー（`box-shadow` 多重）を付与し、ホログラム調の発光リングを作る。
- 位置遷移アニメーション: 切り抜き位置の移動は 220ms `cubic-bezier(0.22, 1, 0.36, 1)`。

### 8.2 スポットライト枠
- 内側 1px のシアン精密枠＋外側 6px のぼかしシアングロー（二重リング）。
- 枠はベタ単色ではなく、シアン → 紫の微グラデーションを適用する。
- 「呼吸」アニメ: 1.6 秒周期でグロー幅 4px ↔ 12px を `ease-in-out` で往復させる。

### 8.3 吹き出し（コーチマーク）
- 幅: 400px（タブレットでは画面幅 -32px に縮小）。
- パディング 24px、角丸 16px。
- 背景: `rgba(15,23,42,0.78)` ＋ `backdrop-filter: blur(20px) saturate(150%)`。
- 枠: 1px のグラデーション枠（シアン → 紫）。`border-image` または擬似要素で実現する。
- 構成:
  - ヘッダー: 左にステップバッジ（タブラー数字「03 / 26」モノスペース、シアンの細フレーム）／右にタイトル（22px / weight 600 / 字間 -0.01em）。
  - 本文: 15px / 行間 1.7 / 色 `rgba(203,213,225,0.95)`。
  - フッター: 左に「スキップ」（控えめテキストリンク）／右に「戻る」「次へ」または「クリックして次へ」案内。
- ハイライト対象との接続: 三角ノッチではなく、1px の発光する細線コネクタ（シアン）で吹き出しと対象を結ぶ。
- 入場アニメ: slide + fade 240ms `cubic-bezier(0.22, 1, 0.36, 1)`。

### 8.4 ポインタアニメ（タップ誘導）
- **ユーザー操作ステップ**（§3.6）でのみ表示する。説明のみ・自動入力ステップでは出さない。
- 構成: 中央にシアン発光ドット（直径 12px）＋ 同心円パルスリング 3 層（外側に広がりながらフェード）。
- アニメーション: 1.4 秒ループ、3 層は遅延 0 / 0.4s / 0.8s で時間差発火させる。
- 旧 `touch_app` Material アイコンは廃止する。
- ポインタはマスクの切り抜きの上に描画し、対象要素のクリック判定は阻害しない（`pointer-events: none`）。

### 8.5 進行バー
- 高さ 48px、`backdrop-filter: blur(16px)` のフローティングバー（背景 `rgba(15,23,42,0.72)`）。
- 左端: 「PRACTICE MODE」ラベル（モノスペース／シアンの細フレーム）。
- 中央: ステップ表記「03 / 26」（モノスペース・タブラー数字）。
- 中央右: 注記「練習モードのため、入力内容や作成したアンケートは保存されません。」（12px / 補助色）。
- 右端: プログレスゲージ（シアン → 紫のグラデーション塗り、発光）。

### 8.6 スキップ確認モーダル
- §8.3 と同じグラスモーフィズム＋グラデーション枠で統一する。
- ボタンも控えめなグラス調とし、ベタ塗り CTA は使わない。

### 8.7 タイポグラフィ
- 見出し: 22px / weight 600 / 字間 -0.01em。
- 本文: 15px / 行間 1.7。
- 数値（ステップ番号など）: `Inter` / `JetBrains Mono` 等のタブラー数字対応フォントを使用する。

## 9. 離脱の扱い

### 9.1 スキップ（意図的離脱）
- 吹き出しの「スキップ」押下 → スキップ確認モーダル（§3.5）。
- 「終了する」確定で完了フラグを立て、進行状態を破棄し、`?tutorial=1` を外して `02_dashboard/index.html` を再読み込み。
- 一度スキップした場合は強制再生対象から外れる。再閲覧導線は次フェーズで提供（§13.7）。

### 9.2 非意図的離脱
- 進行状態（`speedad-tutorial-progress`）が `localStorage` に保存されているため、以下いずれの離脱でも次回アクセス時に**中断したステップから再開**する。

| 離脱パターン | 再開挙動 |
| --- | --- |
| リロード（F5） | 同一 URL の `?tutorial=1` でロードされるため、同一ステップから自動再開（マスク・吹き出しも復元） |
| `tutorial/` ページで `?tutorial=1` を外したアドレスバー操作 | `05_support/tutorial/index.html` が `?tutorial=1` を自動付与し、進行状態に応じたステップで再開する。実 `02_dashboard/` 画面へ移動した場合はチュートリアルを離脱し、再開は次回ログイン時の §7 エントリーポイント判定に委ねる |
| タブ閉じ・ブラウザ閉じ | 次回ログイン時に強制リダイレクトされ、保存ステップから再開。当該ステップがブロック C 以降なら `05_support/tutorial/surveyCreation.html?tutorial=1&step=N` へ自動転送 |
| ブラウザ「戻る」「進む」 | URL に `?tutorial=1` が付いていればその場のステップから再開、外れていれば直前項目と同様 |

### 9.3 進行状態のリセット契機
- スキップ確定
- 完了画面の「完了」押下
- 上記以外で `localStorage` を手動クリアした場合（QA 用途）

リセット後は次回アクセス時にブロック A の先頭から開始する。

## 10. 非機能要件

- PC（横幅 ≥ 1024px）を主対象とする。タブレット（≥ 768px）は閲覧可、スマートフォンは「PC でアクセスしてください」の案内のみ表示する。
- ステップ進行・マスク移動は 60fps を目安に滑らかに動作すること。
- XSS 対策として、吹き出し本文は文字列リテラル定義のみとし、`innerHTML` への動的代入を行わない。
- マスクオーバーレイは `aria-modal="true"` を付与し、フォーカストラップを実装する。
- 「次へ / 戻る / スキップ」と確認モーダルのボタンは Enter / Space で操作可能とする。
- 対応ブラウザは Chrome / Edge / Safari / Firefox の最新 2 バージョンとする。`backdrop-filter`（§8.1 / §8.3 / §8.5）はこの対応範囲で問題なく動作する。
- 実画面側のレイアウト変更でハイライトがズレないよう、ステップ対象要素には専用属性 `data-tutorial-target="<step-key>"` を実画面側に最小限付与し、`steps.js` 側はその属性経由で要素を取得する。ID 直参照は本仕様では補助手段とする。

## 11. 手動確認観点

### 11.1 起動・通過
- 新規アカウントで初回ログインすると、`05_support/tutorial/index.html?tutorial=1&step=1` へ遷移しチュートリアルが起動する。
- ブロック A 〜 D の全 30 ステップを最後まで通過できる。
- ページ遷移を伴うステップ（#8 → #9、#30 → 完了）で進行状態と URL クエリが引き継がれる。
- マスク内の要素はクリックできず、ハイライト対象のみ操作できる。
- 設問タイプ選択・追加ボタン・QR ボタンはユーザー自身がクリックする（自動クリックされない）。
- ステップ 29 で「アンケートを保存する」を押しても、本番アンケート一覧にチュートリアル中のアンケートが残らない（ガード動作）。
- 完了到達後に再ログインしても、チュートリアルは再起動せず通常のダッシュボードへ遷移する。

### 11.2 ゲーム的演出
- ユーザー操作ステップの対象要素上に指差しポインタが脈動表示される。
- スポットライト切り抜きの縁取りが脈動アニメーションする。
- 説明のみ・自動入力ステップでは指差しポインタが出ない。
- 対象要素クリック直後にポインタがフェードアウトしてから次ステップへ進む。

### 11.3 練習モード表示
- 起動中は画面最上部の進行バー左端に「🎓 練習モード」バッジが常時表示される。
- 進行バーに「練習モードのため、入力内容や作成したアンケートは保存されません。」の注記が常時表示される。
- ステップ 8（モーダルの「作成する」）と ステップ 29（「アンケートを保存する」）の吹き出し本文に、「練習のため保存されない」旨が明示されている。
- ステップ 29 押下後に別タブで本番ダッシュボードを開いても、アンケート一覧に新規レコードが現れない。

### 11.4 スキップ
- 吹き出しの「スキップ」押下で確認モーダルが表示される。
- 「続ける」でモーダルが閉じ、押下直前のステップから継続できる。
- 「終了する」で本番ダッシュボードへ遷移し、以降のログインでチュートリアルが再起動しない。
- スキップ時にチュートリアル中の入力アンケートが本番一覧に残らない。

### 11.5 離脱からの再開
- 任意のステップでリロードしても、同じステップが復元される。
- 任意のステップでタブを閉じた後、再ログインすると保存ステップから再開する（ブロック C 以降なら `05_support/tutorial/surveyCreation.html?tutorial=1&step=N` へ自動遷移）。
- ブロック B のモーダル表示中に離脱しても、再開時にモーダルが開き直された状態から再開する。
- `05_support/tutorial/index.html` で `?tutorial=1` を外しても、チュートリアルページが自動でクエリを付与し、進行状態に応じたステップで再開する。
- `localStorage` の `speedad-tutorial-progress` を手動で削除すると、次回アクセス時にブロック A の先頭から開始する。

## 13. 本番化要件

### 13.1 新規アカウント判定と強制遷移

- 初回ログイン判定はサーバー側のユーザー属性（例: `first_login: true`）で行う。ログイン認証成功時に判定し、`true` の場合は `05_support/tutorial/index.html?tutorial=1&step=1`（または進行状態に応じた step）へ遷移する。
- 実 `02_dashboard/` 画面はチュートリアル誘導を持たないため、チュートリアルへの誘導はログイン認証成功時の遷移先判定に一本化する。初回ログイン未完了ユーザーが実 `02_dashboard/` 画面を直接開いた場合の扱い（チュートリアルへ誘導するか、そのまま閲覧を許容するか）は開発会社と確認する。
- ログインルートとリダイレクト先の整合も同時に確認する。
- チュートリアルバンドル `05_support/tutorial/` は `05_support` の独立サブドメインで配信する。チュートリアル完了・スキップ後に実ダッシュボード（`02_dashboard/`、アプリ側ドメイン）へ戻る遷移はクロスドメインとなるため、誘導元（ログイン）・退出先の絶対 URL と、完了フラグをドメイン跨ぎで共有する方式（サーバー側 `first_login` フラグ）を開発会社と確認する。

### 13.2 完了状態とスキップ状態の保存
- 完了画面到達時・スキップ確定時の両方でサーバー API へ通知し、`first_login` フラグを `false` に更新する。
- **完了とスキップは同一フラグ（完了相当）に集約する**。§6 の状態モデルと一致させる。再閲覧導線（§13.7）の設計時に別ステータス化が必要と判明した場合のみ、その時点で分離を検討する。
- 通信失敗時の扱い（ローカル完了フラグのみ立てて次回再試行 / 再生継続 等）は開発会社と確認する。
- 端末・ブラウザを跨いで「1 度完了したら再表示しない」を実現するため、フラグはサーバー側に保持する。

### 13.3 進行状態のサーバー保存
- モックアップ段階では `localStorage` で進行状態を保持するが、本番では別端末・別ブラウザでも中断地点から再開できるよう、サーバー側ユーザー領域に保存する。
- 保存粒度（ステップ番号のみ／自動入力値も含めるか）は開発会社と確認する。
- モーダル→作成画面の引継ぎ用キー `speedad-tutorial-handoff` は、本番ではモーダル送信時に作成画面側へクエリパラメータまたはサーバー API（下書き保存）経由で渡すモデルに置換する。

### 13.4 ログ
- チュートリアル開始・完了・スキップ・各ステップ通過の操作ログを記録する。ユーザー ID・ステップ番号・経過時間を含む。保管期間は開発会社と確認する。
- リロード・タブ閉じによる中断回数もログに残し、UX 改善の指標として利用する。

### 13.5 文言の運用
- 吹き出し文言・進行バー表記は初版ではソースコード内に直接記述する。将来的に文言管理基盤（CMS / 翻訳ファイル）へ外出しする想定。

### 13.6 多言語対応
- 初版は日本語のみ。多言語対応はダッシュボード本体の多言語化整備に合わせて検討する。

### 13.7 再閲覧導線（次フェーズ）
- 完了済みユーザー・スキップ済みユーザーがチュートリアルを再閲覧できる導線（ヘルプセンター内リンク、ヘッダーメニュー等）は本書のスコープ外。次フェーズで定義する。
- 再閲覧時も同一の `?tutorial=1` 起動方式を流用する想定。
