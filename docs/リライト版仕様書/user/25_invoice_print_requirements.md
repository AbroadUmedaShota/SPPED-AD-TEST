# 請求書印刷用画面 要件定義書

| 項目 | 値 |
| --- | --- |
| 対象画面 | `02_dashboard/invoice-print.html` |
| 正本区分 | 本書を正本とする |
| owner | product |
| last_reviewed | 2026-04-27 |

## 1. 文書目的

- 本書は、請求書印刷用画面の要件を定義する。
- 本書を正本とする。
- 請求書詳細画面（`invoice-detail.html`）とは別に、印刷に特化した表示を扱う。

## 2. 画面概要

- 利用者が請求書を印刷するための専用画面。
- URL クエリ `id` で指定された請求書を取得し、印刷向けレイアウトで表示する。
- 請求書データ表示後、自動でブラウザの印刷ダイアログを開く。
- 共通ヘッダーやサイドバーは表示せず、帳票本文のみを表示する。

## 3. 画面要件

### 3.1 初期表示

- `invoice-content` に請求書を描画する。
- `id` がない場合は「請求書IDが指定されていません。」を表示する。
- 対象請求書がない場合は「対象の請求書が見つかりませんでした。」を表示する。
- データ取得に失敗した場合は「請求データの取得に失敗しました。」を表示する。

### 3.2 請求書ヘッダー

- タイトル「請求書」を表示する。
- 発行日、請求書番号を表示する。
- 差出人情報を右上に表示する。

### 3.3 宛先

- 請求先会社名を「御中」付きで表示する。
- 担当者名を「様」付きで表示する。
- 請求先情報はユーザー情報またはグループ情報から解決する。

### 3.4 請求サマリー

- 以下の項目を表示する。

| 項目 | 内容 |
| --- | --- |
| 請求対象期間 | `YYYY年MM月DD日 〜 YYYY年MM月DD日` |
| 契約区分 | 個人契約またはグループ契約 |
| 請求内訳 | 明細から判定した主要費目 |
| ステータス | 未入金、入金済、延滞、取消 |
| 小計（課税） | 課税対象金額 |
| 小計（非課税） | 非課税金額 |
| 消費税 | 消費税額 |
| 合計（税込） | 合計請求額 |

### 3.5 振込先

- 振込先銀行、支店名、口座種別、口座番号、口座名義、支払期限を表示する。

### 3.6 請求明細

- 明細表の列は以下とする。
  - No.
  - 項目
  - 内訳
  - 数量
  - 単価
  - 金額
- 明細がない場合は「明細が登録されていません。」を表示する。
- 備考がある場合、1ページ目に備考欄を表示する。
- 明細が多い場合、複数ページに分割する。
- 各ページ右下にページ番号を表示する。

### 3.7 自動印刷

- 請求書描画後、短い待機時間を置いて `window.print()` を実行する。
- エラー表示時は自動印刷しない。

## 4. 状態と挙動

### 4.1 請求書取得

- URL クエリ `id` から請求書 ID を取得する。
- `fetchInvoiceById(id)` で請求書データを取得する。
- ユーザー情報、グループ情報を取得し、請求先会社名と担当者名を補完する。

### 4.2 請求先の決定

- 個人アカウントの場合、ユーザー情報から請求先を決定する。
- 請求先別指定がある場合は、請求先別指定の会社名・担当者名を優先する。
- グループアカウントの場合、グループ請求先または作成者情報から請求先を決定する。

### 4.3 明細の並び順

- プレミアム利用料、追加アカウント料金、名刺データ化費用、お礼メール送信費用、クーポン値引きの順で表示する。
- アンケート単位の明細は同じアンケート名でまとめ、小計行を表示する。
- 契約関連費用は契約関連の小計行を表示する。

### 4.4 ページ分割

- 1ページ目は宛先、サマリー、振込先を含むため、明細表示量を少なめにする。
- 2ページ目以降は明細を中心に表示する。
- 小計行がページ先頭に孤立しないよう、前行との関係を保つ。

## 5. 非機能要件

### 5.1 印刷

- A4 縦向きで印刷する。
- 印刷時は余白、影、不要な背景を抑え、帳票として読める状態にする。
- ページごとに改ページする。

### 5.2 レスポンシブ

- 本画面は印刷用途を主目的とする。
- 画面表示時も横幅 170mm 相当の帳票として中央表示する。

### 5.3 アクセシビリティ

- エラー時のメッセージはテキストで表示する。
- 印刷対象の表は、見出し行とデータ行の対応が分かる構造にする。

### 5.4 セキュリティ

- 本画面はログイン済みユーザー向け画面とする。
- 請求書 ID の URL 直打ちでも、サーバ側で閲覧権限を確認する。
- 画面に出力する値は HTML エスケープして表示する。

## 6. 画面フロー

- 印刷専用画面を採用する場合、請求書詳細画面または請求書一覧から `invoice-print.html?id=<invoiceId>` へ遷移する。
- 現在の請求書詳細画面は PDF ダウンロード中心であり、印刷専用画面への明示導線は表示していない。
- 本画面表示 → 請求データ取得 → 請求書描画 → 印刷ダイアログ表示。
- エラー時はメッセージを表示し、印刷ダイアログは表示しない。

## 7. 既知の課題（実装と要件のギャップ）

| 区分 | 項目 | 内容 |
| --- | --- | --- |
| API | 請求書 API 未接続 | 現在は静的 JSON から請求書データを取得している。 |
| 権限 | 閲覧権限 | 現在の画面単体では、請求書 ID に対する閲覧権限を確認できない。 |
| 導線 | 印刷画面への入口 | 現在の詳細画面は PDF ダウンロード中心で、印刷専用画面への明示導線は画面外の確認事項。 |
| 差出人 | 会社情報 | 印刷画面の差出人表記の正式値は画面外の確認事項。 |
| 保守 | 詳細画面との重複 | 詳細画面と印刷画面で帳票生成ロジックが重複している。整理要否は画面外の確認事項。 |

## 8. 本番化要件

本セクションでは、完成済み画面を本番環境へ移す際に、画面からは直接確定できないため開発会社と確認する事項をまとめる。API・DB・ログ・権限などの詳細は、画面上の確定仕様ではなく実装時の確認観点である。

### 8.1 API 契約

以下は確認用のたたき台であり、画面上の確定仕様ではない。詳細スキーマ・パスは開発会社と合意して確定する。

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/invoices/{id}` | 印刷用請求書データ取得 | セッション Cookie | 200 `InvoiceDetail` ／ 403 権限なし ／ 404 なし |

- API はログイン済みセッション必須とする。
- 印刷用に必要な請求先、明細、金額、振込先、備考を返却する。
- 印刷用 PDF をサーバで生成する場合の方式は画面外の確認事項とする。

### 8.2 権限・認可

| 利用者種別 | 取扱 |
| --- | --- |
| 未ログイン | ログイン画面へリダイレクト |
| 個人アカウント | 自分の請求書のみ印刷可 |
| グループ管理者 | 管理対象グループの請求書を印刷可 |
| 一般メンバー | 請求書印刷可否は画面外の確認事項 |

### 8.3 業務エラー時の挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| ID 未指定 | 「請求書IDが指定されていません。」 | 印刷しない |
| 請求書なし | 「対象の請求書が見つかりませんでした。」 | 印刷しない |
| 権限なし | 「この請求書を表示する権限がありません。」 | 印刷しない |
| 取得失敗 | 「請求データの取得に失敗しました。」 | 印刷しない |

### 8.4 データ永続化

- 本画面では請求データを更新しない。
- 請求書情報、請求先、明細、入金状態はサーバ側の請求データを正とする。
- 印刷履歴を保存するかは画面外の確認事項とする。

### 8.5 ログ・監査

- 操作ログ: 印刷用画面表示、印刷ダイアログ起動、取得失敗。
- 監査ログの要否は画面外の確認事項とする。
- 請求金額、請求先、口座情報は不要にログへ出力しない。

### 8.6 受け入れ基準

- [ ] `id` 指定ありで印刷用請求書が表示される
- [ ] `id` 未指定時にエラーメッセージが表示され、自動印刷されない
- [ ] 発行日、請求書番号、宛先、金額、振込先、支払期日が表示される
- [ ] 明細が品名、数量、単価、金額付きで表示される
- [ ] 明細が多い場合にページ分割される
- [ ] 正常表示後にブラウザの印刷ダイアログが開く
- [ ] 権限のない請求書は URL 直打ちでも表示できない
