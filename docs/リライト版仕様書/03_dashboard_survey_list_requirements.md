## 1. 文書目的

- 本書は、ダッシュボードTOPである「アンケート一覧画面」の要件を定義する。
- 画面モックアップ（`02_dashboard/index.html` およびペア実装）を正本とする。
- アンケート作成の起点・運用中アンケートの俯瞰・各アクションへの導線集約という、本画面の業務上の役割を明文化する。

## 2. 画面概要

- SPEED AD にログイン後、最初に遷移するダッシュボードTOP画面である。
- 登録済みアンケートを一覧表示し、運用担当者が会期状況・回答数・データ化進捗を俯瞰できる。
- 本画面を起点として、アンケートの新規作成・編集・複製・プレビュー・QR配布・SPEEDレビュー・データダウンロード・詳細参照へ遷移する。
- プラン区分（Free / Standard / Premium）により、表示要素および利用可能な機能に差を設ける。

## 3. 画面要件

### 3.1 上部情報
- パンくずと画面タイトル「アンケート一覧」を表示する。
- 画面右側に「アンケート新規作成」プライマリ CTA を配置し、本画面唯一のプライマリ階層とする。
- チュートリアル実行中は CTA クリックを抑制する。

### 3.2 フィルター・検索エリア
- 3 列グリッドで以下を配置する。
  - **キーワード検索**: アンケート名およびアンケートID に対する部分一致検索。入力は debounce（約 300ms）で確定させる。大小文字は区別しない。
  - **ステータスフィルター**: 会期前 / 会期中 / 会期終了 / 会期終了（データ化中）/ 会期終了（データ化完了）/ 会期終了（DL期限終了）を選択可能とする。`削除済み` は選択肢に含めない。
  - **期間フィルター**: 開始日・終了日の 2 つの日付ピッカー。開始日変更時は終了日ピッカーの最小値が連動する。両端 inclusive でマッチさせる。
- **フィルターリセット**ボタンをフィルターエリア内に配置し、キーワード／ステータス／期間を一括クリアする。グループ選択はリセット対象外とする。
- リセット実行時はトーストで完了を通知する。

### 3.3 アンケートテーブル
- 以下の列をこの順序で表示する。

| 列 | ソート | 備考 |
| --- | --- | --- |
| アクション | 不可 | 4 種の行アクションボタンを横並びで配置 |
| アンケートID | 可（初期 desc） | 画面初期表示はこの列の降順 |
| アンケート名 | 可 | 多言語名は現在言語優先で表示 |
| ステータス | 可 | バッジ表示、色はステータス別に固定 |
| 回答数 | 可 | 数値、欠損は 0 とみなす |
| 展示会会期 | 可 | 開始日 〜 終了日 |

- ソート可能な列はヘッダークリック／Enter／Space で昇順降順をトグルし、`aria-sort` 属性を更新する。
- **Premium プランのアンケート行はハイライト表示**（背景色を区別）する。
- 行クリック（ボタン以外の領域）でアンケート詳細モーダル（`surveyDetailsModal`）を開く。
- データ取得中はローディングインジケーターを表示する。取得失敗時は取得失敗オーバーレイを表示し、「データの再取得」「ページをリロード」導線を提供する。
- 該当行が 0 件の場合は空状態メッセージ（検索条件変更 or 新規作成の案内）を表示する。

### 3.4 行アクション
- 各行のアクション列に以下 4 ボタンを配置する。

| ボタン | 動作 |
| --- | --- |
| 編集 | アンケート作成画面へ遷移（対象アンケートを編集対象として開く） |
| QRコード表示 | QR コードモーダルを開く |
| SPEEDレビュー | SPEED レビュー画面へ遷移 |
| データダウンロード | データダウンロードモーダルを開く |

- 各ボタンには tooltip（`title` 属性）と `aria-label` を付与する。
- プラン制限・ライフサイクル状態によっては非活性化する（§4.5 参照）。

### 3.5 ページネーション
- テーブル下に表示件数セレクト（10 / 25 / 50）、前／次ボタン、ページ番号、件数表示（`N - M / 全 X件`）を配置する。
- 表示件数変更時は現在ページを 1 にリセットする。
- ページ番号は現在ページを中心に最大 5 個表示する。

### 3.6 新規作成導線
- 「アンケート新規作成」CTA から **新規作成モーダル**（`newSurveyModal`）を開く。
- モーダル内で必須情報を入力後、アンケート作成画面（`surveyCreation.html`）へ遷移する。

### 3.7 複製導線
- アンケート詳細モーダル内の「複製」ボタンから **複製モーダル**（`duplicateSurveyModal`）を開く。
- 新しいアンケート名・会期開始日・会期終了日を入力し、複製を実行する。
- 複製されたアンケートは一覧の先頭に追加され、ステータス `会期前` / 回答数 0 で初期化される。
- 新しいアンケートID は規定の採番ルール（ユーザーID・年度・連番の組み合わせ）で自動採番する。連番の上限到達時は管理者への問い合わせを促すメッセージを表示する。

### 3.8 サイドバー連動
- サイドバーのグループ選択に応じて一覧を絞り込む。
- 選択中のグループは永続化し、次回ログイン時にも復元する。
- 「個人ビュー」選択時はグループ未設定のアンケートのみを表示する。

## 4. 状態と挙動

### 4.1 初期表示
- ログイン直後または他画面からの戻り時に、サーバーからアンケート一覧を取得して表示する。
- 初期ソートは **アンケートID 降順**（新しいアンケートが上）とする。
- **`削除済み` ステータスのアンケートは一覧およびフィルター選択肢から除外**する。
- 取得失敗時はエラーオーバーレイと失敗トーストを表示し、再取得とページリロードの導線を提供する。

### 4.2 ページ復帰時
- 他画面から本画面へ戻った際、一覧を再取得して最新状態に同期する。
- 他画面で更新された差分（セッションストレージ経由）をマージして反映する。
- 初回訪問時は重複取得を避け、2 回目以降の訪問でのみ再取得する。

### 4.3 フィルタリング
- キーワード・ステータス・期間・グループの 4 条件を AND で適用する。
- フィルター条件を変更すると即時にテーブルを再描画する。
- フィルターリセットは 3 条件（キーワード／ステータス／期間）を一括クリアし、グループ選択は維持する。

### 4.4 言語切替
- ヘッダーで言語を切り替えると、テーブル内の多言語項目（アンケート名など）が即時に再描画される。
- ヘルプポップオーバー文言も言語切替に追従する。

### 4.5 プラン別表示制限
- Premium プランのアンケート行はハイライト表示し、Premium 固有機能（将来: 高度な分析・追加出力形式など）の利用可否を示す。
- プラン制限によりボタンが非活性化される場合は、tooltip で理由を提示する。
- プラン能力定義（`plan-capabilities.json`）に基づくゲーティングは Phase 2 で本画面に統合予定。現行モックアップではハイライトのみ実装済み。

### 4.6 チュートリアル連動
- チュートリアル状態が未完了（`pending` / `main-running` / `modal-running`）の場合、初期データ取得をスキップし空状態で表示する。
- チュートリアル再開バナーを画面上部に表示し、「続きから再開」「終了する」の 2 導線を提供する。
- `?start_tutorial=true` 付き URL でアクセスすると、遅延起動でチュートリアルを自動開始する。

## 5. 詳細モーダル（surveyDetailsModal）

### 5.1 起動
- 一覧の行クリック（ボタン以外の領域）で起動する。

### 5.2 表示内容
- **ヘッダー**: アンケート名、ステータスバッジ（tooltip にステータス説明）。
- **基本情報**: アンケートID、プラン、社内管理用名称、表示タイトル、メモ、展示会会期。
- **データ・請求**: 回答数、リアルタイム差分、データ化完了予定日、ダウンロード期限、見込み／確定請求額。
- **名刺データ化設定**: データ化プラン名、見込み件数、名刺データ化設定画面への遷移ボタン。
- **お礼メール設定**: 設定種別（自動送信／手動送信／送信しない／設定なし）、お礼メール設定画面への遷移ボタン。
- **アクセス情報**: アンケート URL（クリックでクリップボードコピー）、QR コード画像、QR ダウンロードボタン。

### 5.3 アクションボタン

| ボタン | 表示条件 | 動作 |
| --- | --- | --- |
| 編集 | ステータスが「会期前」のときのみ | アンケート作成画面へ遷移 |
| 名刺データ化設定へ | 常時 | 名刺データ化設定画面へ遷移 |
| お礼メール設定へ | 常時 | お礼メール設定画面へ遷移 |
| SPEEDレビューへ | 常時 | SPEED レビュー画面へ遷移 |
| ダウンロード | 常時 | データダウンロードモーダルを開く |
| 複製 | 常時 | 複製モーダルを開く |
| 削除 | 常時（Phase 2） | 確認モーダル後に論理削除を実行予定 |

### 5.4 閉じる挙動
- 閉じるボタン、背景オーバーレイクリック、Esc キー の 3 経路で閉じる。

## 6. プレビュー・QR・ダウンロード

### 6.1 アンケートプレビュー
- 詳細モーダルのアンケート URL から回答者向け画面（`survey-answer.html`）へ遷移してプレビューする。
- URL はクリップボードコピー可能とし、コピー成功時はトーストで通知する。

### 6.2 QR コードモーダル
- 行アクションの「QR コード表示」から開く。
- 対象アンケートの回答用 URL と QR コード画像を表示する。
- URL コピー、QR 画像ダウンロードの 2 導線を提供する。

### 6.3 データダウンロードモーダル
- 行アクションまたは詳細モーダルの「ダウンロード」ボタンから開く。
- 以下 4 種別のデータについて、それぞれ利用可否を判定して表示する。

| 種別 | 可用条件 |
| --- | --- |
| 回答データ | 回答数 > 0 |
| 名刺入力＋回答データ | 回答データと名刺入力データの両方が可用 |
| 名刺入力データ | 名刺データ化を利用中かつデータ化完了済みかつダウンロード期限内 |
| 画像データ | 名刺データ化を利用中かつダウンロード期限内 |

- 利用不可種別は `利用不可` バッジと不可理由を tooltip で表示する。
- 期間種別として「全期間」または「カスタム期間」を選択可能とする。カスタム選択時は開始日時・終了日時を入力する。
- 未選択で送信した場合、または利用不可種別を選択した場合はトーストで案内する。

## 7. ヘルプポップオーバー

- アクション列ヘッダー、アンケート名列ヘッダー、展示会会期列ヘッダー の 3 箇所にヘルプトリガを配置する。
- クリックで説明ポップオーバーを表示し、Esc または外側クリックで閉じる。
- 文言は多言語対応し、画面言語に追従して切り替える。

## 8. 削除・非表示の扱い

- `削除済み` ステータスのアンケートは、一覧およびステータスフィルターから除外する。
- 詳細モーダルの削除ボタンは現行配置のみで、実動作は Phase 2 で実装する。
- 削除は論理削除とし、物理削除・復元機能の要否は Phase 2 で検討する。

## 9. 受入条件

### 9.1 機能受入
- 初期表示でアンケートID 降順の一覧が表示され、`削除済み` は含まれない。
- キーワード入力で 300ms 後にアンケート名／ID の部分一致で絞り込まれる。
- ステータス／期間フィルターが単独および組み合わせで動作する。
- フィルターリセットでキーワード／ステータス／期間がクリアされ、グループ選択は維持される。
- 「アンケート新規作成」CTA から新規作成モーダルが開く。
- 行アクションの 4 ボタンが正しい遷移／モーダル起動を行う。
- 行クリックで詳細モーダルが開き、「会期前」ステータスのときのみ編集ボタンが表示される。
- 詳細モーダルの複製ボタンから複製モーダルを経て新しいアンケートが一覧先頭に追加される。
- 表示件数変更時に現在ページが 1 にリセットされる。
- 他画面から戻った際、編集差分が一覧に反映される。
- サイドバーのグループ切替で一覧が即時絞り込まれ、選択状態が永続化される。
- 一覧取得失敗時にエラーオーバーレイと再取得導線が動作する。

### 9.2 アクセシビリティ受入
- ソート可能ヘッダーはキーボード（Enter／Space）で操作できる。
- 現在のソート状態が `aria-sort` で表現される。
- 全アクションボタンに `aria-label` が設定されている。
- Esc キーで全モーダルおよびモバイルサイドバーが閉じる。

### 9.3 性能・レジリエンス
- 1000 件規模の一覧でもキーワード入力からの再描画が体感的に遅延しない。
- セッションストレージの破損データは安全にスキップされ、処理が継続する。
- 複製時の連番上限到達は運用に検知可能な形で通知される。

## 10. 将来計画（Phase 2）

| 項目 | 内容 |
| --- | --- |
| データダウンロード実装 | サーバー経由の CSV/ZIP 生成。現行はモーダル送信時に「未実装」トースト。 |
| アンケート削除 | 確認モーダル後に論理削除 API を呼び出し、一覧から除外。 |
| QR コード画像生成 | 動的な QR 画像生成（現行はサンプル画像固定）。 |
| プラン別機能制限 | `plan-capabilities.json` 参照による編集・複製・ダウンロード等の可否制御。 |
| ダウンロードボタン非活性化 | ライフサイクル状態に応じたボタン非活性化と理由 tooltip。 |
| 編集ボタン状態制御 | 行アクション側も「会期前」以外で非活性化。 |
| ステータス選択肢の動的同期 | ステータス定義と UI 選択肢の二重管理を解消。 |
| リアルタイム回答数表示 | 一覧での `(+N)` 差分表示を有効化。 |
| 検索フィールド文字数上限 | 入力文字数制限を規定。 |
| ページ番号省略記号 | 総ページ数が多い場合の `…` 表示を導入。 |
| 保存系 API 連動 | 複製・削除・ステータス更新・お礼メール設定種別のサーバー永続化。 |
