## 1. 文書目的

- 本書は、サンクス画面設定画面（アンケート作成者が、回答完了後に回答者へ表示するサンクス画面を設定する画面）の要件を定義する。
- 本書を正本とする。
- 回答者側の表示仕様は [10_thank_you_completion_screen.md](10_thank_you_completion_screen.md) に定義し、本画面は 20 番の描画結果を忠実に制御できる設定 UI を提供する責務を負う。20 番を正、本画面（08）はそれに追従する。

## 2. 画面概要

- アンケート単位で、回答完了後に表示されるサンクス画面の文言と連続回答可否を設定する画面。
- 設定者はダッシュボード利用者（アンケート作成者 / 運用担当者）であり、回答者ではない。
- アンケート作成画面の「追加設定」から遷移する。
- 多言語アンケートの場合、言語ごとのメッセージを個別に編集できる。
- 一部機能はプレミアムプラン連動とし、契約プランに応じて編集可否を切り替える。

## 3. 画面要件

### 3.1 上部情報
- パンくずと画面タイトルを表示する。
- 対象アンケートの識別情報（アンケート名）をタイトルに反映する。

### 3.2 サンクスメッセージ編集
- 回答者に表示するメッセージを編集するテキストエリアを提供する。
- 文字数上限を設ける（現行値: 500 文字）。リアルタイムカウンタを表示する。
- 上限超過時はエラー表示を行い、そのままの保存はさせない。
- 空欄保存を許容し、空欄時は既定文言（20 番側のフォールバック）が表示される旨を補足説明として明示する。
- プランに応じて編集可否を切り替える（§5 参照）。編集不可時はテキストエリアを非活性化し、プラン案内バナーを表示する。

### 3.3 多言語対応
- 対象アンケートの有効言語（`activeLanguages`）が複数ある場合、言語切替タブを表示する。
- 基準言語は日本語（`ja`）で、常にタブ先頭に固定し削除不可とする。
- 同時編集できる言語数に上限を設ける（現行値: 基準言語を含め 3 言語）。
- 未入力の言語タブには「未入力」バッジを表示する。
- 基準言語以外を編集中で、当該言語の値が空かつ基準言語に値がある場合、参照ヒントとして基準言語の値を表示する。
- 保存時の値は言語別オブジェクト（`{ ja: string, en: string, ... }`）として永続化する。
- 単一言語アンケートの場合は言語タブを表示せず、基準言語のみを編集対象とする。

### 3.4 連続回答モード
- 「連続回答を許可する」トグル（`allowContinuousAnswer`）を設ける。
- 既定値はオフとする。
- ON/OFF 変更時は利用者にフィードバック（トースト等）を返す。
- トグルの状態変化は編集中状態としてメモリ上に保持し、保存ボタン押下で永続化する。
- 20 番側は保存された `allowContinuousAnswer` に基づき、回答完了画面に「続けて回答する」導線を表示する。

### 3.5 アクションボタン
- `キャンセル` / `保存する` を画面下部に配置する。
- 未保存変更があるままキャンセル（または離脱）する場合は確認モーダルを表示する。
- 保存時は入力値をバリデーションしたうえで永続化し、成功フィードバックを返す。

### 3.6 モーダル・補助導線
- プレミアムプラン案内バナー
- 未保存離脱確認モーダル
- 入力エラー表示（上限超過、必須未入力等）

## 4. 状態と挙動

### 4.1 初期表示
- 画面表示時に以下を取得する。
  - アンケート基本情報（ID、名称、有効言語、契約プラン）
  - 既存のサンクス画面設定（サンクスメッセージ多言語オブジェクト、連続回答可否）
- 取得に優先順位を設ける。
  1. 永続化済みのサンクス画面設定
  2. アンケート定義内のサンクス画面設定
  3. 既定値（空メッセージ、連続回答オフ）
- 取得失敗時はエラーメッセージとフォーム無効化を行い、呼出元画面への復帰導線を確保する。

### 4.2 編集状態
- 入力・タブ切替・トグル操作は即時にメモリ上の編集状態へ反映する。
- 初期値のスナップショットを保持し、差分の有無で未保存変更を判定する。
- 言語タブ切替時は、現在の編集言語値を保持したまま表示中の言語を切り替える。

### 4.3 バリデーション
- サンクスメッセージの文字数が上限を超える場合はエラー表示する。
- 上限超過のまま保存ボタンが押された場合はエラーフィードバックを返し、保存処理を中断する。
- 文字数超過の自動切り詰めは行わない。

### 4.4 保存
- 保存時は入力値を検証したうえで、サンクスメッセージ（多言語オブジェクト）と連続回答可否をアンケート単位で永続化する。
- 保存成功後は編集状態のスナップショットを更新し、未保存判定をリセットする。
- アンケート ID 未発行状態（作成途中のドラフト）でも一時保存できる経路を用意し、保存後は呼出元のアンケート作成画面へ復帰させる。
- 保存失敗時は原因に応じたメッセージを表示し、入力値は保持する。

### 4.5 未保存離脱
- 変更がある場合、画面離脱前およびキャンセル押下時に確認する。

### 4.6 20 番との整合
- 本画面が保存した値は、回答完了画面（20 番）が以下のとおり参照する。
  - サンクスメッセージ: 言語別オブジェクトから回答者のロケールに対応する値を選択し、フォールバックして描画する。
  - 連続回答可否: `continuous=true` の URL クエリ条件と合わせて、連続回答導線の表示可否を決定する。
- したがって本画面は、20 番が期待するデータ構造（多言語オブジェクト、boolean）で必ず保存する。

## 5. プラン連動要件

### 5.1 編集可否
- **サンクスメッセージ編集**: プレミアムプランのみ編集可とする。非プレミアムはテキストエリア非活性化＋プラン案内バナー表示。
- **連続回答トグル**: 現行はプラン制限なしで全プラン操作可。プラン連動の要否は業務判断事項とし、仕様化時に [11_plan_feature_restrictions.md](../画面設計/仕様/11_plan_feature_restrictions.md) を参照する。

### 5.2 プラン判定
- プラン連動は契約プラン情報を参照して行う。判定条件は `plan-capabilities.json` 等の共通定義に揃える。
- 非プレミアム時のバナー文言・アップグレード導線はプラン連動仕様書に従う。

## 6. 連携要件（参考）

### 6.1 初期表示に必要な情報
- アンケート基本情報（名称・ID・有効言語・契約プラン）
- 既存のサンクス画面設定（サンクスメッセージ多言語オブジェクト、連続回答可否）

### 6.2 保存時に必要な情報
- サンクスメッセージ（言語別オブジェクト）、連続回答可否

### 6.3 設計上の論点
- エンドポイント命名、HTTP メソッド、競合制御方式、エラー応答形式
- 一時保存（ドラフト）とアンケート確定後保存の経路統合
- 回答者側（20 番）との契約（データ構造・言語解決順）の永続化仕様

## 7. 失敗系要件

- **バリデーションエラー**: サンクスメッセージの文字数上限超過、空欄許容可否
- **認可エラー**: プラン制限に抵触する操作（非プレミアムでのメッセージ編集等）は入力不可とし、理由を表示
- **一時障害**: アンケート情報取得・保存失敗時に再試行可能とする
- **データ欠損**: 呼出元からアンケート ID が渡されない場合の代替経路（ドラフト編集）を確保

## 8. 非機能要件

- 入力・タブ切替の操作に対して利用者を待たせない応答時間で反応する。
- 画面再読込後も保存済み設定が正しく再現される。
- 回答者側（20 番）と UI 言語切替側の双方で、多言語オブジェクトの言語解決が矛盾なく行われる。
- プラン連動（契約プランによる表示制限）に対応する。

## 9. 手動確認観点

- 初期表示で既存のメッセージとトグル状態が正しく表示される。
- 多言語アンケートで言語タブを切り替えると、各言語の値が正しく保持・表示される。
- 未入力バッジ、参照ヒントが仕様どおりに表示される。
- 文字数上限超過時のエラー表示、保存中断が仕様どおり機能する。
- 連続回答トグル ON/OFF の操作感（フィードバック）。
- 保存後に 20 番（回答完了画面）側で、設定したメッセージ・連続回答導線が期待どおり反映される。
- プレミアム / 非プレミアムの切替でメッセージ編集欄の活性/非活性、バナー表示が切り替わる。
- 未保存離脱時の確認モーダルが表示される。

## 10. 本番実装で対応すること

画面外の確認事項は以下のとおり。

- **保存先がクライアント localStorage のみ** — サーバ永続化へ移行（アンケート別キーで保存中）
- **プラン連動が部分的** — メッセージ編集はプラン連動済み、連続回答トグルは要業務判断
- **多言語カタログの一部未整備** — 一部対応言語で UI 文字列カタログが未提供
- **UI 言語切替時の自動再描画未対応** — 画面再読込で反映される運用
- **テストシナリオ切替 UI が本番表示のまま** — 本番ビルドでは除外する
- **楽観ロック画面外の確認事項** — 他ユーザー更新との競合検知なし
- **20 番との契約（データ構造）整合の明文化が未完** — 本書 §3.3 / §4.6 で多言語オブジェクトを正と定義、実装側を追従させる

## 11. 本番化要件

本セクションでは、完成済み画面を本番環境へ移す際に、画面からは直接確定できないため開発会社と確認する事項をまとめる。API・DB・ログ・権限などの詳細は、画面上の確定仕様ではなく実装時の確認観点である。

### 11.1 API 契約

以下は確認用のたたき台であり、画面上の確定仕様ではない。詳細スキーマ・パスは開発会社と合意して確定する。

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/surveys/{surveyId}/thank-you-screen` | 初期表示（設定・契約プラン・有効言語・version を取得） | — | 200 `{ settings, activeLanguages, plan, version }` ／ 403 ／ 404 |
| `PUT /api/surveys/{surveyId}/thank-you-screen` | 設定保存 | `{ thankYouMessage: { ja, en, ... }, allowContinuousAnswer: boolean, version }` | 200 `{ settings, version }` ／ 409 ／ 400 ／ 403 |
| `POST /api/survey-drafts/{draftId}/thank-you-screen` | アンケート ID 未発行時の一時保存 | `{ thankYouMessage, allowContinuousAnswer }` | 200 `{ draftId }` ／ 400 |

- 保存値の構造は 09/10 番（回答完了画面）との契約：`thankYouMessage` は言語キー → 文字列の多言語オブジェクト、`allowContinuousAnswer` は boolean。この構造は API 応答にもそのまま反映する。
- プラン連動（サンクスメッセージ編集可否）はサーバ側でも認可判定し、非プレミアムによる保存を拒否する。

### 11.2 権限・認可

| 利用者種別 | 取扱 |
| --- | --- |
| アンケート所有者・編集権限保持者 | 本画面の全操作可（プラン制限範囲内） |
| 閲覧のみ権限 | 表示のみ、保存は非活性 |
| 未ログイン | アクセス不可 |
| 非プレミアムプラン | サンクスメッセージ編集不可、連続回答トグルは可（連動要否は §5.1 のとおり業務判断事項） |

### 11.3 業務エラー時の挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| 初期取得失敗 | エラーメッセージ＋呼出元への復帰導線 | フォーム無効化 |
| 文字数上限超過（クライアント検証） | テキストエリア下にエラー表示 | 保存ボタン非活性化 |
| 保存バリデーション（400） | 該当項目にエラー表示 | 入力値保持 |
| 楽観ロック競合（409） | 競合通知＋再読込導線 | 入力退避を促す |
| プラン制限違反（403） | プラン案内バナー、アップグレード導線 | 該当操作を非活性 |
| ネットワーク切断 | 「ネットワークに接続できません。接続を確認してから再度お試しください。」 | 自動リトライなし |

### 11.4 データ永続化

- サンクス画面設定は既存アンケートテーブルに項目追加、または専用テーブルを持つ（どちらでも可）。
  - `thank_you_message`（JSON: 多言語オブジェクト）
  - `allow_continuous_answer`（boolean）
  - `version`（楽観ロック）
- 一時保存（ドラフト）はアンケート作成ドラフトと同じ保存経路に統合する（アンケート作成側の既存ドラフト機構に乗せる）。
- 現在の画面の localStorage 保存（アンケート別キー）はサーバ永続化へ完全移行し、クライアント側は保持しない。

### 11.5 ログ・監査

- 操作ログ: 設定保存（誰が／いつ／どのアンケート／前後値のダイジェスト）。
- 本画面は個人情報を扱わないため、監査・失敗ログは最小限（API 5xx 応答のみ）。
- 保管期間は運用方針に従う（開発会社と確認）。

### 11.6 対応ブラウザ・デバイス

- 想定対応: 最新版 Chrome / Edge / Safari / Firefox（直近 1 年以内のメジャーバージョン）。
- レスポンシブ対応: PC（横幅 ≥ 1024px）／タブレット／スマートフォン（横幅 ≥ 375px）。
- IE および非 Evergreen ブラウザは対象外。

### 11.7 受け入れ基準

- [ ] 多言語アンケートで言語タブ切替時、各言語値が正しく保持される
- [ ] 保存値が多言語オブジェクト形式で永続化され、10 番（回答完了画面）が同構造で参照できる
- [ ] 文字数上限超過時は保存がブロックされる（クライアント＋サーバ両方で検証）
- [ ] 非プレミアムプランでサンクスメッセージ編集欄が非活性化し、サーバ側保存も 403 で拒否される
- [ ] 連続回答トグルの値が 10 番に反映され、`continuous=true` クエリと組み合わせて表示制御される
- [ ] 未保存離脱時の確認モーダルが表示される
- [ ] 他ユーザー更新との楽観ロック競合（409）が検知され、再読込導線が出る
- [ ] 保存後の再読込で設定値が完全再現される（クライアント localStorage に依存しない）
