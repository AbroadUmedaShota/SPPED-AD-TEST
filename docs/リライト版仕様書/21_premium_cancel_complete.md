## 1. 文書目的

- 本書は、プレミアムプラン解約完了画面の要件を定義する。
- 本書を正本とする。
- 解約手続き画面（`premium_cancel.html`）からの完了遷移先として扱う。

## 2. 画面概要

- プレミアムプランの自動更新停止が完了したことを利用者へ通知する画面。
- 契約期間終了日まではプレミアム機能を利用できること、期間終了後は Standard へ移行することを案内する。
- ダッシュボードへ戻る導線を提供する。
- 共通ヘッダー、サイドバー、フッターは `src/main.js` により placeholder へ注入される。

## 3. 画面要件

### 3.1 完了カード

- 画面中央に白基調の状態カードを1枚表示する。
- カード上部に感謝を想起させるアイコンを表示する。
- 見出し「これまでのご利用、ありがとうございました」を表示する。
- 補助ラベルとして `AUTO-RENEWAL OFF` を表示する。
- 状態カード内に「自動更新の停止手続きが完了しました」のメッセージ帯を表示し、これまでの利用への感謝と再開可能であることを伝える。
- 利用可能期限、期限後のStandard移行、再開導線は、同じ状態カード内の情報行と操作ボタンで示す。
- 問い合わせフォームとサポートセンターへの導線は、同じ状態カード内の補助リンクとして表示する。
- 説明文として以下を表示する。
  - プレミアムプランの自動更新を停止したこと
  - 現在の契約期間終了日まではプレミアム機能を利用できること

### 3.2 今後の利用案内

- 状態カード内の情報行で「今後のご利用について」を表示する。
- 以下の内容を案内する。
  - 契約期間終了後、自動的に Standard へ移行する
  - 期間終了までは Premium を継続利用できる
  - 再開は `premium_signup_new.html` から行える

### 3.3 お礼文

- 完了カードの主見出しとして「これまでのご利用、ありがとうございました」を表示する。
- メッセージ帯で「SPEED AD Premiumをお選びいただきありがとうございました」と伝える。

### 3.4 ダッシュボード戻り導線

- 「ダッシュボードへ戻る」ボタンを表示する。
- 押下で `index.html` へ遷移する。
- 必要に応じて `premium_signup_new.html` への再開導線を小さなリンクとして併記する。

## 4. 状態と挙動

- 初期表示時に完了カードを表示する。
- 本画面内に入力項目や再送信操作は持たない。
- 戻るボタンで解約手続き画面へ戻った場合の再実行制御は画面外の確認事項とする。

## 5. 非機能要件

### 5.1 レスポンシブ

- PC、タブレット、スマートフォンで完了カードが画面内に収まること。
- 主要文言と「ダッシュボードへ戻る」ボタンがファーストビュー内で確認できることを目標とする。

### 5.2 アクセシビリティ

- 完了見出しを `h1` として扱う。
- チェックアイコンは装飾扱いとし、スクリーンリーダーで重複読み上げしない。
- 戻り導線はキーボード操作でフォーカスできる。

### 5.3 セキュリティ

- 本画面はログイン済みユーザー向け画面とする。
- 解約完了状態の正しさはサーバ側契約状態を正とする。

## 6. 画面フロー

- `premium_cancel.html` の確認モーダル「自動更新を停止する」成功後 → 本画面。
- 本画面「ダッシュボードへ戻る」 → `index.html`。

## 7. 既知の課題（実装と要件のギャップ）

| 区分 | 項目 | 内容 |
| --- | --- | --- |
| 状態確認 | 完了状態の検証 | 本画面は解約手続き成功後に表示する。現在の画面単体では、直 URL で到達した場合に解約完了済みか確認できない。 |
| 表示 | 契約期間終了日 | 現在の画面には具体日付が表示されない。表示要否は画面外の確認事項。 |
| 導線 | 自動更新再開 | 完了画面上には自動更新再開導線がない。必要な場合はプレミアム案内画面側で扱う。 |

## 8. 本番化要件

本セクションでは、完成済み画面を本番環境へ移す際に、画面からは直接確定できないため開発会社と確認する事項をまとめる。API・DB・ログ・権限などの詳細は、画面上の確定仕様ではなく実装時の確認観点である。

### 8.1 API 契約

以下は確認用のたたき台であり、画面上の確定仕様ではない。詳細スキーマ・パスは開発会社と合意して確定する。

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/subscription/me` | 解約完了状態と契約終了日の取得 | セッション Cookie | 200 `{ plan, status, currentPeriodEnd }` ／ 401 未認証 |

- 本画面表示前に解約 API が成功していることを前提とする。
- 直 URL アクセス時に契約状態を再確認するかは画面外の確認事項とする。

### 8.2 権限・認可

| 利用者種別 | 取扱 |
| --- | --- |
| 未ログイン | ログイン画面へリダイレクト |
| 解約予約中ユーザー | 本画面を表示可 |
| アクティブなプレミアム会員 | 解約手続き画面へ誘導するかは画面外の確認事項 |
| Standard ユーザー | ダッシュボードへ戻す |

### 8.3 業務エラー時の挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| 契約状態取得失敗 | 「契約情報の取得に失敗しました。時間をおいて再度お試しください。」 | ダッシュボード戻り導線は表示 |
| セッション切れ | ログイン画面へ遷移 | 現在 URL を戻り先として保持 |

### 8.4 ログ・監査

- 操作ログ: 解約完了画面表示、ダッシュボード戻り押下。
- 監査ログは解約 API 側で記録する。本画面では追加の契約更新を行わない。

### 8.5 受け入れ基準

- [ ] 解約 API 成功後に本画面へ遷移する
- [ ] 見出し「これまでのご利用、ありがとうございました」が表示される
- [ ] 「自動更新の停止手続きが完了しました」のメッセージ帯が表示される
- [ ] `AUTO-RENEWAL OFF` が表示される
- [ ] 契約期間終了日まではプレミアム機能を利用できる旨が表示される
- [ ] Standard へ移行する旨が表示される
- [ ] `premium_signup_new.html` への再開導線が表示される
- [ ] 「ダッシュボードへ戻る」押下で `index.html` へ遷移する
- [ ] 未ログインユーザーは URL 直打ちしても画面に到達できない
