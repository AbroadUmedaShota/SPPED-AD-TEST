## 1. 文書目的

- 本書は、プレミアムプラン加入促進ページの要件を定義する。
- 本書を正本とする。
- 加入申込フォーム（`premium_registration_spa.html`）および解約フロー（`premium_cancel.html`）は別書の対象とし、本書では遷移先としてのみ扱う。

## 2. 画面概要

- SPEED AD ダッシュボード内で、プレミアムプランの料金・特典・プラン比較・FAQ を案内し、加入申込フォームへ誘導するマーケティング兼サービス案内ページ。
- 想定利用者:
  - 未課金の Standard ユーザー（新規加入候補）
  - 過去に解約した再加入候補ユーザー
  - 既存プレミアム会員（契約内容・請求書・解約導線の確認）
  - 無料トライアル中ユーザー
  - 解約予約中（期間満了待ち）ユーザー
- ユーザー状態に応じて、同一 URL でヒーローセクション・主要導線・課金キャンペーン表示が動的に切り替わる（§4）。
- ヘッダー / サイドバー / フッターは `src/main.js` により placeholder へ共通部品として注入される。

## 3. 画面要件

### 3.1 ヘッダーエリア（ヒーローセクション）

- パンくず「ホーム → プレミアムプラン」を表示する。
- ヒーロー見出し「事務作業とデータ入力時間を、ゼロへ。」を表示する。小さく「※現在検討中」のサブ注記を併記する。
- ラベル「PREMIUM PLAN」を見出し上部に表示する。
- 3 枚のアイコンカードで主要価値を示す。
  - 「データ最長1年保存」: 契約期間中、最長 1 年間保管
  - 「プロフェッショナル仕様」: 高度な設問設計と外部連携
  - 「高速処理」: 即日利用開始可能
- ユーザー状態別の表示差替えは §4 を参照。

### 3.2 料金セクション

- 見出し「プラン料金」を表示する。
- 月額 `¥10,000`（税別）、補足「1 日あたり約 330 円」を表示する。
- 「✨ 今なら初月無料！」バッジを表示する（ID: `first-month-free-message`）。
- 付帯条件を箇条書きで表示する。
  - 初期費用 0 円
  - 請求書払い（月末締め翌月末払い）
- 右側に CTA ボタン「今すぐ申し込む」を配置し、`premium_registration_spa.html` へ遷移する。
- トラストバッジ「データ最長1年保存」「いつでも解約可能」を CTA 直下に表示する。
- 料金セクション下部に「初月無料キャンペーン」詳細コンテナ（ID: `integrated-campaign-container`）を配置する。新規ユーザーのみ JS で動的に描画する（§4）。

### 3.3 特別機能カード

- 見出し「プレミアムプランだけの特別機能」を表示する。
- 6 枚のカードで機能を訴求する。
  1. 最長1年データ保存: 名刺データを最長 1 年間保存・活用
  2. 高度な入力対応: 手書き文字や画像のデータ化
  3. Excel レポート: 集計データを自由に加工・分析
  4. SPEED レビュー: 入力データと画像を並べて確認・修正
  5. 自社ブランディング: 自社ドメイン送信、ロゴ非表示
  6. 外部システム連携: Slack や SFA/CRM との自動連携
- レスポンシブ: SP 2 列 / Desktop 3 列。

### 3.4 プラン比較表

- 見出し「プラン比較」を表示する。
- Standard（無料） / Premium の 2 列で 9 項目を比較する。

| # | 機能 | Standard (無料) | Premium |
| --- | --- | --- | --- |
| 1 | 追加設問・機能 | × | 設問分岐・画像アップロード・手書き |
| 2 | アンケート構成 | 1 ページ完結 | 改ページ・進捗バー表示 |
| 3 | 多言語対応 | × | 作成・回答表示 |
| 4 | お礼メール | 制限あり | 自動配信・独自ドメイン・条件分岐 |
| 5 | データ化スピード | 通常 | オンデマンド利用可（※別途費用） |
| 6 | 分析・レポート | SPEED レビュー（閲覧のみ） | SPEED レビュー + Excel/CSV 出力 |
| 7 | 特殊データ DL | × | 名刺・画像結合データ出力 |
| 8 | データ保存期間 | 30 日 | 最長 1 年 |
| 9 | 外部連携 | Slack | Slack, SFA/CRM |

- 横スクロール対応。SP では左右スクロールで閲覧する。

### 3.5 CTA セクション（FAQ 直前）

- 見出し「今すぐプレミアムプランを始めませんか？」を表示する。
- 補足「業務効率が劇的に向上します。最短 1 分で登録完了。」を表示する。
- CTA ボタン「今すぐ申し込む」で `premium_registration_spa.html` へ遷移する。
- トラストバッジ「データ最長1年保存」「いつでも解約可能」「ビジネス機能完備」を併記する。

### 3.6 FAQ セクション

- 見出し「よくある質問」を表示する。
- アコーディオン形式の FAQ を配置する。初期状態は開。
  - Q. プレミアムプランの主な機能は何ですか？
  - A. データ保存期間が最長 1 年になるほか、手書き設問、画像アップロード、多言語対応、Excel/CSV ダウンロード、外部連携（Slack/CRM）、自社ドメイン送信、ロゴ非表示などの高度な機能が月額料金内で全て利用可能です。
- 追加の疑問はサポートセンター（`https://support.speed-ad.com/help/`）への導線を提示する。
- 最下部にお問い合わせ導線（`bug-report.html`）を配置する。

### 3.7 モーダル群

| モーダル ID | 役割 | 表示条件 |
| --- | --- | --- |
| `contactModal` | お問い合わせフォーム（氏名・メール・件名・本文） | `open-support-modal` リンク押下（現状は遷移リンクとして機能、§7） |
| `resumeModal` | 自動更新再開の確認モーダル | 解約予約中ユーザーが「自動更新を再開する」カードを押下 |
| `resumeSuccessModal` | 自動更新再開の完了モーダル（次回更新日を表示） | `resumeModal` の「再開する」押下後 |

- `resumeSuccessModal` の「ダッシュボードに戻る」押下でシナリオを `premium-member`（解約予約解除）へ更新し、ページをリロードする。

## 4. 状態と挙動

### 4.1 ユーザー状態の判定

- 判定フラグ（JS では `localStorage.simulationUserData` に格納）:
  - `is_premium_member`: 既存プレミアム会員かどうか
  - `is_free_trial`: 無料トライアル中かどうか
  - `is_cancelled`: 解約予約中かどうか
  - `is_rejoining_user`: 再加入候補ユーザーかどうか
- 判定優先順位（上が優先）:
  1. `is_premium_member`（`is_cancelled` の併存可）
  2. `is_free_trial`
  3. `is_rejoining_user`
  4. 新規ユーザー（デフォルト）

### 4.2 状態別の表示内容

| ユーザー状態 | ヒーロー見出し | メイン領域 | キャンペーン | CTA |
| --- | --- | --- | --- | --- |
| 新規ユーザー（全情報あり／一部あり／情報なし） | デフォルト「事務作業とデータ入力時間を、ゼロへ。」 | 料金・機能比較・FAQ 標準表示 | 初月無料キャンペーン表示 | 「今すぐ申し込む」→ `premium_registration_spa.html` |
| 再加入ユーザー | 「おかえりなさい！ 再びご利用ありがとうございます！」 | 再加入課金注意書き（当月分発生、契約期間を動的表示） | 非表示 | 「再加入する」→ `premium_registration_spa.html` |
| 既存プレミアム会員（Active） | 氏名表示＋Status: Active、現プラン・次回更新予定日（当月末） | 請求書確認 / プラン解約のクイックアクション 2 枚＋サポート導線 | 非表示 | CTA ボタンは表示しない |
| 既存プレミアム会員（解約予約中） | 氏名表示＋Status: Auto-Renewal Off、利用期限＋注意書き（翌日以降フリープランへ） | 請求書確認 / 自動更新を再開する の 2 枚＋サポート導線 | 非表示 | 自動更新再開モーダル（§3.7） |
| 無料トライアル会員 | 氏名表示＋Status: Free Trial、トライアル期限・初回請求予定日 | 請求書確認（空状態）/ トライアル終了 の 2 枚＋サポート導線 | 非表示 | CTA ボタンは表示しない |

### 4.3 日付計算ルール

- 「次回更新予定日」「利用期限」「トライアル期限」: 閲覧日の**当月末日**を動的に算出する。
- 「初回請求予定日」: 閲覧日の**翌月 1 日**を算出する。
- 再加入者向け「契約期間」: 本日 〜 当月末日。
- サーバ連携時は閲覧時刻ではなくサーバ側の契約日付を利用する（§8）。

### 4.4 初月無料キャンペーン描画

- `integrated-campaign-container` は以下の条件を**すべて満たす**場合のみ JS で描画する。
  - `is_rejoining_user` が false
  - `is_premium_member` が false
- 描画内容: 登録月（当月、¥0）と翌月（¥10,000）のタイムライン、および「現在のご登録なら `{当月末日}` まで無料」のサマリ。

### 4.5 FAQ アコーディオン

- `.faq-question` クリックで `aria-expanded` をトグルし、`.faq-answer` の開閉クラス（`open`）を切り替える。

### 4.6 シミュレーションシナリオ切替（モック専用）

- 画面右下の歯車アイコンボタン `scenario-switcher-button` および `scenario-switcher-menu` は**モック確認用の UI であり、本番では非表示**とする。
- 切替値は `localStorage.currentScenario` / `localStorage.simulationUserData` に保存し、リロードにより UI へ反映する。
- シナリオ一覧:
  - 完全新規 - 全情報あり / 一部情報あり / 情報なし
  - 再加入ユーザー
  - 既存プレミアム会員
  - 無料トライアル会員
  - 解約予約中（期間満了待ち）
- 本番移行時は当スイッチャを撤去し、ユーザー状態はサーバ API から取得する（§8）。

## 5. 非機能要件

### 5.1 レスポンシブ

- SP（≥ 375px） / Tablet / Desktop（≥ 1024px）で表示崩れなし。
- プラン比較表は横スクロールで閲覧可能とする。
- 特別機能カードは SP 2 列 / Desktop 3 列。

### 5.2 アクセシビリティ

- FAQ アコーディオンは `aria-expanded` を状態に応じて切替える。
- 装飾アイコンは `aria-hidden="true"` を付与する（現状は一部のみ対応、§7）。
- モーダルは `role="dialog"` / `aria-modal="true"` / `aria-labelledby` を備える（現状未付与、§7）。
- コントラスト比 4.5:1 以上を目標とする。

### 5.3 パフォーマンス

- クライアント完結のため主要操作は 1 秒以内に完結する前提。
- 画像・外部フォントは CDN 経由。サーバ連携時はユーザー状態取得を初回描画前に完了させる。

## 6. 画面フロー

- ダッシュボードのサイドバーやアップグレード導線 → 本画面。
- 本画面 → `premium_registration_spa.html`（新規 / 再加入の CTA 押下）。
- 本画面（既存会員）→ `invoiceList.html`（請求書確認）。
- 本画面（既存会員 Active） → `premium_cancel.html`（解約手続き）。
- 本画面（既存会員 トライアル） → `premium_cancel.html`（トライアル終了）。
- 本画面（解約予約中） → 自動更新再開モーダル → 再開完了モーダル → ダッシュボードへ戻る（状態更新）。
- 本画面 → `bug-report.html`（お問い合わせ）。
- 本画面 → `https://support.speed-ad.com/help/`（サポートセンター）。

本画面の現状観察から確認された未実装・改修必要項目を列挙する。

| 区分 | 項目 | 内容 |
| --- | --- | --- |
| 未実装 | ユーザー状態取得 API | 現状は `localStorage.simulationUserData` を唯一のソースとしており、実ユーザーの課金状態・トライアル状態・解約予約状態をサーバから取得していない。 |
| モック専用 | シナリオ切替ボタン | `scenario-switcher-*` はモック確認用。本番では DOM ごと除去する。 |
| 未実装 | 自動更新再開処理 | `confirmResumeBtn` 押下は `localStorage` のフラグ更新のみで、サーバへの解約予約キャンセル API 呼出は行っていない（`console.log` のみ）。 |
| 未実装 | お問い合わせモーダル送信 | `contactModal` 内フォームはバリデーション・送信処理が未結線（`sendContactFormBtn` のハンドラなし）。現状は `bug-report.html` への遷移が優先される。 |
| デッドコード | Sticky CTA | `premium_signup_new.js` に `#sticky-cta` のスクロール連動コードがあるが、HTML 側に `#sticky-cta` 要素が存在せず動作しない。 |
| デッドコード | `open-support-modal` のモーダル起動 | `a#open-support-modal` は `href="bug-report.html"` のリンクだが、同 ID を対象としたモーダル起動コードの名残が JS に残っている。 |

## 8. 本番化要件

本書はモックアップ作成段階の要件として作成したが、最終的に本番環境構築を担う開発会社が実装に着手するために必要な情報を本セクションに集約する。

### 8.1 API 契約

想定エンドポイントと入出力。詳細スキーマは実装着手時に確定する。

| エンドポイント | 用途 | リクエスト | 主要レスポンス |
| --- | --- | --- | --- |
| `GET /api/subscription/me` | 閲覧ユーザーの課金状態取得（`is_premium_member` / `is_free_trial` / `is_cancelled` / `is_rejoining_user` / `next_renewal_at` / `trial_end_at` 等） | セッション Cookie | 200 `SubscriptionStatus` ／ 401 未認証 |
| `POST /api/subscription/resume` | 解約予約のキャンセル（自動更新再開） | — | 200 `{ next_renewal_at }` ／ 409 状態不正（解約予約中でない） |
| `POST /api/help/contact`（お問い合わせモーダル採用時） | お問い合わせ送信 | `{ name, email, subject, message }` | 200 `{ ok: true }` ／ 400 バリデーション |

- 全エンドポイントはログイン認証済みセッション必須。
- 内部フラグ名は API レスポンスと画面制御で統一する（キャメルケース／スネークケースの揺れを解消）。
- 「当月末日」等の日付はクライアント計算ではなく API レスポンスを正とする。

### 8.2 権限・認可

| 利用者種別 | 取扱 |
| --- | --- |
| 未ログイン | ログイン画面へリダイレクト |
| Standard（未課金）ユーザー | 新規ユーザービュー |
| 過去解約済みユーザー | 再加入ユーザービュー |
| 無料トライアル中ユーザー | トライアルビュー |
| アクティブなプレミアム会員 | 既存会員ビュー（Active） |
| 解約予約中ユーザー | 既存会員ビュー（自動更新停止中） |

### 8.3 業務エラー時の挙動

| エラー種別 | ユーザー表示 | システム挙動 |
| --- | --- | --- |
| 課金状態取得失敗（5xx） | 「プラン情報の取得に失敗しました。時間をおいて再度お試しください。」 | 自動リトライなし。再読込導線を提示 |
| 自動更新再開失敗（409 / 5xx） | 「自動更新の再開に失敗しました。サポートへお問い合わせください。」 | モーダル内でエラー表示、成功モーダルへは遷移しない |
| ネットワーク切断 | 「ネットワークに接続できません。」 | CTA・再開ボタンは無効化しない（再押下で再試行可能） |
| セッション切れ（401） | ログイン画面へ自動遷移 | 現在 URL を戻り先として保持 |

### 8.4 データ永続化

- 課金状態・契約期間・解約予約日等はサーバ側サブスクリプションテーブルに保存する。本画面は読み取り専用。
- クライアント側 `localStorage` 保存は**モック専用**（`currentScenario` / `simulationUserData`）。本番では撤去する。
- お問い合わせモーダル採用時、送信内容のサーバ保管期間は運用方針に従う（要確定）。

### 8.5 ログ・監査

- 操作ログ: CTA 押下（加入申込ページへの遷移）、自動更新再開、お問い合わせ送信。
- 監査ログ: 自動更新再開はユーザー ID・実行時刻・再開前後状態を記録する。
- 失敗ログ: 自動更新再開 API の 409 / 5xx を記録し、サポート照会に備える。

### 8.6 想定データ量

- 1 ユーザーあたりの API 呼出は画面表示時 1 回（`GET /api/subscription/me`）。軽量。
- 自動更新再開は稀操作（月数十件想定、要確定）。性能要件は標準で足りる。

### 8.7 対応ブラウザ・デバイス

- 想定対応: 最新版 Chrome / Edge / Safari / Firefox（直近 1 年以内のメジャーバージョン）。
- レスポンシブ対応: PC（横幅 ≥ 1024px）／タブレット／スマートフォン（横幅 ≥ 375px）。
- IE および非 Evergreen ブラウザは対象外。

### 8.8 受け入れ基準

- [ ] ユーザー状態が `GET /api/subscription/me` から取得され、`localStorage.simulationUserData` への依存が撤去される
- [ ] シナリオ切替ボタン（`scenario-switcher-*`）が本番ビルドから除去される
- [ ] 「次回更新予定日」「利用期限」「初回請求予定日」が API レスポンス値で表示される
