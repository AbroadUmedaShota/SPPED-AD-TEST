# テスト設計 — 名刺データ化フロー シナリオテスト

作成日: 2026-06-03
根拠: `テスト分析.md`、`シナリオテスト実行結果.md`（実機実測セレクタ）

---

## 1. 設計方針

- ユースケース（UC-01〜UC-05）を軸にテストデザインを展開する
- 1 テストデザイン = 1 シナリオの「どう試すか」の骨格
- 各 TD はテストケース（TC-SC-*）に展開する

---

## 2. テストデザイン一覧

### TD-SC-01: 通常プラン（324）回答+名刺投入シナリオ

**対応観点**: VP-01, VP-02, VP-03, VP-04, VP-10
**対応 UC**: UC-01

**設計意図**:
通常プランのアンケート（324）に対して、回答者が設問の全タイプを埋め、ダミー名刺を表面アップロードして送信した場合に、サンクスページへ到達し、管理側の回答件数が増加することを確認する。

**操作概要**:
1. `/questionnaire_answer?id=324` を開く
2. ラジオ・チェック・textarea・number・select・date/time の各設問を入力
3. 手書き canvas が存在する場合は scrollIntoView してマウス描画
4. `#bizcard-open-modal-button` でモーダルを開く
5. `#frontUpload` でダミー名刺画像（`.pwtest/fixtures/dummy-meishi.png`）をアップロード
6. 「裏面をスキップ」→「アップロード完了」
7. `#submit-survey-button` 押下
8. `/survey-thanks` への遷移を確認
9. 管理者画面で 324 の回答件数が +1 されていることを確認

**期待結果**:
- サンクスページ（`/survey-thanks`）に遷移する
- 管理側 324 の回答件数が増加する

**プラン差の確認ポイント**:

### TD-SC-03: 名刺データ化キュー反映確認シナリオ

**対応観点**: VP-05
**対応 UC**: UC-02

**設計意図**:
TD-SC-01/02 完了後、管理者の名刺データ化キュー（`/admin/data_input_list`）のグループ1（メールアドレス）の作業可能件数が、投入した件数分増加していることを確認する。

**操作概要**:
1. Lv4 管理者アカウントで `/admin/login` にログイン
2. `/admin/data_input_list` を開く
3. グループ1「メールアドレス」の作業可能件数を記録する
4. 件数が TD-SC-01/02 の投入分（324+325 = 2件以上）に相当することを確認する

### TD-SC-04: 名刺データ化作業画面 正常系（確定: Ctrl+Enter）シナリオ

**対応観点**: VP-06, VP-07
**対応 UC**: UC-03（主フロー）

**設計意図**:
グループ1 作業画面で Ctrl+Enter による確定操作が正常に動作することを確認する。REG-001 との比較検証として位置づける。

**操作概要**:
1. `/admin/data_input_list` → グループ1 リンク押下 → `/admin/data_input_screen?id=<n>` 表示
2. 名刺画像の表示を確認する
3. メールアドレス入力欄に値を入力する
4. `Ctrl+Enter` を押下する
5. 405 エラー画面に遷移しないこと（画面維持または次フローへの遷移）を確認する

---

### TD-SC-05: 名刺データ化作業画面 異常系（Enter 誤 submit: REG-001）シナリオ

**対応観点**: VP-08, VP-12
**対応 UC**: UC-03（例外フロー）

**設計意図**:
REG-001 の回帰テスト。メールアドレス入力欄で素の Enter を押した場合に HTTP 405 エラー（Laravel デバッグ画面=カーネル画面）が発生することを再現確認する。

**操作概要**:
1. `/admin/data_input_screen?id=<n>` を開く（グループ1 作業画面）
2. メールアドレス入力欄にフォーカスし値を入力する
3. **素の Enter キー**を押下する
4. ページ全体が `MethodNotAllowedHttpException`（HTTP 405）の Laravel デバッグ画面に遷移することを確認する

**副次確認**:
- 405 デバッグ画面にスタックトレース・ソースパスが表示されていること（SEC-R2-1 根拠）

### TD-SC-06: 管理者側 名刺画像 zip ダウンロードシナリオ

**対応観点**: VP-09
**対応 UC**: UC-04

**設計意図**:
管理者が `admin/surveys` の「ダウンロード」ボタンから名刺画像 zip を取得できることを確認する。

**操作概要**:
1. `/admin/surveys` で検索 ID に `324` を入力して検索する
2. 該当行の「ダウンロード」（緑ボタン）を押下する
3. `00191003324ncpics.zip` のダウンロードを確認する（ファイルサイズ > 0）
4. 同様に `325` でも実施し `00191003325ncpics.zip` を確認する
