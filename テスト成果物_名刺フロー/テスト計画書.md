# テスト計画書 — 名刺データ化フロー シナリオテスト

作成日: 2026-06-03
対象環境: `https://stg.speed-ad.com`
テスト種別: シナリオ（ユースケース）テスト — 回答者→管理者をまたぐ多段業務フロー
認証情報: 環境変数のみ・本書に値なし

---

## 1. 目的

名刺データ化フローの業務シナリオ全体（回答者による設問回答・名刺アップロード → 管理者による名刺データ化キュー確認・データ入力作業 → 管理者による名刺画像ダウンロード）が、プラン差（通常/オンデマンド）を含めて正常動作することを確認する。

あわせて、既知の不具合 REG-001（名刺データ化作業画面における Enter キー誤 submit による 405 エラー）を回帰テストとして再現確認する。

## 3. テスト範囲

### 3.1 対象（スコープ内）

- 公開回答ページ（`/questionnaire_answer?id=<surveyId>`）での設問回答・名刺アップロード・送信
- 管理者画面（`/admin/data_input_list`）での名刺データ化キュー確認
- 管理者画面（`/admin/data_input_screen?id=<n>`）での名刺データ入力作業（REG-001 再現テスト含む）
- 管理者画面（`/admin/surveys`）からの名刺画像ダウンロード（zip）
- 通常プランとオンデマンドプランのフロー差異

### 3.2 対象外（スコープ外）

- ユーザー側（`/login`）からのデータ・画像ダウンロード（所有アカウント資格情報なし）
- 照合（matching）フロー
- データ化完了後の CSV アップロード・ユーザー DL 解禁フロー（324: 2026-06-15 以降、325: データ化完了後）
- グループ2〜8（氏名・会社名・住所等）の個別入力作業（グループ1=メールアドレスのみ実施）

| リスクID | リスク内容 | 影響範囲 | 重大度 | テストでの対応 |

| PR-01 | 名刺データ化作業画面の Enter 誤 submit（405） | 名刺データ化業務の全グループ（入力担当者全員） | **高** | TC-SC-05 で REG-001 として再現確認 |

## 5. テストアプローチ

### 5.1 テスト種別

シナリオ（ユースケース）テスト。ロール（回答者→管理者）をまたぐ多段の業務フローを 1 シナリオとして通し実行する。

### 5.2 実行方式

- 手動実行（ブラウザ操作 + Playwright 補助）
- 実ステージング環境（`stg.speed-ad.com`）に対して実行
- 副作用あり（実データ更新）: ユーザー承認済み

### 5.3 副作用方針

- 324/325 各1件のテスト回答・ダミー名刺画像（架空: テスト商事/検証太郎/example.com）を投入済み
- テストデータが不要になった場合、運営側で当該テスト回答を削除することを推奨
- ユーザー側 DL の検証は、所有アカウント資格情報が提供された時点で追加実施

### 5.4 設問種別カバレッジ

回答シナリオは以下の設問タイプを網羅することを目標とする:

| 設問タイプ | セレクタ |
|---|---|
| ラジオボタン | `name=q_*`, `name=likert_*`, `name=matrix_*` |
| チェックボックス | `name=q_*` |
| テキストエリア | `textarea` |
| 数値入力 | `input[type=number]` |
| セレクトボックス | `select` |
| 日付 | `input[type=date]` |
| 時刻 | `input[type=time]` |
| 手書き canvas | scrollIntoView 後にマウス描画 |
| 名刺アップロード | `#bizcard-open-modal-button` → `#frontUpload` / `#backUpload` |
