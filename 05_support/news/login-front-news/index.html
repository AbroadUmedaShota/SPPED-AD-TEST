ログイン前ページのお知らせ欄を準備中です

ログイン前トップに表示するお知らせ欄について、本番運用に向けて構成と連携の整備を進めています。サブドメインの公開 JSON を正本とし、ログイン前ページには最新3件を抜粋表示する想定です。掲載基準・差し替え手順を含めて固まり次第、正式運用へ切り替えます。

想定する正本構成

お知らせの正本データは、サポートサブドメイン（support.speed-ad.com）配下の静的 JSON として配置する予定です。ログイン前ページからは fetch で読み込み、最新の上位3件のみを表示します。

  • /news.json の items 配列を date 降順で取得
  • ログイン前ページには上位3件のみを抜粋（タイトル＋日付）
  • 詳細リンクは 05_support/news/<slug>/ に接続
  • 取得失敗時はお知らせセクションを非表示にし、ログイン導線を阻害しない

掲載基準（運用ルール）

すべてのお知らせを無条件に正本へ載せるのではなく、以下の基準で掲載対象を選別します。

  • サービスの利用に直接関係する情報（アップデート・メンテナンス・障害情報）
  • 導入企業・対外向けの広報情報（プレスリリース・出展案内）
  • 内部進捗の報告・予告など、対外向けに整っていないものは原則対象外

差し替え・更新フロー

正本 JSON の更新は、コンテンツ担当者がリポジトリの 05_support/assets/data/news.json を編集し、PR レビューを経て本番反映する流れを想定しています。Push 後、CDN キャッシュは最大10分で切り替わります。

現状の表示について

本ページを含め、現時点のお知らせ各記事は本番運用のモックアップとして用意したものです。内容・日付・件数はいずれもサンプルであり、正式運用開始時に整理し直します。
