# 見積・設定ロジック

## 設定項目

| 項目 | 用途 |
| --- | --- |
| 名刺データ化を行うか | データ化対象かどうかの基礎 |
| プラン | 単価、納期目安、対応項目の前提 |
| 想定枚数 | 見積金額、想定件数通知、最低請求の前提 |
| オプション | 多言語対応など追加要件の前提 |
| クーポン | アンケート単位の割引適用 |
| 社内メモ | 運用補足、例外対応の記録 |

## 現行モックで見える計算観点

- 名刺データ化を行わない場合は、見積を0として扱う。
- プランに応じた単価と想定枚数から基本金額を計算する。
- 一部オプションは件数や選択数に応じて加算される。
- 最低請求の考え方があり、想定枚数に対する一定割合を最低請求対象として扱う。
- クーポンは画面上の見積では扱うが、最終的な値引き反映は請求側と整合が必要です。

## 注意点

- HOMEでは単価などの課金数値を最小限にし、計算の流れと正本参照先を優先します。
- 本番の金額計算はサーバー側で行う必要があります。
- 画面モックの計算式は、正式な課金ポリシーではなく現行実装の補助根拠として扱います。
