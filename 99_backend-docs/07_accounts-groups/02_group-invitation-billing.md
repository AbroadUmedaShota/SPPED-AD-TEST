# グループ招待と請求関係

## グループの役割

グループは、複数ユーザーでアンケートや請求情報を扱うための共同利用単位です。招待、所属、権限、請求先との関係がバックエンドで管理対象になります。

## 招待パターン

| パターン | 通知 | 説明 |
| --- | --- | --- |
| 既存ユーザー招待 | グループ招待メール | 既存アカウントへ参加承認URLを送る |
| 新規ユーザー招待 | アカウント作成依頼メール | アカウント作成とグループ参加を接続する |

## 請求との関係

| 観点 | 内容 |
| --- | --- |
| 請求主体 | 作成者請求かグループ請求かを判定する |
| 追加アカウント | グループ内の人数が請求明細に影響する可能性がある |
| 招待状態 | 招待中、参加済み、停止、削除の扱いが請求・権限に影響する |
| アンケート共有 | グループ内でどのアンケートを扱えるかを制御する |

## 未確認事項

- 招待URLの有効期限、再送、取消の本番仕様。
- グループ脱退・削除時のアンケート所有権と請求先の扱い。
- 請求対象人数の確定タイミング。
