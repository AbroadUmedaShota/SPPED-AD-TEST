# BUG-002 Backlog起票テンプレート

## 起票条件

Backlogへ起票できるのは、次の条件を満たす代表ケースだけです。

- `defect_cases` に代表ケースがある。
- 少なくとも1件の人間確認済み observation が紐づいている。
- `status` が `duplicate` または `rejected` ではない。
- AI観測のみ、または未確認AI観測のみではない。

## 件名

```md
[不具合][環境][画面/機能] 概要
```

例:

```md
[不具合][stg][アンケート回答] 必須エラー後に回答送信できない
```

## 本文

```md
## 概要
（代表ケースのsummary）

## 発生環境
- 環境: stg / production / local / dev
- 画面/機能: （画面名）
- 症状キー: （重複判定に使う短いキー）
- 深刻度: 緊急 / 高 / 中 / 低

## 再現・観測
- 日時 / 役割 / 再現可否 / 確認状態
  - 具体的な再現手順、補足、観測メモ

## 期待結果
（期待される動作）

## 実際結果
（実際に起きたこと）

## 証跡
- screenshot: URLまたはパス
  - 概要:
  - マスク状態:

## 切り分け状況
- DB受付正本からBacklog対応課題として起票。
- AI観測は人間確認済みのものだけ本文生成対象。

## 関連DB
- case_id:
- backlog_key:
- duplicate_of_case_id:
```

## 深刻度の目安

| DB値 | 表示 | Backlog優先度候補 | 目安 |
| --- | --- | --- | --- |
| `Critical` | 緊急 | 高 | サービス停止、データ損失、送信不能 |
| `High` | 高 | 高 | 主要導線が使えない、回避が難しい |
| `Medium` | 中 | 中 | 一部機能の不具合、回避可能 |
| `Low` | 低 | 低 | 表示揺れ、軽微な文言・UI差分 |

## 追加報告コメント

同じ代表ケースへ追加する場合は、Backlogに新規課題を作らず、既存課題へ次の形式でコメントします。

```md
## 追加報告
- 追加報告ロール:
- 確認日:
- 環境差分:
- 再現可否:
- 既存手順との差分:
- 追加証跡:
- 影響補足:
```

## 起票前チェック

- [ ] 既存Backlog課題に同じ現象がない。
- [ ] 代表ケースに人間確認済み observation がある。
- [ ] 添付や証跡に個人情報、トークン、Cookieが写っていない。
- [ ] AI観測だけを根拠にしていない。
- [ ] 仕様確認なのか不具合なのかを暫定分類している。
