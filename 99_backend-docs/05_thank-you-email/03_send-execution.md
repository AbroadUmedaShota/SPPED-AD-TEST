# 送信実行と失敗系

## 送信実行の流れ

1. 利用者がお礼メール設定を保存する。
2. 会期終了後、名刺データ化または対象者確定処理が進む。
3. 送信可能状態になったら、利用者が送信対象者を確認する。
4. 利用者が送信実行ボタンを押す。
5. システムが対象者リストに基づいてメールを送信する。
6. 送信結果、失敗件数、送信済み状態を更新する。

## 失敗系

| 失敗系 | 必要な扱い |
| --- | --- |
| 期限切れ | 送信不可または例外承認が必要 |
| 対象者なし | 実行不可、理由表示 |
| メールアドレス不足 | 対象者単位で除外または警告 |
| 送信失敗 | 失敗件数、失敗理由、再送可否を記録 |
| 二重送信 | 冪等性キーや送信済み判定で防止 |
| 部分成功 | 成功・失敗を分けて表示、補正方針を決める |

## 未確認事項

- 実送信基盤、送信履歴DB、監査ログの本番設計。
- 再送を許可する条件。
- 利用者操作と管理者補正の責務分担。
