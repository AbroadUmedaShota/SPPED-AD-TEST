# 整理の前提と凡例

## 対象

Backlog SPDAD2026 の課題から、STG/本番環境の不具合・挙動不正・環境差に該当するものを抽出した、6/23時点の整理です。タイトルに「バグ」と入っていなくても、本文が不具合・期待挙動との不一致・環境差を示すものは含めています。

## 抽出条件

- ステータス: 未対応のみ。完了・処理済み・処理中は除外。
- 種別: 運用バグ修正を中心に、仕様メモ・仕様整理・既存品質改善でも内容が不具合性を持つものを含む。
- 純粋な仕様確認の問い合わせ、新機能の仕様策定、モック確認は対象外。

## 本番／STG 列の凡例

| 表記 | 意味 |
| --- | --- |
| あり | その環境で再現を確認 |
| なし | その環境では発生しない（確認済み） |
| 未確認 | その環境ではまだ検証していない |
| 要確認 | 発生環境を特定中 |

## 注意

- ステータスは Backlog の6/23時点の値です。Backlog 側が動いたらこの一覧も更新が必要です。
- 同一事象を別チケットで管理しているものは一覧側でキーを併記しています（例: 名刺画像DLの404は STG=156 / 本番=169）。
- 担当者の割り当ては Backlog 側で管理し、この共有ビューには個人名を記載しません。
