# ステータス判定の全体像

## 目的

アンケート案件の状態を、利用者に見せる言葉とバックエンド上の判定観点に分けて整理します。

## 利用者向けの主要状態

| 状態 | 意味 | 主な制御 |
| --- | --- | --- |
| 会期前 | アンケート開始前 | 設定確認、会期前通知 |
| 会期中 | 回答受付中 | 回答受付、名刺画像登録、件数監視 |
| 会期終了 | 会期終了後でデータ化対象外または不要 | 回答受付終了、編集制御 |
| 会期終了（データ化中） | 名刺データ化処理中 | CSV/名刺DL不可、完了待ち |
| 会期終了（データ化完了） | 名刺データ化完了 | CSV/名刺DL可能、お礼メール送信準備 |
| 会期終了（DL期限終了） | 取得期限超過 | 名刺データDL不可 |
| 削除済み | 削除扱い | 通常一覧から除外または制限 |

## 現行モックで見える判定順

1. 削除済み、アーカイブなどの明示状態を確認する。
2. 会期開始日前なら `会期前`。
3. 会期開始日から会期終了日までは `会期中`。
4. 会期終了後、DL期限を過ぎていれば `会期終了（DL期限終了）`。
5. 名刺データ化対象でなければ `会期終了`。
6. 名刺データ化が完了していれば `会期終了（データ化完了）`。
7. それ以外は `会期終了（データ化中）`。

## 注意点

- `公開中` は内部・旧表現として残る可能性がありますが、利用者向けには `会期中` を優先します。
- 本番ではサーバー側の状態判定を正とし、画面側だけで状態を確定しない設計が必要です。
- 管理者向けには、未割当、照合差異あり、送信失敗、請求確認待ちなどの補助状態が必要になる可能性があります。
