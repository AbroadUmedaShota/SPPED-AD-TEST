# 請求の流れ

## 基本フロー

1. アンケート案件ごとの利用実績が確定する。
2. 名刺データ化件数、お礼メール送信対象、追加アカウントなどの請求対象を整理する。
3. 月次で請求書を作成する。
4. 利用者へ請求通知メールを送る。
5. 利用者が請求書画面またはPDFで明細を確認する。
6. 管理側で請求状態、入金状態、補正メモを管理する。

## 請求に接続する主要情報

| 情報 | 用途 |
| --- | --- |
| アカウントID | 請求先単位 |
| グループ請求設定 | グループ請求か作成者請求かの判断 |
| アンケート案件 | アンケート依存の明細表示 |
| 名刺データ化件数 | 名刺データ化費用の数量 |
| お礼メール送信対象件数 | お礼メール費用の数量計算 |
| クーポン | 値引き行または割引計算 |
| 税区分 | 課税・非課税、消費税計算 |

## 未確認事項

- 請求確定のタイミングと再計算可否。
- 請求書単位とアンケート単位の正規関係。
- 管理者補正と利用者向け請求書表示の同期方法。
