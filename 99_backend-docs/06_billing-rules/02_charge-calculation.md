# 課金計算の考え方

## HOMEでは数値を最小限にする

課金数値は契約・価格・法務判断に近いため、この資料では計算の流れを中心に整理します。具体的な単価や契約条件は正本資料または管理側資料で確認します。

## 代表的な明細

| 明細 | 数量の考え方 | 備考 |
| --- | --- | --- |
| 名刺データ化費用 | 名刺データ化件数 | プランや速度で単価が変わる可能性あり |
| お礼メール送信費用 | 送信対象件数から無料枠を差し引いた件数 | 送信対象件数は名刺データ化件数を超えない前提 |
| 追加アカウント料金 | 請求対象アカウント数 | 表示上は内部無料数を露出しない方針 |
| プレミアム関連 | 契約状態に応じた月額・オプション | private管理との境界に注意 |
| クーポン | 値引きまたは割引行 | 請求確定時の適用条件が必要 |

## 請求書表示の考え方

- 請求書の表示ラベルは具体課金名を使い、曖昧な `従量課金` 表示は避ける。
- アンケート依存の行は、品目1にアンケート名、品目2に具体課金名を表示する。
- 主導線の請求書は、1請求書 = 1アカウントとして読める状態を維持する。
