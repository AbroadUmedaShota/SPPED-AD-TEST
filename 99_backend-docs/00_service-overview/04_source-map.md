# 情報源マップ

## 扱いの優先度

| 種別 | 位置づけ | 例 |
| --- | --- | --- |
| 正本仕様 | 実装・共有時に優先する仕様 | `docs/画面設計/仕様/` |
| 管理者正本 | 管理画面の責務整理 | `docs/画面設計/仕様/admin/` |
| 補助資料 | 正本の理解を助ける資料 | `docs/テンプレート/`, `docs/リファレンス/` |
| 現行モック | 実装済みの挙動確認 | `02_dashboard/`, `03_admin/`, `data/` |
| 旧Wiki・アーカイブ | 過去判断やバッチ時刻の補助根拠 | `docs/アーカイブ/` |
| 実配信観測 | 実際に届いたメール文面の確認 | Gmail観測例をプレースホルダー化 |

## この資料群の立ち位置

`99_backend-docs` は、サービス全体の仕組みを把握するための整理ビューです。正式仕様ではありません。

既存資料、現行モック、旧Wiki、実配信例を横断して、バックエンド側でどの処理が必要そうかを見える化します。正式仕様へ昇格する場合は、背景や観測ログではなく、実装に必要な結論だけを正本資料へ転記します。

## 記載しないもの

- 実名、個人宛先、GmailメッセージID
- 実トークンURL、パスワードハッシュ、認証情報
- 非共有の価格、契約、法務判断
- 本番DB、API、ジョブキューの未確認実装を断定する記述

## 参照時の注意

- 「旧Wiki」は時刻や処理一覧の補助根拠として扱います。
- 「現行モック」は画面やデモデータの実装確認には有効ですが、本番仕様とは限りません。
- 「Gmail実配信例」は観測済み文面の確認であり、送信条件の正本ではありません。
