# E2E-003 実行環境と前提データ

## 基本情報

| 項目 | 内容 |
| --- | --- |
| 対象 | E2E確認時の環境、ブラウザ、データ前提 |
| 現行実行方法 | リポジトリ直下をローカルHTTPサーバーで配信 |
| 推奨ブラウザ | Chrome、Firefox、Edge、Safari確認可能環境 |
| 注意 | `file://` ではなくHTTPサーバー経由で確認する |

## 環境の扱い

| 環境 | URL | 主な用途 | 注意 |
| --- | --- | --- | --- |
| local | `http://127.0.0.1:8001/` など | 静的モックと資料ビューの確認 | サーバー起動ポートに合わせる |
| dev | `https://dev.speed-ad.com/` | 開発環境の実装確認 | 自動操作可否は要確認 |
| stg | `https://stg.speed-ad.com/` | リリース前の結合確認 | データ初期化や通知送信に注意 |
| prod | `https://speed-ad.com/` | 本番確認 | 破壊的操作や実送信は避ける |

## 前提データ

| データ | 用途 | 確認観点 |
| --- | --- | --- |
| `data/surveys/enquete/*.json` | 回答画面の設問表示 | `surveyId` とJSONファイルの対応 |
| `data/core/surveys.json` | ダッシュボード一覧 | アンケート名、会期、ステータス表示 |
| `data/core/plan-capabilities.json` | プラン制限 | Free/Premium等の機能可否 |
| 回答系モックデータ | SPEEDレビュー、グラフ、DL導線 | 集計値や表示文言 |
| シナリオ切替UI | モック上の状態再現 | 本番要件ではなく検証補助として扱う |

## 事前準備

1. リポジトリ直下で `python -m http.server 8001` などを起動する。
2. `http://127.0.0.1:8001/index.html` または対象ページを開く。
3. テスト前にブラウザのローカルストレージをクリアする。
4. 対象JSONが存在し、画面で読み込まれていることを確認する。
5. コンソールとネットワークで取得エラーがないことを確認する。

## ブラウザ幅

| 幅 | 用途 |
| --- | --- |
| desktop | 一覧、表、詳細モーダル、複数カラム表示の確認 |
| tablet | サイドバーやカードの折り返し確認 |
| mobile | タップ操作、縦積み、横スクロール、テキストはみ出し確認 |

## 未確認事項

- dev/stg/prodで使用してよいテストアカウントの範囲。
- 実メール送信、請求確定、データ削除など副作用のある操作を自動化対象から除外する具体ルール。
- シナリオ切替UIを本番ビルドから除外する方式。
