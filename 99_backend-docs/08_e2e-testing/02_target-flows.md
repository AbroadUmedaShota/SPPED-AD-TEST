# E2E-002 対象フロー

## 基本情報

| 項目 | 内容 |
| --- | --- |
| 対象 | SPEED ADの代表的な利用者導線 |
| 主な登場人物 | 企業ユーザー、来場者、事務局、システム |
| 参照元 | `docs/フロー定義/01_user_flow.md` |
| 注意 | 実名や個人宛先を含めず、画面導線と状態だけを扱う |

| --- | --- | --- | --- |

| E2E-FLOW-05 | お礼メール設定 | `02_dashboard/thankYouEmailSettings.html` | メール文面、送信対象、送信期限の状態表示を確認する |

## ログインから回答までの最小シナリオ

| ステップ | 操作 | 期待結果 |
| --- | --- | --- |
| 1 | ログイン画面のテスト用アカウントから標準ユーザーを選択し、ログインする | ダッシュボードが開き、アンケート一覧が表示される |
| 2 | 任意のアンケート行を開き、アンケートURL相当の情報を確認する | 詳細モーダルにアンケート概要とURLが表示される |
| 3 | `survey-answer.html?surveyId={surveyId}` を開く | 回答画面が対象JSONに基づいて表示される |
| 4 | 必須設問を入力して送信する | 完了メッセージが表示され、再送信できない状態になる |
| 5 | コンソールとネットワークを確認する | JSON取得や画面処理のエラーがない |

## 回答送信からデータ受け取りまでの通しシナリオ

| ステップ | 操作 | 期待結果 |
| --- | --- | --- |
| 1 | 回答URLを開き、必須設問と名刺撮影または名刺なし分岐を入力する | 送信条件が満たされ、送信直前状態を確認できる |
| 2 | 明示許可がある場合のみ回答を本送信する | 回答完了画面が表示され、回答データがアンケート案件に紐づく |
| 3 | 管理者側で対象アンケートの名刺処理状態、実件数、入力/照合入口を確認する | 回答が名刺データ化対象として追跡できる |
| 4 | 完了CSVアップロードまたは完了済みテストデータで、データ化完了状態を確認する | 作成者が取得可能な状態へ進む |
| 5 | 作成者側で詳細モーダル、DL導線、DL期限、お礼メール対象者確認を確認する | データ化完了データを受け取る導線が成立する |

## フロー分岐

- 新規ユーザーは初回チュートリアルへ遷移するため、ダッシュボード操作前にチュートリアル完了またはスキップが必要。
- 管理者は管理画面へ遷移するため、回答導線の確認は標準ユーザーで実施する。
- URLパラメータ欠落や不正IDでは、エラーメッセージが表示されることを確認する。
- 多言語設定があるアンケートでは、言語切替で設問文や完了メッセージが切り替わることを確認する。

## 未確認事項

- アンケート公開前の事務局レビューや承認をE2E対象に含めるか。
- 名刺データ化の実作業画面は、確定操作を伴うため許可なしでは入口と直前状態までをE2E対象にする。
- 請求確定やメール送信の実処理を、モック上の表示確認とどう分けるか。
