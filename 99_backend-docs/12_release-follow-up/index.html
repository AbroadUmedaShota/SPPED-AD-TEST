顧客利用、営業活動、運用価値への影響が大きいもの。

通常ロードマップとして順序と仕様確定時期を決めるもの。

1. リリース必達だった残件

リリースに間に合わせたかったが、間に合わなかった機能

品質・移管対応として確認

名刺画像・回答データ・CSVダウンロードの安定化

既存アンケートのデータ取得に直結。大容量DLは改善確認済みだが、処理時間や完了通知遅延は残論点。

関連: SPDAD2026-160 / SPDAD2026-162 / SPDAD2026-164 / SPDAD2026-155 / SPDAD2026-156 / SPDAD2026-146

名刺データ化設定まわり

営業開始、顧客利用、データ納品に関わる前提領域。設定、DL、権限制御の確認が必要。

関連: SPDAD2026-111 / SPDAD2026-155 / SPDAD2026-156 / SPDAD2026-157

ログイン後ダッシュボードTOP・主要導線

営業開始前に整えたかった領域。ダッシュボードTOPと主要導線の対象範囲、Backlog紐付けを確認する。

関連: 要Backlog紐付け確認

本番利用前に潰したかった基本不具合

アカウント情報モーダル保存など、6月上旬時点でリリース可否に影響していた項目。

関連: 要Backlog紐付け確認

ログイン・サインアップ画面の本番化最低限対応

一部は反映済みだが、Googleログインや導入事例は非表示運用。復帰条件と残実装期限を確認する。

関連: SPDAD2026-147

2. 短期導入したい機能

リリースには間に合わないと聞いていたが、早く導入したい機能

営業・顧客利用への影響順

Google認証

ログイン、会員登録のGoogle認証。導入可能時期と既存ログインとの併用を確認。

SPDAD2026-1 / SPDAD2026-3

導入事例表示・公開事例連携

公開NG記事の整理後、営業面で早めに復帰したい領域。

SPDAD2026-166 / SPDAD2026-147

プレミアムプランの未実装機能

サンクス文言設定、名刺データ化項目追加など。未実装表示のまま残す範囲も決める。

SPDAD2026-111 / SPDAD2026-114

お礼メール設定・自動送信・エラー検知

会期後フォロー品質に直結。送信条件、エラー時通知、手動運用との切替を確認。

SPDAD2026-108

SPEEDレビュー機能

表示不具合は修正報告あり。機能としての本格導入と編集導線は確認が必要。

SPDAD2026-109

ヘルプ・バグ報告の本番実装整理

実体は反映済みの可能性あり。Backlogクローズと運用開始条件を整理する。
