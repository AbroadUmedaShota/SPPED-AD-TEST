# 通知起点

## バッチ起点のシステムメール

| 起点 | 対象メール | 備考 |
| --- | --- | --- |
| 会期前日 | 会期前日メール | Excel・Gmail確認、本文差分あり |
| 会期開始日 | 会期開始メール | Excel・Gmail確認 |
| 会期終了日 | 会期終了メール | Excel・Gmail確認 |
| データ化完了 | 名刺データ化完了メール | Excel・Gmail確認 |
| データ化完了後 | 御礼メール送信依頼 | Excel・Gmail確認 |
| DL期限前 | ダウンロード期限メール | Excelのみ確認 |
| 月末請求 | 請求確定・請求書送付 | Excel・Gmail確認、既存テンプレート差分あり |

## 操作起点の通知

| 起点 | 対象メール |
| --- | --- |
| アカウント作成 | 新規アカウント作成時 |
| パスワード再設定 | パスワード再設定時 |
| 名刺データ化申込 | 名刺データ化申込時 |
| グループ招待 | グループ招待、新規ユーザー招待 |

## 設計上の論点

- バッチの再実行でメールが重複送信されないよう、送信済み判定が必要です。
- 状態更新に成功したが通知に失敗した場合の再送・補正手段が必要です。
- 実配信とExcelテンプレートに差分があるため、正本テンプレートを別途確定する必要があります。
