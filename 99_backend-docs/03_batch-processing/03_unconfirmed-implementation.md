# 未確認の実装論点

## 確認が必要なこと

| 論点 | 確認内容 |
| --- | --- |
| ジョブ基盤 | cron、キュー、クラウドスケジューラなど、どこで実行するか |
| 冪等性 | 同じ処理が複数回実行された場合の重複更新・重複送信防止 |
| リトライ | 失敗時に自動再実行するか、手動補正するか |
| 監視 | 実行成功、失敗、遅延をどこで検知するか |
| 履歴 | 状態更新、通知送信、請求作成の履歴をどの粒度で残すか |
| タイムゾーン | 会期、期限、月末処理をどのタイムゾーンで判定するか |
| 営業日 | 名刺データ化やお礼メール期限で営業日計算をどう扱うか |

## 正式化時の分け方

- 状態更新ジョブ
- 通知ジョブ
- 請求作成ジョブ
- 実績集計ジョブ
- リトライ・補正オペレーション
