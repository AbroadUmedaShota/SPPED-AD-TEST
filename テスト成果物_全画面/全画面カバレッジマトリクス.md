## 凡例

| 列名 | 説明 |
| --- | --- |
| 実ルート | 実測ルート（実測値のみ記載。未特定の場合は「要マッピング」） |
| スモーク HTTP | 認証済みクロールまたはsmokeテストで取得したHTTPステータス |
| コンソールerr | 実測コンソールエラー件数 |
| スモーク結果 | スモーク層（到達性+コンソールerr+表示）の実施状況 |
| 機能テスト | 機能テスト（観点別）の実施状況 |
| 今後の機能テスト観点 | 未実施の深い機能テスト観点（実施済みは「スモークのみ」と記載） |

スモーク結果の値:

- `済`: 実測でHTTP200+コンソールerr0を確認
- `済(要確認)`: 到達したが確認事項あり（本文参照）
- `済(異常)`: 実測で異常（4xx/5xx/コンソールerr）を確認
- `未`: 未実施
- `該当なし`: 独立ルートなし（モーダル等）

機能テストの値:

- `実施済(観点別)`: run3機能テストで観点別チェック実施（PASS/WARN/SKIP/FAILあり）
- `機能テスト未実施`: 今回未実施
- `該当なし`: 独立ルートなし（モーダル等）
